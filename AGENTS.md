# AGENTS

1. 始终使用中文对话
2. 遵守.kiro/specs/realtime-crypto-datawarehouse的设计规范
2. 生成的代码按功能拆分，能复用的功能方法进行复用，不要重复实现相同功能
3. 添加新的方法、调用新的外部接口等要先确保可用，可以先写脚本测试没问题再使用
3. 我把一些关键信息保存在本地环境变量，你可以去查看，需要的时候获取使用，简单的方法是echo 某个变量获取到 
4. 我是初学者，代码尽量加注释，尤其每次改动要加注释说明下 
5. 任务完成后检查下是否符合要求
6. 每次完成一次对话、解决一次问题，都要添加一个说明文档，用来说明本次解决问题的思路、步骤、有效方法等信息
6. 生成的说明文档统一放到docs目录,文件名称格式{日期如202601241230}-{功能，几个字}-{具体解决的问题}.md，方便后续排查问题搜索、参考
7. 同时每一个新解决的问题都要更新到根目录下的问题解决汇总.md文件里，并且对解决的问题进行归类，方便后续排除
8. 测试文件、临时文件统一放到test目录 
9. skills安装地址在~/.agent/skills或~/.kiro/skills或~/.claude/skills 
10. 所有关于时间的都是中国时区，默认东八区
11. 如果有什么需要重复实现的操作，并且需要遵循一定规律，可以沉淀下来，生成skills，放到项目的skills目录下，方便后续复用
12. 涉及到输入、输出的操作尽量进行封装，统一管理，并且考虑通用性
13. 把使用过的命令，进行归类放到commands文件夹下，方便学习，如果已经有了就不要重复生成，不过可以合并、完善命令的不同用法
14. 一个文件不要太长，尽量把功能拆分到多个文件
15. 涉及到数字计算的，尽量打印详细的日志，包括原始数据和计算后数据，方便排查问题
16. 常量统一放到一个常量文件中，方便调用
