## 📊 数据仓库分层

### ODS 层 (Operational Data Store)

**表名**: `ods_crypto_ticker_rt`

**用途**: 存储原始行情数据，不做任何转换

**表结构**:
```sql
CREATE TABLE IF NOT EXISTS crypto_dw.ods_crypto_ticker_rt (
    inst_id VARCHAR(50),           -- 交易对 (如 BTC-USDT)
    last DECIMAL(20, 8),           -- 最新成交价
    last_sz DECIMAL(20, 8),        -- 最新成交量
    ask_px DECIMAL(20, 8),         -- 卖一价
    ask_sz DECIMAL(20, 8),         -- 卖一量
    bid_px DECIMAL(20, 8),         -- 买一价
    bid_sz DECIMAL(20, 8),         -- 买一量
    open_24h DECIMAL(20, 8),       -- 24小时开盘价
    high_24h DECIMAL(20, 8),       -- 24小时最高价
    low_24h DECIMAL(20, 8),        -- 24小时最低价
    vol_ccy_24h DECIMAL(20, 8),    -- 24小时成交量 (计价货币)
    vol_24h DECIMAL(20, 8),        -- 24小时成交量 (交易货币)
    ts BIGINT,                     -- 时间戳 (毫秒)
    ts_ms VARCHAR(20),             -- 时间戳字符串
    trade_date DATE,               -- 交易日期
    create_time DATETIME           -- 创建时间
) DUPLICATE KEY(inst_id, ts)
PARTITION BY RANGE(trade_date) ()
DISTRIBUTED BY HASH(inst_id) BUCKETS 4;
```

**数据来源**: Kafka Topic `crypto-ticker`

**更新频率**: 实时 (约 100ms 一次)

### DWD 层 (Data Warehouse Detail)

**表名**: `dwd_crypto_ticker_detail`

**用途**: 清洗后的明细数据，包含衍生字段

**表结构**:
```sql
CREATE TABLE IF NOT EXISTS crypto_dw.dwd_crypto_ticker_detail (
    inst_id VARCHAR(50),           -- 交易对
    last_price DECIMAL(20, 8),     -- 最新成交价
    last_size DECIMAL(20, 8),      -- 最新成交量
    ask_price DECIMAL(20, 8),      -- 卖一价
    ask_size DECIMAL(20, 8),       -- 卖一量
    bid_price DECIMAL(20, 8),      -- 买一价
    bid_size DECIMAL(20, 8),       -- 买一量
    spread DECIMAL(20, 8),         -- 价差 (ask_price - bid_price)
    spread_rate DECIMAL(10, 6),    -- 价差率 (spread / last_price)
    open_24h DECIMAL(20, 8),       -- 24小时开盘价
    high_24h DECIMAL(20, 8),       -- 24小时最高价
    low_24h DECIMAL(20, 8),        -- 24小时最低价
    vol_ccy_24h DECIMAL(20, 8),    -- 24小时成交量 (计价货币)
    vol_24h DECIMAL(20, 8),        -- 24小时成交量 (交易货币)
    trade_date DATE,               -- 交易日期
    trade_time TIME,               -- 交易时间
    trade_hour INT,                -- 交易小时
    ts BIGINT,                     -- 时间戳 (毫秒)
    data_source VARCHAR(10),       -- 数据源 (OKX)
    create_time DATETIME           -- 创建时间
) DUPLICATE KEY(inst_id, ts)
PARTITION BY RANGE(trade_date) ()
DISTRIBUTED BY HASH(inst_id) BUCKETS 4;
```

**数据来源**: ODS 层 `ods_crypto_ticker_rt`

**数据转换**:
- 时间字段转换: `ts` → `trade_date`, `trade_time`, `trade_hour`
- 价格字段类型转换: `String` → `Decimal`
- 衍生字段计算: `spread`, `spread_rate`
- 添加数据源标识: `data_source = 'OKX'`

**更新频率**: 实时 (约 1 秒一次)

### DWS 层 (Data Warehouse Summary)

**表名**: `dws_crypto_kline_1min`

**用途**: 1分钟 K 线聚合数据

**表结构**:
```sql
CREATE TABLE IF NOT EXISTS crypto_dw.dws_crypto_kline_1min (
    inst_id VARCHAR(50),           -- 交易对
    trade_date DATE,               -- 交易日期
    kline_time DATETIME,           -- K线时间 (分钟级别)
    open_price DECIMAL(20, 8),     -- 开盘价
    high_price DECIMAL(20, 8),     -- 最高价
    low_price DECIMAL(20, 8),      -- 最低价
    close_price DECIMAL(20, 8),    -- 收盘价
    volume DECIMAL(20, 8),         -- 成交量
    amount DECIMAL(20, 8),         -- 成交额
    trade_count BIGINT,            -- 成交笔数
    create_time DATETIME           -- 创建时间
) AGGREGATE KEY(inst_id, trade_date, kline_time)
PARTITION BY RANGE(trade_date) ()
DISTRIBUTED BY HASH(inst_id) BUCKETS 4;
```

**数据来源**: DWD 层 `dwd_crypto_ticker_detail`

**聚合逻辑**:
- 窗口: 1分钟滚动窗口
- 开盘价: `FIRST_VALUE(last_price)`
- 最高价: `MAX(last_price)`
- 最低价: `MIN(last_price)`
- 收盘价: `LAST_VALUE(last_price)`
- 成交量: `SUM(last_size)`
- 成交额: `SUM(last_price * last_size)`
- 成交笔数: `COUNT(*)`

**更新频率**: 每分钟一次

**日志配置**:
- 滚动策略: 每 3 小时或 100MB

- 保留时间: 3 天
- 日志级别: INFO (应用), DEBUG (开发)

## 🔍 故障排查

### 常见问题

#### 1. Kafka 连接失败

**症状**: 数据采集器无法连接到 Kafka

**排查步骤**:
```bash
# 检查 Kafka 是否运行
docker ps --filter "name=kafka"

# 检查 Kafka 端口
netstat -an | grep 9092

# 测试 Kafka 连接
docker exec -it kafka kafka-broker-api-versions --bootstrap-server localhost:9092

# 查看 Kafka 日志
docker logs kafka
```

**解决方案**:
- 确保 Kafka 容器正在运行
- 检查端口映射是否正确
- 重启 Kafka: `docker restart kafka`

#### 2. Doris 写入失败

**症状**: Flink 作业报错 "Connection reset" 或 "unknowndatabases"

**排查步骤**:
```bash
# 检查 Doris 是否运行
docker ps --filter "name=doris"

# 检查 BE 节点状态
mysql -h 127.0.0.1 -P 9030 -u root -e "SHOW BACKENDS;"

# 测试 BE HTTP 端口
curl http://127.0.0.1:8040/api/health

# 查看 Doris 日志
docker logs doris-fe
docker logs doris-be
```

**解决方案**:
- 确保 Doris FE 和 BE 都在运行
- 检查 BE 节点是否 Alive
- 验证数据库和表是否存在
- 重启 Doris: `bash restart-doris.sh`

#### 3. Flink 作业失败

**症状**: Flink 作业启动失败或运行中崩溃

**排查步骤**:
```bash
# 查看应用日志
tail -100 logs/app/flink-app.log

# 查看错误日志
tail -100 logs/error/error.log

# 检查 Flink Web UI
open http://localhost:8081

# 查看 JVM 内存
jstat -gc <pid>
```

**解决方案**:
- 检查配置文件是否正确
- 确保所有依赖服务都在运行
- 增加 JVM 内存: `-Xmx2g`
- 查看详细错误信息并根据提示修复

#### 4. 监控数据缺失

**症状**: Grafana 面板显示 "No data"

**排查步骤**:
```bash
# 检查 Prometheus 是否运行
docker ps --filter "name=prometheus"

# 检查 Pushgateway 指标
curl http://localhost:9091/metrics | grep flink

# 检查 Prometheus 配置
curl http://localhost:9090/api/v1/targets

# 测试 PromQL 查询
curl 'http://localhost:9090/api/v1/query?query=flink_taskmanager_job_task_numRecordsInPerSecond'
```

**解决方案**:
- 确保 Flink 作业配置了 Metrics Reporter
- 检查 Pushgateway 是否接收到指标
- 验证 Prometheus 是否正确采集数据
- 重启监控服务: `bash restart-monitoring.sh`

#### 5. StreamPark 无法连接 Flink

**症状**: StreamPark 添加 Flink 集群时提示连接失败

**排查步骤**:
```bash
# 检查 Flink 是否运行
docker ps --filter "name=flink"

# 测试网络连通性
docker exec streampark ping -c 3 jobmanager

# 测试 Flink REST API
docker exec streampark curl -s http://jobmanager:8081/overview

# 检查 StreamPark 网络
docker inspect streampark --format '{{range $key, $value := .NetworkSettings.Networks}}{{$key}} {{end}}'
```

**解决方案**:
- 确保 StreamPark 连接到 flink-net 网络
- 使用容器名称 `jobmanager:8081` 而不是 `localhost:8081`
- 重启 StreamPark: `docker restart streampark`

## 🚀 生产环境部署

### 部署架构

**推荐配置**:
- Kafka: 3 节点集群 (Broker)
- Flink: Standalone 集群 (1 JobManager + 3 TaskManager)
- Doris: 1 FE + 3 BE
- Prometheus + Grafana: 独立部署
- StreamPark: 独立部署

### 部署清单

**硬件要求**:
- CPU: 8 核以上
- 内存: 16GB 以上
- 磁盘: 500GB 以上 (SSD 推荐)
- 网络: 千兆网卡

**软件要求**:
- 操作系统: Linux (CentOS 7+, Ubuntu 18.04+)
- Docker: 20.10+
- Docker Compose: 2.0+
- JDK: 8 或 11
- Maven: 3.6+

### 部署步骤

1. **准备环境**
   ```bash
   # 安装 Docker
   curl -fsSL https://get.docker.com | bash
   
   # 安装 Docker Compose
   sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
   sudo chmod +x /usr/local/bin/docker-compose
   
   # 安装 JDK
   sudo yum install java-1.8.0-openjdk-devel
   
   # 安装 Maven
   sudo yum install maven
   ```

2. **配置环境变量**
   ```bash
   # 编辑 .env 文件
   cp .env.example .env
   vim .env
   
   # 设置生产环境配置
   APP_ENV=prod
   KAFKA_BOOTSTRAP_SERVERS=kafka1:9092,kafka2:9092,kafka3:9092
   DORIS_FE_HTTP_URL=http://doris-fe:8030
   # ... 其他配置
   ```

3. **部署服务**
   ```bash
   # 启动 Kafka 集群
   docker-compose -f docker-compose-kafka-cluster.yml up -d
   
   # 启动 Doris 集群
   docker-compose -f docker-compose-doris-cluster.yml up -d
   
   # 启动 Flink 集群
   docker-compose -f docker-compose-flink-cluster.yml up -d
   
   # 启动监控服务
   docker-compose -f docker-compose-monitoring.yml up -d
   
   # 启动 StreamPark
   docker-compose -f docker-compose-streampark.yml up -d
   ```

4. **初始化数据库**
   ```bash
   # 创建 Doris 数据库和表
   mysql -h doris-fe -P 9030 -u root < sql/create_tables.sql
   
   # 初始化 StreamPark 数据库
   bash setup-streampark-mysql.sh
   ```

5. **部署应用**
   ```bash
   # 编译打包
   mvn clean package -DskipTests
   
   # 提交 Flink 作业到集群
   flink run -c com.crypto.dw.flink.FlinkODSJobDataStream target/crypto-dw-1.0-SNAPSHOT.jar
   flink run -c com.crypto.dw.flink.FlinkDWDJobSQL target/crypto-dw-1.0-SNAPSHOT.jar
   flink run -c com.crypto.dw.flink.FlinkDWSJob1MinSQL target/crypto-dw-1.0-SNAPSHOT.jar
   ```

6. **配置监控告警**
   ```bash
   # 配置 Prometheus 告警规则
   vim monitoring/prometheus/alerts.yml
   
   # 配置 Grafana 告警通知
   # 访问 Grafana → Alerting → Notification channels
   ```

### 高可用配置

**Kafka 高可用**:
- 3 个 Broker 节点
- 副本因子: 3
- Min ISR: 2

**Flink 高可用**:
- 使用 Zookeeper 或 Kubernetes HA
- 配置 Savepoint 自动恢复
- 启用 Checkpoint

**Doris 高可用**:
- 3 个 FE 节点 (1 Master + 2 Follower)
- 3 个 BE 节点
- 副本数: 3
