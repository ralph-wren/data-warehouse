# 套利作业白名单改黑名单

## 修改说明

将 `FlinkADSArbitrageJob` 的白名单逻辑改为黑名单逻辑。

## 修改内容

### 1. 逻辑变更

**原逻辑（白名单）：**
- 只处理白名单中的交易对
- 不在白名单中的交易对会被过滤掉
- Redis 中没有数据时使用默认白名单

**新逻辑（黑名单）：**
- 过滤掉黑名单中的交易对
- 不在黑名单中的交易对会被处理
- Redis 中没有数据时不过滤任何交易对（黑名单为空）

### 2. 代码修改

#### 类名修改
- `RedisWhitelistSource` → `RedisBlacklistSource`
- `WhitelistFilter` → `BlacklistFilter`

#### 状态描述符修改
```java
// 原代码
MapStateDescriptor<String, Boolean> whitelistStateDescriptor = new MapStateDescriptor<>(
    "whitelist-state",
    BasicTypeInfo.STRING_TYPE_INFO,
    BasicTypeInfo.BOOLEAN_TYPE_INFO
);

// 新代码
MapStateDescriptor<String, Boolean> blacklistStateDescriptor = new MapStateDescriptor<>(
    "blacklist-state",
    BasicTypeInfo.STRING_TYPE_INFO,
    BasicTypeInfo.BOOLEAN_TYPE_INFO
);
```

#### 过滤逻辑修改

**原逻辑（白名单）：**
```java
Boolean isWhitelisted = whitelistState.get(opportunity.symbol);

if (isWhitelisted != null && isWhitelisted) {
    // 在白名单中，输出数据
    out.collect(opportunity);
} else {
    // 不在白名单中，过滤掉
    logger.debug("过滤非白名单交易对: {}", opportunity.symbol);
}
```

**新逻辑（黑名单）：**
```java
Boolean isBlacklisted = blacklistState.get(opportunity.symbol);

if (isBlacklisted != null && isBlacklisted) {
    // 在黑名单中，过滤掉
    logger.debug("过滤黑名单交易对: {}", opportunity.symbol);
} else {
    // 不在黑名单中，输出数据
    out.collect(opportunity);
}
```

#### Redis Key 修改
```java
// 原代码
private static final String WHITELIST_KEY = "crypto:whitelist";

// 新代码
private static final String BLACKLIST_KEY = "crypto:blacklist";
```

#### 默认行为修改

**原逻辑（白名单）：**
```java
// 如果 Redis 中没有数据，返回默认白名单
if (whitelist == null || whitelist.isEmpty()) {
    logger.warn("Redis 中没有白名单数据（Key: {}），使用默认白名单", WHITELIST_KEY);
    whitelist = getDefaultWhitelist();
}

private Set<String> getDefaultWhitelist() {
    Set<String> defaultWhitelist = new HashSet<>();
    defaultWhitelist.add("BTC-USDT");
    defaultWhitelist.add("ETH-USDT");
    defaultWhitelist.add("SOL-USDT");
    defaultWhitelist.add("BNB-USDT");
    defaultWhitelist.add("XRP-USDT");
    return defaultWhitelist;
}
```

**新逻辑（黑名单）：**
```java
// 如果 Redis 中没有数据，返回空黑名单（不过滤任何交易对）
if (blacklist == null || blacklist.isEmpty()) {
    logger.info("Redis 中没有黑名单数据（Key: {}），不过滤任何交易对", BLACKLIST_KEY);
    return new HashSet<>();
}

// 移除了 getDefaultWhitelist() 方法
```

### 3. Redis 数据结构

#### 原数据结构（白名单）
```bash
# 添加白名单
SADD crypto:whitelist "BTC-USDT"
SADD crypto:whitelist "ETH-USDT"
SADD crypto:whitelist "SOL-USDT"

# 查看白名单
SMEMBERS crypto:whitelist

# 删除白名单
SREM crypto:whitelist "SOL-USDT"
```

#### 新数据结构（黑名单）
```bash
# 添加黑名单（需要过滤的交易对）
SADD crypto:blacklist "SHIB-USDT"
SADD crypto:blacklist "DOGE-USDT"
SADD crypto:blacklist "PEPE-USDT"

# 查看黑名单
SMEMBERS crypto:blacklist

# 删除黑名单
SREM crypto:blacklist "DOGE-USDT"

# 清空黑名单（处理所有交易对）
DEL crypto:blacklist
```

## 使用说明

### 1. 添加黑名单交易对

如果某些交易对不想参与套利计算（例如波动太大、流动性差等），可以添加到黑名单：

```bash
# 连接 Redis
redis-cli

# 添加到黑名单
SADD crypto:blacklist "SHIB-USDT"
SADD crypto:blacklist "DOGE-USDT"
SADD crypto:blacklist "PEPE-USDT"

# 验证
SMEMBERS crypto:blacklist
```

### 2. 移除黑名单交易对

如果某个交易对恢复正常，可以从黑名单中移除：

```bash
# 从黑名单移除
SREM crypto:blacklist "DOGE-USDT"

# 验证
SMEMBERS crypto:blacklist
```

### 3. 清空黑名单

如果想处理所有交易对，可以清空黑名单：

```bash
# 删除整个黑名单
DEL crypto:blacklist

# 或者逐个移除
SREM crypto:blacklist "SHIB-USDT"
SREM crypto:blacklist "PEPE-USDT"
```

## 行为对比

### 白名单模式（原逻辑）

| 场景 | 行为 |
|------|------|
| 交易对在白名单中 | ✅ 处理 |
| 交易对不在白名单中 | ❌ 过滤 |
| Redis 中没有白名单 | ✅ 使用默认白名单（BTC、ETH、SOL、BNB、XRP） |
| Redis 连接失败 | ✅ 使用默认白名单 |

### 黑名单模式（新逻辑）

| 场景 | 行为 |
|------|------|
| 交易对在黑名单中 | ❌ 过滤 |
| 交易对不在黑名单中 | ✅ 处理 |
| Redis 中没有黑名单 | ✅ 处理所有交易对 |
| Redis 连接失败 | ✅ 处理所有交易对 |

## 优势

1. **更灵活**：默认处理所有交易对，只需要配置需要排除的
2. **更简单**：不需要维护一个完整的白名单列表
3. **更安全**：Redis 故障时不会影响正常交易对的处理
4. **更直观**：黑名单通常比白名单更容易理解和维护

## 注意事项

1. 黑名单数据每 60 秒从 Redis 刷新一次
2. 修改黑名单后，最多需要等待 60 秒才会生效
3. 如果 Redis 中没有黑名单数据，作业会处理所有交易对
4. 建议在 Redis 中维护一个黑名单，避免处理不合适的交易对
