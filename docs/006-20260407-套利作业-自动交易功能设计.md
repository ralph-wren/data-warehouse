# 套利作业自动交易功能设计

## 功能概述

在 `FlinkADSArbitrageJob` 中添加自动交易功能，当发现套利机会时自动下单，套利机会消失时自动平仓。

## 业务逻辑

### 1. 套利策略

#### 策略 A：做多现货 + 做空合约
**触发条件**：合约价格 > 现货价格，价差率 > 阈值

**开仓操作**：
1. 买入现货（做多）
2. 卖出合约（做空）

**平仓操作**：
1. 卖出现货
2. 买入合约（平空仓）

#### 策略 B：做空现货 + 做多合约
**触发条件**：现货价格 > 合约价格，价差率 > 阈值

**开仓操作**：
1. 卖出现货（做空）
2. 买入合约（做多）

**平仓操作**：
1. 买入现货
2. 卖出合约（平多仓）

### 2. 持仓状态管理

使用 Flink ValueState 管理每个交易对的持仓状态：

```java
public class PositionState {
    String symbol;              // 交易对
    String direction;           // 套利方向：LONG_SPOT_SHORT_SWAP / SHORT_SPOT_LONG_SWAP
    BigDecimal amount;          // 持仓金额
    BigDecimal entrySpread;     // 开仓时的价差率
    long openTime;              // 开仓时间
    String spotOrderId;         // 现货订单ID
    String swapOrderId;         // 合约订单ID
    boolean isOpen;             // 是否持仓中
}
```

### 3. 风险控制

#### 3.1 开仓条件
- ✅ 价差率超过阈值（默认 0.5%）
- ✅ 不在黑名单中
- ✅ 当前没有该交易对的持仓
- ✅ 配置了 OKX API 密钥
- ✅ 单笔交易金额在限制范围内

#### 3.2 平仓条件
- ✅ 价差率回归到阈值以下
- ✅ 持仓时间超过最大持仓时间（防止长期占用资金）
- ✅ 亏损超过止损线（可选）

#### 3.3 资金管理
- 单笔交易金额：可配置（默认 100 USDT）
- 最大持仓数量：可配置（默认 5 个）
- 杠杆倍数：可配置（默认 1x，不使用杠杆）

## 技术实现

### 1. 核心组件

#### 1.1 OKXTradingService
交易服务，封装 OKX API 调用：
- `buySpot()` - 买入现货
- `sellSpot()` - 卖出现货
- `longSwap()` - 做多合约
- `shortSwap()` - 做空合约
- `closeLongSwap()` - 平多仓
- `closeShortSwap()` - 平空仓

#### 1.2 ArbitrageTrader
套利交易处理函数，使用 KeyedProcessFunction：
- 管理持仓状态（ValueState）
- 判断开仓/平仓条件
- 调用交易服务下单
- 记录交易日志

#### 1.3 PositionState
持仓状态数据结构

### 2. 数据流

```
套利机会流
    │
    ├─→ KeyedProcessFunction (按交易对分组)
    │       │
    │       ├─→ 读取持仓状态
    │       │
    │       ├─→ 判断是否开仓
    │       │   ├─ 是 → 调用交易服务 → 更新持仓状态
    │       │   └─ 否 → 跳过
    │       │
    │       ├─→ 判断是否平仓
    │       │   ├─ 是 → 调用交易服务 → 清空持仓状态
    │       │   └─ 否 → 跳过
    │       │
    │       └─→ 输出交易记录
    │
    └─→ Doris Sink (记录交易历史)
```

### 3. 配置项

在 `application-dev.yml` 中添加：

```yaml
# 套利交易配置
arbitrage:
  trading:
    enabled: true                    # 是否启用自动交易
    trade-amount: 100                # 单笔交易金额（USDT）
    max-positions: 5                 # 最大持仓数量
    leverage: 1                      # 杠杆倍数
    open-threshold: 0.005            # 开仓阈值（0.5%）
    close-threshold: 0.002           # 平仓阈值（0.2%）
    max-hold-time-minutes: 60        # 最大持仓时间（分钟）
    stop-loss-rate: -0.02            # 止损率（-2%）

# OKX API 配置
okx:
  api:
    key: ${OKX_API_KEY}
    secret: ${OKX_SECRET_KEY}
    passphrase: ${OKX_PASSPHRASE}
    simulated: true                  # 是否模拟交易
```

## 使用示例

### 1. 配置 API 密钥

```bash
# 设置环境变量
export OKX_API_KEY="your-api-key"
export OKX_SECRET_KEY="your-secret-key"
export OKX_PASSPHRASE="your-passphrase"
```

### 2. 启用自动交易

修改 `application-dev.yml`：

```yaml
arbitrage:
  trading:
    enabled: true
    trade-amount: 100
    max-positions: 5
```

### 3. 启动作业

```bash
cd data-warehouse
bash run-flink-ads-arbitrage.sh
```

### 4. 监控交易

```bash
# 查看交易日志
tail -f logs/flink-ads-arbitrage.log | grep "交易"

# 查看持仓状态
redis-cli HGETALL arbitrage:positions

# 查看交易历史
mysql -h 127.0.0.1 -P 9030 -u root -e "SELECT * FROM crypto_dw.ads_arbitrage_trades ORDER BY timestamp DESC LIMIT 10"
```

## 注意事项

### 1. 风险提示
- ⚠️ 自动交易有风险，建议先在模拟环境测试
- ⚠️ 合理设置交易金额和止损线
- ⚠️ 密切监控持仓状态
- ⚠️ 确保账户有足够的资金

### 2. 技术限制
- OKX API 有频率限制（每秒最多 20 次请求）
- 市价单可能有滑点
- 网络延迟可能影响套利效果
- Flink 状态恢复时需要重新加载持仓

### 3. 最佳实践
- 从小金额开始测试
- 设置合理的开仓和平仓阈值
- 定期检查持仓状态
- 保留足够的日志用于分析
