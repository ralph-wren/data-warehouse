## 改进的交易流程

### 原流程(不可靠)
```
发现套利机会 → 下单 → 假设成交 → 保存持仓 → 监控 → 平仓
```

**问题**:
- 不确定订单是否真的成交
- 可能只有一个订单成交,另一个失败
- 使用预估价格而非实际成交价格

### 新流程(可靠)
```
发现套利机会
  ↓
同时下单(现货 + 合约)
  ↓
注册待匹配订单
  ↓
WebSocket 接收订单更新
  ↓
现货订单成交 ✓
  ↓
合约订单成交 ✓
  ↓
订单匹配成功 → 确认开仓 → 保存持仓状态
  ↓
监控套利机会
  ↓
价差回归 → 同时平仓(现货 + 合约)
```

**优势**:
- ✅ 等待订单实际成交
- ✅ 确保现货和合约都成交才确认开仓
- ✅ 使用实际成交价格计算价差
- ✅ 同时平仓减少风险

DataStream<PositionState> confirmedPositionStream = orderStream
    .keyBy(message -> {
        // 从订单消息中提取交易对作为 key
        try {
            JsonNode rootNode = OBJECT_MAPPER.readTree(message);
            if (rootNode.has("data") && rootNode.get("data").isArray()) {
                JsonNode order = rootNode.get("data").get(0);
                String instId = order.get("instId").asText();
                return instId.replace("-SWAP", "");
            }
        } catch (Exception e) {
            logger.error("解析订单消息失败: {}", e.getMessage());
        }
        return "UNKNOWN";
    })
    .process(new OrderMatchProcessor())
    .name("Order Match Processor");

## 注意事项

1. **默认关闭自动交易**: `arbitrage.trading.enabled` 默认为 `false`,需要手动开启
2. **模拟交易模式**: `okx.api.simulated` 默认为 `true`,建议先在模拟环境测试
3. **API 密钥安全**: 不要将 API 密钥提交到代码仓库,使用环境变量
4. **订单超时**: 目前未实现超时处理,需要后续添加
5. **部分成交**: 目前未处理部分成交情况,需要后续添加
6. **网络延迟**: WebSocket 可能有延迟,需要容忍一定的时间差
7. **状态恢复**: Flink 重启后需要重新订阅 WebSocket
