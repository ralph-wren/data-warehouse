### 步骤 1: 创建数据库表

在 Doris 中执行:

```sql
-- 交易记录表
CREATE TABLE IF NOT EXISTS crypto_dw.ads_arbitrage_trades (
    symbol VARCHAR(50) COMMENT '交易对',
    action VARCHAR(20) COMMENT '操作类型: OPEN_PENDING/OPEN_CONFIRMED/CLOSE',
    direction VARCHAR(50) COMMENT '套利方向',
    amount DECIMAL(20, 8) COMMENT '交易金额',
    spread_rate DECIMAL(10, 6) COMMENT '价差率(%)',
    spot_price DECIMAL(20, 8) COMMENT '现货价格',
    swap_price DECIMAL(20, 8) COMMENT '合约价格',
    profit DECIMAL(20, 8) COMMENT '盈亏(USDT)',
    close_reason VARCHAR(100) COMMENT '平仓原因',
    hold_time_ms BIGINT COMMENT '持仓时间(毫秒)',
    timestamp BIGINT COMMENT '时间戳'
)
DUPLICATE KEY(symbol, timestamp)
DISTRIBUTED BY HASH(symbol) BUCKETS 10
PROPERTIES (
    "replication_num" = "1"
);
```

### 步骤 2: 更新配置文件

在 `data-warehouse/src/main/resources/config/application-dev.yml` 中添加:

```yaml
# 套利交易配置
arbitrage:
  trading:
    enabled: false                   # 是否启用自动交易(默认关闭,测试通过后再开启)
    trade-amount: 100                # 单笔交易金额(USDT)
    max-positions: 5                 # 最大持仓数量
    open-threshold: 0.005            # 开仓阈值(0.5%)
    close-threshold: 0.002           # 平仓阈值(0.2%)
    order-timeout-seconds: 30        # 订单超时时间(秒)

# OKX API 配置
okx:
  api:
    key: ${OKX_API_KEY}              # 从环境变量读取
    secret: ${OKX_SECRET_KEY}        # 从环境变量读取
    passphrase: ${OKX_PASSPHRASE}    # 从环境变量读取
    simulated: true                  # 是否模拟交易(默认开启,建议先测试)
```

## 完整集成(后续步骤)

完整的集成需要实现 `ArbitrageTraderV2` 类,包含:

1. **处理套利机会**:
   - 检查是否有持仓
   - 检查价差率是否超过开仓阈值
   - 下单(现货 + 合约)
   - 注册待匹配订单

2. **处理确认的持仓**:
   - 接收 OrderMatchProcessor 输出的持仓状态
   - 保存到 Flink 状态
   - 输出开仓记录

3. **监控持仓并平仓**:
   - 接收新的套利机会数据
   - 检查价差率是否回归
   - 同时平仓(现货 + 合约)
   - 计算盈亏
   - 输出平仓记录

详细实现代码请参考 `008-20260407-套利作业-订单流集成方案.md` 文档。

## 常见问题

### Q1: WebSocket 连接失败
**A**: 检查网络连接,确保能够访问 `wss://ws.okx.com:8443`

### Q2: 登录认证失败
**A**: 检查 API Key、Secret、Passphrase 是否正确,检查签名算法是否正确

### Q3: 订阅失败
**A**: 检查订阅消息格式是否正确,检查 instType 是否支持

### Q4: 收不到订单消息
**A**: 确保有实际的订单成交,可以手动下单测试

### Q5: 订单匹配失败
**A**: 检查 key 提取逻辑是否正确,检查订单消息格式是否正确
