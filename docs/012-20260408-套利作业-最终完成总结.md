## 🎯 核心改进

### 原流程(不可靠)
```
发现套利机会 → 下单 → 假设成交 → 保存持仓 → 监控 → 平仓
```

**问题**:
- ❌ 不确定订单是否真的成交
- ❌ 可能只有一个订单成交
- ❌ 使用预估价格而非实际成交价格

### 新流程(可靠)
```
发现套利机会
  ↓
同时下单(现货 + 合约)
  ↓
注册待匹配订单
  ↓
WebSocket 订阅订单状态
  ↓
等待成交确认
  ↓
现货订单成交 ✓
  ↓
合约订单成交 ✓
  ↓
订单匹配成功 → 确认开仓 → 保存持仓状态
  ↓
监控套利机会
  ↓
价差回归 / 超时 / 止损 → 同时平仓(现货 + 合约)
```

**优势**:
- ✅ 等待订单实际成交
- ✅ 确保现货和合约都成交才确认开仓
- ✅ 使用实际成交价格计算价差
- ✅ 同时平仓减少风险
- ✅ 支持超时平仓
- ✅ 支持止损平仓

## 🔧 故障排查

### Q1: WebSocket 连接失败
**原因**: 网络问题或 API 配置错误

**解决**:
1. 检查网络连接
2. 确认能够访问 `wss://ws.okx.com:8443`
3. 检查 API Key、Secret、Passphrase 是否正确
4. 检查环境变量是否正确设置

### Q2: 登录认证失败
**原因**: API 密钥错误或签名算法错误

**解决**:
1. 确认 API Key、Secret、Passphrase 正确
2. 检查 API 权限是否包含交易权限
3. 检查签名算法实现

### Q3: 收不到订单消息
**原因**: 没有实际订单或订阅失败

**解决**:
1. 确保有实际的订单成交
2. 手动下单测试
3. 检查订阅的 instType 是否正确
4. 查看日志确认订阅是否成功

### Q4: 订单匹配失败
**原因**: Key 提取逻辑错误或订单消息格式错误

**解决**:
1. 检查 key 提取逻辑
2. 打印订单消息查看格式
3. 确认现货和合约订单的 instId 格式

### Q5: 下单失败
**原因**: API 权限不足或参数错误

**解决**:
1. 检查 API 权限
2. 检查交易金额是否符合最小限制
3. 检查账户余额是否充足
4. 查看 API 返回的错误信息
