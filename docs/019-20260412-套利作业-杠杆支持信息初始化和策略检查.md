# 套利作业 - 杠杆支持信息初始化和策略检查

**日期**: 2026-04-12  
**功能**: 套利作业  
**问题**: 添加杠杆支持信息初始化逻辑和开仓前策略检查

---

## 一、问题背景

套利作业需要根据币种是否支持杠杆来决定是否可以执行特定策略：

- **策略A**：现货卖出 + 合约买入（需要杠杆借币，只有杠杆倍数 > 0 的币种才支持）
- **策略B**：现货买入 + 合约卖出（不需要杠杆，所有币种都支持）

需要实现：
1. 启动时从 OKX API 查询所有现货币对的杠杆支持信息
2. 将杠杆信息存储到 Redis，供后续快速查询
3. 开仓前检查币种是否支持对应的策略

---

## 二、解决方案

### 2.1 架构设计

```
FlinkTickerCollectorJob (启动时)
    ↓
1. 获取所有现货交易对
    ↓
2. 遍历查询杠杆信息
    ↓
3. 存储到 Redis (okx:margin:support)
    ↓
FlinkADSArbitrageJob (运行时)
    ↓
4. MarginSupportCache 从 Redis 加载到内存
    ↓
5. TradingDecisionProcessor 开仓前检查策略支持
```

### 2.2 Redis 数据结构

```
Key: okx:margin:support
Type: Hash
Field: symbol (如 "BTC-USDT")
Value: leverage (杠杆倍数，如 "10" 或 "0" 表示不支持杠杆)
```

**示例数据**:
```
BTC-USDT -> "10"   (支持10倍杠杆)
ETH-USDT -> "10"   (支持10倍杠杆)
DOGE-USDT -> "0"   (不支持杠杆)
```

## 三、代码实现

### 3.1 FlinkTickerCollectorJob - 初始化杠杆信息

**文件**: `src/main/java/com/crypto/dw/flink/FlinkTickerCollectorJob.java`

**新增方法**: `initializeMarginSupport(ConfigLoader config)`

**功能**:
1. 通过 `OKXRestApiSource.fetchSpotPrices()` 获取所有现货交易对
2. 遍历所有交易对，调用 `OKXTradingService.getMarginLeverage()` 查询杠杆信息
3. 将结果存储到 Redis `okx:margin:support`
4. 支持杠杆的存储杠杆倍数（如 "10"），不支持的存储 "0"
5. 每次查询间隔 100ms，避免请求过快
6. 输出详细的统计信息

**调用位置**: 在 `main` 方法中，计算价差之前调用

```java
// 在计算价差之前，初始化杠杆支持信息
initializeMarginSupport(config);
```

**关键代码**:
```java
private static void initializeMarginSupport(ConfigLoader config) {
    try {
        logger.info("🔄 开始初始化杠杆支持信息...");
        
        // 1. 获取所有现货交易对
        OKXRestApiSource apiSource = new OKXRestApiSource(config, 0, 10);
        Map<String, BigDecimal> spotPrices = apiSource.fetchSpotPrices();
        
        // 2. 提取币种符号列表
        List<String> symbols = new ArrayList<>(spotPrices.keySet());
        
        // 3. 创建 OKXTradingService 和 Redis 连接
        OKXTradingService tradingService = new OKXTradingService(config, null);
        RedisConnectionManager redisManager = new RedisConnectionManager(config);
        
        // 4. 清空旧数据
        redisManager.del("okx:margin:support");
        
        // 5. 查询每个币种的杠杆信息并存储
        int successCount = 0;
        int noMarginCount = 0;
        int failCount = 0;
        
        for (String symbol : symbols) {
            try {
                String instId = symbol + "-USDT";
                String leverage = tradingService.getMarginLeverage(instId);
                
                if (leverage != null && !leverage.isEmpty()) {
                    redisManager.hset("okx:margin:support", instId, leverage);
                    successCount++;
                } else {
                    redisManager.hset("okx:margin:support", instId, "0");
                    noMarginCount++;
                }
                
                Thread.sleep(100);  // 避免请求过快
                
            } catch (Exception e) {
                failCount++;
                logger.warn("⚠️ 查询 {} 杠杆信息失败: {}", symbol, e.getMessage());
            }
        }
        
        // 6. 输出统计信息
        logger.info("✅ 杠杆支持信息初始化完成:");
        logger.info("   - 总数: {} 个币种", symbols.size());
        logger.info("   - 支持杠杆: {} 个", successCount);
        logger.info("   - 不支持杠杆: {} 个", noMarginCount);
        logger.info("   - 查询失败: {} 个", failCount);
        
    } catch (Exception e) {
        logger.error("❌ 初始化杠杆支持信息失败: {}", e.getMessage(), e);
        logger.warn("⚠️ 杠杆信息初始化失败不影响主流程，作业将继续运行");
    }
}
```

### 3.2 MarginSupportCache - 从 Redis 加载数据

**文件**: `src/main/java/com/crypto/dw/trading/MarginSupportCache.java`

**功能**:
- 启动时从 Redis 加载所有杠杆支持信息到内存
- 提供快速查询接口（O(1) 时间复杂度）
- 线程安全，支持并发访问

**关键方法**:
```java
// 检查是否支持策略A（现货卖出 + 合约买入，需要杠杆）
public boolean supportsStrategyA(String symbol) {
    String leverage = cache.get(symbol);
    if (leverage == null || leverage.isEmpty() || "0".equals(leverage)) {
        return false;
    }
    try {
        int lev = Integer.parseInt(leverage);
        return lev > 0;
    } catch (NumberFormatException e) {
        return false;
    }
}

// 检查是否支持策略B（现货买入 + 合约卖出，不需要杠杆）
public boolean supportsStrategyB(String symbol) {
    // 策略B不需要杠杆，所有币种都支持
    return cache.containsKey(symbol);
}

// 获取杠杆倍数
public int getLeverage(String symbol) {
    String leverage = cache.get(symbol);
    if (leverage == null || leverage.isEmpty()) {
        return 0;
    }
    try {
        return Integer.parseInt(leverage);
    } catch (NumberFormatException e) {
        return 0;
    }
}
```

### 3.3 TradingDecisionProcessor - 开仓前策略检查

**文件**: `src/main/java/com/crypto/dw/processor/TradingDecisionProcessor.java`

**修改方法**: `openPosition(ArbitrageOpportunity opp, Collector<TradeRecord> out)`

**检查逻辑**:
```java
// ⭐ 第一步：检查策略支持（在检查持仓数量之前）
String standardSymbol = opp.symbol + "-USDT";

// 判断套利方向对应的策略
boolean isStrategyA = opp.arbitrageDirection.contains("做空现货");  // 策略A：现货卖出 + 合约买入（需要杠杆）
boolean isStrategyB = opp.arbitrageDirection.contains("做多现货");  // 策略B：现货买入 + 合约卖出（不需要杠杆）

// 检查策略A支持
if (isStrategyA && !marginCache.supportsStrategyA(standardSymbol)) {
    logger.warn("⚠️ 币种 {} 不支持策略A（现货卖出+合约买入，需要杠杆借币），拒绝开仓", opp.symbol);
    
    // 写入拒绝日志
    String timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
    String rejectLog = String.format("%s,%s,OPEN,REJECTED,N/A,N/A,N/A,N/A,REJECTED,不支持策略A（需要杠杆借币）",
        timestamp, opp.symbol);
    orderDetailWriter.write(rejectLog);
    return;
}

// 检查策略B支持
if (isStrategyB && !marginCache.supportsStrategyB(standardSymbol)) {
    logger.warn("⚠️ 币种 {} 不支持策略B（现货买入+合约卖出），拒绝开仓", opp.symbol);
    
    // 写入拒绝日志
    String timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
    String rejectLog = String.format("%s,%s,OPEN,REJECTED,N/A,N/A,N/A,N/A,REJECTED,不支持策略B",
        timestamp, opp.symbol);
    orderDetailWriter.write(rejectLog);
    return;
}

// 记录策略信息
if (isStrategyA) {
    int leverage = marginCache.getLeverage(standardSymbol);
    logger.info("✅ 策略A检查通过: {} | 杠杆倍数: {}x", opp.symbol, leverage);
} else if (isStrategyB) {
    logger.info("✅ 策略B检查通过: {} | 不需要杠杆", opp.symbol);
}

// ⭐ 第二步：检查全局持仓数量
// ... 后续逻辑
```

## 四、工作流程

### 4.1 启动流程

```
1. 启动 FlinkTickerCollectorJob
   ↓
2. 调用 initializeMarginSupport()
   ↓
3. 查询所有现货币对的杠杆信息
   ↓
4. 存储到 Redis (okx:margin:support)
   ↓
5. 继续执行主流程（计算价差、订阅 WebSocket）
```

### 4.2 套利作业流程

```
1. 启动 FlinkADSArbitrageJob
   ↓
2. MarginSupportCache 从 Redis 加载数据到内存
   ↓
3. 检测到套利机会
   ↓
4. TradingDecisionProcessor.openPosition()
   ↓
5. 检查策略支持
   - 策略A：检查 supportsStrategyA()
   - 策略B：检查 supportsStrategyB()
   ↓
6. 检查通过 → 继续下单
   检查失败 → 拒绝开仓，记录日志
```

## 六、性能优化

### 6.1 内存缓存

- 使用 `ConcurrentHashMap` 存储杠杆信息
- 查询时间复杂度 O(1)
- 避免频繁访问 Redis

### 6.2 批量初始化

- 启动时一次性查询所有币对
- 避免运行时动态查询
- 减少 API 请求次数

### 6.3 请求限流

- 每次查询间隔 100ms
- 避免触发 OKX API 限流
- 保证初始化成功率

## 七、错误处理

### 7.1 初始化失败

- 不影响主流程，作业继续运行

### 7.2 策略检查失败

- 拒绝开仓，记录拒绝日志
