# 套利作业 - 杠杆支持信息查询时自动存储和策略检查

## 问题描述

之前的实现中,杠杆支持信息的查询和存储逻辑分散在多个地方:
1. `FlinkTickerCollectorJob.initializeMarginSupport()` 方法负责初始化杠杆信息到 Redis
2. `OKXRestApiSource.calculateSpreads()` 方法中有杠杆过滤逻辑
3. `TradingDecisionProcessor.openPosition()` 方法中需要检查策略支持

这种设计存在以下问题:
- 代码重复,维护困难
- 杠杆过滤逻辑分散,不统一
- 初始化方法和查询方法分离,容易遗漏

## 解决方案

### 1. 修改 OKXTradingService.getMarginLeverage() 方法

在查询杠杆信息时自动存储到 Redis,无需单独的初始化方法。

**修改位置**: `src/main/java/com/crypto/dw/trading/OKXTradingService.java`

**关键代码**:
```java
public String getMarginLeverage(String symbol) {
    // ... 查询 OKX API ...
    
    // 自动存储到 Redis
    if (redisManager != null) {
        try {
            String instId = symbol + "-USDT";
            redisManager.hset("okx:margin:support", instId, leverageValue);
            logger.debug("✓ 已存储杠杆信息到 Redis: {} -> {}", instId, leverageValue);
        } catch (Exception e) {
            logger.warn("⚠️ 存储杠杆信息到 Redis 失败: {}", e.getMessage());
        }
    }
    
    return leverageValue;
}
```

**存储格式**:
- Redis Hash: `okx:margin:support`
- Key: 交易对名称 (如 `BTC-USDT`)
- Value: 杠杆倍数 (如 `10`) 或 `0` (不支持杠杆)

### 2. 删除 FlinkTickerCollectorJob.initializeMarginSupport() 方法

由于 `getMarginLeverage()` 方法已经自动存储,不再需要单独的初始化方法。

**删除位置**: `src/main/java/com/crypto/dw/flink/FlinkTickerCollectorJob.java`

**删除内容**:
- `initializeMarginSupport()` 方法定义
- `main()` 方法中对 `initializeMarginSupport()` 的调用

### 3. 移除 OKXRestApiSource 中的杠杆过滤逻辑

将杠杆过滤逻辑统一到 `FlinkTickerCollectorJob.refreshCache()` 方法中处理。

**修改位置**: `src/main/java/com/crypto/dw/flink/source/OKXRestApiSource.java`

**删除内容**:
- 成员变量 `filterMarginOnly` 和 `marginCache`
- 构造函数中的 `filterMarginOnly` 参数
- `open()` 方法中的 `marginCache` 初始化逻辑
- `calculateSpreads()` 方法中的杠杆过滤逻辑 (第 301-310 行)

**修改后的 calculateSpreads() 方法**:
```java
public List<PriceSpreadInfo> calculateSpreads(
        Map<String, BigDecimal> spotPrices, 
        Map<String, BigDecimal> swapPrices) {
    
    List<PriceSpreadInfo> spreadList = new ArrayList<>();
    
    // 遍历所有现货价格
    for (Map.Entry<String, BigDecimal> entry : spotPrices.entrySet()) {
        String symbol = entry.getKey();
        BigDecimal spotPrice = entry.getValue();
        
        // 检查是否有对应的合约价格
        if (!swapPrices.containsKey(symbol)) {
            continue;
        }
        
        BigDecimal swapPrice = swapPrices.get(symbol);
        
        // 计算价差 (合约价格 - 现货价格)
        BigDecimal spread = swapPrice.subtract(spotPrice);
        
        // 计算价差率 (价差 / 现货价格)
        BigDecimal spreadRate = spread.divide(spotPrice, 6, BigDecimal.ROUND_HALF_UP);
        
        // 创建价差信息对象
        PriceSpreadInfo info = new PriceSpreadInfo();
        info.symbol = symbol;
        info.spotPrice = spotPrice;
        info.swapPrice = swapPrice;
        info.spread = spread;
        info.spreadRate = spreadRate;
        info.timestamp = System.currentTimeMillis();
        
        spreadList.add(info);
    }
    
    // 按价差率绝对值降序排序
    spreadList.sort((a, b) -> b.spreadRate.abs().compareTo(a.spreadRate.abs()));
    
    return spreadList;
}
```

### 4. 在 TradingDecisionProcessor 中添加策略支持检查

在开仓前检查币对是否支持当前套利方向的策略。

**修改位置**: `src/main/java/com/crypto/dw/processor/TradingDecisionProcessor.java`

**关键逻辑**:
```java
// 1. 策略支持检查
boolean strategySupported = false;
if (decision.getArbitrageDirection() == ArbitrageDirection.SPOT_SELL_SWAP_BUY) {
    // 策略A: 现货卖出 + 合约买入 (需要杠杆借币)
    strategySupported = marginCache.supportsStrategyA(decision.getSymbol());
    if (!strategySupported) {
        logger.warn("⚠️ 拒绝开仓: {} 不支持策略A (现货卖出+合约买入,需要杠杆)", decision.getSymbol());
        return;
    }
    logger.info("✓ 策略检查通过: {} 支持策略A (现货卖出+合约买入)", decision.getSymbol());
} else {
    // 策略B: 现货买入 + 合约卖出 (不需要杠杆,所有币种都支持)
    strategySupported = marginCache.supportsStrategyB(decision.getSymbol());
    logger.info("✓ 策略检查通过: {} 支持策略B (现货买入+合约卖出)", decision.getSymbol());
}

// 2. 全局持仓数量检查

```

## 策略说明

### 策略A: 现货卖出 + 合约买入
- 需要杠杆借币支持
- 只有支持杠杆的币种才能使用
- Redis 中 `okx:margin:support` 的 value 不为 "0"

### 策略B: 现货买入 + 合约卖出
- 不需要杠杆
- 所有币种都支持
- 无需检查 Redis 中的杠杆信息

## 数据流转

```
启动时:
1. FlinkTickerCollectorJob.main()
   └─> startCacheRefreshTask()
       └─> refreshCache()
           └─> OKXTradingService.getMarginLeverage()
               └─> 查询 OKX API
               └─> 自动存储到 Redis (okx:margin:support)

定时刷新:
1. ScheduledExecutorService (每 5 分钟)
   └─> refreshCache()
       └─> OKXTradingService.getMarginLeverage()
           └─> 查询 OKX API
           └─> 自动存储到 Redis (okx:margin:support)

开仓检查:
1. TradingDecisionProcessor.openPosition()
   └─> 策略支持检查
       ├─> 策略A: marginCache.supportsStrategyA()
       │   └─> 检查 Redis (okx:margin:support) value != "0"
       └─> 策略B: marginCache.supportsStrategyB()
           └─> 始终返回 true (所有币种都支持)
```

## 优化效果

1. **代码简化**: 删除了重复的初始化方法,减少了约 50 行代码
2. **逻辑统一**: 杠杆信息的查询和存储在同一个方法中完成
3. **维护性提升**: 只需要维护一个方法,降低了出错概率
4. **自动化**: 查询时自动存储,无需手动调用初始化方法

## 注意事项

1. Redis 中 `okx:margin:support` 的 value 格式为实际杠杆倍数 (如 "10") 或 "0" (不支持杠杆)
2. 策略A (现货卖出+合约买入) 需要杠杆支持,策略B (现货买入+合约卖出) 不需要
3. 杠杆信息会在启动时和定时任务中自动刷新,无需手动初始化
4. 开仓前会自动检查策略支持,不支持的策略会被拒绝
