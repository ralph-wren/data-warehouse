# 套利作业 - 套利阈值从配置文件读取

## 问题描述

`ArbitrageCalculator` 类中的套利阈值 `ARBITRAGE_THRESHOLD` 是硬编码的常量,无法灵活调整。

**原代码**:
```java
// 套利阈值：价差超过 0.5% 才认为有套利机会
private static final BigDecimal ARBITRAGE_THRESHOLD = new BigDecimal("0.005");
```

**问题**:
- 阈值硬编码,无法根据市场情况灵活调整
- 修改阈值需要重新编译代码
- 不同环境(开发/测试/生产)无法使用不同的阈值

## 解决方案

修改 `ArbitrageCalculator` 类,从配置文件中读取套利阈值。

### 1. 修改 ArbitrageCalculator 类

**修改位置**: `src/main/java/com/crypto/dw/flink/processor/ArbitrageCalculator.java`

**关键修改**:

```java
// 修改前：硬编码常量
private static final BigDecimal ARBITRAGE_THRESHOLD = new BigDecimal("0.005");

// 修改后：从配置文件读取
private final BigDecimal arbitrageThreshold;

/**
 * 构造函数
 * 
 * @param config 配置加载器
 */
public ArbitrageCalculator(ConfigLoader config) {
    // 从配置文件读取套利阈值，默认 0.005 (0.5%)
    String thresholdStr = config.getString("arbitrage.trading.open-threshold", "0.005");
    this.arbitrageThreshold = new BigDecimal(thresholdStr);
    logger.info("✓ 套利阈值配置: {}% (从配置文件读取)", 
        arbitrageThreshold.multiply(new BigDecimal("100")));
}
```

**使用位置修改**:
```java
// 修改前
if (spreadRate.compareTo(ARBITRAGE_THRESHOLD) > 0) {

// 修改后
if (spreadRate.compareTo(arbitrageThreshold) > 0) {
```

### 2. 修改 FlinkADSArbitrageJob 类

**修改位置**: `src/main/java/com/crypto/dw/flink/FlinkADSArbitrageJob.java`

**关键修改**:

```java
// 修改前：无参构造
.process(new ArbitrageCalculator())

// 修改后：传入配置
.process(new ArbitrageCalculator(config))  // 传入配置,从配置文件读取套利阈值
```

### 3. 配置文件

**配置位置**: `src/main/resources/config/application-dev.yml`

**配置项**:
```yaml
arbitrage:
  trading:
    open-threshold: 0.005  # 开仓阈值(0.5%)
    close-threshold: 0.002  # 平仓阈值(0.2%)
```

**说明**:
- `open-threshold`: 开仓阈值,价差率超过此值才认为有套利机会
- `close-threshold`: 平仓阈值,价差率低于此值时平仓
- 默认值: 0.005 (0.5%)
- 单位: 小数形式 (0.005 = 0.5%)

## 技术要点

### 1. 配置读取

使用 `ConfigLoader.getString()` 方法读取配置:
```java
String thresholdStr = config.getString("arbitrage.trading.open-threshold", "0.005");
this.arbitrageThreshold = new BigDecimal(thresholdStr);
```

**为什么不使用 `getBigDecimal()`?**
- `ConfigLoader` 类没有 `getBigDecimal()` 方法
- 使用 `getString()` 读取字符串,然后转换为 `BigDecimal`
- 提供默认值 "0.005",避免配置缺失时出错

### 2. 构造函数注入

通过构造函数注入 `ConfigLoader`:
```java
public ArbitrageCalculator(ConfigLoader config) {
    // 读取配置
}
```

**优点**:
- 依赖注入,便于测试
- 配置在对象创建时加载,避免运行时读取
- 线程安全,配置不可变

### 3. 日志输出

在构造函数中输出配置信息:
```java
logger.info("✓ 套利阈值配置: {}% (从配置文件读取)", 
    arbitrageThreshold.multiply(new BigDecimal("100")));
```

**输出示例**:
```
✓ 套利阈值配置: 0.5% (从配置文件读取)
```

## 注意事项

1. 阈值单位是小数形式 (0.005 = 0.5%)
2. 阈值越低,套利机会越多,但收益可能较低
3. 阈值越高,套利机会越少,但收益可能较高
4. 建议根据历史数据和回测结果调整阈值
5. 修改配置后需要重启作业才能生效
