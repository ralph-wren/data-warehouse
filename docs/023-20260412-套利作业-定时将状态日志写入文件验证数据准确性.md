## 数据验证要点

### 1. 持仓状态验证
- 检查 `position_status` 是否正确
- 验证 `hold_time_seconds` 是否递增
- 确认 `direction` 与开仓方向一致

### 2. 价格验证
- 对比 `entry_spot_price` 和 `current_spot_price` 的变化
- 对比 `entry_swap_price` 和 `current_swap_price` 的变化
- 验证 `entry_spread_rate` 和 `current_spread_rate` 的计算

### 3. 成本验证
- 验证 `spot_cost = spot_fill_price × spot_fill_quantity`
- 验证 `futures_cost = futures_fill_price × futures_fill_quantity`
- 验证 `total_cost = spot_cost + futures_cost`
- 验证 `total_expense = total_cost + total_fee`

### 4. 利润验证
- 验证 `unrealized_profit` 的计算逻辑
- 验证 `profit_rate = unrealized_profit / amount_usdt × 100`
- 检查利润是否随价差变化而变化

### 5. 配置验证
- 确认 `trading_enabled` 状态
- 验证 `open_threshold` 和 `close_threshold` 配置
- 检查 `max_hold_time_minutes` 和 `max_loss` 配置

## 使用方法

### 1. 启动作业
状态日志会自动写入 `./logs/trading/status_log_YYYYMMDD.csv`

### 2. 查看日志
```bash
# 查看最新的状态日志
tail -f ./logs/trading/status_log_$(date +%Y%m%d).csv

# 查看特定币对的状态
grep "BTC" ./logs/trading/status_log_$(date +%Y%m%d).csv

# 统计持仓时间
awk -F',' '{print $5}' ./logs/trading/status_log_$(date +%Y%m%d).csv | sort -n
```

### 3. 数据分析
可以使用 Excel、Python pandas 等工具分析 CSV 数据:

```python
import pandas as pd

# 读取状态日志
df = pd.read_csv('./logs/trading/status_log_20260412.csv')

# 查看持仓币对
print(df[df['position_status'] == 'OPEN']['symbol'].unique())

# 计算平均利润率
print(df['profit_rate'].mean())

# 绘制利润率变化曲线
df['profit_rate'].plot()
```

## 输出频率

- 状态日志每 30 秒输出一次（由 `STATUS_LOG_INTERVAL_MS` 常量控制）
- 只有持仓状态的币对才会输出状态日志
- 黑名单中的币对不会输出状态日志

## 注意事项

1. **文件大小**: 状态日志会持续写入,注意磁盘空间
2. **性能影响**: CSV 写入使用线程池,不会阻塞主流程
3. **数据完整性**: 如果某些字段为空,会显示 "N/A"
4. **时间精度**: 所有时间字段精确到秒
5. **价格精度**: 价格字段保留完整精度,不截断
