# 套利作业 - 修复状态日志使用实际成交价格和成本

## 问题描述

通过分析 `status_log_2026041222.csv` 和订单明细日志,发现状态日志中的数据计算存在问题:

### 发现的问题

1. **使用预估价格而非实际成交价格**:
   - `entry_spot_price`: 0.10225 (预估价格)
   - 实际成交价: 0.1020897869353332
   - 差异: 约 0.0001 USDT

2. **USDT 金额计算错误**:
   - 显示: 6.14 USDT
   - 实际: 6.125387216119992 USDT (60 × 0.1020897869353332)

3. **成本字段全部为 N/A**:
   - `spot_cost`, `futures_cost`, `total_cost`, `total_fee`, `total_expense` 全部未更新

### 根本原因

1. **开仓时使用预估价格**: `position.setEntrySpotPrice(opp.spotPrice)` 使用的是套利机会的价格,不是订单实际成交价
2. **订单详情未同步到 PositionState**: 查询订单详情后更新了 `ArbitrageOpportunity`,但没有同步到 `PositionState`
3. **状态日志读取错误**: 状态日志从 `ArbitrageOpportunity` 读取成本,但该对象是局部变量,没有持久化

## 解决方案

### 1. 扩展 PositionState 类

添加实际成交价格和成本字段:

```java
// ⭐ 新增: 实际成交价格和成本字段（用于精确计算利润）

/**
 * 现货实际成交价格（从订单详情获取）
 */
private BigDecimal actualSpotPrice;

/**
 * 合约实际成交价格（从订单详情获取）
 */
private BigDecimal actualSwapPrice;

/**
 * 现货成本（USDT）= 实际成交价 × 成交数量
 */
private BigDecimal spotCost;

/**
 * 合约成本（USDT）= 实际成交价 × 成交数量
 */
private BigDecimal futuresCost;

/**
 * 总成本（USDT）= 现货成本 + 合约成本
 */
private BigDecimal totalCost;

/**
 * 现货手续费（USDT）
 */
private BigDecimal spotFee;

/**
 * 合约手续费（USDT）
 */
private BigDecimal futuresFee;

/**
 * 总手续费（USDT）= 现货手续费 + 合约手续费
 */
private BigDecimal totalFee;

/**
 * 总费用（USDT）= 总成本 + 总手续费
 */
private BigDecimal totalExpense;
```

### 2. 添加订单详情同步方法

在 `processElement` 方法中,持仓状态下同步订单详情:

```java
// ⭐ 同步订单详情到 PositionState（如果 ArbitrageOpportunity 包含订单详情）
syncOrderDetailsToPosition(opportunity, position);
```

实现 `syncOrderDetailsToPosition` 方法:

```java
/**
 * 同步订单详情到 PositionState
 * 从 ArbitrageOpportunity 中提取订单详情并更新到 PositionState
 */
private void syncOrderDetailsToPosition(ArbitrageOpportunity opp, PositionState position) throws Exception {
    // 检查是否有订单详情需要同步
    boolean hasSpotDetail = opp.spotFillPrice != null && opp.spotFillPrice.compareTo(BigDecimal.ZERO) > 0;
    boolean hasFuturesDetail = opp.futuresFillPrice != null && opp.futuresFillPrice.compareTo(BigDecimal.ZERO) > 0;
    
    // 检查是否已经同步过（避免重复同步）
    boolean alreadySynced = position.getActualSpotPrice() != null && position.getActualSwapPrice() != null;
    
    if ((hasSpotDetail || hasFuturesDetail) && !alreadySynced) {
        // 同步现货订单详情
        if (hasSpotDetail) {
            position.setActualSpotPrice(opp.spotFillPrice);
            position.setSpotCost(opp.spotCost);
            position.setSpotFee(opp.spotFee);
        }
        
        // 同步合约订单详情
        if (hasFuturesDetail) {
            position.setActualSwapPrice(opp.futuresFillPrice);
            position.setFuturesCost(opp.futuresCost);
            position.setFuturesFee(opp.futuresFee);
        }
        
        // 计算总成本和总费用
        if (hasSpotDetail && hasFuturesDetail) {
            position.setTotalCost(opp.totalCost);
            position.setTotalFee(opp.totalFee);
            position.setTotalExpense(opp.totalExpense);
        }
        
        // 更新状态
        positionState.update(position);
    }
}
```

### 3. 修改状态日志写入逻辑

优先从 `PositionState` 读取实际成交价格和成本:

```java
// 开仓价格相关字段（优先使用实际成交价格）
String entrySpotPrice = "N/A";
String entrySwapPrice = "N/A";
if (position != null && position.isOpen()) {

    if (position.getActualSpotPrice() != null) {

    } else {
        entrySpotPrice = formatPrice(position.getEntrySpotPrice());
    }
    
    if (position.getActualSwapPrice() != null) {
        entrySwapPrice = formatPrice(position.getActualSwapPrice());
    } else {

    }

// 金额相关字段（使用实际成交价格计算）

    }
    // ... 其他字段类似
```

## 修复后的数据流程

### 1. 开仓阶段
```
1. 发现套利机会 (使用预估价格)
2. 提交订单
3. 创建临时 PositionState (使用预估价格)
4. 异步查询订单详情
5. 更新 ArbitrageOpportunity (实际成交价格和成本)
```

### 2. 持仓阶段
```
1. 接收新的套利机会数据
2. 检查 ArbitrageOpportunity 是否包含订单详情
3. 如果包含,同步到 PositionState
4. 更新预估利润
5. 输出状态日志 (使用实际成交价格和成本)
```

### 3. 状态日志输出
```
1. 优先从 PositionState 读取实际成交价格
2. 如果没有,使用预估价格
3. 使用实际成交价格计算 USDT 金额
4. 从 PositionState 读取成本和手续费
5. 写入 CSV 文件
```

## 注意事项

1. **异步同步**: 订单详情是异步查询的,可能需要几秒钟才能同步到 `PositionState`
2. **首次日志**: 开仓后的第一条状态日志可能还是使用预估价格,后续日志会使用实际成交价格
3. **状态持久化**: `PositionState` 是 Flink 状态,会自动持久化到 Checkpoint
4. **精度问题**: 所有价格和金额字段保留 8 位小数,确保精度
