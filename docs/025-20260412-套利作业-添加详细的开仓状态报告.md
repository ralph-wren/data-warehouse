# 套利作业 - 添加详细的开仓状态报告

## 需求描述

用户要求在开仓后打印详细的币对状态报告,包含:
1. 确定开仓时现货合约价格
2. 下单金额信息
3. 实际成交的价格、数量、费用信息
4. 总成本信息(双边实际投入金额)
5. 预估利润、预估利润率信息
6. 当前价格、当前价差等各种信息
7. 能通过信息确认计算的指标是否正确

## 解决方案

### 1. 添加开仓状态报告方法

在 `TradingDecisionProcessor` 中添加 `printOpenPositionReport` 方法,在查询完订单详情后自动打印详细报告。

### 2. 报告内容结构

报告分为 7 个部分:

#### 【1. 开仓决策信息】
- 发现价差率
- 现货价格(预估)
- 合约价格(预估)
- 下单金额

#### 【2. 现货订单成交信息】
- 订单ID
- 成交价格(实际)
- 成交数量
- 成交金额
- 手续费

#### 【3. 合约订单成交信息】
- 订单ID
- 成交价格(实际)
- 成交数量
- 成交金额
- 手续费

#### 【4. 成本汇总信息】
- 现货成本
- 合约成本
- 总成本(双边投入)
- 总手续费
- 总费用(成本+手续费)

#### 【5. 实际成交价差信息】
- 实际价差
- 实际价差率
- 预期价差率
- 价差偏差

#### 【6. 预估利润信息】
- 预估利润(假设价差完全收敛)
- 预估利润率(基于总成本)

#### 【7. 验证计算公式】
- 现货成本 = 现货成交价 × 现货成交量
- 合约成本 = 合约成交价 × 合约成交量
- 总成本 = 现货成本 + 合约成本
- 总手续费 = 现货手续费 + 合约手续费
- 预估利润 = 价差 × 数量 - 总手续费

## 关键信息说明

### 1. 成本计算

**双边投入成本** = 现货成本 + 合约成本

这是实际投入的资金总额,用于计算利润率。

### 2. 手续费说明

OKX API 返回的 `fee` 字段已经是总手续费,不需要乘以数量:
- 现货手续费: 0.00612539 USDT (已扣除)
- 合约手续费: 0.00305220 USDT (已扣除)

### 3. 价差计算

**实际价差** = 合约成交价 - 现货成交价

- 正价差: 合约价格高于现货价格(正向套利)
- 负价差: 合约价格低于现货价格(反向套利)

### 4. 利润计算

**预估利润** = 价差 × 数量 - 总手续费

这是假设价差完全收敛时的理论利润。

**预估利润率** = 预估利润 / 总成本 × 100%

基于双边投入成本计算的利润率。

### 5. 价差偏差

**价差偏差** = 实际价差率 - 预期价差率

- 正偏差: 实际价差更优,利润空间更大
- 负偏差: 实际价差更差,利润空间更小

## 触发时机

报告在以下时机自动打印:
1. 开仓订单提交成功
2. 延迟 2 秒后查询订单详情
3. 现货和合约订单详情都查询成功
4. 计算完总成本和费用后
5. 自动打印详细报告

## 验证方法

### 1. 验证成本计算

```python
# 现货成本
spot_cost = spot_fill_price * spot_fill_quantity
# 6.12538722 = 0.1020897869353332 × 60

# 合约成本
futures_cost = futures_fill_price * futures_fill_quantity
# 0.61044 = 0.10174 × 6

# 总成本
total_cost = spot_cost + futures_cost
# 6.73582722 = 6.12538722 + 0.61044
```

### 2. 验证手续费

```python
# 总手续费
total_fee = spot_fee + futures_fee
# 0.00917759 = 0.00612539 + 0.00305220
```

### 3. 验证利润

```python
# 实际价差
actual_spread = futures_fill_price - spot_fill_price
# -0.03468979 = 0.10174 - 0.1020897869353332

# 预估利润
estimated_profit = actual_spread * spot_fill_quantity - total_fee
# -2.0906 = -0.03468979 × 60 - 0.00917759

# 预估利润率
estimated_profit_rate = estimated_profit / total_cost * 100
# -31.03% = -2.0906 / 6.73582722 × 100
```

## 注意事项

1. **报告时机**: 报告在订单详情查询完成后打印,可能有 2-3 秒延迟
2. **价格精度**: 所有价格保留完整精度,不截断
3. **负利润**: 如果价差偏差过大,可能出现负利润(亏损)
4. **合约数量**: OKX 合约数量单位是"张",不是币的数量
5. **双边成本**: 总成本包含现货和合约两边的投入

## 总结

通过添加详细的开仓状态报告,可以:
1. 清晰了解每笔交易的完整信息
2. 验证所有计算公式的正确性
3. 及时发现价差偏差和潜在问题
4. 方便后续分析和优化策略

报告采用表格格式,信息结构清晰,易于阅读和验证。
