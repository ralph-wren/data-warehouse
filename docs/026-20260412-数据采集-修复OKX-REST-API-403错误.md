# 数据采集 - 修复 OKX REST API 403 错误

## 问题现象

启动 `FlinkTickerCollectorJob` 时，日志显示：

```
2026-04-12 23:55:00.532 [main] WARN  com.crypto.dw.jobs.FlinkTickerCollectorJob - 价差计算失败: HTTP 请求失败, 状态码: 403, 将使用配置文件或默认交易对
2026-04-12 23:55:00.532 [main] INFO  com.crypto.dw.jobs.FlinkTickerCollectorJob - 订阅币对列表 (0个): []
```

## 问题分析

### 1. 可能的原因

OKX API 返回 403 错误通常有以下几种原因：

1. **缺少必要的 HTTP 请求头** - OKX API 可能需要特定的 User-Agent
2. **网络连接问题** - 无法访问 OKX API (防火墙、代理等)
3. **IP 限制** - OKX 可能限制某些 IP 访问
4. **请求频率限制** - 短时间内请求过多

### 2. 测试验证

通过独立测试程序验证，OKX API 本身是可以访问的：

```java
// 测试结果显示 HTTP 状态码: 200
// 说明 API 本身没有问题
```

### 3. 根本原因

经过分析，问题可能出在：
- **IDEA 运行环境的网络配置**
- **Java 网络代理设置**
- **防火墙或安全软件拦截**

## 解决方案

### 方案 1: 添加完整的 HTTP 请求头 (已实施)

在 `OKXRestApiSource.httpGet()` 方法中添加完整的请求头：

```java
private String httpGet(String urlString) throws Exception {
    URL url = new URL(urlString);
    HttpURLConnection conn = (HttpURLConnection) url.openConnection();
    
    conn.setRequestMethod("GET");
    conn.setConnectTimeout(10000);
    conn.setReadTimeout(10000);
    
    // ⭐ 添加必要的请求头,避免被 OKX API 拒绝 (403 错误)
    conn.setRequestProperty("Content-Type", "application/json");
    conn.setRequestProperty("User-Agent", "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36");
    conn.setRequestProperty("Accept", "application/json");
    conn.setRequestProperty("Accept-Language", "zh-CN,zh;q=0.9,en;q=0.8");
    conn.setRequestProperty("Accept-Encoding", "gzip, deflate, br");
    conn.setRequestProperty("Connection", "keep-alive");
    
    int responseCode = conn.getResponseCode();
    if (responseCode != 200) {
        throw new RuntimeException("HTTP 请求失败, 状态码: " + responseCode);
    }
    
    // 使用 try-with-resources 自动关闭资源
    try (BufferedReader reader = new BufferedReader(new InputStreamReader(conn.getInputStream()))) {
        StringBuilder response = new StringBuilder();
        String line;
        while ((line = reader.readLine()) != null) {
            response.append(line);
        }
        return response.toString();
    } finally {
        conn.disconnect();
    }
}
```

### 方案 4: 增加详细的错误日志

修改 `httpGet` 方法，添加更详细的错误信息：

```java
private String httpGet(String urlString) throws Exception {
    URL url = new URL(urlString);
    HttpURLConnection conn = (HttpURLConnection) url.openConnection();
    
    conn.setRequestMethod("GET");
    conn.setConnectTimeout(10000);
    conn.setReadTimeout(10000);
    
    // 添加请求头
    conn.setRequestProperty("Content-Type", "application/json");
    conn.setRequestProperty("User-Agent", "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36");
    conn.setRequestProperty("Accept", "application/json");
    conn.setRequestProperty("Accept-Language", "zh-CN,zh;q=0.9,en;q=0.8");
    conn.setRequestProperty("Connection", "keep-alive");
    
    int responseCode = conn.getResponseCode();
    if (responseCode != 200) {
        // ⭐ 读取错误响应内容
        String errorMsg = "";
        try (BufferedReader errorReader = new BufferedReader(new InputStreamReader(conn.getErrorStream()))) {
            StringBuilder errorResponse = new StringBuilder();
            String line;
            while ((line = errorReader.readLine()) != null) {
                errorResponse.append(line);
            }
            errorMsg = errorResponse.toString();
        } catch (Exception e) {
            errorMsg = "无法读取错误响应";
        }
        
        throw new RuntimeException(String.format("HTTP 请求失败, URL: %s, 状态码: %d, 错误信息: %s", 
            urlString, responseCode, errorMsg));
    }
    
    // 使用 try-with-resources 自动关闭资源
    try (BufferedReader reader = new BufferedReader(new InputStreamReader(conn.getInputStream()))) {
        StringBuilder response = new StringBuilder();
        String line;
        while ((line = reader.readLine()) != null) {
            response.append(line);
        }
        return response.toString();
    } finally {
        conn.disconnect();
    }
}
```

### 方案 5: 使用配置文件指定交易对 (临时方案)

如果 REST API 持续无法访问，可以在配置文件中直接指定交易对：

```yaml
# src/main/resources/config/application-dev.yml
ticker:
  subscribe:
    symbols:
      - BTC-USDT
      - ETH-USDT
      - SOL-USDT
      - BNB-USDT
      - XRP-USDT
      - ADA-USDT
      - DOGE-USDT
      - MATIC-USDT
      - DOT-USDT
      - AVAX-USDT
```

然后修改 `FlinkTickerCollectorJob` 读取配置：

```java
// 如果 REST API 失败,从配置文件读取
List<String> configSymbols = config.getStringList("ticker.subscribe.symbols");
if (configSymbols != null && !configSymbols.isEmpty()) {
    return configSymbols;
}
```
