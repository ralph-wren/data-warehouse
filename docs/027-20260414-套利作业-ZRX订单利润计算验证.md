## 计算逻辑分析

### 1. 代码位置

利润计算逻辑位于 `TradingDecisionProcessor.updateUnrealizedProfit()` 方法：

```java
// 计算当前价差
BigDecimal entrySpread = pos.getEntrySwapPrice().subtract(pos.getEntrySpotPrice());
BigDecimal currentSpread = opp.futuresPrice.subtract(opp.spotPrice);
BigDecimal spreadDiff = currentSpread.subtract(entrySpread);

// 根据方向调整价差差异
if ("SHORT_SPOT_LONG_SWAP".equals(pos.getDirection())) {
    spreadDiff = spreadDiff.negate();
}

// 计算未实现利润 = 价差变化 × 币数量
BigDecimal unrealizedProfitFromSpread = spreadDiff.multiply(pos.getAmount());

// 计算手续费（基于 USDT 金额）
BigDecimal usdtAmount = pos.getAmount().multiply(pos.getEntrySpotPrice());
BigDecimal fee = usdtAmount.multiply(new BigDecimal("0.002"));

// 最终未实现利润 = 价差利润 - 手续费
BigDecimal unrealizedProfit = unrealizedProfitFromSpread.subtract(fee);
```

### 2. 手动计算验证

#### 步骤 1: 计算价差

```
开仓价差 = 合约价 - 现货价
        = 0.10166 - 0.10225
        = -0.00059

当前价差 = 合约价 - 现货价
        = 0.10166 - 0.10225
        = -0.00059

价差变化 = 当前价差 - 开仓价差
        = -0.00059 - (-0.00059)
        = 0
```

#### 步骤 2: 调整方向

因为是 `SHORT_SPOT_LONG_SWAP` (做空现货 + 做多合约)，需要对价差变化取反：

```
调整后价差变化 = -0 = 0
```

#### 步骤 3: 计算价差利润

```
价差利润 = 价差变化 × 数量
        = 0 × 60
        = 0 USDT
```

#### 步骤 4: 计算手续费

```
USDT 金额 = 数量 × 开仓现货价
         = 60 × 0.10225
         = 6.135 USDT

手续费 = USDT 金额 × 0.002
      = 6.135 × 0.002
      = 0.01227 USDT
```

#### 步骤 5: 计算未实现利润

```
未实现利润 = 价差利润 - 手续费
          = 0 - 0.01227
          = -0.01227 USDT

利润率 = (未实现利润 / USDT 金额) × 100
      = (-0.01227 / 6.135) × 100
      = -0.20%
```

### 3. 对比结果


| 项目    | CSV 中的值      | 手动计算值         | 差异           |
| ----- | ------------ | ------------- | ------------ |
| 未实现利润 | -0.0123 USDT | -0.01227 USDT | 0.00003 USDT |
| 利润率   | -0.20%       | -0.20%        | 0%           |


**结论**: ✅ 计算结果基本一致，微小差异来自精度舍入。

## 计算逻辑说明

### 1. 为什么未实现利润是负数？

因为当前价格没有变化（开仓价格 = 当前价格），所以价差利润为 0，但需要扣除手续费，导致未实现利润为负。

### 2. 手续费计算

- 手续费率: 0.2% (0.002)
- 计算基础: USDT 金额（数量 × 现货价格）
- 这是一个预估值，实际手续费需要从订单详情中获取

### 3. SHORT_SPOT_LONG_SWAP 方向说明

- **做空现货**: 卖出现货，希望现货价格下跌
- **做多合约**: 买入合约，希望合约价格上涨
- **套利逻辑**: 当合约价格 < 现货价格时开仓，等待价差收敛获利
- **价差变化取反**: 因为是做空现货，价差扩大对我们不利，所以需要取反

## 改进建议

### 1. 使用实际手续费

当前代码使用固定费率 0.2% 计算手续费，建议改为使用订单详情中的实际手续费：

```java
// 当前实现
BigDecimal fee = usdtAmount.multiply(new BigDecimal("0.002"));

// 建议改为
BigDecimal fee = position.getTotalFee();  // 从订单详情中获取
```

### 2. 记录实际成交价格

CSV 日志中应该记录实际成交价格，而不仅仅是开仓时的市场价格：

```java
// 优先使用实际成交价格
if (position.getActualSpotPrice() != null) {
    entrySpotPrice = formatPrice(position.getActualSpotPrice());
} else {
    entrySpotPrice = formatPrice(position.getEntrySpotPrice());
}
```

### 3. 添加滑点记录

记录预期价格和实际成交价格的差异（滑点）：

```java
BigDecimal spotSlippage = actualSpotPrice.subtract(expectedSpotPrice);
BigDecimal swapSlippage = actualSwapPrice.subtract(expectedSwapPrice);
```
