## 问题分析

### 🚨 严重问题：现货订单部分成交

**预期**:
- 现货卖出 60 ZRX
- 合约买入 60 ZRX (6 张合约)

**实际**:
- 现货只卖出了 11.278388 ZRX (18.8%)
- 合约买入了 60 ZRX (100%)

**后果**:
- **持仓不平衡**: 现货少卖了 48.72 ZRX
- **风险敞口**: 合约多头 60 ZRX，但现货空头只有 11.28 ZRX
- **套利失效**: 无法形成完整的对冲套利

### 可能原因

1. **现货流动性不足**
   - ZRX-USDT 现货市场深度不够
   - 市价单无法完全成交

2. **杠杆借币限制**
   - 使用全仓杠杆模式 (tdMode: cross)
   - 可能借币额度不足，导致无法卖出全部数量

3. **账户余额不足**
   - 现货账户 ZRX 余额可能不足 60 个
   - 借币额度已用完

## 改进建议

### 1. 添加订单成交验证

在开仓后立即查询订单详情，验证是否完全成交：

```java
// 查询现货订单详情
String spotDetailJson = tradingService.getOrderDetail(spotInstId, spotOrderId);
JsonNode spotDetail = mapper.readTree(spotDetailJson);
String spotState = spotDetail.path("data").get(0).path("state").asText();
BigDecimal spotFillSz = new BigDecimal(spotDetail.path("data").get(0).path("accFillSz").asText());

// 检查是否完全成交
if (!spotState.equals("filled") || spotFillSz.compareTo(expectedSize) < 0) {
    logger.error("❌ 现货订单未完全成交: 预期={}, 实际={}", expectedSize, spotFillSz);
    // 取消合约订单或调整策略
    return;
}
```

### 2. 使用实际成交数据

```java
// 使用实际成交价格和数量
position.setAmount(spotFillSz);  // 实际成交数量
position.setActualSpotPrice(new BigDecimal(spotDetail.path("data").get(0).path("avgPx").asText()));
position.setActualSwapPrice(new BigDecimal(swapDetail.path("data").get(0).path("avgPx").asText()));
position.setSpotFee(new BigDecimal(spotDetail.path("data").get(0).path("fee").asText()).abs());
position.setSwapFee(new BigDecimal(swapDetail.path("data").get(0).path("fee").asText()).abs());
```

### 3. 处理部分成交

```java
// 如果现货部分成交，调整合约持仓
if (spotFillSz.compareTo(expectedSize) < 0) {
    logger.warn("⚠️ 现货部分成交，调整合约持仓");
    
    // 计算需要平掉的合约数量
    BigDecimal excessSwap = expectedSize.subtract(spotFillSz);
    
    // 平掉多余的合约持仓
    tradingService.closePosition(swapInstId, excessSwap);
}
```

### 4. 添加流动性检查

在开仓前检查市场深度：

```java
// 查询订单簿深度
String orderbookJson = tradingService.getOrderbook(instId);
// 检查是否有足够的流动性
if (availableLiquidity.compareTo(tradeAmount) < 0) {
    logger.warn("⚠️ 流动性不足，跳过开仓");
    return;
}
```
