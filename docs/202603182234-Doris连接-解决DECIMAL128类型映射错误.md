# Doris Flink Connector DECIMAL128 类型映射错误解决方案

## 问题描述

在使用 Flink Doris Connector (版本 24.0.0) 写入数据到 Doris 时,遇到以下错误:

```
Exception in thread "main" org.apache.doris.flink.exception.DorisRuntimeException: 
org.apache.doris.flink.exception.DorisException: Doris FE's response cannot map to schema.
```

错误发生在 `DorisSink` 初始化时调用 `checkKeyType` 方法,该方法无法正确解析 Doris 表中的 `DECIMAL128` 类型。

## 问题根源

1. **DECIMAL 类型映射问题**: Doris 表中定义的 `DECIMAL(20,8)` 和 `DECIMAL(30,8)` 在 Doris 内部被映射为 `DECIMAL128` 类型
2. **Connector 版本兼容性**: Flink Doris Connector 24.0.0 版本在进行 schema 映射时,无法正确识别 `DECIMAL128` 类型
3. **强制 schema 检查**: `DorisSink.Builder.build()` 方法会自动调用 `checkKeyType()` 进行表键类型检查,导致初始化失败

## 解决方案

### 方案选择

采用**自定义 Sink 函数**方案,直接使用 Doris 的 HTTP Stream Load API 写入数据,完全绕过 schema 映射检查。

### 实现步骤

#### 1. 创建自定义 DorisStreamLoadSink 类

创建文件: `src/main/java/com/crypto/dw/flink/sink/DorisStreamLoadSink.java`

核心功能:
- 继承 `RichSinkFunction<String>`,实现自定义 Sink
- 使用缓冲区批量写入,提高性能
- 直接调用 Doris HTTP Stream Load API
- 支持批量大小和时间间隔两种刷新策略

关键代码:
```java
// 构建 Stream Load URL
String loadUrl = String.format("%s/api/%s/%s/_stream_load", 
                              feHttpUrl, database, table);

// 设置 HTTP 请求头
conn.setRequestProperty("format", "json");
conn.setRequestProperty("read_json_by_line", "true");
conn.setRequestProperty("strip_outer_array", "false");
conn.setRequestProperty("max_filter_ratio", "0.1");
```

#### 2. 修改 FlinkODSJobDataStream 类

主要改动:
- 移除 Doris Connector 相关的 import
- 添加自定义 Sink 的 import
- 创建 `createDorisStreamLoadSink()` 方法替代原来的 `createDorisSink()`
- 使用 `addSink()` 替代 `sinkTo()` 方法

修改前:
```java
DorisSink<String> dorisSink = createDorisSink(config);
odsStream.sinkTo(dorisSink).name("Doris ODS Sink");
```

修改后:
```java
DorisStreamLoadSink dorisSink = createDorisStreamLoadSink(config);
odsStream.addSink(dorisSink).name("Doris ODS Sink");
```

#### 3. 升级 Doris Connector 版本 (可选)

在 `pom.xml` 中将版本从 24.0.0 升级到 24.0.1:
```xml
<dependency>
    <groupId>org.apache.doris</groupId>
    <artifactId>flink-doris-connector-1.18</artifactId>
    <version>24.0.1</version>
</dependency>
```

## 技术要点

### 1. Stream Load API 参数说明

| 参数 | 说明 | 值 |
|------|------|-----|
| format | 数据格式 | json |
| read_json_by_line | 按行读取 JSON | true |
| strip_outer_array | 去除外层数组 | false |
| max_filter_ratio | 最大容错率 | 0.1 (10%) |

### 2. 批量写入策略

采用双重触发机制:
- **批量大小触发**: 缓冲区达到 batchSize (默认 1000 条)
- **时间间隔触发**: 距离上次刷新超过 batchIntervalMs (默认 5000ms)

### 3. HTTP 认证

使用 Basic Authentication:
```java
String auth = username + ":" + password;
String encodedAuth = Base64.getEncoder().encodeToString(
    auth.getBytes(StandardCharsets.UTF_8));
conn.setRequestProperty("Authorization", "Basic " + encodedAuth);
```

## 优势分析

### 相比官方 Connector 的优势

1. **完全绕过 schema 检查**: 不依赖 Connector 的类型映射机制
2. **更好的兼容性**: 支持所有 Doris 数据类型,包括 DECIMAL128
3. **更灵活的配置**: 可以自定义所有 Stream Load 参数
4. **更清晰的错误处理**: 直接获取 HTTP 响应,便于调试

### 性能考虑

1. **批量写入**: 减少网络请求次数
2. **异步刷新**: 不阻塞数据处理流程
3. **可配置缓冲**: 根据实际情况调整批量大小

## 验证结果

编译成功,无错误:

mvn clean compile -DskipTests

## 注意事项

1. **数据格式**: 确保发送的 JSON 格式与 Doris 表结构匹配
2. **错误处理**: Stream Load 失败时会抛出异常,需要配置 Flink 的重试策略
3. **Checkpoint**: 自定义 Sink 需要配合 Flink Checkpoint 保证数据一致性
4. **性能调优**: 根据数据量调整 batchSize 和 batchIntervalMs 参数
