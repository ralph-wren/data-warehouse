# Doris Stream Load "100-continue" 错误解决方案

## 问题描述

在使用自定义 `DorisStreamLoadSink` 写入数据到 Doris 时,遇到以下错误:

```json
{
  "status": "FAILED",
  "msg": "There is no 100-continue header"
}
```

虽然 HTTP 响应码是 200,但 Doris 返回的状态是 FAILED。

## 错误日志

```
2026-03-18 23:00:40,074 INFO  DorisStreamLoadSink:144 - HTTP 响应码: 200
2026-03-18 23:00:40,075 INFO  DorisStreamLoadSink:161 - 响应内容: {"status":"FAILED","msg":"There is no 100-continue header"}
```

## 问题分析

### HTTP 100-continue 机制

HTTP/1.1 协议定义了 `Expect: 100-continue` 机制:

1. **客户端行为**: 当发送大量数据时,`HttpURLConnection` 会自动添加 `Expect: 100-continue` 请求头
2. **服务器响应**: 服务器应该先返回 `100 Continue` 响应,然后客户端再发送请求体
3. **Doris 限制**: Doris Stream Load API 不支持 `Expect: 100-continue` 机制

### 为什么会出现这个问题?

```java
// 当数据量较大时 (通常 > 1KB)
conn.setDoOutput(true);  // 这会触发 HttpURLConnection 自动添加 Expect: 100-continue
```

Java 的 `HttpURLConnection` 在以下情况会自动添加 `Expect: 100-continue` 头:
- 使用 PUT 或 POST 方法
- 设置了 `setDoOutput(true)`
- 请求体大小超过一定阈值 (通常是 1KB)

## 解决方案

### 方法 1: 禁用 Expect 头 (推荐)

在发送请求前,显式设置 `Expect` 头为空字符串:

```java
// 关键修复: 禁用 Expect: 100-continue 头
conn.setRequestProperty("Expect", "");
```

**完整代码**:

```java
// 发送 HTTP 请求
HttpURLConnection conn = (HttpURLConnection) new URL(loadUrl).openConnection();
conn.setRequestMethod("PUT");
conn.setDoOutput(true);
conn.setConnectTimeout(30000);
conn.setReadTimeout(30000);

// 关键修复: 禁用 Expect: 100-continue 头,避免 Doris 报错
conn.setRequestProperty("Expect", "");

// 设置认证
String auth = username + ":" + password;
String encodedAuth = Base64.getEncoder().encodeToString(auth.getBytes(StandardCharsets.UTF_8));
conn.setRequestProperty("Authorization", "Basic " + encodedAuth);

// 设置 Stream Load 参数
conn.setRequestProperty("format", "json");
conn.setRequestProperty("read_json_by_line", "true");
conn.setRequestProperty("strip_outer_array", "false");
conn.setRequestProperty("max_filter_ratio", "0.1");
```

### 方法 2: 使用 Apache HttpClient (备选)

如果方法 1 不起作用,可以使用 Apache HttpClient 库:

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.apache.httpcomponents</groupId>
    <artifactId>httpclient</artifactId>
    <version>4.5.14</version>
</dependency>
```

```java
// 使用 Apache HttpClient
CloseableHttpClient httpClient = HttpClients.custom()
    .setDefaultRequestConfig(RequestConfig.custom()
        .setExpectContinueEnabled(false)  // 禁用 100-continue
        .build())
    .build();

HttpPut httpPut = new HttpPut(loadUrl);
httpPut.setEntity(new StringEntity(data.toString(), StandardCharsets.UTF_8));
```

### 方法 3: 使用 OkHttp (备选)

项目中已经有 OkHttp 依赖,也可以使用它:

```java
OkHttpClient client = new OkHttpClient.Builder()
    .connectTimeout(30, TimeUnit.SECONDS)
    .readTimeout(30, TimeUnit.SECONDS)
    .build();

RequestBody body = RequestBody.create(
    data.toString(), 
    MediaType.parse("application/json; charset=utf-8")
);

Request request = new Request.Builder()
    .url(loadUrl)
    .put(body)
    .addHeader("Authorization", "Basic " + encodedAuth)
    .addHeader("format", "json")
    .addHeader("read_json_by_line", "true")
    .build();

Response response = client.newCall(request).execute();
```

## 技术细节

### HTTP 100-continue 工作流程

```
客户端                                服务器
  |                                    |
  |-- PUT /api/db/table (Expect: 100-continue) -->|
  |                                    |
  |<-- 100 Continue -------------------|
  |                                    |
  |-- [Request Body] ----------------->|
  |                                    |
  |<-- 200 OK -------------------------|
```

### Doris 不支持的原因

Doris Stream Load API 设计为:
1. 一次性接收完整的请求体
2. 不需要 100-continue 协商
3. 简化服务器实现,提高性能

## 相关问题

### Q1: 为什么 HTTP 响应码是 200 但状态是 FAILED?

**答**: Doris Stream Load API 的设计是:
- HTTP 层面: 请求成功接收,返回 200
- 业务层面: 通过 JSON 响应体中的 `Status` 字段表示导入结果

### Q2: 其他 HTTP 客户端会遇到这个问题吗?

**答**: 
- `HttpURLConnection`: 会自动添加 Expect 头,需要手动禁用
- `Apache HttpClient`: 默认启用,需要配置禁用
- `OkHttp`: 默认不添加 Expect 头,无需处理
- `curl`: 默认不添加,除非使用 `-H "Expect: 100-continue"`

### Q3: 为什么不是所有请求都会出现这个问题?

**答**: 只有当请求体较大时 (通常 > 1KB),`HttpURLConnection` 才会自动添加 `Expect: 100-continue` 头。小数据量的请求不会触发。

## 最佳实践

### 1. 始终禁用 Expect 头

在所有 Doris Stream Load 请求中,都应该禁用 Expect 头:

```java
conn.setRequestProperty("Expect", "");
```

### 2. 检查响应状态

不要只检查 HTTP 响应码,还要检查 JSON 响应体中的 Status 字段:

```java
if (responseCode == 200 && responseJson != null) {
    String status = responseJson.getString("Status");
    if ("Success".equalsIgnoreCase(status)) {
        // 成功
    } else {
        // 失败
    }
}
```

