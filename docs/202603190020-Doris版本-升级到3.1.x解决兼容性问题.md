# Doris 版本升级 - 从 4.0.1-rc02 到 3.1.x

## 问题总结

在使用 Doris 4.0.1-rc02 版本时遇到多个严重问题:

1. **DECIMAL128 类型不兼容** - Flink Doris Connector 1.6.2 无法识别 DECIMAL128 类型
2. **Stream Load API Bug** - HTTP 协议处理有问题,无论如何设置都报错
3. **Connection Reset** - BE 节点频繁连接重置
4. **Schema 解析失败** - Connector 无法解析 Doris 4.0.x 返回的 schema 格式

## 解决方案

### 升级到 Doris 3.1.x

Doris 3.1.x 是当前最稳定的版本,与 Flink Doris Connector 1.6.2 完全兼容。

### 版本对应关系

| 组件 | 推荐版本 | 说明 |
|-----|---------|------|
| Doris | 3.1.x | 稳定版,生产可用 |
| Flink | 1.17.2 | LTS 版本 |
| Flink Doris Connector | 1.6.2 | 与 Doris 3.1.x 完全兼容 |
| Flink Kafka Connector | 3.0.2-1.17 | 匹配 Flink 1.17 |

## 升级步骤

### 1. 停止旧版本 Doris

```bash
docker stop doris-fe doris-be
docker rm doris-fe doris-be
```

### 2. 启动 Doris 3.1.x

```bash
# 拉取 Doris 3.1.x 镜像
docker pull apache/doris:fe-3.1.0
docker pull apache/doris:be-3.1.0

# 启动 FE
docker run -d \
  --name doris-fe \
  -p 8030:8030 \
  -p 9030:9030 \
  apache/doris:fe-3.1.0

# 启动 BE
docker run -d \
  --name doris-be \
  -p 8040:8040 \
  -p 9050:9050 \
  apache/doris:be-3.1.0
```

### 3. 创建数据库和表

```bash
# 创建数据库
mysql -h 127.0.0.1 -P 9030 -u root -e "CREATE DATABASE IF NOT EXISTS crypto_dw;"

# 创建表
mysql -h 127.0.0.1 -P 9030 -u root crypto_dw < sql/create_tables.sql
```

### 4. 验证表结构

```bash
mysql -h 127.0.0.1 -P 9030 -u root -e "DESC crypto_dw.ods_crypto_ticker_rt;"
```

## 关键改进

### 1. 数据类型兼容性

**Doris 4.0.1-rc02**:
```sql
last_price DECIMAL128(20, 8)  -- Flink Connector 不支持
```

**Doris 3.1.x**:
```sql
last_price DECIMAL(20, 8)  -- 标准类型,完全兼容
```

### 2. Schema 格式

**Doris 4.0.1-rc02** 返回的 schema 包含额外字段:
```json
{
  "schema_version": 0,
  "materialized_indexes": {...},
  ...
}
```

**Doris 3.1.x** 返回标准格式:
```json
{
  "status": 200,
  "keysType": "DUP_KEYS",
  "properties": [...]
}
```

### 3. Stream Load API

Doris 3.1.x 的 Stream Load API 更稳定:
- 正确处理 HTTP 协议
- 支持 Expect: 100-continue
- 连接更稳定,无频繁 reset

## 常见问题

### Q1: BE 节点连接失败

**错误**: `172.19.0.3:8040 failed to respond`

**原因**: Docker 容器使用内部 IP,但端口已映射到 127.0.0.1

**解决**: 
1. 检查端口映射: `docker ps | grep doris`
2. 确认 BE 健康: `curl http://127.0.0.1:8040/api/health`
3. 等待 BE 完全启动 (约 30 秒)

### Q2: Schema 解析失败

**错误**: `Unrecognized field "schema_version"`

**原因**: Doris 4.0.x 的 schema 格式与 Connector 不兼容

**解决**: 升级到 Doris 3.1.x

### Q3: DECIMAL128 类型错误

**错误**: `Doris FE's response cannot map to schema`

**原因**: Flink Connector 不支持 DECIMAL128

**解决**: 使用 Doris 3.1.x 的标准 DECIMAL 类型
