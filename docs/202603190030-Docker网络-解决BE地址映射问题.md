# Docker 网络问题 - 解决 BE 地址映射

## 问题描述

Flink 作业启动时报错:

```
org.apache.http.NoHttpResponseException: 172.19.0.3:8040 failed to respond
```

## 根本原因

Doris BE 在 Docker 容器中运行时,注册到 FE 的是 Docker 内部 IP (172.19.0.x),而 Flink 作业运行在宿主机上,无法直接访问 Docker 内部网络。

### 问题流程

1. Flink 连接 FE (127.0.0.1:8030) ✅
2. FE 返回 BE 地址 (172.19.0.3:8040) ⚠️
3. Flink 尝试连接 BE (172.19.0.3:8040) ❌ 无法访问

## 解决方案

### 方案 1: 配置 BE 节点地址 (推荐,快速)

在 `application.yml` 中直接指定 BE 地址:

```yaml
doris:
  fe:
    http-url: http://127.0.0.1:8030
    username: root
    password: ""
  
  # 关键配置: 直接指定 BE 地址
  be:
    nodes: "127.0.0.1:8040"  # 使用宿主机地址
  
  database: crypto_dw
```

### 方案 2: 使用 Docker host 网络模式 (推荐,生产环境)

#### 2.1 停止现有容器

```bash
docker stop doris-fe doris-be
docker rm doris-fe doris-be
```

#### 2.2 使用 host 网络启动

```bash
# 启动 FE
docker run -d \
  --name doris-fe \
  --network host \
  -v doris-fe-data:/opt/apache-doris/fe/doris-meta \
  apache/doris:fe-3.1.0

# 等待 FE 启动
sleep 20

# 启动 BE
docker run -d \
  --name doris-be \
  --network host \
  -e FE_SERVERS="127.0.0.1:9010" \
  -e BE_ADDR="127.0.0.1:9050" \
  -v doris-be-data:/opt/apache-doris/be/storage \
  apache/doris:be-3.1.0
```

#### 2.3 或使用提供的脚本

```bash
./restart-doris.sh
```

### 方案 3: 手动修改 BE 注册地址

```bash
# 连接 Doris
mysql -h 127.0.0.1 -P 9030 -u root

# 添加新的 BE (使用宿主机地址)
ALTER SYSTEM ADD BACKEND '127.0.0.1:9050';

# 删除旧的 BE (使用 DROPP 强制删除)
ALTER SYSTEM DROPP BACKEND '172.19.0.3:9050' FORCE;

# 查看 BE 状态
SHOW BACKENDS\G
```

## 代码修改

### 更新 Flink 作业代码

在 `FlinkODSJobDataStream.java` 和 `FlinkODSJobDataStream2.java` 中添加 BE 节点配置:

```java
// 关键修复: 如果 BE 使用 Docker 内部 IP,直接指定 BE 地址
String beNodes = config.getString("doris.be.nodes", "");
if (!beNodes.isEmpty()) {
    dorisBuilder.setBenodes(beNodes);
    logger.info("使用配置的 BE 节点: {}", beNodes);
}
```

## 常见问题

### Q1: 为什么会出现这个问题?

Docker 容器默认使用桥接网络,容器内部有独立的 IP 地址。BE 注册时使用的是容器 IP,但宿主机无法直接访问。

### Q2: host 网络模式有什么影响?

- 优点: 容器直接使用宿主机网络,无 IP 映射问题
- 缺点: 端口直接暴露在宿主机,可能有端口冲突

### Q3: 配置 BE 地址后还是失败?

检查:
1. BE 端口是否正确映射: `docker ps | grep doris-be`
2. BE 是否正常运行: `curl http://127.0.0.1:8040/api/health`
3. 配置文件是否生效: 查看 Flink 日志

### Q4: 多个 BE 节点怎么配置?

```yaml
doris:
  be:
    nodes: "127.0.0.1:8040,127.0.0.1:8041,127.0.0.1:8042"
```
