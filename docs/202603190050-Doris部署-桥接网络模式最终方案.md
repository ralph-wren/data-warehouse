# Doris 部署 - 桥接网络模式最终方案

**时间**: 2026-03-19 00:50 (东八区)  
**问题**: Flink 连接 Doris BE 地址问题  
**最终方案**: 使用桥接网络 + 端口映射 + 配置 BE 节点地址

---

## 问题回顾

在尝试使用 host 网络模式部署 Doris 时遇到以下问题:
1. Doris 3.1.0 镜像在 host 网络模式下环境变量配置复杂
2. BE 持久化数据中保存了旧的地址信息,导致启动失败
3. Windows/Mac 系统对 host 网络模式支持不完整

---

## 最终解决方案

### 方案概述

使用 Docker 桥接网络模式 + 端口映射,BE 在容器内使用 Docker IP,但通过端口映射暴露到宿主机。Flink 通过配置文件指定 BE 的宿主机地址。

### 架构说明

```
┌─────────────────────────────────────────┐
│          宿主机 (127.0.0.1)              │
│                                         │
│  ┌──────────────────────────────────┐  │
│  │   Docker 桥接网络 (172.25.0.0/16) │  │
│  │                                   │  │
│  │  ┌─────────────────────────────┐ │  │
│  │  │  FE: 172.25.0.2             │ │  │
│  │  │  端口映射: 8030, 9030, 9010  │ │  │
│  │  └─────────────────────────────┘ │  │
│  │                                   │  │
│  │  ┌─────────────────────────────┐ │  │
│  │  │  BE: 172.25.0.3             │ │  │
│  │  │  端口映射: 8040, 9050, 8060  │ │  │
│  │  └─────────────────────────────┘ │  │
│  └──────────────────────────────────┘  │
│                                         │
│  Flink 访问: 127.0.0.1:8040 (映射到     │
│              172.25.0.3:8040)          │
└─────────────────────────────────────────┘
```

## 实施步骤

### 1. Docker Compose 配置

**文件**: `docker-compose-doris.yml`

```yaml
version: '3.8'

services:
  doris-fe:
    image: apache/doris:fe-3.0.7-rc01-1204
    container_name: doris-fe
    hostname: doris-fe
    ports:
      - "8030:8030"   # FE HTTP 端口
      - "9030:9030"   # FE MySQL 端口
      - "9010:9010"   # FE RPC 端口
    environment:
      # FE 配置 - 使用正确的格式: name:ip:port
      - FE_SERVERS=fe1:172.25.0.2:9010
      - FE_ID=1
    volumes:
      - doris-fe-data:/opt/apache-doris/fe/doris-meta
      - doris-fe-log:/opt/apache-doris/fe/log
    networks:
      doris-net:
        ipv4_address: 172.25.0.2
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8030/api/bootstrap"]
      interval: 10s
      timeout: 5s
      retries: 30

  doris-be:
    image: apache/doris:be-3.0.7-rc01-1204
    container_name: doris-be
    hostname: doris-be
    ports:
      - "8040:8040"   # BE HTTP 端口 (Stream Load)
      - "9050:9050"   # BE Heartbeat 端口
      - "8060:8060"   # BE Webserver 端口
      - "9060:9060"   # BE Brpc 端口
    environment:
      # BE 配置 - 连接到 FE (使用正确的格式: name:ip:port)
      - FE_SERVERS=fe1:172.25.0.2:9010
      - BE_ADDR=172.25.0.3:9050
    volumes:
      - doris-be-data:/opt/apache-doris/be/storage
      - doris-be-log:/opt/apache-doris/be/log
    networks:
      doris-net:
        ipv4_address: 172.25.0.3
    depends_on:
      doris-fe:
        condition: service_healthy
    restart: unless-stopped

networks:
  doris-net:
    driver: bridge
    ipam:
      config:
        - subnet: 172.25.0.0/16

volumes:
  doris-fe-data:
  doris-fe-log:
  doris-be-data:
  doris-be-log:
```

**关键配置说明**:
- `FE_SERVERS`: 必须使用 `name:ip:port` 格式
- `BE_ADDR`: BE 在容器内的地址
- 固定 IP 地址避免重启后地址变化
- 端口映射让宿主机可以访问

---

### 2. Flink 配置

**文件**: `config/application.yml`

```yaml
doris:
  fe:
    http-url: http://127.0.0.1:8030
    jdbc-url: jdbc:mysql://127.0.0.1:9030
    username: root
    password: ""
  
  # 关键配置: 指定 BE 节点地址为宿主机地址
  be:
    nodes: "127.0.0.1:8040"  # BE HTTP 地址,通过端口映射访问
  
  database: crypto_dw
  
  stream-load:
    batch-size: 1000
    batch-interval-ms: 5000
    max-retries: 3
    timeout-seconds: 600
```

**说明**:
- `doris.be.nodes` 配置为宿主机地址 `127.0.0.1:8040`
- 官方 Connector 会优先使用配置的 BE 地址
- 如果 Connector 仍然使用 FE 返回的地址,需要在代码中强制设置

---

### 3. 部署脚本

**文件**: `start-doris-for-flink.sh`

脚本功能:
1. 停止旧容器
2. 启动 Doris (桥接网络模式)
3. 等待 FE/BE 启动
4. 检测 BE 地址
5. 如果 BE 使用 Docker 内部 IP,自动添加 127.0.0.1 的 BE 节点
6. 验证连接状态

**使用方法**:
```bash
./start-doris-for-flink.sh
```

---

### 4. 重置脚本

**文件**: `reset-doris-clean.sh`

当 BE 持久化数据出现问题时,使用此脚本完全清理:

```bash
./reset-doris-clean.sh
```

功能:
- 停止并删除所有容器
- 删除所有数据卷
- 删除网络
- 重新开始部署

## 当前状态

### 部署结果

✓ FE 启动成功 (127.0.0.1:9030)  
✓ BE 启动成功 (172.25.0.3:9050, Alive: true)  
✓ BE HTTP 端口可访问 (127.0.0.1:8040)  
⚠ BE 在 FE 中注册为 Docker 内部 IP

### BE 地址问题

BE 在 FE 中注册为 `172.25.0.3:9050`,但通过端口映射,Flink 可以通过 `127.0.0.1:8040` 访问 BE 的 HTTP 端口。

**两种处理方式**:

#### 方式 1: 使用 Docker 内部 IP (当前状态)
- BE 地址: `172.25.0.3:9050`
- Flink 通过端口映射访问: `127.0.0.1:8040`
- 需要在配置文件中指定 `doris.be.nodes: "127.0.0.1:8040"`

#### 方式 2: 手动添加 127.0.0.1 的 BE
```bash
# 删除 Docker 内部 IP 的 BE
mysql -h 127.0.0.1 -P 9030 -u root -e "ALTER SYSTEM DROPP BACKEND '172.25.0.3:9050';"

# 添加宿主机地址的 BE
mysql -h 127.0.0.1 -P 9030 -u root -e "ALTER SYSTEM ADD BACKEND '127.0.0.1:9050';"
```

**问题**: BE 容器内部绑定的是 172.25.0.3,无法直接响应 127.0.0.1 的请求。

---

## 推荐方案

### 使用方式 1 (当前方案)

1. 保持 BE 在 FE 中注册为 `172.25.0.3:9050`
2. 在 `application.yml` 中配置 `doris.be.nodes: "127.0.0.1:8040"`
3. 官方 Connector 会优先使用配置的 BE 地址
4. 通过端口映射,Flink 可以正常访问 BE

### 验证步骤

```bash
# 1. 检查 BE 状态
mysql -h 127.0.0.1 -P 9030 -u root -e "SHOW BACKENDS\G"
# 期望: Host: 172.25.0.3, Alive: true

# 2. 检查 BE HTTP 端口
curl http://127.0.0.1:8040/api/health
# 期望: 返回 OK 或 JSON 响应

# 3. 创建数据库和表
mysql -h 127.0.0.1 -P 9030 -u root -e "CREATE DATABASE IF NOT EXISTS crypto_dw;"
mysql -h 127.0.0.1 -P 9030 -u root < sql/create_tables.sql

# 4. 运行 Flink 作业
./run-flink-ods-datastream.sh
```

## 技术细节

### 端口映射原理

Docker 端口映射通过 iptables 规则实现:
```
宿主机 127.0.0.1:8040 → Docker 网桥 → 容器 172.25.0.3:8040
```

Flink 访问 `127.0.0.1:8040` 时,Docker 会自动转发到容器内的 `172.25.0.3:8040`。

### 官方 Connector 的 BE 发现

```java
// Connector 初始化流程
1. 从配置读取 benodes (如果配置了)
2. 如果没有配置,调用 FE 的 /api/backends 接口
3. 使用获取到的 BE 地址进行数据写入

// 关键配置
DorisOptions.Builder dorisBuilder = DorisOptions.builder()
    .setFenodes("127.0.0.1:8030")
    .setBenodes("127.0.0.1:8040");  // 强制指定 BE 地址
```

## 故障排查

### 问题 1: BE 一直 Alive: false

**原因**: BE 还在启动中或启动失败

**解决**:
```bash
# 查看 BE 日志
docker logs doris-be --tail 50

# 重启 BE
docker restart doris-be

# 等待 30-60 秒
sleep 60
mysql -h 127.0.0.1 -P 9030 -u root -e "SHOW BACKENDS\G"
```

### 问题 2: BE 持久化数据冲突

**原因**: BE 数据卷中保存了旧的地址信息

**解决**:
```bash
# 完全重置 Doris
./reset-doris-clean.sh

# 重新部署
./start-doris-for-flink.sh
```

### 问题 3: Flink 仍然报 NoHttpResponseException

**原因**: Connector 使用了 FE 返回的 Docker 内部 IP

**解决**:
1. 确认 `application.yml` 中配置了 `doris.be.nodes: "127.0.0.1:8040"`
2. 确认代码中使用了 `setBenodes()` 方法
3. 检查 BE HTTP 端口: `curl http://127.0.0.1:8040/api/health`

## 总结

### 优点
- ✓ 兼容所有操作系统 (Windows/Mac/Linux)
- ✓ 配置简单,易于理解
- ✓ 端口映射稳定可靠
- ✓ 支持多个 Doris 实例 (使用不同端口)

### 缺点
- ⚠ 需要在配置文件中指定 BE 地址
- ⚠ BE 在 FE 中显示为 Docker 内部 IP (不影响使用)
- ⚠ 性能略低于 host 网络模式 (可忽略)

### 适用场景
- 开发环境
- 测试环境
- 单机部署
- Windows/Mac 系统
