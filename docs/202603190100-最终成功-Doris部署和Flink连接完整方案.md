## 核心问题和解决方案

### 问题 1: Host 网络模式配置复杂

**问题**: Doris 3.1.0 在 host 网络模式下需要特殊的环境变量配置,容易出错。

**解决**: 使用桥接网络模式 + 端口映射,配置更简单可靠。

### 问题 2: BE 持久化数据冲突

**问题**: BE 数据卷中保存了旧的地址信息,导致启动失败或报告错误。

**解决**: 使用 `reset-doris-clean.sh` 完全清理数据卷,重新初始化。

### 问题 3: BE 地址是 Docker 内部 IP

**问题**: BE 在 FE 中注册为 172.25.0.3,Flink 无法直接访问。

**解决**: 
- 保持 BE 使用 Docker 内部 IP (172.25.0.3)
- 通过端口映射,Flink 访问 127.0.0.1:8040
- 在配置文件中指定 `doris.be.nodes: "127.0.0.1:8040"`

### 问题 4: BE 磁盘容量为 0

**问题**: BE 刚启动时磁盘容量显示为 0,无法创建表。

**解决**: 
- 添加 `PRIORITY_NETWORKS` 环境变量
- 等待 BE 完成磁盘扫描 (约 30-60 秒)
- 确保 BE 完全初始化后再创建表

## 关键技术点

### 1. PRIORITY_NETWORKS 参数

```yaml
environment:
  - PRIORITY_NETWORKS=172.25.0.0/16
```

**作用**: 告诉 Doris 使用哪个网络接口进行通信,避免多网卡环境下的地址混乱。

### 2. FE_SERVERS 格式

```yaml
environment:
  - FE_SERVERS=fe1:172.25.0.2:9010  # 格式: name:ip:port
```

**注意**: 必须使用 `name:ip:port` 格式,否则启动失败。

### 3. 端口映射原理

```
Flink (宿主机) → 127.0.0.1:8040 
    ↓ (Docker 端口映射)
BE (容器内) → 172.25.0.3:8040
```

### 4. 官方 Connector 配置

```java
DorisOptions.Builder dorisBuilder = DorisOptions.builder()
    .setFenodes("127.0.0.1:8030")
    .setBenodes("127.0.0.1:8040");  // 强制指定 BE 地址
```
