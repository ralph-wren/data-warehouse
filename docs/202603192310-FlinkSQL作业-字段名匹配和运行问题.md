# Flink SQL DWD 作业 - 字段名匹配和运行问题

## 问题描述

Flink DWD SQL 作业虽然能够启动，但无法从 Kafka 消费数据并写入 Doris DWD 层。

### 问题现象

1. **作业启动成功**：显示 "✓ Flink DWD SQL 作业已启动"
2. **无日志输出**：启动后没有任何数据处理日志
3. **Kafka 消费组无活跃成员**：`flink-dwd-consumer-sql` 消费组没有活跃成员
4. **DWD 层无数据**：Doris DWD 表中没有任何数据

## 根本原因分析

### 1. 字段名不匹配问题 ✅ 已解决

**问题**：Kafka 消息使用驼峰命名，但 SQL 定义使用下划线命名

**Kafka JSON 格式**：
```json
{
  "instId": "BTC-USDT",
  "last": "69554.3",
  "askPx": "69554.3",
  "bidPx": "69554.2",
  "ts": "1773932328371",
  "open24h": "71288",
  "high24h": "72008",
  "low24h": "69089.1",
  "volCcy24h": "544708880.40368634"
}
```

**原 SQL 定义（错误）**：
```sql
CREATE TABLE kafka_ods_source (
    inst_id STRING,          -- 错误：应该是 instId
    `timestamp` BIGINT,      -- 错误：应该是 ts
    last_price DECIMAL(...), -- 错误：应该是 last
    bid_price DECIMAL(...),  -- 错误：应该是 bidPx
    ask_price DECIMAL(...)   -- 错误：应该是 askPx
)
```

**修复后的 SQL 定义**：
```sql
CREATE TABLE kafka_ods_source (
    instId STRING,           -- 匹配 JSON 字段名
    ts BIGINT,               -- 匹配 JSON 字段名
    `last` STRING,           -- 匹配 JSON 字段名（使用反引号因为是关键字）
    bidPx STRING,            -- 匹配 JSON 字段名
    askPx STRING,            -- 匹配 JSON 字段名
    bidSz STRING,
    askSz STRING,
    volCcy24h STRING,
    high24h STRING,
    low24h STRING,
    open24h STRING
)
```

### 2. 数据类型问题 ✅ 已解决

**问题**：Kafka JSON 中所有数值都是 STRING 类型，需要转换为 DECIMAL

**解决方案**：在 INSERT SQL 中添加类型转换
```sql
SELECT 
    instId as inst_id,
    ts as `timestamp`,
    CAST(`last` AS DECIMAL(20, 8)) as last_price,
    CAST(bidPx AS DECIMAL(20, 8)) as bid_price,
    CAST(askPx AS DECIMAL(20, 8)) as ask_price,
    ...
FROM kafka_ods_source
```

### 3. Flink SQL 本地运行问题 ⚠️ 未解决

**问题**：使用 `mvn exec:java` 运行 Flink SQL 作业时，作业无法正常消费 Kafka

**可能原因**：
1. **类加载问题**：`mvn exec:java` 的类加载器可能与 Flink 不兼容
2. **本地模式限制**：Flink SQL 在本地模式下可能存在限制
3. **异步执行问题**：`executeSql()` 返回后作业在后台运行，但可能没有正确初始化

**证据**：
- Kafka 消费组显示 "has no active members"
- CURRENT-OFFSET 始终为 0，说明从未消费过数据
- 作业进程存在，但没有任何数据处理日志

## 技术要点总结

### Flink SQL 字段名匹配规则

1. **JSON 格式**：字段名必须完全匹配（区分大小写）
2. **关键字处理**：使用反引号包裹关键字（如 `last`）
3. **类型转换**：JSON 中的 STRING 需要显式转换为数值类型

### Flink SQL 本地运行的限制

1. **类加载器**：`mvn exec:java` 可能导致类加载问题
2. **异步执行**：`executeSql()` 异步执行，需要调用 `await()`
3. **资源管理**：本地模式的资源管理可能不完整

### 推荐的开发流程

1. **开发阶段**：使用 DataStream API，便于调试
2. **测试阶段**：使用 Flink standalone 集群
3. **生产阶段**：使用 Flink on YARN/K8s
