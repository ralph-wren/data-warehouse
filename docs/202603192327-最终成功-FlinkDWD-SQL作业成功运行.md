## 解决的关键问题

### 1. JSON 依赖缺失 ✅

**问题**：`Could not find any factory for identifier 'json'`

**解决方案**：添加 flink-json 依赖
```xml
<dependency>
    <groupId>org.apache.flink</groupId>
    <artifactId>flink-json</artifactId>
    <version>${flink.version}</version>
</dependency>
```

### 2. Doris Sink 配置参数错误 ✅

**问题**：`Unsupported options: sink.batch.interval, sink.batch.size`

**解决方案**：使用正确的参数名
```java
'sink.buffer-flush.max-rows' = '1000',
'sink.buffer-flush.interval' = '5s',
```

### 3. 字段名不匹配 ✅

**问题**：Kafka JSON 使用驼峰命名，SQL 定义使用下划线命名

**Kafka JSON 格式**：
```json
{
  "instId": "BTC-USDT",
  "last": "69554.3",
  "ts": "1773932328371",
  "bidPx": "69554.2",
  "askPx": "69554.3"
}
```

**解决方案**：修改 SQL 定义匹配 JSON 字段名
```sql
CREATE TABLE kafka_ods_source (
    instId STRING,      -- 匹配 JSON
    ts BIGINT,          -- 匹配 JSON
    `last` STRING,      -- 匹配 JSON（使用反引号）
    bidPx STRING,       -- 匹配 JSON
    askPx STRING        -- 匹配 JSON
)
```

### 4. 数据类型转换 ✅

**问题**：Kafka JSON 中所有数值都是 STRING 类型

**解决方案**：在 INSERT SQL 中添加类型转换
```sql
SELECT 
    instId as inst_id,
    ts as `timestamp`,
    CAST(`last` AS DECIMAL(20, 8)) as last_price,
    CAST(bidPx AS DECIMAL(20, 8)) as bid_price,
    CAST(askPx AS DECIMAL(20, 8)) as ask_price,
    ...
FROM kafka_ods_source
```

### 5. BE 地址问题 ✅

**问题**：Doris Connector 自动从 FE 获取 BE 的 Docker 内部 IP `172.25.0.3:8040`

**解决方案**：在 Doris Sink DDL 中指定 benodes 参数
```java
'benodes' = '127.0.0.1:8040',  // 覆盖自动发现的 BE 地址
```

### 6. Doris 分区问题 ✅

**问题**：`no partition for this tuple` - 数据日期 2026-03-18 没有对应分区

**解决方案**：
1. 临时禁用动态分区
2. 手动添加历史分区
3. 重新启用动态分区

```sql
ALTER TABLE crypto_dw.dwd_crypto_ticker_detail SET ("dynamic_partition.enable" = "false");
ALTER TABLE crypto_dw.dwd_crypto_ticker_detail ADD PARTITION p20260318 VALUES [('2026-03-18'), ('2026-03-19'));
ALTER TABLE crypto_dw.dwd_crypto_ticker_detail SET ("dynamic_partition.enable" = "true");
```

或者启用历史分区自动创建：
```sql
ALTER TABLE crypto_dw.dwd_crypto_ticker_detail SET ('dynamic_partition.create_history_partition' = 'true');
```

```java
           "    inst_id STRING,\n" +
           "    `timestamp` BIGINT,\n" +
           "    trade_date DATE,\n" +
           "    trade_hour INT,\n" +
           "    ...\n" +
           ") WITH (\n" +
           "    'connector' = 'doris',\n" +
           "    'fenodes' = '" + feNodes + "',\n" +
           "    'benodes' = '" + beNodes + "',\n" +  // 关键参数
           "    'table.identifier' = '" + database + "." + table + "',\n" +
           "    'sink.buffer-flush.max-rows' = '1000',\n" +
           "    'sink.buffer-flush.interval' = '5s',\n" +
           "    'sink.max-retries' = '3',\n" +
           "    'sink.properties.format' = 'json',\n" +
           "    'sink.properties.read_json_by_line' = 'true'\n" +
           ")";
}

// 3. INSERT SQL（类型转换）
private static String createInsertSQL() {
    return "INSERT INTO doris_dwd_sink\n" +
           "SELECT \n" +
           "    instId as inst_id,\n" +
           "    ts as `timestamp`,\n" +
           "    CAST(TO_TIMESTAMP(FROM_UNIXTIME(ts / 1000)) AS DATE) as trade_date,\n" +
           "    CAST(EXTRACT(HOUR FROM TO_TIMESTAMP(FROM_UNIXTIME(ts / 1000))) AS INT) as trade_hour,\n" +
           "    CAST(`last` AS DECIMAL(20, 8)) as last_price,\n" +
           "    CAST(bidPx AS DECIMAL(20, 8)) as bid_price,\n" +
           "    CAST(askPx AS DECIMAL(20, 8)) as ask_price,\n" +
           "    CAST((CAST(askPx AS DECIMAL(20, 8)) - CAST(bidPx AS DECIMAL(20, 8))) AS DECIMAL(20, 8)) as spread,\n" +
           "    ...\n" +
           "FROM kafka_ods_source\n" +
           "WHERE CAST(`last` AS DECIMAL(20, 8)) > 0";
}
```

## 技术要点总结

### 1. Flink SQL 字段名匹配

- JSON 格式的字段名必须完全匹配（区分大小写）
- 使用反引号包裹 SQL 关键字（如 `last`）
- 字段名转换在 SELECT 中进行（如 `instId as inst_id`）

### 2. Flink SQL 类型转换

- JSON 中的 STRING 需要显式 CAST 为数值类型
- 时间戳转换：`TO_TIMESTAMP(FROM_UNIXTIME(ts / 1000))`
- 日期提取：`CAST(... AS DATE)`
- 小时提取：`CAST(EXTRACT(HOUR FROM ...) AS INT)`

### 3. Doris Connector 配置

- `fenodes`：FE 节点地址（用于元数据查询）
- `benodes`：BE 节点地址（用于数据写入，覆盖自动发现）
- `sink.buffer-flush.max-rows`：批量写入行数
- `sink.buffer-flush.interval`：批量写入时间间隔

### 4. Doris 动态分区管理

- `dynamic_partition.start = -7`：保留过去 7 天
- `dynamic_partition.end = 3`：预创建未来 3 天
- `dynamic_partition.create_history_partition = true`：自动创建历史分区
- 手动添加分区需要先禁用动态分区
