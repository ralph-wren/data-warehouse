# 日志配置 - 滚动日志和 Flink Web UI 配置

**时间**: 2026-03-20 00:00  
**问题类型**: 日志管理和监控  
**严重程度**: 中 - 改善运维体验

---

## 需求描述

1. **日志滚动**：每 3 小时生成一个新日志文件
2. **日志分类**：每种日志一个目录
3. **日志清理**：3 天过期自动删除
4. **Web UI**：本地启动能看到 Flink Web UI

## 实施方案

### 1. 日志配置（Log4j2）

#### 目录结构

```
logs/
├── app/          # 应用日志
│   ├── flink-app.log
│   ├── flink-app-2026-03-20-00-0.log.gz
│   └── flink-app-2026-03-20-03-0.log.gz
├── flink/        # Flink 框架日志
│   ├── flink.log
│   └── flink-2026-03-20-00-0.log.gz
├── doris/        # Doris 连接器日志
│   ├── doris.log
│   └── doris-2026-03-20-00-0.log.gz
├── kafka/        # Kafka 连接器日志
│   ├── kafka.log
│   └── kafka-2026-03-20-00-0.log.gz
└── error/        # 错误日志（ERROR 级别）
    ├── error.log
    └── error-2026-03-20-00-0.log.gz
```

#### Log4j2 配置文件

**文件位置**: `src/main/resources/log4j2.properties`

**关键配置**:

```properties
# 1. 滚动文件 Appender
appender.app.type = RollingFile
appender.app.name = AppFileAppender
appender.app.fileName = logs/app/flink-app.log
appender.app.filePattern = logs/app/flink-app-%d{yyyy-MM-dd-HH}-%i.log.gz

# 2. 滚动策略：每 3 小时或文件大小超过 100MB
appender.app.policies.type = Policies
appender.app.policies.time.type = TimeBasedTriggeringPolicy
appender.app.policies.time.interval = 3  # 3 小时
appender.app.policies.time.modulate = true  # 对齐到整点
appender.app.policies.size.type = SizeBasedTriggeringPolicy
appender.app.policies.size.size = 100MB

# 3. 删除策略：保留 3 天
appender.app.strategy.type = DefaultRolloverStrategy
appender.app.strategy.delete.type = Delete
appender.app.strategy.delete.basePath = logs/app
appender.app.strategy.delete.maxDepth = 1
appender.app.strategy.delete.ifLastModified.type = IfLastModified
appender.app.strategy.delete.ifLastModified.age = 3d  # 3 天
```

#### 日志分类

| 日志类型 | 目录 | 包名 | 级别 |
|---------|------|------|------|
| 应用日志 | logs/app/ | com.crypto.dw | DEBUG |
| Flink 日志 | logs/flink/ | org.apache.flink | INFO |
| Doris 日志 | logs/doris/ | org.apache.doris | INFO |
| Kafka 日志 | logs/kafka/ | org.apache.kafka | WARN |
| 错误日志 | logs/error/ | 所有包 | ERROR |

```properties
logger.crypto.additivity = false
logger.crypto.appenderRef.console.ref = ConsoleAppender

logger.flink.appenderRef.console.ref = ConsoleAppender
logger.flink.appenderRef.flink.ref = FlinkFileAppender

```

### 2. Flink Web UI 配置

#### 依赖配置

**文件位置**: `pom.xml`

```xml
<!-- Flink Runtime Web (用于本地启动 Web UI) -->
<dependency>
    <groupId>org.apache.flink</groupId>
    <artifactId>flink-runtime-web</artifactId>
    <version>${flink.version}</version>
    <scope>compile</scope>
</dependency>

<!-- Flink Clients (用于本地执行环境) -->
<dependency>
    <groupId>org.apache.flink</groupId>
    <artifactId>flink-clients</artifactId>
    <version>${flink.version}</version>
    <scope>compile</scope>
</dependency>
```

#### 应用配置

**文件位置**: `config/application.yml`

```yaml
flink:
  # Web UI 配置
  web:
    enabled: true  # 启用 Web UI
    port: 8081  # Web UI 端口
    address: 0.0.0.0  # 监听所有网络接口
    submit:
      enable: true  # 允许通过 Web UI 提交作业
    cancel:
      enable: true  # 允许通过 Web UI 取消作业
    refresh-interval: 3000  # 刷新间隔（毫秒）
```

#### 代码配置

**文件位置**: `src/main/java/com/crypto/dw/flink/FlinkDWDJobSQL.java`

```java
// 创建 Flink 执行环境（启用 Web UI）
Configuration flinkConfig = new Configuration();

// 启用 Web UI
flinkConfig.setBoolean("web.submit.enable", true);
flinkConfig.setBoolean("web.cancel.enable", true);
flinkConfig.setInteger("rest.port", 8081);
flinkConfig.setString("rest.address", "localhost");
flinkConfig.setInteger("rest.bind-port", "8081");

StreamExecutionEnvironment env = StreamExecutionEnvironment.createLocalEnvironmentWithWebUI(flinkConfig);
```

## 配置说明

### 1. 时间滚动策略

```properties
appender.app.policies.time.type = TimeBasedTriggeringPolicy
appender.app.policies.time.interval = 3  # 滚动间隔（小时）
appender.app.policies.time.modulate = true  # 对齐到整点
```

**说明**：
- `interval = 3`：每 3 小时滚动一次
- `modulate = true`：对齐到 0:00, 3:00, 6:00, 9:00, 12:00, 15:00, 18:00, 21:00
- 文件名格式：`flink-app-2026-03-20-00-0.log.gz`（年-月-日-小时-序号）

### 2. 大小滚动策略

```properties
appender.app.policies.size.type = SizeBasedTriggeringPolicy
appender.app.policies.size.size = 100MB
```

**说明**：
- 当文件大小超过 100MB 时，即使未到 3 小时也会滚动
- 避免单个文件过大

### 3. 删除策略

```properties
appender.app.strategy.delete.type = Delete
appender.app.strategy.delete.basePath = logs/app
appender.app.strategy.delete.maxDepth = 1
appender.app.strategy.delete.ifLastModified.type = IfLastModified
appender.app.strategy.delete.ifLastModified.age = 3d
```

**说明**：
- `basePath`：扫描的基础目录
- `maxDepth = 1`：只扫描一级子目录
- `age = 3d`：删除 3 天前的文件

### 4. 压缩配置

```properties
appender.app.filePattern = logs/app/flink-app-%d{yyyy-MM-dd-HH}-%i.log.gz
```

**说明**：
- `.gz` 后缀：自动使用 GZIP 压缩
- 压缩比约 10:1，节省磁盘空间

### 5. Web UI 端口

```java
flinkConfig.setInteger("rest.port", 8081);
```

**说明**：
- 默认端口：8081
- 如果端口被占用，可以修改为其他端口（如 8082, 8083）
- 确保防火墙允许该端口

## 故障排查

### 问题 1：日志文件未滚动

**检查**：
```bash
# 查看日志文件修改时间
ls -lh logs/app/

# 查看 Log4j2 配置
cat src/main/resources/log4j2.properties | grep -A 5 "policies"
```

**可能原因**：
1. 时间间隔配置错误
2. Log4j2 配置未生效
3. 文件权限问题

**解决**：
```bash
# 重新编译项目
mvn clean compile

# 检查文件权限
chmod 755 logs/
chmod 644 logs/*/*.log
```

### 问题 2：旧日志未删除

**检查**：
```bash
# 查看 3 天前的日志
find logs/ -name "*.log.gz" -mtime +3

# 查看删除策略配置
cat src/main/resources/log4j2.properties | grep -A 5 "delete"
```

**可能原因**：
1. 删除策略配置错误
2. 文件权限问题
3. Log4j2 版本不支持

**解决**：
```bash
# 手动删除旧日志
find logs/ -name "*.log.gz" -mtime +3 -delete

# 添加定时任务
crontab -e
# 每天凌晨 2 点清理
0 2 * * * find /path/to/logs/ -name "*.log.gz" -mtime +3 -delete
```

### 问题 3：Web UI 无法访问

**检查**：
```bash
# 检查端口是否被占用
netstat -an | grep 8081

# 检查 Flink 进程
jps -l | grep Flink

# 查看日志
tail -f logs/flink/flink.log | grep "rest"
```

**可能原因**：
1. 端口被占用
2. 防火墙阻止
3. Web UI 未启用

**解决**：
```bash
# 修改端口
# 在代码中修改 rest.port 为其他端口

# 关闭防火墙（Windows）
netsh advfirewall set allprofiles state off

# 检查 Web UI 配置
cat config/application.yml | grep -A 5 "web"
```

### 问题 4：日志文件过大

**检查**：
```bash
# 查看大文件
find logs/ -name "*.log" -size +100M

# 查看总大小
du -sh logs/
```

**可能原因**：
1. 日志级别过低（DEBUG）
2. 滚动策略未生效
3. 日志量过大

**解决**：
```properties
# 调整日志级别
logger.crypto.level = INFO  # 从 DEBUG 改为 INFO

# 减小滚动间隔
appender.app.policies.time.interval = 1  # 从 3 小时改为 1 小时

# 减小文件大小限制
appender.app.policies.size.size = 50MB  # 从 100MB 改为 50MB
```
