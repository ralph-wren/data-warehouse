# 性能优化 - 修复批量配置提升写入速度

**时间**: 2026-03-20 00:45  
**问题**: Doris Stream Load 写入速度慢，363 条数据需要 60 秒  
**状态**: 已解决

---

## 问题现象

从 ODS 作业的日志中发现：

```
"LoadTimeMs": 60357,
"WriteDataTimeMs": 56826,
"NumberTotalRows": 363,
"NumberLoadedRows": 363
```

**关键问题**:
- 363 条数据写入耗时 60 秒
- 写入时间占 56 秒
- 平均每条数据 165ms，性能极差

## 问题分析

### 根本原因

**批量配置错误** - 缓冲区大小设置不当

1. **ODS DataStream 作业**:
```java
// 错误的配置
.setBufferSize(config.getInt("doris.stream-load.batch-size", 1000) * 1024)
```

问题：
- `batch-size` 配置值是 1000（行数）
- 代码中乘以 1024，变成了 1MB 的缓冲区
- 1MB 对于 JSON 数据来说太小，导致频繁 flush
- 每次 flush 都需要一次 HTTP 请求，开销很大

2. **DWD SQL 作业**:
- 缺少 `sink.buffer-size` 配置
- 使用默认的小缓冲区，同样导致频繁 flush

### Doris Connector 批量配置说明

Doris Flink Connector 有三个关键参数控制批量写入：

| 参数 | 类型 | 说明 | 推荐值 |
|-----|------|------|--------|
| `setBufferSize` / `sink.buffer-size` | 字节数 | 缓冲区大小，达到后触发 flush | 10MB (10485760) |
| `setBufferFlushMaxRows` / `sink.buffer-flush.max-rows` | 行数 | 批量行数，达到后触发 flush | 1000-5000 |
| `setBufferFlushIntervalMs` / `sink.buffer-flush.interval` | 毫秒 | 批量间隔，超时后触发 flush | 5000-10000 |

**触发条件**: 满足任一条件即触发 flush
- 缓冲区大小达到 `BufferSize`
- 行数达到 `BufferFlushMaxRows`
- 时间超过 `BufferFlushIntervalMs`

## 解决方案

### 1. 修复 ODS DataStream 作业

**文件**: `src/main/java/com/crypto/dw/flink/FlinkODSJobDataStream.java`

```java
// 批量写入配置
int batchSize = config.getInt("doris.stream-load.batch-size", 1000);  // 批量行数
int batchIntervalMs = config.getInt("doris.stream-load.batch-interval-ms", 5000);  // 批量间隔（毫秒）

DorisExecutionOptions executionOptions = DorisExecutionOptions.builder()
    .setStreamLoadProp(streamLoadProp)
    .setMaxRetries(config.getInt("doris.stream-load.max-retries", 3))
    // 修复：setBufferSize 是字节数，设置为 10MB
    .setBufferSize(10 * 1024 * 1024)  // 缓冲区大小：10MB
    .setBufferCount(3)  // 缓冲区数量
    // 修复：使用 setBufferFlushMaxRows 设置批量行数
    .setBufferFlushMaxRows(batchSize)  // 批量行数：1000 行
    // 修复：使用 setBufferFlushIntervalMs 设置批量间隔
    .setBufferFlushIntervalMs(batchIntervalMs)  // 批量间隔：5000ms
    .setLabelPrefix("flink-ods-" + System.currentTimeMillis())
    .build();
```

**关键修改**:
- ✅ `setBufferSize(10 * 1024 * 1024)` - 10MB 缓冲区
- ✅ `setBufferFlushMaxRows(batchSize)` - 1000 行批量
- ✅ `setBufferFlushIntervalMs(batchIntervalMs)` - 5 秒间隔

## 调优建议

### 1. 缓冲区大小 (BufferSize)

**作用**: 控制内存使用和 flush 频率

| 场景 | 推荐值 | 说明 |
|-----|--------|------|
| 小数据量 | 5-10MB | 减少内存占用 |
| 中等数据量 | 10-50MB | 平衡性能和内存 |
| 大数据量 | 50-100MB | 最大化批量写入 |

**注意**: 
- 太小：频繁 flush，性能差
- 太大：内存占用高，OOM 风险

### 2. 批量行数 (BufferFlushMaxRows)

**作用**: 控制每批写入的行数

| 场景 | 推荐值 | 说明 |
|-----|--------|------|
| 小记录 | 5000-10000 | 每条记录小于 1KB |
| 中等记录 | 1000-5000 | 每条记录 1-10KB |
| 大记录 | 100-1000 | 每条记录大于 10KB |

**注意**:
- 太小：频繁 flush
- 太大：单次请求时间长，超时风险

### 3. 批量间隔 (BufferFlushIntervalMs)

**作用**: 控制最大延迟

| 场景 | 推荐值 | 说明 |
|-----|--------|------|
| 实时性要求高 | 1000-3000ms | 低延迟 |
| 平衡 | 5000-10000ms | 推荐 |
| 吞吐量优先 | 10000-30000ms | 高吞吐 |

**注意**:
- 太小：频繁 flush，性能差
- 太大：延迟高，数据积压
