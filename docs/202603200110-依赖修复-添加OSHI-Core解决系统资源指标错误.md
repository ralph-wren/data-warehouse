# 依赖修复 - 添加 OSHI Core 解决系统资源指标错误

**时间**: 2026-03-20 01:10  
**问题**: 启用系统资源指标时出现 NoClassDefFoundError: oshi/SystemInfo  
**状态**: ✅ 已解决

---

## 问题描述

在集成 Prometheus 监控后，启动 Flink 作业时出现以下错误：

```
[main] SystemResourcesMetricsInitializer:53 - Failed to initialize system resource metrics 
because of missing class definitions. Did you forget to explicitly add the oshi-core optional dependency?

java.lang.NoClassDefFoundError: oshi/SystemInfo
    at org.apache.flink.runtime.metrics.util.SystemResourcesCounter.<init>(SystemResourcesCounter.java:51)
    at org.apache.flink.runtime.metrics.util.SystemResourcesMetricsInitializer.instantiateSystemMetrics(SystemResourcesMetricsInitializer.java:40)
    ...
```

## 问题原因

在 `MetricsConfig.java` 中启用了系统资源指标：

```java
// 启用系统资源指标
config.setBoolean(MetricOptions.SYSTEM_RESOURCE_METRICS, true);
```

但是 `oshi-core` 是 Flink 的可选依赖，需要手动添加到 `pom.xml` 中。

**OSHI (Operating System and Hardware Information)** 是一个用于获取系统和硬件信息的 Java 库，Flink 使用它来收集：
- CPU 使用率
- 内存使用情况
- 磁盘 I/O
- 网络流量
- 系统负载

## 解决方案

### 1. 添加 oshi-core 依赖

在 `pom.xml` 中添加 `oshi-core` 依赖：

```xml
<!-- OSHI Core - 用于系统资源指标收集 -->
<dependency>
    <groupId>com.github.oshi</groupId>
    <artifactId>oshi-core</artifactId>
    <version>6.4.0</version>
</dependency>
```

**版本说明**:
- OSHI 6.4.0 是当前稳定版本
- 兼容 Flink 1.17.2
- 支持 Windows、Linux、macOS

### 2. 重新编译项目

```bash
mvn clean compile -DskipTests
```

**编译结果**:
```
[INFO] BUILD SUCCESS
[INFO] Total time: 7.483 s
```

### 3. 验证修复

启动 Flink 作业后，不再出现 `NoClassDefFoundError` 错误。

## OSHI 依赖说明

```
oshi-core:6.4.0
```

|-----|------|---------|
| CPU 信息 | 处理器型号、核心数、使用率 | Windows, Linux, macOS |
| 内存信息 | 总内存、可用内存、使用率 | Windows, Linux, macOS |
| 磁盘信息 | 磁盘容量、读写速度 | Windows, Linux, macOS |
| 网络信息 | 网络接口、流量统计 | Windows, Linux, macOS |
| 进程信息 | 进程列表、CPU/内存占用 | Windows, Linux, macOS |
| 系统信息 | 操作系统、版本、启动时间 | Windows, Linux, macOS |

## 系统资源指标

启用 OSHI 后，Flink 会收集以下系统资源指标：

### CPU 指标

```
flink_taskmanager_Status_JVM_CPU_Load
flink_taskmanager_Status_JVM_CPU_Time
```

### 内存指标

```
flink_taskmanager_Status_JVM_Memory_Heap_Used
flink_taskmanager_Status_JVM_Memory_Heap_Max
flink_taskmanager_Status_JVM_Memory_Heap_Committed
flink_taskmanager_Status_JVM_Memory_NonHeap_Used
flink_taskmanager_Status_JVM_Memory_NonHeap_Max
```

### 线程指标

```
flink_taskmanager_Status_JVM_Threads_Count
```

### GC 指标

```
flink_taskmanager_Status_JVM_GarbageCollector_G1_Young_Generation_Count
flink_taskmanager_Status_JVM_GarbageCollector_G1_Young_Generation_Time
flink_taskmanager_Status_JVM_GarbageCollector_G1_Old_Generation_Count
flink_taskmanager_Status_JVM_GarbageCollector_G1_Old_Generation_Time
```

## 配置选项

### 启用/禁用系统资源指标

```java
// 启用（默认）
config.setBoolean(MetricOptions.SYSTEM_RESOURCE_METRICS, true);

// 禁用（如果不需要系统资源指标）
config.setBoolean(MetricOptions.SYSTEM_RESOURCE_METRICS, false);
```

### 调整采集间隔

```java
// 默认 5 秒
config.setLong(MetricOptions.SYSTEM_RESOURCE_METRICS_PROBING_INTERVAL, 5000L);

// 增加到 10 秒（减少开销）
config.setLong(MetricOptions.SYSTEM_RESOURCE_METRICS_PROBING_INTERVAL, 10000L);
```

## 替代方案

如果不想添加 OSHI 依赖，可以禁用系统资源指标：

### 方案 1: 修改 MetricsConfig.java

```java
public static void configureCommonMetrics(Configuration config) {
    // 禁用系统资源指标
    config.setBoolean(MetricOptions.SYSTEM_RESOURCE_METRICS, false);
    
    // 其他配置保持不变
    config.setString("metrics.scope.jm", "<host>.jobmanager");
    // ...
}
```

### 方案 2: 只使用 JVM 指标

JVM 指标不需要 OSHI，由 Flink 内置提供：
- JVM 堆内存使用
- JVM GC 统计
- JVM 线程数

但无法获取：
- 系统 CPU 使用率
- 系统内存使用率
- 磁盘 I/O
- 网络流量

## 最佳实践

### 1. 生产环境建议

**启用系统资源指标**:
- 可以全面监控系统状态
- 及时发现资源瓶颈
- 性能开销可接受（< 1% CPU）

### 2. 开发环境建议

**可选**:
- 如果只关注业务指标，可以禁用
- 减少依赖和启动时间

### 3. 监控告警

基于系统资源指标设置告警：

```promql
# CPU 使用率 > 80%
flink_taskmanager_Status_JVM_CPU_Load > 0.8

# 堆内存使用率 > 90%
flink_taskmanager_Status_JVM_Memory_Heap_Used / 
flink_taskmanager_Status_JVM_Memory_Heap_Max > 0.9

# GC 时间占比 > 30%
rate(flink_taskmanager_Status_JVM_GarbageCollector_G1_Young_Generation_Time[1m]) / 1000 > 0.3
```

## 故障排查

### 1. 依赖下载失败

**问题**: Maven 无法下载 oshi-core

**解决**:
```bash
# 清理本地仓库
rm -rf ~/.m2/repository/com/github/oshi

# 重新下载
mvn dependency:resolve
```

### 2. 版本冲突

**问题**: oshi-core 与其他依赖冲突

**解决**:
```bash
# 查看依赖树
mvn dependency:tree | grep oshi

# 排除冲突依赖
<dependency>
    <groupId>com.github.oshi</groupId>
    <artifactId>oshi-core</artifactId>
    <version>6.4.0</version>
    <exclusions>
        <exclusion>
            <groupId>org.slf4j</groupId>
            <artifactId>slf4j-api</artifactId>
        </exclusion>
    </exclusions>
</dependency>
```

### 3. 平台不支持

**问题**: 某些平台可能不支持 OSHI

**解决**: 禁用系统资源指标
```java
config.setBoolean(MetricOptions.SYSTEM_RESOURCE_METRICS, false);
```
