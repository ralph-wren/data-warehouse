# 配置读取 - 解决启动脚本覆盖配置文件问题

**时间**: 2026-03-20 02:10  
**问题**: 采集脚本不带参数启动时，无法从配置文件读取多个交易对  
**状态**: ✅ 已解决

## 问题描述

用户在 `application-dev.yml` 配置文件中设置了 4 个交易对：
```yaml
okx:
  symbols: BTC-USDT,ETH-USDT,SOL-USDT,BNB-USDT
```

但是不带参数运行 `bash run-collector.sh` 时，只订阅了 `BTC-USDT`，配置文件未生效。

## 根本原因

`run-collector.sh` 脚本在第 60-64 行有问题：

```bash
if [ $# -eq 0 ]; then
    echo "订阅交易对: BTC-USDT (默认)"
    SYMBOLS="BTC-USDT"  # ❌ 强制设置为 BTC-USDT
else
    echo "订阅交易对: $@"
    SYMBOLS="$*"
fi

# 运行主程序
mvn exec:java \
    -Dexec.mainClass="com.crypto.dw.collector.DataCollectorMain" \
    -Dexec.args="$SYMBOLS" \  # ❌ 总是传递参数
    -Dexec.cleanupDaemonThreads=false
```

问题：
1. 不带参数时，脚本强制设置 `SYMBOLS="BTC-USDT"`
2. 然后通过 `-Dexec.args="$SYMBOLS"` 传递给 Java 程序
3. Java 程序的 `getSymbols` 方法优先使用命令行参数，导致配置文件被忽略

## 解决方案

### 1. 修改启动脚本

修改 `run-collector.sh`，不带参数时不传递任何参数给 Java 程序：

```bash
# 运行数据采集器
echo "启动数据采集器..."
if [ $# -eq 0 ]; then
    # 不带参数时，从配置文件读取交易对
    echo "订阅交易对: 从配置文件读取 (application-$APP_ENV.yml)"
    echo ""
    echo "按 Ctrl+C 停止"
    echo "=========================================="
    echo ""
    
    # 运行主程序（不传递参数）
    mvn exec:java \
        -Dexec.mainClass="com.crypto.dw.collector.DataCollectorMain" \
        -Dexec.cleanupDaemonThreads=false
else
    # 带参数时，使用命令行参数
    echo "订阅交易对: $@"
    echo ""
    echo "按 Ctrl+C 停止"
    echo "=========================================="
    echo ""
    
    # 运行主程序（传递参数）
    mvn exec:java \
        -Dexec.mainClass="com.crypto.dw.collector.DataCollectorMain" \
        -Dexec.args="$*" \
        -Dexec.cleanupDaemonThreads=false
fi
```

关键改动：
- 不带参数时：不传递 `-Dexec.args`，让 Java 程序的 `args` 数组为空
- 带参数时：传递 `-Dexec.args="$*"`，使用命令行参数

### 2. 添加调试日志

在 `DataCollectorMain.java` 的 `getSymbols` 方法中添加调试日志：

```java
// 尝试从配置文件读取（支持逗号分隔的字符串）
String symbolsConfig = config.getString("okx.symbols", "");
logger.info("Reading okx.symbols from config: '{}'", symbolsConfig); // 添加调试日志

if (!symbolsConfig.isEmpty()) {
    // 支持逗号分隔的多个交易对
    String[] symbolArray = symbolsConfig.split(",");
    List<String> symbols = new ArrayList<>();
    for (String symbol : symbolArray) {
        String trimmed = symbol.trim();
        if (!trimmed.isEmpty()) {
            symbols.add(trimmed);
        }
    }
    if (!symbols.isEmpty()) {
        logger.info("Using symbols from config file: {}", symbols);
        return symbols;
    }
}
```

## 配置优先级

Java 程序的配置读取优先级：

1. **命令行参数**（最高优先级）
   ```bash
   bash run-collector.sh BTC-USDT ETH-USDT
   ```

2. **配置文件** `application-dev.yml`
   ```yaml
   okx:
     symbols: BTC-USDT,ETH-USDT,SOL-USDT,BNB-USDT
   ```

3. **默认值**（最低优先级）
   ```java
   List<String> defaultSymbols = Arrays.asList("BTC-USDT", "ETH-USDT", "SOL-USDT", "BNB-USDT");
   ```

## 使用方式

### 方式 1: 从配置文件读取（推荐）
```bash
# 不带参数，从 application-dev.yml 读取
bash run-collector.sh
```

### 方式 2: 使用命令行参数
```bash
# 带参数，覆盖配置文件
bash run-collector.sh BTC-USDT ETH-USDT SOL-USDT BNB-USDT
```

### 方式 3: 修改配置文件
编辑 `config/application-dev.yml` 或 `src/main/resources/config/application-dev.yml`：
```yaml
okx:
  symbols: BTC-USDT,ETH-USDT,SOL-USDT,BNB-USDT,XRP-USDT,ADA-USDT
```

## 关于 Kafka 分区分布

### Hash 分区机制
Kafka 使用 key 的 hash 值决定分区：
```
partition = hash(key) % partition_count
```

### 当前分布
4 个交易对 hash 到 2 个分区：
- BTC-USDT → 分区 2
- ETH-USDT, SOL-USDT, BNB-USDT → 分区 1

### 如何使用所有 4 个分区
需要订阅更多交易对（8-12 个），让 hash 值分布更均匀：
```yaml
okx:
  symbols: BTC-USDT,ETH-USDT,SOL-USDT,BNB-USDT,XRP-USDT,ADA-USDT,DOGE-USDT,DOT-USDT
```

## 总结

问题的根本原因是启动脚本在不带参数时强制传递了 `BTC-USDT` 参数，导致配置文件被忽略。

解决方案是修改脚本逻辑：
- 不带参数时：不传递任何参数，让 Java 程序从配置文件读取
- 带参数时：传递命令行参数，覆盖配置文件

现在配置文件可以正常工作，4 个交易对成功订阅并分布到 2 个 Kafka 分区。
