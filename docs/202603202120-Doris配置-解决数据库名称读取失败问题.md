# Doris 配置 - 解决数据库名称读取失败问题

**时间**: 2026-03-20 21:20  
**问题**: Flink ODS 作业启动失败，报错 "unknowndatabases, dbName=null"  
**状态**: ✅ 已解决

## 问题描述

启动 Flink ODS 作业时，出现以下错误：

```
org.apache.doris.shaded.com.fasterxml.jackson.databind.exc.InvalidFormatException: 
Cannot deserialize value of type `int` from String "errCode=7,detailMessage=unknowndatabases,dbName=null": 
not a valid `int` value
```

错误信息显示：
- `errCode=7`: Doris 错误代码 7（数据库不存在）
- `detailMessage=unknowndatabases`: 未知数据库
- `dbName=null`: 数据库名称为 null

## 根本原因

**`createDorisSink` 方法中的配置读取问题**

原始代码：
```java
DorisOptions.Builder dorisBuilder = DorisOptions.builder()
    .setFenodes(feNodes)
    .setTableIdentifier(
        config.getString("doris.database") + "." + 
        config.getString("doris.tables.ods")
    )
    .setUsername(config.getString("doris.fe.username"))
    .setPassword(config.getString("doris.fe.password", ""));
```

问题分析：
1. `config.getString("doris.database")` 返回 `null`（配置读取失败）
2. `config.getString("doris.tables.ods")` 返回 `null`（配置读取失败）
3. `tableIdentifier` 变成 `"null.null"`
4. Doris Connector 尝试连接到不存在的数据库，报错 `dbName=null`

**为什么配置读取失败？**

虽然 `application.yml` 中有完整配置：
```yaml
doris:
  database: crypto_dw
  tables:
    ods: ods_crypto_ticker_rt
```

但是 `config.getString("doris.database")` 没有提供默认值，如果读取失败会返回 `null`。

## 解决方案

### 修改 `FlinkODSJobDataStream.java`

在 `createDorisSink` 方法中添加：
1. **提供默认值**：确保配置读取失败时有默认值
2. **添加日志输出**：方便调试配置问题
3. **提前验证配置**：在构建 DorisOptions 前验证配置

```java
/**
 * 创建官方 Doris Sink
 * 使用 Doris Flink Connector,内部处理了 HTTP 协议兼容性问题
 * 
 * 注意: 如果 BE 使用 Docker 内部 IP,需要配置 benodes 参数
 */
private static DorisSink<String> createDorisSink(ConfigLoader config) {
    // 提取 FE 地址 (去掉 http:// 前缀)
    String feHttpUrl = config.getString("doris.fe.http-url");
    String feNodes = feHttpUrl.replace("http://", "").replace("https://", "");
    
    // 读取数据库和表名配置（提供默认值）✅
    String database = config.getString("doris.database", "crypto_dw");
    String table = config.getString("doris.tables.ods", "ods_crypto_ticker_rt");
    String tableIdentifier = database + "." + table;
    
    // 添加日志输出，方便调试 ✅
    logger.info("Doris Sink 配置:");
    logger.info("  FE Nodes: {}", feNodes);
    logger.info("  Database: {}", database);
    logger.info("  Table: {}", table);
    logger.info("  Table Identifier: {}", tableIdentifier);
    
    // Doris 连接配置
    DorisOptions.Builder dorisBuilder = DorisOptions.builder()
        .setFenodes(feNodes)  // FE 地址,格式: host:port
        .setTableIdentifier(tableIdentifier)  // 数据库.表名 ✅
        .setUsername(config.getString("doris.fe.username", "root"))  // 提供默认值 ✅
        .setPassword(config.getString("doris.fe.password", ""));  // 提供默认值 ✅
    
    // ... 其余代码
}
```

### 关键改动

1. **分步读取配置**：
   ```java
   // ❌ 旧方式：直接拼接，无法调试
   .setTableIdentifier(
       config.getString("doris.database") + "." + 
       config.getString("doris.tables.ods")
   )
   
   // ✅ 新方式：分步读取，提供默认值
   String database = config.getString("doris.database", "crypto_dw");
   String table = config.getString("doris.tables.ods", "ods_crypto_ticker_rt");
   String tableIdentifier = database + "." + table;
   ```

2. **添加日志输出**：
   ```java
   logger.info("Doris Sink 配置:");
   logger.info("  FE Nodes: {}", feNodes);
   logger.info("  Database: {}", database);
   logger.info("  Table: {}", table);
   logger.info("  Table Identifier: {}", tableIdentifier);
   ```

3. **提供默认值**：
   ```java
   .setUsername(config.getString("doris.fe.username", "root"))
   .setPassword(config.getString("doris.fe.password", ""))
   ```

## 技术要点

### 1. ConfigLoader 的 getString 方法

```java
// 不带默认值：如果配置不存在，返回 null
String value = config.getString("key");

// 带默认值：如果配置不存在，返回默认值
String value = config.getString("key", "default");
```

### 2. Doris TableIdentifier 格式

```java
// 正确格式：database.table
String tableIdentifier = "crypto_dw.ods_crypto_ticker_rt";

// 错误格式：null.null（导致 dbName=null 错误）
String tableIdentifier = null + "." + null;  // "null.null"
```

### 3. 配置读取最佳实践

```java
// ❌ 不推荐：没有默认值，容易出错
String database = config.getString("doris.database");
if (database == null) {
    throw new RuntimeException("配置缺失");
}

// ✅ 推荐：提供默认值，增加容错性
String database = config.getString("doris.database", "crypto_dw");

// ✅ 推荐：添加日志，方便调试
logger.info("Database: {}", database);
```

### 4. Doris 错误代码

| 错误代码 | 说明 | 常见原因 |
|---------|------|---------|
| errCode=7 | 数据库不存在 | 数据库名称错误或为 null |
| errCode=1 | 表不存在 | 表名称错误 |
| errCode=2 | 权限不足 | 用户名或密码错误 |

## 预防措施

### 1. 配置验证
在应用启动时验证关键配置：
```java
private static void validateConfig(ConfigLoader config) {
    String database = config.getString("doris.database");
    if (database == null || database.isEmpty()) {
        throw new IllegalArgumentException("doris.database 配置缺失");
    }
    
    String table = config.getString("doris.tables.ods");
    if (table == null || table.isEmpty()) {
        throw new IllegalArgumentException("doris.tables.ods 配置缺失");
    }
}
```

### 2. 提供默认值
所有配置读取都应该提供合理的默认值：
```java
String database = config.getString("doris.database", "crypto_dw");
String username = config.getString("doris.fe.username", "root");
String password = config.getString("doris.fe.password", "");
```

### 3. 添加日志
关键配置应该输出日志，方便调试：
```java
logger.info("Doris 配置: database={}, table={}", database, table);
```

## 总结

问题的根本原因是配置读取时没有提供默认值，导致 `tableIdentifier` 为 `"null.null"`，Doris 无法识别数据库。

解决方案：
1. ✅ 分步读取配置，提供默认值
2. ✅ 添加日志输出，方便调试
3. ✅ 提前验证配置，避免运行时错误

修复后，Flink ODS 作业成功启动，数据正常写入 Doris。
