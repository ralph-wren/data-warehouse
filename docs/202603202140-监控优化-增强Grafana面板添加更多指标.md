# 监控优化 - 增强 Grafana 面板添加更多指标

## 问题描述

原有的 Grafana Flink 监控面板只包含 4 个基础指标面板：
- 数据流入速率
- 数据流出速率
- 运行中的作业数
- TaskManager 数量

需要添加更多常用指标，以便更全面地监控 Flink 作业的运行状态。

## 解决方案

### 1. 新增监控指标

增强后的 Grafana 面板包含 15 个监控面板，覆盖以下方面：

#### 数据流指标（2 个面板）
1. **数据流入速率** (Records/s)
   - 指标：`rate(flink_taskmanager_job_task_numRecordsIn[1m])`
   - 说明：每秒接收的记录数
   - 位置：第 1 行左侧

2. **数据流出速率** (Records/s)
   - 指标：`rate(flink_taskmanager_job_task_numRecordsOut[1m])`
   - 说明：每秒输出的记录数
   - 位置：第 1 行右侧

#### Checkpoint 指标（2 个面板）
3. **Checkpoint 持续时间**
   - 指标：`flink_jobmanager_job_lastCheckpointDuration`
   - 说明：最近一次 Checkpoint 的持续时间（毫秒）
   - 阈值：黄色 > 5s，红色 > 10s
   - 位置：第 2 行左侧

4. **Checkpoint 统计**
   - 指标：
     - 完成：`flink_jobmanager_job_numberOfCompletedCheckpoints`
     - 失败：`flink_jobmanager_job_numberOfFailedCheckpoints`
   - 说明：Checkpoint 完成和失败的次数
   - 位置：第 2 行右侧

#### JVM 资源指标（3 个面板）
5. **JVM 堆内存使用率**
   - 指标：`flink_taskmanager_Status_JVM_Memory_Heap_Used / flink_taskmanager_Status_JVM_Memory_Heap_Max * 100`
   - 说明：堆内存使用百分比
   - 阈值：黄色 > 70%，红色 > 85%
   - 位置：第 3 行左侧

6. **JVM CPU 使用率**
   - 指标：`flink_taskmanager_Status_JVM_CPU_Load * 100`
   - 说明：JVM 进程的 CPU 使用率
   - 阈值：黄色 > 60%，红色 > 80%
   - 位置：第 3 行中间

7. **JVM 线程数**
   - 指标：`flink_taskmanager_Status_JVM_Threads_Count`
   - 说明：JVM 中的线程总数
   - 位置：第 3 行右侧

#### 集群状态指标（4 个面板）
8. **运行中的作业数**
   - 指标：`flink_jobmanager_numRunningJobs`
   - 说明：当前运行的 Flink 作业数量
   - 类型：仪表盘
   - 位置：第 4 行第 1 个

9. **TaskManager 数量**
   - 指标：`flink_jobmanager_numRegisteredTaskManagers`
   - 说明：已注册的 TaskManager 数量
   - 类型：仪表盘
   - 位置：第 4 行第 2 个

10. **可用任务槽**
    - 指标：`flink_jobmanager_taskSlotsAvailable`
    - 说明：当前可用的任务槽数量
    - 类型：仪表盘
    - 位置：第 4 行第 3 个

11. **总任务槽**
    - 指标：`flink_jobmanager_taskSlotsTotal`
    - 说明：集群总任务槽数量
    - 类型：仪表盘
    - 位置：第 4 行第 4 个

#### JVM 详细指标（2 个面板）
12. **JVM 堆内存详情**
    - 指标：
      - 已使用：`flink_taskmanager_Status_JVM_Memory_Heap_Used`
      - 最大值：`flink_taskmanager_Status_JVM_Memory_Heap_Max`
    - 说明：堆内存使用的绝对值（字节）
    - 位置：第 5 行左侧

13. **GC 频率**
    - 指标：
      - Young GC：`rate(flink_taskmanager_Status_JVM_GarbageCollector_G1_Young_Generation_Count[1m])`
      - Old GC：`rate(flink_taskmanager_Status_JVM_GarbageCollector_G1_Old_Generation_Count[1m])`
    - 说明：每分钟的 GC 次数
    - 位置：第 5 行右侧

#### 性能和背压指标（2 个面板）
14. **输出缓冲池使用率（背压指标）**
    - 指标：`flink_taskmanager_job_task_buffers_outPoolUsage`
    - 说明：输出缓冲池使用率，接近 1 表示有背压
    - 阈值：黄色 > 0.5，红色 > 0.8
    - 位置：第 6 行左侧

15. **网络吞吐量**
    - 指标：
      - 流入：`rate(flink_taskmanager_job_task_numBytesIn[1m])`
      - 流出：`rate(flink_taskmanager_job_task_numBytesOut[1m])`
    - 说明：每秒的网络字节数
    - 位置：第 6 行右侧

┌────────────┬────────────┬────────────┐

### 3. 配置特性

#### 全局配置
- **刷新间隔**：5 秒（实时监控）
- **时间范围**：最近 15 分钟
- **时区**：Asia/Shanghai（东八区）
- **主题**：Dark（深色主题）
- **Tooltip 模式**：多指标显示

#### 颜色阈值
- **绿色**：正常状态
- **黄色**：警告状态（需要关注）
- **红色**：危险状态（需要处理）

#### 图表类型
- **时间序列图**：数据流、Checkpoint、JVM、性能指标
- **仪表盘**：集群状态指标（更直观）

### 4. 使用方法

#### 启动监控服务
```bash
# 启动 Prometheus + Grafana + Pushgateway
bash start-monitoring.sh

# 等待服务启动（约 10 秒）
sleep 10

# 访问 Grafana
open http://localhost:3000
```

#### 登录 Grafana
- URL: http://localhost:3000
- 用户名: admin
- 密码: admin（首次登录会要求修改）

#### 查看面板
1. 登录后，点击左侧菜单 "Dashboards"
2. 选择 "Flink 实时监控（增强版）"
3. 面板会自动加载并显示指标

#### 自定义面板
1. 点击面板右上角的 "Edit" 按钮
2. 修改查询、阈值、颜色等配置
3. 点击 "Apply" 保存修改
4. 点击右上角的 "Save dashboard" 保存整个面板

### 5. 关键指标说明

#### 数据流指标
- **Records In/Out**：监控数据处理速率
- **正常值**：根据业务需求，一般 100-10000 条/秒
- **异常情况**：
  - 突然降为 0：数据源问题或作业停止
  - 持续下降：可能有背压或性能问题

#### Checkpoint 指标
- **持续时间**：Checkpoint 完成所需时间
- **正常值**：< 5 秒
- **警告**：5-10 秒（可能有性能问题）
- **危险**：> 10 秒（需要优化）
- **失败次数**：应该保持为 0

#### JVM 内存
- **堆内存使用率**：
  - 正常：< 70%
  - 警告：70-85%
  - 危险：> 85%（可能 OOM）
- **优化建议**：
  - 增加堆内存：`-Xmx` 参数
  - 优化代码：减少对象创建

#### CPU 使用率
- **正常值**：< 60%
- **警告**：60-80%
- **危险**：> 80%（可能影响性能）
- **优化建议**：
  - 增加并行度
  - 优化算法复杂度

#### 背压指标
- **缓冲池使用率**：
  - 正常：< 0.5（50%）
  - 警告：0.5-0.8（50-80%）
  - 危险：> 0.8（80%）
- **背压原因**：
  - 下游处理慢（如 Doris 写入慢）
  - 网络带宽不足
  - 资源不足

#### GC 频率
- **Young GC**：正常，频率较高
- **Old GC**：应该很少发生
- **异常情况**：
  - Old GC 频繁：内存不足或内存泄漏
  - GC 时间过长：影响作业性能

### 6. 故障排查

#### 数据流入为 0
1. 检查 Kafka 是否有数据：
   ```bash
   bash test/check-kafka-partitions.sh
   ```
2. 检查 Flink 作业状态：
   ```bash
   bash test/check-flink-metrics.sh
   ```
3. 查看作业日志：
   ```bash
   tail -f logs/app/flink-app.log
   ```

#### Checkpoint 失败
1. 查看错误日志：
   ```bash
   tail -f logs/error/error.log | grep Checkpoint
   ```
2. 检查 Doris 连接：
   ```bash
   mysql -h 127.0.0.1 -P 9030 -u root -e "SHOW BACKENDS;"
   ```
3. 增加 Checkpoint 超时时间：
   ```yaml
   flink:
     checkpoint:
       timeout: 300000  # 5 分钟
   ```

#### 内存使用率过高
1. 查看堆内存详情面板
2. 检查是否有内存泄漏
3. 增加堆内存：
   ```bash
   export FLINK_OPTS="-Xmx4g"
   ```

#### 背压问题
1. 查看缓冲池使用率面板
2. 检查下游 Doris 写入速度
3. 优化批量写入配置：
   ```yaml
   doris:
     stream-load:
       batch-size: 2000  # 增加批量大小
       batch-interval-ms: 3000  # 减少批量间隔
   ```

### 7. 监控最佳实践

#### 告警配置
在 Grafana 中配置告警规则：
1. Checkpoint 失败次数 > 0
2. 堆内存使用率 > 85%
3. CPU 使用率 > 80%
4. 缓冲池使用率 > 0.8
5. 数据流入速率 = 0（持续 1 分钟）

#### 定期检查
- 每天检查 Checkpoint 成功率
- 每周检查 GC 频率和时间
- 每月检查资源使用趋势

#### 性能优化
- 根据监控数据调整并行度
- 根据内存使用情况调整堆内存
- 根据背压情况优化下游写入

### 9. 技术要点

#### Prometheus 查询语法
- `rate(metric[1m])`：计算 1 分钟内的平均速率
- `metric1 / metric2 * 100`：计算百分比
- `sum(metric) by (label)`：按标签聚合

#### Grafana 面板类型
- **timeseries**：时间序列图，显示指标随时间变化
- **gauge**：仪表盘，显示当前值和阈值

#### 颜色阈值配置
```json
"thresholds": {
  "steps": [
    {"color": "green", "value": null},  // 默认绿色
    {"color": "yellow", "value": 70},   // >= 70 黄色
    {"color": "red", "value": 85}       // >= 85 红色
  ]
}
```

## 优势

1. **全面监控**：覆盖数据流、Checkpoint、JVM、集群状态、性能等多个方面
2. **实时更新**：5 秒刷新间隔，及时发现问题
3. **可视化**：直观的图表和仪表盘，易于理解
4. **告警支持**：配置阈值，支持告警通知
5. **易于扩展**：可以轻松添加新的指标面板
