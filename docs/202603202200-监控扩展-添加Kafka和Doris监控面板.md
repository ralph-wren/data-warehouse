# 监控扩展 - 添加 Kafka 和 Doris 监控面板

## 问题描述

当前监控系统只包含 Flink 作业的监控面板，缺少 Kafka 和 Doris 的监控。需要添加：
- Kafka 监控：消息流入速率、消费者 LAG、分区状态等
- Doris 监控：查询速率、连接数、SQL 操作、网络吞吐量等

## 解决方案

### 1. 架构设计

```
┌─────────────┐     ┌──────────────┐     ┌────────────┐     ┌─────────┐
│   Kafka     │────>│Kafka Exporter│────>│ Prometheus │────>│ Grafana │
└─────────────┘     └──────────────┘     └────────────┘     └─────────┘
                                               ▲
┌─────────────┐     ┌──────────────┐          │
│   Doris     │────>│MySQL Exporter│──────────┘
│  (MySQL协议)│     └──────────────┘
└─────────────┘
```

### 2. 添加 Exporter 服务

#### Kafka Exporter
- **镜像**：`danielqsj/kafka-exporter:latest`
- **端口**：9308
- **功能**：导出 Kafka 的 Topic、分区、消费者组等指标

#### MySQL Exporter
- **镜像**：`prom/mysqld-exporter:latest`
- **端口**：9104
- **功能**：通过 MySQL 协议连接 Doris，导出数据库指标

### 3. 修改 docker-compose-monitoring.yml

添加两个新服务：

```yaml
  # Kafka Exporter - Kafka 指标导出
  kafka-exporter:
    image: danielqsj/kafka-exporter:latest
    container_name: kafka-exporter
    ports:
      - "9308:9308"
    command:
      - '--kafka.server=host.docker.internal:9092'
      - '--web.listen-address=:9308'
    networks:
      - monitoring-net
    restart: unless-stopped
    extra_hosts:
      - "host.docker.internal:host-gateway"

  # MySQL Exporter - 用于监控 Doris（通过 MySQL 协议）
  mysql-exporter:
    image: prom/mysqld-exporter:latest
    container_name: mysql-exporter
    ports:
      - "9104:9104"
    environment:
      - DATA_SOURCE_NAME=root:@tcp(host.docker.internal:9030)/
    networks:
      - monitoring-net
    restart: unless-stopped
    extra_hosts:
      - "host.docker.internal:host-gateway"
```

**关键配置说明**：
- `host.docker.internal`：Docker 容器访问宿主机的特殊域名
- `extra_hosts`：添加主机映射，让容器能解析 `host.docker.internal`
- Kafka 端口：9092（Kafka Broker）
- Doris 端口：9030（MySQL 协议端口）

### 4. 更新 Prometheus 配置

在 `monitoring/prometheus/prometheus.yml` 中添加抓取任务：

```yaml
  # Kafka Exporter - Kafka 指标
  - job_name: 'kafka'
    static_configs:
      - targets: ['kafka-exporter:9308']

  # MySQL Exporter - Doris 数据库指标
  - job_name: 'doris-mysql'
    static_configs:
      - targets: ['mysql-exporter:9104']
```

### 5. Kafka 监控面板

创建 `monitoring/grafana/dashboards/kafka-monitoring.json`，包含 8 个面板：

#### 面板列表

1. **Kafka 消息流入速率** (12x8)
   - 指标：`rate(kafka_topic_partition_current_offset{topic="crypto-ticker"}[1m])`
   - 说明：每秒流入的消息数
   - 按分区显示

2. **消费者组 LAG** (12x8)
   - 指标：`sum(kafka_consumergroup_lag{topic="crypto-ticker"}) by (consumergroup)`
   - 说明：消费者组的消费延迟
   - 按消费者组聚合

3. **总 LAG** (6x4 仪表盘)
   - 指标：`sum(kafka_consumergroup_lag)`
   - 阈值：黄色 > 100，红色 > 1000

4. **Topic 数量** (6x4 仪表盘)
   - 指标：`count(kafka_topic_partitions)`

5. **总分区数** (6x4 仪表盘)
   - 指标：`sum(kafka_topic_partitions)`

6. **消费者组数量** (6x4 仪表盘)
   - 指标：`count(count by (consumergroup) (kafka_consumergroup_lag))`

7. **分区 Offset** (12x8)
   - 指标：`kafka_topic_partition_current_offset{topic="crypto-ticker"}`
   - 说明：每个分区的当前 offset

8. **网络流入速率** (12x8)
   - 指标：`rate(kafka_topic_partition_in_bytes_total[1m])`
   - 单位：Bytes/s

#### 面板布局

```
第 1 行：消息流入速率 | 消费者组 LAG
第 2 行：总LAG | Topic数量 | 总分区数 | 消费者组数量
第 3 行：分区 Offset | 网络流入速率
```

### 6. Doris 监控面板

创建 `monitoring/grafana/dashboards/doris-monitoring.json`，包含 10 个面板：

#### 面板列表

1. **Doris 查询速率 (QPS)** (12x8)
   - 指标：`rate(mysql_global_status_queries[1m])`
   - 说明：每秒查询数

2. **连接数** (12x8)
   - 指标：
     - 已连接：`mysql_global_status_threads_connected`
     - 运行中：`mysql_global_status_threads_running`

3. **当前连接数** (6x4 仪表盘)
   - 指标：`mysql_global_status_threads_connected`
   - 阈值：黄色 > 100，红色 > 200

4. **运行时间** (6x4 仪表盘)
   - 指标：`mysql_global_status_uptime`
   - 单位：秒

5. **接收字节数** (6x4 仪表盘)
   - 指标：`mysql_global_status_bytes_received`

6. **发送字节数** (6x4 仪表盘)
   - 指标：`mysql_global_status_bytes_sent`

7. **SQL 操作速率** (12x8)
   - 指标：
     - SELECT：`rate(mysql_global_status_com_select[1m])`
     - INSERT：`rate(mysql_global_status_com_insert[1m])`
     - UPDATE：`rate(mysql_global_status_com_update[1m])`
     - DELETE：`rate(mysql_global_status_com_delete[1m])`

8. **网络吞吐量** (12x8)
   - 指标：
     - 接收：`rate(mysql_global_status_bytes_received[1m])`
     - 发送：`rate(mysql_global_status_bytes_sent[1m])`
   - 单位：Bytes/s

9. **慢查询数量** (12x8)
   - 指标：`mysql_global_status_slow_queries`

10. **表锁统计** (12x8)
    - 指标：
      - 立即获取：`mysql_global_status_table_locks_immediate`
      - 等待：`mysql_global_status_table_locks_waited`

#### 面板布局

```
第 1 行：查询速率 (QPS) | 连接数
第 2 行：当前连接数 | 运行时间 | 接收字节数 | 发送字节数
第 3 行：SQL 操作速率 | 网络吞吐量
第 4 行：慢查询数量 | 表锁统计
```

## 关键指标说明

### Kafka 指标

| 指标 | 说明 | 正常值 | 异常情况 |
|-----|------|--------|---------|
| 消息流入速率 | 每秒新增消息数 | 根据业务 | 突然为 0 或异常高 |
| 消费者 LAG | 消费延迟 | < 100 | > 1000 需要关注 |
| 分区 Offset | 每个分区的位置 | 持续增长 | 停止增长表示无新数据 |
| 网络流入 | 网络吞吐量 | 根据业务 | 异常高可能有问题 |

### Doris 指标

| 指标 | 说明 | 正常值 | 异常情况 |
|-----|------|--------|---------|
| QPS | 每秒查询数 | 根据业务 | 突然下降可能有问题 |
| 连接数 | 当前连接数 | < 100 | > 200 需要关注 |
| 慢查询 | 慢查询数量 | 0 或很少 | 持续增加需要优化 |
| 表锁等待 | 锁等待次数 | 0 或很少 | 频繁等待影响性能 |

## 故障排查

### 问题 1：Kafka Exporter 无法连接 Kafka

**错误信息**：
```
Error: dial tcp: lookup host.docker.internal: no such host
```

**解决方法**：
```bash
# 检查 Kafka 是否运行
netstat -an | grep 9092

# 如果 Kafka 在 Docker 中，修改配置
# 将 host.docker.internal:9092 改为 kafka:9092
# 并确保两个容器在同一网络
```

### 问题 2：MySQL Exporter 无法连接 Doris

**错误信息**：
```
Error 1045: Access denied for user 'root'@'172.x.x.x'
```

**解决方法**：
```bash
# 检查 Doris 是否运行
mysql -h 127.0.0.1 -P 9030 -u root

# 如果需要密码，修改 DATA_SOURCE_NAME
# DATA_SOURCE_NAME=root:password@tcp(host.docker.internal:9030)/
```

### 问题 3：Grafana 面板显示 "No data"

**原因**：Prometheus 还没有抓取到数据

**解决方法**：
```bash
# 1. 检查 Prometheus targets
open http://localhost:9090/targets

# 2. 确认 kafka 和 doris-mysql 的状态为 UP

# 3. 等待 15-30 秒后刷新 Grafana 面板
```

### 问题 4：Kafka LAG 指标为空

**原因**：没有消费者组或消费者组没有消费数据

**解决方法**：
```bash
# 启动 Flink 作业，创建消费者组
bash run-flink-ods-datastream.sh

# 等待 15 秒后查看面板
```

## 测试验证

### 测试脚本
创建 `test/test-kafka-doris-monitoring.sh`：

```bash
#!/bin/bash
echo "=========================================="
echo "Kafka 和 Doris 监控测试"
echo "=========================================="

# 1. 检查 Kafka Exporter
echo "1. 检查 Kafka Exporter..."
if curl -s http://localhost:9308/metrics | grep -q "kafka_brokers"; then
    echo "   ✓ Kafka Exporter 正常"
else
    echo "   ✗ Kafka Exporter 异常"
fi

# 2. 检查 MySQL Exporter
echo "2. 检查 MySQL Exporter..."
if curl -s http://localhost:9104/metrics | grep -q "mysql_up"; then
    echo "   ✓ MySQL Exporter 正常"
else
    echo "   ✗ MySQL Exporter 异常"
fi

# 3. 检查 Prometheus targets
echo "3. 检查 Prometheus targets..."
KAFKA_UP=$(curl -s http://localhost:9090/api/v1/targets | jq -r '.data.activeTargets[] | select(.labels.job=="kafka") | .health')
DORIS_UP=$(curl -s http://localhost:9090/api/v1/targets | jq -r '.data.activeTargets[] | select(.labels.job=="doris-mysql") | .health')

echo "   Kafka target: $KAFKA_UP"
echo "   Doris target: $DORIS_UP"

# 4. 查询 Kafka 指标
echo "4. 查询 Kafka 指标..."
KAFKA_TOPICS=$(curl -s "http://localhost:9090/api/v1/query?query=kafka_topic_partitions" | jq -r '.data.result | length')
echo "   Topic 数量: $KAFKA_TOPICS"

# 5. 查询 Doris 指标
echo "5. 查询 Doris 指标..."
DORIS_CONN=$(curl -s "http://localhost:9090/api/v1/query?query=mysql_global_status_threads_connected" | jq -r '.data.result[0].value[1]')
echo "   Doris 连接数: $DORIS_CONN"

echo ""
echo "测试完成！"
```

## 优势

1. **全面监控**：覆盖 Flink、Kafka、Doris 三大组件
2. **实时更新**：5 秒刷新间隔
3. **统一平台**：所有监控在一个 Grafana 中
4. **易于扩展**：可以轻松添加更多指标
5. **开箱即用**：使用官方 Exporter，稳定可靠

## 注意事项

### 1. Exporter 性能影响

- Kafka Exporter：对 Kafka 性能影响很小（< 1% CPU）
- MySQL Exporter：对 Doris 性能影响很小（< 1% CPU）
- 建议在生产环境中监控 Exporter 的资源使用

### 2. 网络连接

- 确保 Exporter 能访问 Kafka 和 Doris
- 使用 `host.docker.internal` 访问宿主机服务
- 如果服务在 Docker 中，使用容器名称

### 3. 数据保留

- Prometheus 默认保留 15 天数据
- 如需长期保存，考虑使用 Thanos 或 Cortex
- 定期备份 Grafana 面板配置

### 4. 告警配置

建议配置以下告警：
- Kafka LAG > 1000
- Doris 连接数 > 200
- Doris 慢查询持续增加
- Kafka 消息流入速率为 0（持续 5 分钟）
