# Kafka 和 Doris 监控数据显示问题修复

**时间**: 2026-03-20 22:30  
**问题**: Kafka 和 Doris 监控面板没有数据显示  
**状态**: ✅ 已解决

## 问题描述

在配置了 Kafka 和 Doris 监控面板后,Grafana 中两个面板都显示 "No data",需要排查并修复监控数据采集问题。

## 问题分析

### 1. Kafka Exporter 问题

**现象**:
- Kafka Exporter 容器启动,但日志显示大量连接错误
- 错误信息: `dial tcp [::1]:9092: connect: connection refused`

**根本原因**:
- Kafka 容器的 `KAFKA_ADVERTISED_LISTENERS` 配置为 `localhost:9092`
- Kafka Exporter 连接到 Kafka 后,Kafka 返回的 broker 地址是 `localhost:9092`
- Kafka Exporter 尝试连接 `localhost`,但在容器内 localhost 指向容器自己,而不是宿主机的 Kafka

**解决方案**:
- 使用 `host.docker.internal` 让 Kafka Exporter 从容器访问宿主机的 Kafka
- 虽然仍有部分分区连接错误,但基本指标(broker 数量、topic 分区数等)可以正常采集

### 2. MySQL Exporter 问题

**现象**:
- MySQL Exporter 无法连接 Doris
- 错误信息: `Unknown system variable 'lock_wait_timeout'`

**根本原因**:
- MySQL Exporter 在初始化时会查询 `lock_wait_timeout` 等 MySQL 特定的系统变量
- Doris 虽然兼容 MySQL 协议,但不支持所有 MySQL 系统变量
- 即使禁用了各种采集器,exporter 在连接阶段就会失败

**解决方案**:
- 放弃使用 MySQL Exporter
- 直接使用 Doris 自带的 Prometheus metrics 端点
- Doris FE 和 BE 都提供了 `/metrics` 接口,返回 Prometheus 格式的指标

## 解决步骤

### 1. 修复 Kafka Exporter 配置

修改 `docker-compose-monitoring.yml`:

```yaml
kafka-exporter:
  image: danielqsj/kafka-exporter:latest
  container_name: kafka-exporter
  ports:
    - "9308:9308"
  # 使用 host.docker.internal 访问宿主机的 Kafka
  command:
    - '--kafka.server=host.docker.internal:9092'
    - '--log.level=info'
  networks:
    - monitoring-net
  restart: unless-stopped
  extra_hosts:
    - "host.docker.internal:host-gateway"
```

### 2. 移除 MySQL Exporter,使用 Doris 原生 Metrics

修改 `monitoring/prometheus/prometheus.yml`:

```yaml
# 移除 MySQL Exporter target
# - job_name: 'doris-mysql'
#   static_configs:
#     - targets: ['mysql-exporter:9104']

# Doris 已经在配置中,直接使用其原生 metrics 端点
- job_name: 'doris-fe'
  static_configs:
    - targets: ['host.docker.internal:8030']
  metrics_path: '/metrics'

- job_name: 'doris-be'
  static_configs:
    - targets: ['host.docker.internal:8040']
  metrics_path: '/metrics'
```

### 3. 重启服务

```bash
# 停止 MySQL Exporter
docker-compose -f docker-compose-monitoring.yml stop mysql-exporter

# 重启 Kafka Exporter
docker-compose -f docker-compose-monitoring.yml up -d kafka-exporter

# 重启 Prometheus 加载新配置
docker-compose -f docker-compose-monitoring.yml restart prometheus
```

### 4. 验证指标采集

```bash
# 检查 Kafka 指标
curl -s http://localhost:9308/metrics | grep kafka_brokers
# 输出: kafka_brokers 1

# 检查 Doris FE 指标
curl -s http://localhost:8030/metrics | grep jvm_heap_size_bytes
# 输出: jvm_heap_size_bytes{type="max"} 8589934592

# 检查 Prometheus 中的指标
curl -s "http://localhost:9090/api/v1/query?query=kafka_brokers"
curl -s "http://localhost:9090/api/v1/query?query=jvm_heap_size_bytes"
```

## 关键经验

### 1. Kafka Advertised Listeners 问题
- Kafka 的 `advertised.listeners` 配置会影响客户端连接
- 如果配置为 `localhost`,外部客户端会尝试连接 localhost 而失败
- 生产环境应配置为实际的 IP 地址或域名

### 2. Doris 与 MySQL 兼容性
- Doris 兼容 MySQL 协议,但不是完全兼容
- 不支持所有 MySQL 系统变量和功能
- 优先使用 Doris 原生的监控接口,而不是通过 MySQL 协议

### 3. Docker 网络模式选择
- Windows 上 Docker 的 `host` 网络模式支持有限
- 使用桥接网络 + `host.docker.internal` 是更可靠的方案
- 需要在 docker-compose 中添加 `extra_hosts` 配置

### 4. Prometheus Exporter 调试
- 先检查 exporter 的 `/metrics` 端点是否有数据
- 再检查 Prometheus targets 状态
- 最后检查 Grafana 面板查询语句

## 总结

通过分析 Kafka Exporter 和 MySQL Exporter 的连接问题,找到了根本原因:
- Kafka 的 advertised.listeners 配置导致 exporter 无法正确连接
- Doris 与 MySQL 的兼容性问题导致 MySQL Exporter 无法初始化

最终采用的解决方案:
- Kafka Exporter 使用 host.docker.internal 访问宿主机 Kafka
- 放弃 MySQL Exporter,直接使用 Doris 原生 metrics 端点

监控系统现已正常工作,Kafka 和 Doris 的关键指标都可以在 Grafana 中查看。
