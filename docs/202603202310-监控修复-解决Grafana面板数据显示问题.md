# Grafana 面板数据显示问题修复

**时间**: 2026-03-20 23:10  
**问题**: Prometheus 有数据但 Grafana 面板显示 "No data"  
**状态**: ✅ 已解决

## 问题描述

在配置了 Kafka 和 Doris 监控面板后:
- Prometheus 能查询到 Kafka 和 Doris 指标 ✅
- Grafana Explore 中能查询到数据 ✅  
- 但 Grafana 监控面板显示 "No data" ❌

## 问题分析

### 1. Kafka Exporter 指标限制 ⭐⭐⭐

**现象**:
- 部分面板有数据(Topic 数量: 2, 分区数量: 54)
- 其他面板显示 "No data"(消息流入速率、LAG、Offset 等)

**根本原因**:

使用的 `danielqsj/kafka-exporter:latest` 只提供基础指标,不提供高级指标:

**可用指标**:
```
kafka_brokers                                    # Broker 数量
kafka_topic_partitions                           # Topic 分区数
kafka_topic_partition_in_sync_replica            # 同步副本数
kafka_topic_partition_leader                     # 分区 Leader
kafka_topic_partition_replicas                   # 副本数
kafka_topic_partition_under_replicated_partition # 未充分复制的分区
kafka_broker_info                                # Broker 信息
```

**不可用指标** (面板中使用但不存在):
```
kafka_topic_partition_current_offset             # 当前 Offset ❌
kafka_topic_partition_oldest_offset              # 最早 Offset ❌
kafka_consumergroup_lag                          # 消费者 LAG ❌
kafka_topic_partition_in_bytes_total             # 流入字节数 ❌
kafka_topic_partition_out_bytes_total            # 流出字节数 ❌
```

### 2. Topic 名称错误

**面板查询**:
```promql
kafka_topic_partition_current_offset{topic="crypto-ticker"}
```

**实际 Topic 名称**:
```
crypto_ticker  # 下划线,不是连字符
```

### 3. 查询语句错误

**错误示例**:
```promql
rate(kafka_topic_partition_current_offset{topic="crypto_ticker"}[1m])
```

**问题**:
- `kafka_topic_partition_current_offset` 是 gauge 类型
- `rate()` 函数只适用于 counter 类型
- 对 gauge 使用 `rate()` 会返回空值或错误结果

## 解决方案

### 方案 2: 升级到 JMX Exporter (未来优化)

要获得完整的 Kafka 监控(消息速率、LAG、吞吐量等),需要使用 JMX Exporter。

**架构**:
```
Kafka (JMX enabled) → JMX Exporter → Prometheus → Grafana
```

**实施步骤**:

#### 1. 启用 Kafka JMX

修改 Kafka 启动配置:

```yaml
# docker-compose.yml (如果使用 Docker)
services:
  kafka:
    image: confluentinc/cp-kafka:latest
    environment:
      # 启用 JMX
      - KAFKA_JMX_OPTS=-Dcom.sun.management.jmxremote 
                       -Dcom.sun.management.jmxremote.port=9999 
                       -Dcom.sun.management.jmxremote.authenticate=false 
                       -Dcom.sun.management.jmxremote.ssl=false
      - JMX_PORT=9999
    ports:
      - "9092:9092"
      - "9999:9999"  # JMX 端口
```

或者在启动脚本中:

```bash
export KAFKA_JMX_OPTS="-Dcom.sun.management.jmxremote \
  -Dcom.sun.management.jmxremote.port=9999 \
  -Dcom.sun.management.jmxremote.authenticate=false \
  -Dcom.sun.management.jmxremote.ssl=false"
export JMX_PORT=9999

bin/kafka-server-start.sh config/server.properties
```

#### 2. 添加 JMX Exporter

```yaml
# docker-compose-monitoring.yml
services:
  jmx-exporter:
    image: bitnami/jmx-exporter:latest
    container_name: kafka-jmx-exporter
    ports:
      - "5556:5556"
    environment:
      - SERVICE_PORT=5556
      - JMX_HOST=host.docker.internal
      - JMX_PORT=9999
    volumes:
      - ./monitoring/jmx-exporter/kafka.yml:/etc/jmx-exporter/config.yml
    command:
      - "5556"
      - "/etc/jmx-exporter/config.yml"
    networks:
      - monitoring-net
    restart: unless-stopped
    extra_hosts:
      - "host.docker.internal:host-gateway"
```

#### 3. 配置 JMX Exporter

```yaml
# monitoring/jmx-exporter/kafka.yml
---
lowercaseOutputName: true
lowercaseOutputLabelNames: true

rules:
  # Broker 指标
  - pattern: kafka.server<type=(.+), name=(.+)><>Value
    name: kafka_server_$1_$2
    type: GAUGE

  # Topic 指标
  - pattern: kafka.server<type=(.+), name=(.+), topic=(.+)><>Value
    name: kafka_server_$1_$2
    type: GAUGE
    labels:
      topic: "$3"

  # 分区指标
  - pattern: kafka.server<type=(.+), name=(.+), topic=(.+), partition=(.+)><>Value
    name: kafka_server_$1_$2
    type: GAUGE
    labels:
      topic: "$3"
      partition: "$4"

  # 消费者组指标
  - pattern: kafka.consumer<type=(.+), client-id=(.+), topic=(.+), partition=(.+)><>(.+)
    name: kafka_consumer_$1_$5
    type: GAUGE
    labels:
      client_id: "$2"
      topic: "$3"
      partition: "$4"

  # 生产者指标
  - pattern: kafka.producer<type=(.+), client-id=(.+), topic=(.+)><>(.+)
    name: kafka_producer_$1_$4
    type: GAUGE
    labels:
      client_id: "$2"
      topic: "$3"

  # 网络指标
  - pattern: kafka.network<type=(.+), name=(.+)><>Value
    name: kafka_network_$1_$2
    type: GAUGE

  # 日志指标
  - pattern: kafka.log<type=(.+), name=(.+), topic=(.+), partition=(.+)><>Value
    name: kafka_log_$1_$2
    type: GAUGE
    labels:
      topic: "$3"
      partition: "$4"
```

#### 4. 更新 Prometheus 配置

```yaml
# monitoring/prometheus/prometheus.yml
scrape_configs:
  # 保留原有的 Kafka Exporter (基础指标)
  - job_name: 'kafka'
    static_configs:
      - targets: ['kafka-exporter:9308']

  # 添加 JMX Exporter (详细指标)
  - job_name: 'kafka-jmx'
    static_configs:
      - targets: ['jmx-exporter:5556']
```

#### 5. 创建完整版监控面板

使用 JMX 指标创建包含以下内容的完整面板:

**消息指标**:
- `kafka_server_brokertopicmetrics_messagesinpersec`: 消息流入速率
- `kafka_server_brokertopicmetrics_bytesinpersec`: 字节流入速率
- `kafka_server_brokertopicmetrics_bytesoutpersec`: 字节流出速率

**消费者指标**:
- `kafka_consumer_fetch_manager_metrics_records_lag`: 消费者 LAG
- `kafka_consumer_fetch_manager_metrics_records_consumed_rate`: 消费速率

**性能指标**:
- `kafka_server_replicamanager_underreplicatedpartitions`: 未充分复制的分区
- `kafka_server_replicamanager_partitioncount`: 分区数量
- `kafka_server_replicamanager_leadercount`: Leader 数量

**网络指标**:
- `kafka_network_requestmetrics_totaltimems`: 请求处理时间
- `kafka_network_requestmetrics_requestqueuetimems`: 请求队列时间

## 关键经验

### 1. Exporter 选择很重要

不同的 exporter 提供的指标不同:

| Exporter | 优点 | 缺点 | 适用场景 |
|---------|------|------|---------|
| danielqsj/kafka-exporter | 简单易用,无需配置 | 指标有限 | 基础监控 |
| JMX Exporter | 指标完整,高度可定制 | 配置复杂,需要启用 JMX | 生产环境 |
| Confluent Exporter | 官方支持,指标标准 | 仅支持 Confluent Platform | Confluent 用户 |

### 2. Grafana 面板调试步骤

```
1. 在 Prometheus 中验证指标存在
   http://localhost:9090/graph
   查询: kafka_brokers

2. 在 Grafana Explore 中测试查询
   Explore → Prometheus → 输入查询 → Run query

3. 确认查询语句正确
   - 检查指标名称拼写
   - 检查标签值(如 topic 名称)
   - 检查函数使用(rate/increase/sum 等)

4. 检查时间范围设置
   - 确保时间范围包含数据
   - 尝试 "Last 5 minutes" 或 "Last 15 minutes"

5. 导入到面板
   - 复制工作的查询语句
   - 粘贴到面板配置中
```

### 3. 常见错误及解决方法

| 错误 | 原因 | 解决方法 |
|-----|------|---------|
| No data | 指标不存在 | 检查 exporter 的 /metrics 端点 |
| No data | 标签值错误 | 在 Prometheus 中查询确认标签值 |
| No data | 时间范围不对 | 调整时间范围或刷新 |
| 查询错误 | 函数使用错误 | 确认指标类型,使用正确的函数 |
| 数据不更新 | 刷新间隔太长 | 调整刷新间隔(如 5s) |

### 4. Prometheus 指标类型

| 类型 | 说明 | 适用函数 | 示例 |
|-----|------|---------|------|
| Counter | 只增不减的计数器 | rate(), increase() | 消息总数 |
| Gauge | 可增可减的数值 | 直接使用,聚合函数 | 当前连接数 |
| Histogram | 分布统计 | _bucket, _sum, _count | 请求延迟 |
| Summary | 摘要统计 | _sum, _count, 分位数 | 响应时间 |

**使用示例**:

```promql
# Counter - 使用 rate() 计算速率
rate(kafka_messages_total[1m])

# Gauge - 直接使用或聚合
kafka_brokers
sum(kafka_topic_partitions)

# Histogram - 计算分位数
histogram_quantile(0.95, rate(kafka_request_duration_seconds_bucket[5m]))

# Summary - 获取分位数
kafka_request_duration_seconds{quantile="0.95"}
```
