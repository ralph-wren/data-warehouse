# JMX 连接 - 解决 Docker 网络 RMI 回调问题

**时间**: 2026-03-21 00:10  
**问题类型**: 监控集成  
**严重程度**: ⭐⭐⭐ 高  
**状态**: ✅ 已解决

## 问题描述

在升级 Kafka 监控系统时，JMX Exporter 容器无法连接到 Kafka 的 JMX 端口，导致无法获取详细的 Kafka 指标。

### 错误信息

```
java.rmi.ConnectException: Connection refused to host: host.docker.internal; 
nested exception is: java.net.ConnectException: Connection refused

Caused by: java.net.ConnectException: Connection refused
```

### 问题现象

1. **JMX Exporter HTTP 服务器启动成功**（端口 5556）
2. **但无法连接到 Kafka JMX**（端口 9999）
3. **JMX Exporter 指标端点只有自身的 JVM 指标，没有 Kafka 指标**
4. **TCP 连接测试成功，但 RMI 连接失败**

## 根本原因

### RMI 连接机制问题 ⭐⭐⭐

**Java RMI (Remote Method Invocation) 的两阶段连接机制**:

1. **第一阶段**: 客户端连接到 RMI Registry（端口 9999）
2. **第二阶段**: RMI Registry 返回 RMI Server 的地址，客户端连接到 RMI Server

**问题所在**:
- Kafka 的 `java.rmi.server.hostname` 设置不正确
- RMI Registry 返回的地址是 JMX Exporter 容器无法访问的地址
- 导致第二阶段连接失败

### 尝试的方案

#### 方案 1: 使用 host.docker.internal ❌

```bash
# Kafka JMX 配置
-Djava.rmi.server.hostname=host.docker.internal

# JMX Exporter 配置
hostPort: host.docker.internal:9999
```

**问题**: 
- `host.docker.internal` 在 Docker 容器内部无法正确解析
- RMI 回调时仍然失败

#### 方案 2: 使用 localhost ❌

```bash
# Kafka JMX 配置
-Djava.rmi.server.hostname=localhost

# JMX Exporter 配置
hostPort: host.docker.internal:9999
```

**问题**:
- RMI Registry 返回 `localhost`
- JMX Exporter 容器中的 `localhost` 指向容器自己，而不是 Kafka

## 解决方案

### 最终方案: Docker 网络 + 容器名称 ✅ ⭐⭐⭐

**核心思路**: 让 Kafka 和 JMX Exporter 在同一个 Docker 网络中，使用容器名称进行通信

### 1. 修改 Kafka JMX 配置

**文件**: `restart-kafka-with-jmx.sh`

```bash
# 关键环境变量
-e KAFKA_JMX_HOSTNAME=kafka \  # Confluent 镜像特有的环境变量
-e KAFKA_JMX_PORT=9999 \
-e KAFKA_JMX_OPTS="-Dcom.sun.management.jmxremote \
    -Dcom.sun.management.jmxremote.port=9999 \
    -Dcom.sun.management.jmxremote.rmi.port=9999 \
    -Dcom.sun.management.jmxremote.authenticate=false \
    -Dcom.sun.management.jmxremote.ssl=false \
    -Djava.rmi.server.hostname=kafka \  # 使用容器名称
    -Djava.net.preferIPv4Stack=true"
```

**关键参数说明**:
- `KAFKA_JMX_HOSTNAME=kafka`: Confluent 镜像会使用这个值设置 RMI hostname
- `java.rmi.server.hostname=kafka`: 确保 RMI Registry 返回容器名称
- `jmxremote.rmi.port=9999`: 固定 RMI 端口，避免动态端口

### 2. 修改 JMX Exporter 配置

**文件**: `monitoring/jmx-exporter/kafka.yml`

```yaml
# JMX 连接配置
# 使用 Kafka 容器名称直接连接（需要在同一 Docker 网络中）
hostPort: kafka:9999
```

### 3. 连接 Kafka 到监控网络

**在启动脚本中添加**:

```bash
# 步骤 2.5: 连接 Kafka 到监控网络
echo ""
echo "步骤 2.5: 连接 Kafka 到监控网络..."
# 检查监控网络是否存在
if docker network ls | grep -q "monitoring-net"; then
    # 尝试连接到监控网络（如果已连接会报错，忽略）
    docker network connect data-warehouse_monitoring-net kafka 2>/dev/null || \
        echo "✓ Kafka 已在监控网络中"
    echo "✓ Kafka 已连接到监控网络"
else
    echo "⚠ 监控网络不存在，请先启动监控服务"
fi
```

### 4. Docker Compose 配置

**文件**: `docker-compose-monitoring.yml`

```yaml
services:
  jmx-exporter:
    image: bitnami/jmx-exporter:latest
    container_name: kafka-jmx-exporter
    ports:
      - "5556:5556"
    volumes:
      - ./monitoring/jmx-exporter/kafka.yml:/etc/jmx-exporter/config.yml:ro
    command:
      - "5556"
      - "/etc/jmx-exporter/config.yml"
    networks:
      - monitoring-net  # 在监控网络中
    restart: unless-stopped
    extra_hosts:
      - "host.docker.internal:host-gateway"  # 保留，用于其他用途

networks:
  monitoring-net:
    driver: bridge
```

## 技术要点

### 1. RMI 连接机制

**RMI 两阶段连接**:
```
客户端 → RMI Registry (9999) → 获取 RMI Server 地址 → 连接 RMI Server
```

**关键配置**:
- `jmxremote.port`: RMI Registry 端口
- `jmxremote.rmi.port`: RMI Server 端口（必须固定）
- `java.rmi.server.hostname`: RMI Server 的主机名（客户端用这个地址连接）

### 2. Docker 网络通信

**容器名称解析**:
- 同一 Docker 网络中的容器可以通过容器名称互相访问
- Docker 内置 DNS 服务器会解析容器名称到容器 IP

**网络连接命令**:
```bash
# 将容器连接到网络
docker network connect <network-name> <container-name>

# 查看网络中的容器
docker network inspect <network-name>
```

### 3. Confluent Kafka 镜像特性

**KAFKA_JMX_HOSTNAME 环境变量**:
- Confluent 镜像特有的环境变量
- 会自动设置 `java.rmi.server.hostname`
- 优先级高于 `KAFKA_JMX_OPTS` 中的设置

### 4. JMX Exporter 配置

**hostPort 格式**:
```yaml
# 格式: hostname:port
hostPort: kafka:9999

# 支持的格式:
# - localhost:9999 (本地)
# - 192.168.1.100:9999 (IP 地址)
# - kafka:9999 (容器名称，需要在同一网络)
# - host.docker.internal:9999 (宿主机，Windows/Mac)
```

## 关键经验

### 1. RMI 连接需要正确的 hostname

- `java.rmi.server.hostname` 必须是客户端可以访问的地址
- 在 Docker 环境中，使用容器名称最可靠
- 不要使用 `localhost` 或 `127.0.0.1`

### 2. Docker 网络是关键

- 容器间通信应该使用 Docker 网络
- 容器名称在同一网络中可以自动解析
- 避免使用 `host.docker.internal`（不稳定）

### 3. RMI 端口必须固定

- `jmxremote.port` 和 `jmxremote.rmi.port` 必须设置为相同值
- 否则 RMI Server 会使用随机端口，导致连接失败

### 4. Confluent 镜像的特殊性

- 使用 `KAFKA_JMX_HOSTNAME` 环境变量
- 比 `KAFKA_JMX_OPTS` 中的设置优先级更高
- 确保两者一致

## 监控指标说明

### 核心指标

**消息吞吐量**:
- `kafka_server_brokertopicmetrics_messagesinpersec_total`: 消息流入速率
- `kafka_server_brokertopicmetrics_bytesinpersec_total`: 字节流入速率
- `kafka_server_brokertopicmetrics_bytesoutpersec_total`: 字节流出速率

**分区和副本**:
- `kafka_server_replicamanager_partitioncount`: 分区总数
- `kafka_server_replicamanager_leadercount`: Leader 分区数
- `kafka_server_replicamanager_underreplicatedpartitions`: 未充分复制的分区

**控制器**:
- `kafka_controller_kafkacontroller_activecontrollercount`: 活跃控制器数量（应该为 1）
- `kafka_controller_kafkacontroller_offlinepartitionscount`: 离线分区数量

**JVM**:
- `jvm_process_cpu_load`: JVM CPU 使用率
- `jvm_memory_heap_used`: 堆内存使用量
- `jvm_gc_collection_count_total`: GC 次数

### 使用 Prometheus 查询

```promql
# 消息流入速率（每秒）
rate(kafka_server_brokertopicmetrics_messagesinpersec_total[1m])

# 字节流入速率（每秒）
rate(kafka_server_brokertopicmetrics_bytesinpersec_total[1m])

# 未充分复制的分区
kafka_server_replicamanager_underreplicatedpartitions

# JVM 堆内存使用率
jvm_memory_heap_used / jvm_memory_heap_max * 100
```
