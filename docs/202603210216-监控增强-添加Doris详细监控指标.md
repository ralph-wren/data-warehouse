## 需求描述

用户要求为 Doris 监控添加更详细、更必要的监控指标，以便更全面地了解 Doris 的运行状态和性能表现。

## 解决方案

### 创建增强版监控面板

**文件**: `monitoring/grafana/dashboards/doris-monitoring-enhanced.json`

在原有 13 个面板的基础上，扩展到 38 个面板，覆盖以下监控维度：

## 新增监控指标分类

### 1. FE 查询性能 (面板 1-5)

**核心指标**:
- `doris_fe_qps`: 每秒查询数
- `doris_fe_rps`: 每秒请求数
- `doris_fe_query_latency_ms`: 查询延迟（P95/P99）
- `doris_fe_query_err_rate`: 查询错误率 ⭐ 新增
- `doris_fe_query_err`: 查询错误总数 ⭐ 新增
- `doris_fe_query_instance_num`: 查询实例数 ⭐ 新增

**监控价值**:
- 实时了解查询负载和性能
- 快速发现查询错误和异常
- 监控查询实例数量，避免资源耗尽


### 2. FE 连接管理 (面板 6-7)

**核心指标**:
- `doris_fe_connection_total`: 连接总数（按用户分组）⭐ 新增
- `sum(doris_fe_connection_total)`: 活跃连接总数 ⭐ 新增

**监控价值**:
- 监控不同用户的连接数
- 及时发现连接泄漏或连接池耗尽
- 设置连接数阈值告警

### 3. FE Edit Log (面板 8-9)

**核心指标**:
- `doris_fe_edit_log`: Edit Log 大小 ⭐ 新增
- `doris_fe_edit_log_write_latency_ms`: Edit Log 写入延迟 ⭐ 新增

**监控价值**:
- Edit Log 是 Doris 元数据变更的日志
- 监控 Edit Log 大小，避免过大影响性能
- 监控写入延迟，发现元数据操作瓶颈

### 4. FE RPC 性能 (面板 10-11)

**核心指标**:
- `doris_fe_thrift_rpc_latency_ms`: Thrift RPC 延迟（按服务分组）⭐ 新增
- `rate(doris_fe_thrift_rpc_total[1m])`: RPC 请求速率 ⭐ 新增

**监控价值**:
- Thrift RPC 是 FE 和 BE 之间的通信协议
- 监控 RPC 延迟，发现网络或服务瓶颈
- 监控 RPC 请求速率，了解集群通信负载

### 5. FE 线程池 (面板 12-14)

**核心指标**:
- `doris_fe_thread_pool{type="task_num"}`: 线程池任务数 ⭐ 新增
- `doris_fe_thread_pool{type="queue_size"}`: 线程池队列大小 ⭐ 新增
- `doris_fe_thread_pool{type="active_thread_num"}`: 活跃线程数 ⭐ 新增

**监控价值**:
- 监控各个线程池的负载情况
- 发现线程池队列积压
- 优化线程池配置

### 6. FE Cache (面板 15-16)

**核心指标**:
- `doris_fe_cache_hit / (doris_fe_cache_hit + doris_fe_cache_miss)`: Cache 命中率 ⭐ 新增
- `rate(doris_fe_cache_added[1m])`: Cache 添加速率 ⭐ 新增

**监控价值**:
- 监控 Cache 命中率，优化查询性能
- Cache 命中率低可能需要调整 Cache 大小
- 监控 Cache 添加速率，了解 Cache 更新频率


### 7. FE 事务 (面板 17-18)

**核心指标**:
- `rate(doris_fe_txn_counter[1m])`: 事务速率（begin/success/failed）
- `doris_fe_txn_num`: 运行中事务数
- `doris_fe_publish_txn_num`: 发布中事务数

**监控价值**:
- 监控事务成功率和失败率
- 发现事务积压问题
- 优化事务处理性能

### 8. FE JVM (面板 19-20)

**核心指标**:
- `jvm_heap_size_bytes`: JVM 堆内存使用
- `rate(jvm_gc[1m])`: GC 频率

**监控价值**:
- 监控 JVM 内存使用，避免 OOM
- 监控 GC 频率，发现内存问题
- 优化 JVM 参数配置

### 9. FE 集群状态 (面板 21-26)

**核心指标**:
- `node_info{type="is_master"}`: FE Master 状态
- `node_info{type="be_node_num",state="alive"}`: BE 节点数
- `doris_fe_internal_database_num`: 数据库数量
- `doris_fe_internal_table_num`: 表数量
- `doris_fe_tablet_status_count`: Tablet 状态统计

**监控价值**:
- 监控集群整体健康状态
- 及时发现节点故障
- 监控数据规模增长

### 10. BE Stream Load (面板 27-28)

**核心指标**:
- `rate(doris_be_stream_load_txn_request[1m])`: Stream Load 请求速率 ⭐ 新增
- `doris_be_stream_load_current_processing`: 当前处理中的 Stream Load ⭐ 新增

**监控价值**:
- 监控数据导入速率
- 发现导入瓶颈
- 避免导入任务积压

### 11. BE Compaction (面板 29-31)

**核心指标**:
- `doris_be_compaction_task_state_total`: Compaction 任务状态 ⭐ 新增
- `rate(doris_be_compaction_rows_total[1m])`: Compaction 处理行数 ⭐ 新增
- `doris_be_compaction_max_score`: 最大 Compaction Score ⭐ 新增

**监控价值**:
- Compaction 是 Doris 数据合并和优化的关键过程
- 监控 Compaction 任务状态，发现积压
- Compaction Score 过高会影响查询性能
- 优化 Compaction 配置


### 12. BE Cache (面板 32)

**核心指标**:
- `doris_be_cache_hit_ratio`: BE Cache 命中率 ⭐ 新增

**监控价值**:
- 监控 BE 端的 Cache 命中率
- 优化数据访问性能
- 调整 Cache 配置

### 13. BE 网络 (面板 33-34)

**核心指标**:
- `rate(doris_be_network_send_packets[1m])`: 网络发送包速率 ⭐ 新增
- `rate(doris_be_network_receive_packets[1m])`: 网络接收包速率 ⭐ 新增

**监控价值**:
- 监控网络流量
- 发现网络瓶颈
- 优化网络配置

### 14. BE 内存 (面板 35-36)

**核心指标**:
- `rate(doris_be_memory_pgpgin[1m])`: 内存页面换入速率 ⭐ 新增
- `doris_be_memory_memtable_consumption`: Memtable 内存消耗 ⭐ 新增

**监控价值**:
- 监控内存使用情况
- 发现内存压力
- 优化内存配置

### 15. BE 磁盘 (面板 37-38)

**核心指标**:
- `doris_be_disks_total_capacity`: 磁盘总容量
- `doris_be_disks_avail_capacity`: 磁盘可用容量
- `(doris_be_disks_total_capacity - doris_be_disks_avail_capacity) / doris_be_disks_total_capacity * 100`: 磁盘使用率 ⭐ 新增

**监控价值**:
- 监控磁盘空间使用
- 及时发现磁盘空间不足
- 规划容量扩展

## 关键监控指标说明

### 需要重点关注的指标

#### 1. 查询性能指标
- **doris_fe_query_err_rate**: 查询错误率应该接近 0
- **doris_fe_query_latency_ms (P99)**: P99 延迟应该在可接受范围内
- **doris_fe_qps**: 了解系统负载

#### 2. 连接管理指标
- **doris_fe_connection_total**: 连接数过高可能导致资源耗尽
- 建议设置告警阈值（如 > 100）

#### 3. Edit Log 指标
- **doris_fe_edit_log**: Edit Log 过大会影响 FE 启动速度
- 建议定期清理旧的 Edit Log

#### 4. 线程池指标
- **doris_fe_thread_pool (queue_size)**: 队列积压表示线程池不足
- 需要调整线程池大小或优化任务处理

#### 5. Cache 指标
- **Cache 命中率**: 命中率低会影响查询性能
- 建议命中率 > 80%

#### 6. Compaction 指标
- **doris_be_compaction_max_score**: Score > 100 表示 Compaction 积压严重
- 需要调整 Compaction 配置或增加资源

#### 7. 磁盘使用率
- **磁盘使用率**: 应该 < 80%
- 超过 80% 需要扩容或清理数据

## 告警建议

### 建议配置的告警规则

#### 1. 查询错误率告警
```yaml
- alert: DorisHighQueryErrorRate
  expr: doris_fe_query_err_rate > 0.01
  for: 5m
  annotations:
    summary: "Doris 查询错误率过高"
    description: "查询错误率 {{ $value }}% 超过 1%"
```

#### 2. 连接数告警
```yaml
- alert: DorisHighConnectionCount
  expr: sum(doris_fe_connection_total) > 100
  for: 5m
  annotations:
    summary: "Doris 连接数过高"
    description: "当前连接数 {{ $value }} 超过 100"
```

#### 3. Compaction Score 告警
```yaml
- alert: DorisHighCompactionScore
  expr: doris_be_compaction_max_score > 100
  for: 10m
  annotations:
    summary: "Doris Compaction Score 过高"
    description: "Compaction Score {{ $value }} 超过 100"
```

#### 4. 磁盘使用率告警
```yaml
- alert: DorisHighDiskUsage
  expr: (doris_be_disks_total_capacity - doris_be_disks_avail_capacity) / doris_be_disks_total_capacity * 100 > 80
  for: 5m
  annotations:
    summary: "Doris 磁盘使用率过高"
    description: "磁盘使用率 {{ $value }}% 超过 80%"
```


## 技术要点

### 1. 指标选择原则

**选择标准**:
- ✅ 能反映系统健康状态
- ✅ 能帮助定位性能瓶颈
- ✅ 能预警潜在问题
- ✅ 有明确的优化方向

**避免的指标**:
- ❌ 过于底层，难以理解
- ❌ 变化不大，没有监控价值
- ❌ 无法采取行动的指标

### 2. 面板设计原则

**布局原则**:
- 相关指标放在一起
- 重要指标放在上方
- 使用合适的可视化类型（时序图、仪表盘、统计值）

**颜色和阈值**:
- 绿色: 正常状态
- 黄色: 警告状态（需要关注）
- 红色: 危险状态（需要立即处理）

### 3. 时间聚合

**使用 rate() 函数**:
- 适用于计数器类型的指标
- 计算每秒的变化率
- 示例: `rate(doris_fe_txn_counter[1m])`

**使用 avg_over_time() 函数**:
- 适用于瞬时值类型的指标
- 计算时间窗口内的平均值
- 示例: `avg_over_time(doris_fe_qps[5m])`

### 4. 标签过滤

**按类型过滤**:
```promql
doris_fe_thread_pool{type="task_num"}
```

**按状态过滤**:
```promql
node_info{type="be_node_num",state="alive"}
```

**按数据库过滤**:
```promql
doris_fe_txn_num{db="crypto_dw"}
```

## 故障排查

### 1. 面板显示 "No data"

**可能原因**:
- Prometheus 未采集到指标
- 查询语句错误
- 时间范围内没有数据

**排查步骤**:
```bash
# 1. 检查 Prometheus targets
curl -s http://localhost:9090/api/v1/targets | grep doris

# 2. 在 Prometheus UI 中测试查询
# 访问 http://localhost:9090
# 输入查询语句测试

# 3. 检查 Doris metrics 端点
curl http://127.0.0.1:8030/metrics | grep <metric_name>
curl http://127.0.0.1:8040/metrics | grep <metric_name>
```

### 2. 指标值异常

**可能原因**:
- Doris 配置问题
- 系统资源不足
- 数据倾斜或热点

**排查步骤**:
```bash
# 1. 检查 Doris 日志
docker logs doris-fe
docker logs doris-be

# 2. 检查系统资源
docker stats doris-fe doris-be

# 3. 检查 Doris 内部状态
# 连接到 Doris FE
mysql -h 127.0.0.1 -P 9030 -u root

# 查看 BE 状态
SHOW BACKENDS;

# 查看表状态
SHOW TABLET STATUS;
```


## 性能优化建议

### 基于监控指标的优化方向

#### 1. 查询性能优化
- **现象**: 查询延迟高、QPS 低
- **优化方向**:
  - 优化 SQL 查询语句
  - 添加合适的索引
  - 调整分区和分桶策略
  - 增加 BE 节点

#### 2. 连接管理优化
- **现象**: 连接数过高
- **优化方向**:
  - 使用连接池
  - 及时关闭不用的连接
  - 调整连接超时时间
  - 增加 FE 节点

#### 3. Cache 优化
- **现象**: Cache 命中率低
- **优化方向**:
  - 增加 Cache 大小
  - 优化查询模式
  - 预热热点数据

#### 4. Compaction 优化
- **现象**: Compaction Score 高
- **优化方向**:
  - 增加 Compaction 线程数
  - 调整 Compaction 策略
  - 优化数据导入频率
  - 增加磁盘 I/O 性能

#### 5. 内存优化
- **现象**: 内存使用率高、频繁 GC
- **优化方向**:
  - 增加 JVM 堆内存
  - 优化查询，减少内存使用
  - 调整 GC 参数
  - 增加物理内存

#### 6. 磁盘优化
- **现象**: 磁盘使用率高
- **优化方向**:
  - 清理过期数据
  - 调整数据保留策略
  - 增加磁盘容量
  - 使用数据压缩
