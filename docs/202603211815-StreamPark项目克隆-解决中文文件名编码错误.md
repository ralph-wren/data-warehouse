# StreamPark 项目克隆 - 解决中文文件名编码错误

## 问题描述

在 StreamPark Web UI 中添加 Git 项目时，克隆失败并报错：

```
java.nio.file.InvalidPathException: Malformed input or input contains unmappable characters: 
/tmp/streampark/project/100001/data-warehouse-main/docs/._202603182301-HTTP??-??100-continue??.md
```

## 问题现象

1. **Git clone 失败**：StreamPark 无法克隆包含中文文件名的项目
2. **文件名乱码**：中文字符显示为 `??`
3. **路径无效**：JGit 无法处理乱码的文件路径

## 根本原因 ⭐⭐

**StreamPark 容器默认字符编码不支持中文**

1. **容器默认编码**：StreamPark Docker 镜像默认使用 ASCII 或 Latin-1 编码
2. **中文文件名**：项目中的文档文件名包含中文字符（如 "HTTP请求-解决100-continue错误.md"）
3. **Git clone 失败**：JGit 在 checkout 文件时无法正确处理中文文件名
4. **编码转换失败**：`UnixPath.encode()` 方法无法将中文字符转换为有效的文件路径

## 解决方案

### 方案 1：配置 StreamPark 容器字符编码（推荐）✅

**修改 `docker-compose-streampark.yml`**：

```yaml
services:
  streampark:
    image: apache/streampark:v2.1.5
    container_name: streampark
    environment:
      - TZ=Asia/Shanghai  # 中国时区
      - DATASOURCE_DIALECT=h2
      # ✅ 字符编码配置 - 支持中文文件名
      - LANG=C.UTF-8
      - LC_ALL=C.UTF-8
      - JAVA_TOOL_OPTIONS=-Dfile.encoding=UTF-8
```

**环境变量说明**：

| 环境变量 | 作用 | 值 |
|---------|------|-----|
| `LANG` | 系统默认语言和编码 | `C.UTF-8` |
| `LC_ALL` | 覆盖所有本地化设置 | `C.UTF-8` |
| `JAVA_TOOL_OPTIONS` | JVM 启动参数 | `-Dfile.encoding=UTF-8` |

**重启 StreamPark**：

```bash
# 停止容器
docker-compose -f docker-compose-streampark.yml down

# 启动容器（应用新配置）
docker-compose -f docker-compose-streampark.yml up -d

# 等待启动完成（约 30 秒）
sleep 30

# 验证编码配置
docker exec streampark env | grep -E "(LANG|LC_ALL|JAVA_TOOL)"
```

**预期输出**：
```
JAVA_TOOL_OPTIONS=-Dfile.encoding=UTF-8
LANG=C.UTF-8
LC_ALL=C.UTF-8
```

### 方案 2：重命名文件为英文（不推荐）❌

将所有包含中文的文件名改为英文：

```bash
# 示例
mv "docs/202603182301-HTTP请求-解决100-continue错误.md" \
   "docs/202603182301-HTTP-Request-Fix-100-Continue-Error.md"
```

**缺点**：
- 需要重命名大量文件（50+ 个）
- 失去中文描述的可读性
- 不符合项目规范（AGENTS.md 要求使用中文）

## 技术要点

### UTF-8 编码的重要性

1. **国际化支持**：UTF-8 可以表示所有 Unicode 字符，包括中文、日文、韩文等
2. **Git 兼容性**：Git 默认使用 UTF-8 编码存储文件名
3. **Java 默认编码**：Java 在不同平台上的默认编码不同
   - Linux: 通常是 UTF-8
   - Windows: GBK 或 GB2312
   - Docker 容器: 取决于基础镜像

### 编码配置层级

```
系统级别 (LANG, LC_ALL)
    ↓
JVM 级别 (JAVA_TOOL_OPTIONS, file.encoding)
    ↓
应用级别 (代码中指定编码)
```

### JGit 编码处理

JGit 在处理文件路径时：
1. 从 Git 对象数据库读取文件名（UTF-8 编码）
2. 使用 `UnixPath.encode()` 转换为系统路径
3. 如果系统编码不支持，抛出 `InvalidPathException`

## 常见问题

### Q1: 为什么使用 `C.UTF-8` 而不是 `zh_CN.UTF-8`？

**A**: 
- `C.UTF-8` 是通用的 UTF-8 locale，所有 Linux 发行版都支持
- `zh_CN.UTF-8` 需要安装中文语言包，可能在容器中不可用
- `C.UTF-8` 提供 UTF-8 编码支持，但使用 C locale 的排序和格式规则

### Q2: `JAVA_TOOL_OPTIONS` 和 `JAVA_OPTS` 有什么区别？

**A**:
- `JAVA_TOOL_OPTIONS`: JVM 自动识别，适用于所有 Java 进程
- `JAVA_OPTS`: 需要应用程序显式使用，不是 JVM 标准变量
- StreamPark 使用 `JAVA_TOOL_OPTIONS` 更可靠

### Q3: 修改编码后需要重新克隆项目吗？

**A**: 
- 是的，之前失败的克隆需要删除并重新添加
- 或者在 StreamPark 中删除项目，然后重新添加

### Q4: 这个配置会影响性能吗？

**A**:
- 不会，UTF-8 编码处理的性能开销可以忽略不计
- 现代 JVM 对 UTF-8 编码有很好的优化

## 最佳实践

### 1. 文件命名规范

虽然现在支持中文文件名，但建议：
- 关键配置文件使用英文命名
- 文档文件可以使用中文命名（提高可读性）
- 避免使用特殊字符（如 `<>:"/\|?*`）

### 2. Git 配置

确保 Git 仓库配置正确：
```bash
# 设置 Git 使用 UTF-8 编码
git config --global core.quotepath false
git config --global gui.encoding utf-8
git config --global i18n.commitencoding utf-8
git config --global i18n.logoutputencoding utf-8
```

### 3. 容器编码标准化

所有 Docker 容器都应该配置 UTF-8 编码：
```yaml
environment:
  - LANG=C.UTF-8
  - LC_ALL=C.UTF-8
```

## 故障排查

### 问题：编码配置后仍然失败

**检查步骤**：

1. **验证环境变量**：
   ```bash
   docker exec streampark env | grep -E "(LANG|LC_ALL|JAVA_TOOL)"
   ```

2. **检查 JVM 编码**：
   ```bash
   docker exec streampark java -XshowSettings:properties -version 2>&1 | grep file.encoding
   ```

3. **查看详细日志**：
   ```bash
   docker logs streampark --tail 100 | grep -i "encoding\|charset"
   ```

4. **测试文件系统**：
   ```bash
   docker exec streampark bash -c "echo 'test' > /tmp/中文测试.txt && cat /tmp/中文测试.txt"
   ```

### 问题：容器重启后配置丢失

**原因**：环境变量没有持久化到 docker-compose.yml

**解决**：确保 `docker-compose-streampark.yml` 中包含编码配置，并使用 `docker-compose up -d` 启动

通过配置 StreamPark 容器的字符编码为 UTF-8，成功解决了中文文件名导致的 Git clone 失败问题。

**关键配置**：
- `LANG=C.UTF-8`
- `LC_ALL=C.UTF-8`
- `JAVA_TOOL_OPTIONS=-Dfile.encoding=UTF-8`

**技术要点**：
- Docker 容器默认编码可能不支持中文
- JGit 需要正确的字符编码才能处理中文文件名
- UTF-8 是国际化应用的标准编码

**最佳实践**：
- 所有容器都应该配置 UTF-8 编码
- 文档文件可以使用中文命名（提高可读性）
- 关键配置文件建议使用英文命名
