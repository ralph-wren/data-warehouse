# StreamPark 权限配置 - 解决系统设置无权限问题

## 问题描述

用户登录 StreamPark 后，访问系统设置页面时提示"无权限"或"internal server error"。

## 问题分析

1. **权限配置检查**：
   - admin 用户已创建（user_id=100000, user_type=1）
   - Admin 角色已创建（role_id=100001）
   - 81 个菜单权限已全部分配给 Admin 角色
   - 用户已添加到 default 团队

2. **系统设置相关权限**：
   ```
   menu_id | menu_name            | perms
   --------|----------------------|----------------
   130000  | menu.setting         | NULL
   130100  | setting.system       | NULL
   130101  | view                 | setting:view
   130102  | setting update       | setting:update
   130200  | setting.alarm        | NULL
   130300  | setting.flinkHome    | NULL
   130400  | setting.flinkCluster | menu:view
   130500  | setting.externalLink | menu:view
   130600  | setting.yarnQueue    | menu:view
   ```

3. **根本原因**：
   - 权限配置完全正确
   - 问题在于 **Shiro 权限缓存** 和 **浏览器 Session 缓存**
   - 新分配的权限需要重启容器和重新登录才能生效

## 解决方案

### 方案 1：重启 StreamPark 容器（推荐）

```bash
# 使用提供的脚本重启
./test/restart-streampark-refresh-permissions.sh
```

脚本会自动：
1. 停止 StreamPark 容器
2. 等待容器完全停止
3. 重新启动容器
4. 等待容器完全启动（约 30 秒）

### 方案 2：清除浏览器缓存

1. **清除缓存**：
   - Chrome/Edge: 按 `Ctrl+Shift+Delete`
   - 选择"Cookie 和其他网站数据"
   - 选择"缓存的图片和文件"
   - 点击"清除数据"

2. **使用无痕模式**：
   - Chrome/Edge: 按 `Ctrl+Shift+N`
   - 访问 http://localhost:10000

3. **重新登录**：
   - 用户名: `admin`
   - 密码: `streampark`

### 方案 3：手动重启容器

```bash
# 停止容器
docker-compose -f docker-compose-streampark.yml down

# 启动容器
docker-compose -f docker-compose-streampark.yml up -d

# 等待 30 秒后访问
sleep 30
```

## 技术细节

### StreamPark 权限模型

```
用户 (t_user)
  ↓
成员关系 (t_member)
  ↓
角色 (t_role)
  ↓
角色菜单关系 (t_role_menu)
  ↓
菜单权限 (t_menu)
```

### 权限配置

```sql
-- 用户配置
user_id: 100000
username: admin
user_type: 1 (管理员)
last_team_id: 100000

-- 团队配置
team_id: 100000
team_name: default

-- 角色配置
role_id: 100001
role_name: Admin

-- 权限配置
menu_count: 81 (所有菜单权限)
```

### Shiro 权限缓存

StreamPark 使用 Apache Shiro 进行权限管理：
- 权限信息会被缓存在内存中
- 修改权限后需要重启容器或清除缓存
- Session 中也会缓存用户权限信息

## 相关文件

- `test/restart-streampark-refresh-permissions.sh` - 重启容器刷新权限
- `test/verify-streampark-permissions.sh` - 验证权限配置
- `test/init-streampark-complete.sh` - 完整初始化脚本
- `docker-compose-streampark.yml` - StreamPark 配置

## 注意事项

1. **首次登录**：初始化后首次登录可能需要等待容器完全启动
2. **权限缓存**：修改权限后务必重启容器或清除浏览器缓存
3. **Session 过期**：长时间未操作可能导致 Session 过期，需要重新登录
4. **数据库连接**：确保 MySQL 数据库正常运行且可连接

## 总结

权限配置完全正确，所有 81 个菜单权限都已分配给 Admin 角色。问题是由于 Shiro 权限缓存和浏览器 Session 缓存导致的，重启容器并清除浏览器缓存后即可正常访问系统设置页面。
