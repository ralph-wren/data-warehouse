#### 整体架构
```
数据源层 → 消息队列层 → 流处理层 → 数据仓库层 → 监控层 → 管理层
```

#### 详细组件架构

展示了所有组件的详细信息：

**数据采集层 (本地运行)**:
- OKX WebSocket Client
- 订阅实时行情数据
- 发送到 Kafka

**消息队列层 (Docker)**:
- Apache Kafka 7.6.0
- Topic: crypto-ticker (4 分区)
- 端口: 9092, 9093

**流处理层 (本地/Docker)**:
- Apache Flink 1.17.2
- ODS 作业 (端口 8081)
- DWD 作业 (端口 8082)
- DWS 作业 (端口 8083)
- Flink Standalone 集群 (可选)

**数据仓库层 (Docker)**:
- Apache Doris 3.0.7
- FE (端口 8030, 9030)
- BE (端口 8040, 9050)
- 数据分层: ODS, DWD, DWS

**监控层 (Docker)**:
- Prometheus 2.45.0 (端口 9090)
- Pushgateway 1.6.0 (端口 9091)
- Grafana 10.0.0 (端口 3000)

**管理层 (Docker)**:
- StreamPark 2.1.0 (端口 10000)

**数据库层 (本地)**:
- MySQL 8.0 (端口 3306)

### 2. 数据流转图

详细描述了数据从 OKX API 到 Doris 数据仓库的完整流程：

```
OKX API (WebSocket)
  ↓
数据采集器 (OKXWebSocketClient)
  ↓ Kafka Producer
Kafka Topic: crypto-ticker (4 分区)
  ↓ Kafka Consumer
Flink ODS 作业 (FlinkODSJobDataStream)
  ↓ Doris Stream Load
Doris ODS 层: ods_crypto_ticker_rt
  ↓ Flink Table API
Flink DWD 作业 (FlinkDWDJobSQL)
  ↓ Doris Sink
Doris DWD 层: dwd_crypto_ticker_detail
  ↓ Flink Table API
Flink DWS 作业 (FlinkDWSJob1MinSQL)
  ↓ Doris Sink
Doris DWS 层: dws_crypto_kline_1min
```

**关键信息**:
- 数据格式: JSON
- 消费模式: earliest (从头开始)
- 批量写入: 1000条/批 或 5秒
- 窗口聚合: 1分钟滚动窗口

### 3. 网络通信图

展示了所有 Docker 网络和端口映射：

**Docker 网络**:
- `data-warehouse_default`: Kafka, Zookeeper
- `data-warehouse_doris-net`: Doris FE, BE (172.25.0.0/16)
- `data-warehouse_flink-net`: Flink JobManager, TaskManager
- `data-warehouse_monitoring-net`: Prometheus, Grafana, Pushgateway
- `data-warehouse_streampark-net`: StreamPark (连接多个网络)

**端口映射**:
- 数据层: 9092 (Kafka), 8030/9030 (Doris FE), 8040 (Doris BE)
- 计算层: 8081 (Flink)
- 监控层: 9090 (Prometheus), 9091 (Pushgateway), 3000 (Grafana)
- 管理层: 10000 (StreamPark), 3306 (MySQL)

**通信协议**:
- OKX API → 数据采集器: WebSocket (WSS)
- 数据采集器 → Kafka: Kafka Protocol
- Flink → Kafka: Kafka Protocol
- Flink → Doris: HTTP Stream Load / JDBC
- Flink → Pushgateway: HTTP POST
- Prometheus → 数据源: HTTP GET
- Grafana → Prometheus: HTTP
- StreamPark → Flink: HTTP REST API
- StreamPark → MySQL: JDBC
