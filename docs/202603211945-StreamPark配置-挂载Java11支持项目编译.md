# StreamPark 配置 - 挂载 Java 11 支持项目编译

## 更新时间

2026-03-21 19:45 (东八区)

## 问题描述

项目需要使用 Java 11 编译，但 StreamPark 容器内置的是 Java 8，无法编译 Java 11 项目。

## 解决方案

通过 Docker 卷挂载的方式，将宿主机的 Java 11 挂载到 StreamPark 容器中。

## 实施步骤

### 1. 下载和安装 Java 11

#### 方案 A: 使用 OpenJDK 11 (Temurin) - 推荐 ✅

**下载地址**: https://adoptium.net/temurin/releases/?version=11

**下载命令**:
```bash
# 下载 OpenJDK 11 (Temurin)
curl -L -o /tmp/openjdk11.zip 'https://github.com/adoptium/temurin11-binaries/releases/download/jdk-11.0.22%2B7/OpenJDK11U-jdk_x64_windows_hotspot_11.0.22_7.zip'

# 解压到 Java 目录
unzip /tmp/openjdk11.zip -d '/e/DataFiles/Java/'

# 重命名目录
mv '/e/DataFiles/Java/jdk-11.0.22+7' '/e/DataFiles/Java/jdk-11'
```

#### 方案 B: 使用 Oracle JDK 11

**下载地址**: https://www.oracle.com/java/technologies/javase/jdk11-archive-downloads.html

**注意**: Oracle JDK 需要登录才能下载

**推荐下载**: 
- Windows x64 Compressed Archive (jdk-11.0.50_windows-x64_bin.zip)

**安装步骤**:
1. 手动下载 JDK 11 压缩包
2. 解压到 `E:\DataFiles\Java\jdk-11`

### 2. 验证 Java 11 安装

```bash
# 检查 Java 11 目录
ls -la /e/DataFiles/Java/jdk-11

# 验证 Java 版本
/e/DataFiles/Java/jdk-11/bin/java -version

# 预期输出
java version "11.0.x"
Java(TM) SE Runtime Environment (build 11.0.x+x)
Java HotSpot(TM) 64-Bit Server VM (build 11.0.x+x, mixed mode)
```

### 3. 更新 docker-compose-streampark.yml

**修改内容**:

```yaml
services:
  streampark:
    environment:
      # ... 其他环境变量
      
      # Java 11 配置 - 使用挂载的 Java 11
      - JAVA_HOME=/opt/java11
      - PATH=/opt/java11/bin:$PATH
      
    volumes:
      # ... 其他挂载
      
      # 挂载 Java 11 到容器 - 用于编译和运行 Java 11 项目
      - E:\DataFiles\Java\jdk-11:/opt/java11:ro
```

**关键配置说明**:

1. **环境变量**:
   - `JAVA_HOME=/opt/java11`: 设置 Java 11 为默认 Java
   - `PATH=/opt/java11/bin:$PATH`: 将 Java 11 添加到 PATH 最前面

2. **卷挂载**:
   - 源路径: `E:\DataFiles\Java\jdk-11` (宿主机 Java 11 目录)
   - 目标路径: `/opt/java11` (容器内路径)
   - 挂载模式: `:ro` (只读)

### 4. 重启 StreamPark 容器

```bash
# 停止容器
docker-compose -f docker-compose-streampark.yml down

# 启动容器
docker-compose -f docker-compose-streampark.yml up -d

# 等待容器启动
sleep 30

# 查看日志
docker logs streampark
```

### 6. 更新项目 pom.xml

将项目的 Java 版本改回 Java 11:

```xml
<properties>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
    <!-- 使用 Java 11 -->
    <maven.compiler.source>11</maven.compiler.source>
    <maven.compiler.target>11</maven.compiler.target>
    <flink.version>1.17.2</flink.version>
</properties>
```

### 7. 在 StreamPark 中编译项目

1. 访问 StreamPark Web UI: http://localhost:10000
2. 登录 (admin/streampark)
3. 添加项目 (Git 仓库)
4. 构建项目

StreamPark 会自动使用挂载的 Java 11 进行编译。

## 技术要点

### 1. 为什么需要 Java 11

**Flink 1.17.2 支持的 Java 版本**:
- Java 8 ✅
- Java 11 ✅
- Java 17 ✅

**项目使用 Java 11 的原因**:
- 更好的性能
- 更多的语言特性
- 更好的 GC 性能
- 长期支持 (LTS) 版本

### 2. Docker 卷挂载

**挂载方式**:
```yaml
volumes:
  - <宿主机路径>:<容器路径>:<挂载模式>
```

**挂载模式**:
- `ro`: 只读 (推荐，防止容器修改宿主机文件)
- `rw`: 读写 (默认)

**Windows 路径转换**:
- Windows 路径: `E:\DataFiles\Java\jdk-11`
- Docker 路径: `E:\DataFiles\Java\jdk-11` (Docker Desktop 自动处理)
- Git Bash 路径: `/e/DataFiles/Java/jdk-11`

### 3. 环境变量优先级

**PATH 环境变量**:
```bash
PATH=/opt/java11/bin:$PATH
```

- `/opt/java11/bin` 在最前面，优先级最高
- 容器内置的 Java 8 在后面，优先级较低
- 执行 `java` 命令时，会使用 `/opt/java11/bin/java`

**JAVA_HOME 环境变量**:
```bash
JAVA_HOME=/opt/java11
```

- Maven 和其他工具会使用 `$JAVA_HOME/bin/java`
- 确保使用正确的 Java 版本

### 4. Java 版本兼容性

| Java 版本 | Flink 1.17 | StreamPark | 项目编译 | 推荐 |
|----------|-----------|-----------|---------|------|
| Java 8   | ✅ 支持    | ✅ 内置    | ✅ 可用  | ⚠️ 旧版本 |
| Java 11  | ✅ 支持    | ✅ 挂载    | ✅ 推荐  | ✅ 推荐 |
| Java 17  | ✅ 支持    | ✅ 挂载    | ✅ 可用  | ⚠️ 较新 |
| Java 21  | ❌ 不支持  | ❌ 不兼容  | ❌ 不可用 | ❌ 太新 |

## 常见问题

### 1. Java 11 目录未挂载

**症状**:
```
❌ Java 11 目录未挂载
```

**解决方案**:
1. 检查 Java 11 是否已安装: `ls -la /e/DataFiles/Java/jdk-11`
2. 检查 docker-compose-streampark.yml 中的挂载路径
3. 重启容器: `docker-compose -f docker-compose-streampark.yml restart`

### 2. Java 版本不正确

**症状**:
```
⚠️  Java 版本可能不是 11
```

**解决方案**:
1. 验证宿主机 Java 11 版本: `/e/DataFiles/Java/jdk-11/bin/java -version`
2. 检查挂载路径是否正确
3. 确保挂载的是 JDK 而不是 JRE

### 3. 默认 java 不是 Java 11

**症状**:
```
⚠️  默认 java 不是 Java 11
```

**原因**:
- 容器内置的 Java 8 优先级更高
- PATH 环境变量配置不正确

**解决方案**:
1. 检查 PATH 环境变量: `docker exec streampark printenv PATH`
2. 确保 `/opt/java11/bin` 在 PATH 最前面
3. 重启容器应用新配置

### 4. 编译失败

**症状**:
```
❌ Java 11 编译失败
```

**解决方案**:
1. 检查 Java 11 是否完整安装
2. 验证 JDK 目录结构: `docker exec streampark ls -la /opt/java11`
3. 查看详细错误信息: `docker logs streampark`

## 最佳实践

### 1. 使用 OpenJDK

推荐使用 OpenJDK (Temurin) 而不是 Oracle JDK:
- 免费开源
- 无需登录下载
- 与 Oracle JDK 完全兼容
- 长期支持

### 2. 只读挂载

使用 `:ro` 只读挂载模式:
- 防止容器修改宿主机文件
- 提高安全性
- 避免意外删除或修改

### 3. 统一 Java 版本

确保开发环境和 StreamPark 使用相同的 Java 版本:
- 本地开发: Java 11
- StreamPark: Java 11
- Flink 集群: Java 11

### 4. 定期更新

定期更新 Java 11 到最新版本:
- 获取安全补丁
- 获取性能改进
- 获取 bug 修复
