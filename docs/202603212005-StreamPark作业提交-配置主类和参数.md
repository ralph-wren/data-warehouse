# StreamPark 作业提交 - 配置主类和参数

**时间**: 2026-03-21 20:05  
**问题**: StreamPark 提交 Flink 作业失败，报错 `System.exit(1) was called in your flink job`  
**状态**: ✅ 已解决

---

## 问题描述

在 StreamPark Web UI 中提交 Flink 作业时失败，错误信息：

```
java.util.concurrent.CompletionException: java.lang.reflect.InvocationTargetException
Caused by: java.lang.RuntimeException: [flink-submit] Both JobGraph submit plan and Rest API submit plan all failed!
Caused by: org.apache.flink.client.program.ProgramInvocationException: The main method caused an error: System.exit(1) was called in your flink job
```

## 问题分析

### 1. 错误原因

**`System.exit(1)` 被调用**：
- Flink 作业在启动时遇到错误
- 可能是配置加载失败、依赖缺失、连接失败等
- 导致作业异常退出

### 2. 诊断结果

运行诊断脚本 `test/diagnose-streampark-job.sh`：

```bash
bash test/diagnose-streampark-job.sh
```

**发现的问题**：
1. ✅ 配置文件已打包到 JAR：`config/application.yml`、`config/application-dev.yml`
2. ✅ 主类已打包：`FlinkODSJobDataStream.class`
3. ❌ **MANIFEST.MF 中的 Main-Class 配置错误**：
   - 当前配置：`Main-Class: com.crypto.dw.Main`
   - 正确配置：应该在 StreamPark 中指定具体的作业类

### 3. 根本原因

**StreamPark 需要明确指定作业的主类**：
- JAR 包中的 `Main-Class` 只是默认值
- StreamPark 允许一个 JAR 包包含多个 Flink 作业
- 需要在 StreamPark Web UI 中手动指定要运行的主类

---

## 解决方案

### 方案 1：在 StreamPark Web UI 中指定主类（推荐）✅

**步骤**：

1. **访问 StreamPark Web UI**：
   - URL: http://localhost:10000
   - 用户名：`admin`
   - 密码：`streampark`

2. **创建新应用**：
   - 点击左侧菜单 "Application" → "Add New"

3. **填写基本信息**：
   - **Application Name**: `Flink ODS Job`
   - **Development Mode**: `Flink SQL` 或 `Custom Code`（选择 Custom Code）
   - **Execution Mode**: `Remote`（提交到 Flink 集群）
   - **Flink Cluster**: 选择之前添加的 Flink 集群

4. **配置作业参数**：
   
   **Program Jar**：
   - 上传或选择 JAR 文件：`target/realtime-crypto-datawarehouse-1.0.0.jar`
   
   **Main Class**（重要！）：
   ```
   com.crypto.dw.flink.FlinkODSJobDataStream
   ```
   
   **Program Args**（可选）：
   ```
   # 如果需要传递命令行参数
   ```
   
   **Flink Configuration**：
   ```yaml
   # Flink 运行时配置
   taskmanager.numberOfTaskSlots: 4
   parallelism.default: 4
   ```
   
   **Dynamic Properties**（环境变量）：
   ```
   APP_ENV=dev
   ```

5. **保存并启动**：
   - 点击 "Submit" 保存配置
   - 点击 "Start" 启动作业

### 方案 2：修改 pom.xml 的 Main-Class（可选）

如果你希望 JAR 包可以直接运行（不通过 StreamPark），可以修改 `pom.xml`：

```xml
<transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">
    <!-- 修改为你的主类 -->
    <mainClass>com.crypto.dw.flink.FlinkODSJobDataStream</mainClass>
</transformer>
```

**注意**：
- 这种方式只能指定一个主类
- 如果有多个 Flink 作业，建议使用方案 1

**配置要求**：

- 写入 Doris DWS 层表

## StreamPark 作业配置示例

### ODS 作业配置

```yaml
# Application 配置
Application Name: Flink ODS Job
Development Mode: Custom Code
Execution Mode: Remote
Flink Cluster: Flink Docker Cluster

# Program 配置
Program Jar: /path/to/realtime-crypto-datawarehouse-1.0.0.jar
Main Class: com.crypto.dw.flink.FlinkODSJobDataStream

# Flink 配置
Flink Configuration:
  taskmanager.numberOfTaskSlots: 4
  parallelism.default: 4
  state.checkpoints.dir: file:///tmp/flink-checkpoints
  state.savepoints.dir: file:///tmp/flink-savepoints

# 环境变量
Dynamic Properties:
  APP_ENV: dev

# 资源配置
Task Slots: 4
Job Manager Memory: 1024m
Task Manager Memory: 2048m
```

### DWD 作业配置

```yaml
# Application 配置
Application Name: Flink DWD Job
Development Mode: Custom Code
Execution Mode: Remote
Flink Cluster: Flink Docker Cluster

# Program 配置
Program Jar: /path/to/realtime-crypto-datawarehouse-1.0.0.jar
Main Class: com.crypto.dw.flink.FlinkDWDJobSQL

# Flink 配置
Flink Configuration:
  taskmanager.numberOfTaskSlots: 4
  parallelism.default: 4
  table.local-time-zone: Asia/Shanghai  # 重要：设置时区

# 环境变量
Dynamic Properties:
  APP_ENV: dev
```

## 常见问题排查

### 1. 作业启动失败：`ClassNotFoundException`

**原因**：主类名称错误或类不在 JAR 包中

**解决**：
```bash
# 检查 JAR 包中的类
jar tf target/realtime-crypto-datawarehouse-1.0.0.jar | grep FlinkODSJobDataStream

# 应该输出
com/crypto/dw/flink/FlinkODSJobDataStream.class
```

### 2. 作业启动失败：`System.exit(1)`

**原因**：配置加载失败或依赖缺失

**解决**：
1. 检查环境变量是否设置：`APP_ENV=dev`
2. 检查配置文件是否在 JAR 包中：
   ```bash
   jar tf target/realtime-crypto-datawarehouse-1.0.0.jar | grep config/
   ```
3. 检查依赖服务是否运行：
   ```bash
   # Kafka
   docker ps | grep kafka
   
   # Doris
   docker ps | grep doris
   
   # Flink
   curl http://localhost:8081/overview
   ```

### 3. 作业运行但无数据

**原因**：Kafka 消费模式配置错误

**解决**：
- 在配置文件中设置：`kafka.consumer.startup-mode: earliest`
- 或者在 StreamPark 中添加环境变量：`KAFKA_STARTUP_MODE=earliest`

### 4. 连接 Doris 失败

**原因**：网络配置或地址错误

**解决**：
1. 检查 Doris 地址配置：
   - FE HTTP: `http://127.0.0.1:8030`
   - FE JDBC: `jdbc:mysql://127.0.0.1:9030`
   - BE: `127.0.0.1:8040`

2. 测试连接：
   ```bash
   # 测试 FE
   curl http://127.0.0.1:8030/api/bootstrap
   
   # 测试 JDBC
   mysql -h 127.0.0.1 -P 9030 -u root
   ```
