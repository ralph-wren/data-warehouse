# 配置管理 - 添加 Docker 环境配置文件

## 问题描述

项目需要在不同环境下运行：
1. **本地开发环境**：直接在 IDE 中运行，连接本地的 Kafka 和 Doris
2. **Docker 环境**：在 Docker 容器中运行（如 StreamPark），需要使用容器名称访问服务

之前只有一个 `application-dev.yml` 配置文件，混合了本地和 Docker 的配置，导致：
- 本地运行时需要手动修改配置
- Docker 运行时需要手动修改配置
- 容易出错，配置管理混乱

## 解决方案

创建独立的环境配置文件，通过环境变量 `APP_ENV` 来切换：

### 配置文件结构

```
src/main/resources/config/
├── application.yml          # 基础配置（所有环境共享）
├── application-dev.yml      # 开发环境配置（本地运行）
└── application-docker.yml   # Docker 环境配置（容器运行）
```

### 1. application.yml（基础配置）

包含所有环境共享的默认配置：
- OKX WebSocket 配置
- Kafka 基础配置
- Doris 基础配置
- Flink 基础配置
- 应用基础配置

**关键配置**：
```yaml
# 默认使用 localhost
kafka:
  bootstrap-servers: localhost:9092

doris:
  fe:
    http-url: http://127.0.0.1:8030
    jdbc-url: jdbc:mysql://127.0.0.1:9030
```

### 2. application-dev.yml（本地开发环境）

用于本地 IDE 开发，覆盖基础配置：

**关键配置**：
```yaml
# 本地开发环境使用 localhost
kafka:
  bootstrap-servers: localhost:9092

doris:
  fe:
    http-url: http://127.0.0.1:8030
    jdbc-url: jdbc:mysql://127.0.0.1:9030
  be:
    nodes: "127.0.0.1:8040"

application:
  logging:
    level: DEBUG  # 本地开发使用 DEBUG 级别
```

**使用场景**：
- 在 IDE 中直接运行 Flink 作业
- 本地调试和开发
- 连接本地 Docker 部署的 Kafka 和 Doris

**启动方式**：
```bash
# 方式 1：设置环境变量
export APP_ENV=dev
java -jar target/realtime-crypto-datawarehouse-1.0.0.jar

# 方式 2：在 IDE 中设置 VM 参数
-DAPP_ENV=dev

# 方式 3：在启动脚本中设置
APP_ENV=dev bash run-flink-ods-datastream.sh
```

### 3. application-docker.yml（Docker 环境）

用于在 Docker 容器中运行，使用容器名称访问服务：

**关键配置**：
```yaml
# Docker 环境使用容器名称
kafka:
  bootstrap-servers: kafka:9092  # Kafka 容器名称

doris:
  fe:
    http-url: http://doris-fe:8030  # Doris FE 容器名称
    jdbc-url: jdbc:mysql://doris-fe:9030
  be:
    nodes: "doris-be:8040"  # Doris BE 容器名称

flink:
  state:
    # Docker 容器内路径
    checkpoint-dir: file:///opt/flink/checkpoints
    savepoint-dir: file:///opt/flink/savepoints

application:
  logging:
    level: INFO  # Docker 环境使用 INFO 级别
    file: /opt/flink/logs/crypto-dw.log
```

**使用场景**：
- 在 StreamPark 中提交 Flink 作业
- 在 Flink Standalone 集群中运行
- 在 Kubernetes 中部署

**启动方式**：
```bash
# 在 StreamPark 中设置 Dynamic Properties
APP_ENV=docker

# 或在 Docker Compose 中设置环境变量
environment:
  - APP_ENV=docker
```

## 配置加载机制

### ConfigLoader 加载顺序

1. 加载 `application.yml`（基础配置）
2. 根据 `APP_ENV` 环境变量加载对应的配置文件：
   - `APP_ENV=dev` → 加载 `application-dev.yml`
   - `APP_ENV=docker` → 加载 `application-docker.yml`
   - 未设置 → 只使用 `application.yml`
3. 环境特定配置会覆盖基础配置

### 代码实现

```java
// ConfigLoader.java
public class ConfigLoader {
    private static final String BASE_CONFIG = "config/application.yml";
    private static final String ENV_KEY = "APP_ENV";
    
    public static Config load() {
        // 1. 加载基础配置
        Config baseConfig = loadYaml(BASE_CONFIG);
        
        // 2. 获取环境变量
        String env = System.getenv(ENV_KEY);
        if (env == null || env.isEmpty()) {
            env = System.getProperty(ENV_KEY, "");
        }
        
        // 3. 加载环境特定配置
        if (!env.isEmpty()) {
            String envConfig = "config/application-" + env + ".yml";
            Config envSpecificConfig = loadYaml(envConfig);
            // 合并配置（环境配置覆盖基础配置）
            baseConfig = baseConfig.withFallback(envSpecificConfig);
        }
        
        return baseConfig;
    }
}
```

## 最佳实践

### 1. 环境变量优先级

```
命令行参数 > 环境变量 > 配置文件
```

### 2. 敏感信息处理

敏感信息（如 API Key）应该：
- ✅ 使用环境变量：`${OKX_API_KEY}`
- ✅ 不提交到 Git
- ❌ 不硬编码在配置文件中

### 3. 配置文件命名规范

```
application.yml           # 基础配置
application-{env}.yml     # 环境特定配置
```

常见环境名称：
- `dev`: 开发环境（本地）
- `docker`: Docker 容器环境
- `test`: 测试环境
- `staging`: 预发布环境
- `prod`: 生产环境

### 4. 配置验证

在应用启动时验证关键配置：

```java
public class ConfigValidator {
    public static void validate(Config config) {
        // 验证必需配置
        requireNonEmpty(config, "kafka.bootstrap-servers");
        requireNonEmpty(config, "doris.fe.http-url");
        requireNonEmpty(config, "doris.database");
        
        // 验证配置格式
        validateUrl(config.getString("doris.fe.http-url"));
        validateJdbcUrl(config.getString("doris.fe.jdbc-url"));
        
        logger.info("配置验证通过");
    }
}
```

## 故障排查

### 问题 1：配置文件未加载

**现象**：应用使用默认配置，环境特定配置未生效

**排查步骤**：
```bash
# 1. 检查环境变量
echo $APP_ENV

# 2. 检查配置文件是否存在
jar -tf target/realtime-crypto-datawarehouse-1.0.0.jar | grep config

# 3. 查看日志
tail -f logs/flink-app.log | grep "Loading config"
```

**解决方法**：
- 确保设置了 `APP_ENV` 环境变量
- 确保配置文件已打包到 JAR 中
- 检查配置文件名称是否正确

### 问题 2：容器名称无法解析

**现象**：`UnknownHostException: kafka` 或 `doris-fe`

**排查步骤**：
```bash
# 检查容器网络
docker network inspect data-warehouse_default

# 测试容器间连通性
docker exec flink-jobmanager ping -c 3 kafka
docker exec flink-jobmanager ping -c 3 doris-fe
```

**解决方法**：
- 确保容器在同一个 Docker 网络中
- 检查 `docker-compose.yml` 中的 networks 配置
- 使用 `APP_ENV=docker` 配置

### 问题 3：配置覆盖不生效

**现象**：环境配置没有覆盖基础配置

**原因**：ConfigLoader 合并逻辑错误

**解决方法**：
```java
// 正确的合并顺序（环境配置优先）
Config finalConfig = envConfig.withFallback(baseConfig);

// 错误的合并顺序
Config finalConfig = baseConfig.withFallback(envConfig);  // ❌
```
