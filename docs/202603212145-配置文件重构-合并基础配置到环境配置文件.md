# 配置文件重构 - 合并基础配置到环境配置文件

**时间**: 2026-03-21 21:45  
**类型**: 配置管理  
**状态**: ✅ 已完成

## 问题背景

之前的配置文件结构采用"基础配置 + 环境配置"的方式：
- `application.yml` - 基础配置（所有环境共享）
- `application-dev.yml` - 开发环境特定配置
- `application-docker.yml` - Docker 环境特定配置

`ConfigLoader` 会先加载基础配置，再加载环境配置，然后合并两者。

这种方式存在以下问题：
1. 配置分散在多个文件中，不够直观
2. 需要额外的合并逻辑，增加代码复杂度
3. 环境配置文件不完整，无法独立使用

## 解决方案

### 1. 配置文件重构

将 `application.yml` 中的所有配置合并到各个环境配置文件中：

```yaml
# application-dev.yml - 完整的开发环境配置
okx:
  websocket:
    url: wss://ws.okx.com:8443/ws/v5/public
  api:
    key: ${OKX_API_KEY}
  symbols: BTC-USDT,ETH-USDT,...

kafka:
  bootstrap-servers: localhost:9092  # 本地地址
  topic:
    crypto-ticker: crypto_ticker
  ...

doris:
  fe:
    http-url: http://127.0.0.1:8030  # 本地地址
  ...

flink:
  execution:
    parallelism: 4
  checkpoint:
    interval: 60000
  ...
```

```yaml
# application-docker.yml - 完整的 Docker 环境配置
okx:
  # 与 dev 相同的 OKX 配置
  ...

kafka:
  bootstrap-servers: kafka:9092  # Docker 容器名称
  ...

doris:
  fe:
    http-url: http://doris-fe:8030  # Docker 容器名称
  ...

flink:
  # 与 dev 相同的 Flink 配置
  state:
    checkpoint-dir: file:///opt/flink/checkpoints  # Docker 容器内路径
  ...
```

### 2. 代码修改

修改 `ConfigLoader.java` 的 `loadConfig()` 方法：

```java
/**
 * 加载配置文件
 * 
 * 注意：每个环境配置文件（application-{env}.yml）都是完整独立的配置，
 *      不再需要加载基础配置文件（application.yml）并合并
 */
private void loadConfig() {
    // 获取环境参数（优先级：System Property > Environment Variable > 默认值）
    String env = System.getProperty("APP_ENV");
    if (env == null || env.isEmpty()) {
        env = System.getenv("APP_ENV");
    }
    if (env == null || env.isEmpty()) {
        env = "dev";
    }
    
    log.info("Loading configuration for environment: " + env);
    
    // 直接加载环境特定配置文件（完整独立的配置）
    String envConfigFile = "application-" + env + ".yml";
    config = loadYamlFile(envConfigFile);
    
    if (config.isEmpty()) {
        log.error("Error: Environment-specific config file not found: " + envConfigFile);
        config = new HashMap<>();
    } else {
        log.info("Configuration loaded successfully from: " + envConfigFile);
    }
    
    // 解析环境变量引用（如 ${OKX_API_KEY}）
    config = resolveEnvVars(config);
    
    log.info("Total config keys: " + countKeys(config));
}
```

删除不再使用的 `mergeConfig()` 方法。

## 优势

1. **配置更直观**：每个环境配置文件都是完整的，可以独立查看和理解
2. **代码更简洁**：移除了配置合并逻辑，减少代码复杂度
3. **维护更方便**：修改配置时不需要在多个文件之间切换
4. **部署更灵活**：可以单独复制一个环境配置文件到其他环境使用

## 注意事项

1. 每个环境配置文件现在都是完整独立的，修改时需要同步更新所有环境
2. 如果需要添加新的配置项，需要在所有环境配置文件中添加
3. 环境特定的配置（如地址、路径）需要根据实际环境设置
