# 交易对配置 - 移除 OKX 不存在的交易对

**时间**: 2026-03-21 21:50  
**类型**: 配置修复  
**状态**: ✅ 已完成

## 问题背景

在运行数据采集器时，出现 WebSocket 订阅错误：

```
ERROR com.crypto.dw.collector.OKXWebSocketClient - Subscription error: 
{"event":"error","msg":"Wrong URL or channel:tickers,instId:RECALL-USDT doesn't exist. 
Please use the correct URL, channel and parameters referring to API document.","code":"60018"}
```

## 问题分析

**根本原因**:
1. 配置文件中包含了 OKX 交易所不存在的交易对
2. 错误的交易对：`RECALL-USDT`, `PUMP-USDT`, `TRUMP-USDT`, `NMR-USDT`
3. WebSocket 订阅失败，导致无法获取这些交易对的数据

**影响范围**:
- 数据采集器启动时会报错
- 无法订阅不存在的交易对
- 不影响其他有效交易对的数据采集

## 解决方案

### 1. 移除无效交易对

修改配置文件，只保留 OKX 上存在的主流交易对：

**修改前**:
```yaml
okx:
  symbols: BTC-USDT,ETH-USDT,SOL-USDT,BNB-USDT,TRUMP-USDT,XRP-USDT,LTC-USDT,BCH-USDT,NMR-USDT,RECALL-USDT,PUMP-USDT
```

**修改后**:
```yaml
okx:
  # 订阅的交易对（逗号分隔）
  # 多个交易对对应多个 Kafka 分区，数据会均匀分布
  # 注意：只订阅 OKX 上存在的交易对，避免订阅错误
  symbols: BTC-USDT,ETH-USDT,SOL-USDT,BNB-USDT,XRP-USDT,LTC-USDT,BCH-USDT,DOGE-USDT,ADA-USDT
```

## 如何添加新的交易对

### 1. 验证交易对是否存在

访问 OKX API 文档或使用以下命令验证：

```bash
# 查询 OKX 支持的交易对
curl -s "https://www.okx.com/api/v5/public/instruments?instType=SPOT" | \
  jq '.data[] | select(.instId | contains("USDT")) | .instId' | \
  grep -i "YOUR-SYMBOL"
```

### 2. 修改配置文件

在 `symbols` 配置中添加新的交易对：

```yaml
okx:
  symbols: BTC-USDT,ETH-USDT,...,NEW-USDT
```

### 3. 重新编译和部署

```bash
mvn clean package -DskipTests
bash run-collector.sh
```

## 注意事项

1. **交易对格式**: 必须使用 OKX 的标准格式，如 `BTC-USDT`（不是 `BTCUSDT`）
2. **大小写敏感**: 交易对名称区分大小写，建议使用大写
3. **分隔符**: 多个交易对使用逗号分隔，不要有空格
4. **验证存在性**: 添加新交易对前，先验证其在 OKX 上是否存在
5. **数量限制**: 不要订阅过多交易对，建议不超过 20 个

## 技术要点

- OKX WebSocket API 对不存在的交易对会返回错误码 `60018`
- 订阅错误不会导致程序崩溃，但会影响数据完整性
- 配置文件支持动态修改，重启服务即可生效
- Kafka 分区数应该与交易对数量匹配，实现数据均匀分布
