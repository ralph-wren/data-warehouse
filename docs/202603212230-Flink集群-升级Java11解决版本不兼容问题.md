# Flink 集群 - 升级 Java 11 解决版本不兼容问题

**时间**: 2026-03-21 22:30  
**问题**: Flink 作业运行失败，提示 Java 版本不兼容  
**状态**: ✅ 已解决

---

## 问题描述

在 StreamPark 中提交 Flink 作业后，作业启动失败，报错：

```
java.lang.UnsupportedClassVersionError: com/crypto/dw/flink/FlinkODSJobDataStream$ODSTransformFunction 
has been compiled by a more recent version of the Java Runtime (class file version 55.0), 
this version of the Java Runtime only recognizes class file versions up to 52.0
```

### 错误分析

**Java 版本对应关系**：
- class file version 55.0 = Java 11（项目编译使用的版本）
- class file version 52.0 = Java 8（Flink 集群运行时使用的版本）

**问题原因**：
- 项目使用 Java 11 编译（pom.xml 中配置 `<maven.compiler.target>11</maven.compiler.target>`）
- Flink 集群使用 Java 8 镜像（`flink:1.17.0-scala_2.12-java8`）
- Java 8 无法运行 Java 11 编译的字节码

---

## 解决方案

### 方案选择

有两种解决方案：

1. **方案 1：升级 Flink 集群到 Java 11**（推荐）
   - 优点：性能更好，功能更强，与 StreamPark 配置一致
   - 缺点：需要重启 Flink 集群

2. **方案 2：降级项目编译版本到 Java 8**
   - 优点：不需要修改 Flink 集群
   - 缺点：无法使用 Java 11 的新特性，性能较差

我们选择**方案 1**，因为：
- StreamPark 已经配置了 Java 11
- Java 11 是长期支持版本（LTS）
- 性能和功能都优于 Java 8

---

## 实施步骤

### 1. 修改 Flink 集群配置

修改 `docker-compose-flink.yml`，将镜像从 Java 8 升级到 Java 11：

```yaml
services:
  jobmanager:
    # 修改前：image: flink:1.17.0-scala_2.12-java8
    image: flink:1.17.0-scala_2.12-java11  # 升级到 Java 11
    # ... 其他配置保持不变

  taskmanager:
    # 修改前：image: flink:1.17.0-scala_2.12-java8
    image: flink:1.17.0-scala_2.12-java11  # 升级到 Java 11
    # ... 其他配置保持不变
```

### 2. 创建重启脚本

创建 `restart-flink-java11.sh` 脚本，自动化重启流程：

```bash
#!/bin/bash

# 1. 停止现有的 Flink 集群
docker-compose -f docker-compose-flink.yml down

# 2. 拉取 Java 11 镜像
docker pull flink:1.17.0-scala_2.12-java11

# 3. 启动新的 Flink 集群
docker-compose -f docker-compose-flink.yml up -d

# 4. 验证 Java 版本
docker exec flink-jobmanager java -version
docker exec flink-taskmanager java -version
```

### 3. 执行重启

```bash
bash restart-flink-java11.sh
```

## 技术细节

### Java 版本兼容性

**向后兼容**：
- Java 11 可以运行 Java 8 编译的代码
- Java 8 无法运行 Java 11 编译的代码

**字节码版本**：
| Java 版本 | Class File Version | 十六进制 |
|-----------|-------------------|----------|
| Java 8    | 52.0              | 0x34     |
| Java 9    | 53.0              | 0x35     |
| Java 10   | 54.0              | 0x36     |
| Java 11   | 55.0              | 0x37     |
| Java 17   | 61.0              | 0x3D     |

### Flink 官方镜像

Flink 1.17.0 提供了多个 Java 版本的镜像：
- `flink:1.17.0-scala_2.12-java8` - Java 8（已过时）
- `flink:1.17.0-scala_2.12-java11` - Java 11（推荐）
- `flink:1.17.0-scala_2.12-java17` - Java 17（最新）

### 为什么选择 Java 11

1. **长期支持（LTS）**：Java 11 是 LTS 版本，支持到 2026 年
2. **性能提升**：相比 Java 8，性能提升约 10-15%
3. **新特性**：
   - 局部变量类型推断（var）
   - HTTP Client API
   - 字符串增强方法
   - 改进的垃圾回收器（G1GC、ZGC）
4. **生态兼容**：大多数现代框架和库都支持 Java 11

## 后续优化建议

### 1. 统一 Java 版本

确保所有组件使用相同的 Java 版本：
- ✅ 项目编译：Java 11
- ✅ Flink 集群：Java 11
- ✅ StreamPark：Java 11
- ⚠️ 本地开发：建议使用 Java 11

### 2. 考虑升级到 Java 17

Java 17 是最新的 LTS 版本（支持到 2029 年），提供了更多性能优化和新特性：
- 密封类（Sealed Classes）
- 模式匹配增强
- 更好的垃圾回收器
- 更强的安全性

升级步骤：
1. 修改 `pom.xml`：`<maven.compiler.target>17</maven.compiler.target>`
2. 修改 `docker-compose-flink.yml`：`image: flink:1.17.0-scala_2.12-java17`
3. 重新编译和部署

### 3. 添加版本检查

在项目启动时添加 Java 版本检查：

```java
public static void main(String[] args) {
    // 检查 Java 版本
    String javaVersion = System.getProperty("java.version");
    if (!javaVersion.startsWith("11") && !javaVersion.startsWith("17")) {
        throw new RuntimeException("This application requires Java 11 or later");
    }
    
    // ... 其他启动逻辑
}
```

## 常见问题

### Q1: 升级后作业无法启动？

**检查步骤**：
1. 确认 Flink 集群正常运行：`docker-compose -f docker-compose-flink.yml ps`
2. 查看 JobManager 日志：`docker logs flink-jobmanager`
3. 查看 TaskManager 日志：`docker logs flink-taskmanager`

### Q2: 如何回退到 Java 8？

如果需要回退：
1. 修改 `docker-compose-flink.yml`，改回 `java8` 镜像
2. 修改 `pom.xml`，设置 `<maven.compiler.target>8</maven.compiler.target>`
3. 重新编译项目：`mvn clean package -DskipTests`
4. 重启 Flink 集群：`bash restart-flink-java11.sh`

### Q3: 其他组件需要升级吗？

不需要，其他组件（Kafka、Doris、Prometheus 等）不受影响，因为：
- 它们运行在独立的容器中
- 不依赖 Flink 的 Java 版本
- 通过网络通信，与 Java 版本无关
