## 问题解决历程

### 问题 1: 配置文件加载失败 ✅

**错误**: `Config file not found: application-docker.yml`

**原因**: Flink 的 `ChildFirstClassLoader` 无法通过标准方式加载 JAR 包内的配置文件

**解决方案**: 
1. 使用 Java NIO 的 `FileSystem` API 直接打开 JAR 文件
2. 在关闭文件系统之前，将配置文件内容读取到内存中（`ByteArrayInputStream`）
3. 实现三种加载方式的降级策略

**关键代码**:
```java
// 读取文件内容到内存
byte[] fileContent = Files.readAllBytes(configPath);
// 返回基于内存的流
return new ByteArrayInputStream(fileContent);
```

**文档**: `docs/202603212215-配置加载-使用JAR文件系统解决ChildFirstClassLoader问题.md`

### 问题 2: Java 版本不兼容 ✅

**错误**: `UnsupportedClassVersionError: class file version 55.0, this version only recognizes up to 52.0`

**原因**: 
- 项目使用 Java 11 编译（版本 55.0）
- Flink 集群使用 Java 8 运行（只支持版本 52.0）

**解决方案**: 
1. 修改 `docker-compose-flink.yml`，升级镜像到 Java 11
2. 重启 Flink 集群

**关键修改**:
```yaml
services:
  jobmanager:
    image: flink:1.17.0-scala_2.12-java11  # 从 java8 升级到 java11
  taskmanager:
    image: flink:1.17.0-scala_2.12-java11  # 从 java8 升级到 java11
```

**文档**: `docs/202603212230-Flink集群-升级Java11解决版本不兼容问题.md`

## 技术总结

### 关键技术点

1. **JAR 文件系统 API**
   - 使用 `FileSystems.newFileSystem()` 打开 JAR 文件
   - 使用 `Files.readAllBytes()` 读取内容到内存
   - 返回 `ByteArrayInputStream` 避免文件系统关闭问题

2. **ChildFirstClassLoader**
   - Flink 的特殊类加载器
   - 优先加载子类路径的类和资源
   - 需要特殊处理 JAR 包内资源

3. **Java 字节码版本**
   - Java 8 = 52.0
   - Java 11 = 55.0
   - 向后兼容但不向前兼容

4. **Docker 网络**
   - 使用容器名称而不是 localhost
   - 多个网络连接（default, doris-net, flink-net）
   - 确保所有服务在同一网络中

### 最佳实践

1. **配置管理**
   - 使用独立的环境配置文件
   - 通过 Program Args 传递环境参数
   - 添加详细的调试日志

2. **版本统一**
   - 所有组件使用相同的 Java 版本
   - 使用 LTS 版本（Java 11 或 17）
   - 定期更新依赖版本

3. **故障排查**
   - 添加详细的日志输出
   - 使用多种加载方式降级
   - 提供清晰的错误信息
