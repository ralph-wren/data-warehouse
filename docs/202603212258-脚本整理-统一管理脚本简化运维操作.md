# 脚本整理 - 统一管理脚本简化运维操作

## 问题描述

项目中存在大量分散的启动、停止、重启脚本,管理混乱:
- `restart-doris.sh`
- `restart-flink-java11.sh`
- `restart-kafka-with-jmx.sh`
- `restart-kafka-with-network.sh`
- `start-doris-for-flink.sh`
- `start-flink-docker.sh`
- `start-monitoring.sh`
- `start-streampark.sh`
- `stop-flink-docker.sh`
- `stop-monitoring.sh`
- `stop-streampark.sh`

每个组件有多个脚本,不便于维护和使用。

## 解决方案

### 1. 统一管理脚本设计

为每个组件创建一个统一的管理脚本,支持三种操作:
- `start` - 启动服务
- `stop` - 停止服务
- `restart` - 重启服务(默认操作)

### 2. 创建的新脚本

#### 2.1 manage-kafka.sh
```bash
#!/bin/bash
# Kafka 管理脚本
# 功能: 启动/停止/重启 Kafka 服务,并自动创建 Topic
```

特点:
- 集成了 Topic 创建功能
- 包含 JMX 监控配置
- 自动连接到正确的 Docker 网络

#### 2.2 manage-flink.sh
```bash
#!/bin/bash
# Flink 集群管理脚本
# 功能: 启动/停止/重启 Flink 集群
```

特点:
- 使用 Java 11 镜像
- 验证 Java 版本
- 显示 WebUI 访问地址

#### 2.3 manage-doris.sh
```bash
#!/bin/bash
# Doris 管理脚本
# 功能: 启动/停止/重启 Doris 服务
```

特点:
- 桥接网络模式配置
- 等待服务就绪
- 显示连接信息

#### 2.4 manage-monitoring.sh
```bash
#!/bin/bash
# 监控系统管理脚本
# 功能: 启动/停止/重启 Prometheus + Grafana
```

特点:
- 同时管理 Prometheus 和 Grafana
- 显示访问地址和默认密码

#### 2.5 manage-streampark.sh
```bash
#!/bin/bash
# StreamPark 管理脚本
# 功能: 启动/停止/重启 StreamPark 服务
```

特点:
- 使用自定义 Java 11 镜像
- 显示访问地址和默认账号

### 3. 更新 start-all.sh

修改启动脚本使用新的管理脚本:

```bash
# 旧方式
docker-compose up -d
bash kafka-setup.sh

# 新方式
bash manage-kafka.sh start
bash manage-doris.sh start
bash manage-flink.sh start
bash manage-monitoring.sh start
```

### 4. 创建 stop-all.sh

新增统一的停止脚本:

```bash
#!/bin/bash
# 按照依赖关系倒序停止所有服务
bash manage-monitoring.sh stop
bash manage-flink.sh stop
bash manage-doris.sh stop
bash manage-kafka.sh stop
bash manage-streampark.sh stop
```

## 使用方法

### 单个组件管理

```bash
# 启动 Kafka
bash manage-kafka.sh start

# 停止 Flink
bash manage-flink.sh stop

# 重启 Doris (默认操作)
bash manage-doris.sh
# 或
bash manage-doris.sh restart
```

### 所有组件管理

```bash
# 启动所有服务
bash start-all.sh

# 停止所有服务
bash stop-all.sh
```

## 脚本特点

### 1. 统一接口
所有管理脚本使用相同的参数格式:
```bash
./manage-<component>.sh [start|stop|restart]
```

### 2. 智能默认值
不传参数时默认执行 `restart`,最常用的操作

### 3. 详细输出
每个操作都有清晰的提示信息:
- 操作开始提示
- 等待时间说明
- 服务状态确认
- 访问地址显示

### 4. 错误处理
- 检查 Docker 是否运行
- 验证服务启动状态
- 显示错误信息

## 脚本对比

### 旧方式 (多个脚本)
```
restart-kafka-with-network.sh  # 重启 Kafka
kafka-setup.sh                 # 创建 Topic
start-flink-docker.sh          # 启动 Flink
stop-flink-docker.sh           # 停止 Flink
restart-flink-java11.sh        # 重启 Flink (Java 11)
```

### 新方式 (单个脚本)
```
manage-kafka.sh [start|stop|restart]  # 包含 Topic 创建
manage-flink.sh [start|stop|restart]  # 自动使用 Java 11
```

## 优势

1. **简化管理**: 每个组件只需记住一个脚本名
2. **减少文件**: 从 11+ 个脚本减少到 5 个管理脚本
3. **统一接口**: 所有脚本使用相同的参数格式
4. **易于维护**: 修改只需更新一个文件
5. **功能集成**: 相关功能集成在一起(如 Kafka + Topic 创建)
