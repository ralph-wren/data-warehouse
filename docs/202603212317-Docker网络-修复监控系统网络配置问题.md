# Docker 网络 - 修复监控系统网络配置问题

## 问题描述

启动监控系统时报错：

```
a network with name data-warehouse_monitoring-net exists but was not created by compose.
Set `external: true` to use an existing network
network data-warehouse_monitoring-net was found but has incorrect label 
com.docker.compose.network set to "" (expected: "monitoring-net")
```

## 问题现象

监控系统无法启动，提示网络配置错误。

## 根本原因

### Docker Compose 网络管理机制

Docker Compose 对网络有两种管理方式：

1. **内部网络** (`external: false` 或不设置):
   - 由当前 compose 文件创建和管理
   - 网络名称会自动添加项目前缀
   - 网络标签 `com.docker.compose.network` 设置为网络名称

2. **外部网络** (`external: true`):
   - 引用已存在的网络
   - 不由当前 compose 文件管理
   - 不检查网络标签

### 本项目的情况

- `docker-compose-kafka.yml` 创建了 `data-warehouse_monitoring-net` 网络
- `docker-compose-monitoring.yml` 尝试创建同名网络
- 两个 compose 文件都想管理同一个网络，导致冲突

**错误配置** (`docker-compose-monitoring.yml`):
```yaml
networks:
  monitoring-net:
    driver: bridge  # ❌ 尝试创建新网络
```

**正确配置**:
```yaml
networks:
  monitoring-net:
    name: data-warehouse_monitoring-net
    external: true  # ✅ 引用已存在的网络
```

## 解决方案

### 修改 docker-compose-monitoring.yml

```yaml
networks:
  monitoring-net:
    name: data-warehouse_monitoring-net  # 指定完整的网络名称
    external: true  # 标记为外部网络
```

**关键点**:
- `name`: 指定实际的网络名称（包含项目前缀）
- `external: true`: 告诉 Docker Compose 这是外部网络，不要尝试创建

### 重新启动监控系统

```bash
bash manage-monitoring.sh start
```

## 技术要点

### 1. Docker Compose 网络命名规则

```
<project_name>_<network_name>
```

例如：
- 项目名: `data-warehouse`
- 网络名: `monitoring-net`
- 实际名称: `data-warehouse_monitoring-net`

### 2. 查看网络信息

```bash
# 列出所有网络
docker network ls

# 查看网络详情
docker network inspect data-warehouse_monitoring-net

# 查看容器的网络连接
docker inspect <container_name> --format '{{range $key, $value := .NetworkSettings.Networks}}{{$key}} {{end}}'
```

### 3. 网络配置最佳实践

**单一职责原则**:
- 每个网络由一个 compose 文件创建和管理
- 其他 compose 文件使用 `external: true` 引用

**示例**:

创建网络的文件 (`docker-compose-kafka.yml`):
```yaml
networks:
  default:
    name: data-warehouse_default
  monitoring-net:
    name: data-warehouse_monitoring-net
```

引用网络的文件 (`docker-compose-monitoring.yml`):
```yaml
networks:
  monitoring-net:
    name: data-warehouse_monitoring-net
    external: true  # 引用，不创建
```

### 通用解决方案

当遇到网络相关错误时：

1. **检查网络是否存在**:
   ```bash
   docker network ls | grep <network_name>
   ```

2. **检查容器的网络连接**:
   ```bash
   docker inspect <container> --format '{{range $key, $value := .NetworkSettings.Networks}}{{$key}} {{end}}'
   ```

3. **手动连接容器到网络**:
   ```bash
   docker network connect <network_name> <container_name>
   ```

4. **修改 compose 文件使用 external**:
   ```yaml
   networks:
     <network_name>:
       external: true
   ```
