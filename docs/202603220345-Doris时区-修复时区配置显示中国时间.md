# Doris 时区配置 - 修复时区显示中国时间

## 问题描述

Superset 查询 Doris 数据时,时间显示为 UTC 时间(如 17:14),而不是中国时间(应该是 01:14,东八区 +8 小时)。

## 问题原因

1. **Doris 默认时区**: Doris 数据库默认时区为 `Etc/UTC`
2. **字段名错误**: SQL 查询中使用了错误的字段名 `ts`,实际字段名为 `timestamp`
3. **字段名错误**: SQL 查询中使用了错误的字段名 `vol_24h`,实际字段名为 `volume_24h`
4. **SQL 语法不兼容**: Doris 不支持 `(inst_id, timestamp) IN (SELECT ...)` 多列 IN 语法

## 解决方案

### 1. 设置 Doris 全局时区

```bash
# 连接到 Doris FE 容器
docker exec doris-fe mysql -h127.0.0.1 -P9030 -uroot

# 设置全局时区为中国时区
SET GLOBAL time_zone = 'Asia/Shanghai';

# 验证时区设置
SHOW VARIABLES LIKE 'time_zone';
SELECT NOW() AS '当前时间';
```

### 2. 修改 Docker Compose 配置

在 `docker-compose-doris.yml` 中添加环境变量,确保容器重启后时区配置仍然生效:

```yaml
services:
  doris-fe:
    environment:
      - TZ=Asia/Shanghai  # 设置时区为中国时区
  
  doris-be:
    environment:
      - TZ=Asia/Shanghai  # 设置时区为中国时区
```

### 3. 修复 SQL 查询

#### 问题 1: 字段名错误

将所有 SQL 查询中的字段名修正:
- `ts` → `timestamp`
- `vol_24h` → `volume_24h`

#### 问题 2: SQL 语法不兼容

Doris 不支持多列 IN 语法,需要改用窗口函数:

**错误写法**:
```sql
SELECT * FROM table
WHERE (inst_id, timestamp) IN (
    SELECT inst_id, MAX(timestamp)
    FROM table
    GROUP BY inst_id
)
```

**正确写法**:
```sql
SELECT * FROM (
    SELECT 
        *,
        ROW_NUMBER() OVER (PARTITION BY inst_id ORDER BY timestamp DESC) AS rn
    FROM table
) t
WHERE rn = 1
```

## 验证结果

### 1. 验证时区设置

```bash
docker exec doris-fe mysql -h127.0.0.1 -P9030 -uroot \
  -e "SHOW VARIABLES LIKE 'time_zone'; SELECT NOW();"
```

输出:
```
Variable_name   Value           Default_Value   Changed
time_zone       Asia/Shanghai   Etc/UTC         1

当前时间: 2026-03-22 03:41:47
```

### 2. 验证查询结果

```sql
SELECT 
    inst_id AS 币种,
    last_price AS 最新价格,
    FROM_UNIXTIME(timestamp / 1000) AS 更新时间
FROM (
    SELECT 
        inst_id,
        last_price,
        timestamp,
        ROW_NUMBER() OVER (PARTITION BY inst_id ORDER BY timestamp DESC) AS rn
    FROM crypto_dw.dwd_crypto_ticker_detail
) t
WHERE rn = 1
ORDER BY last_price DESC
LIMIT 5;
```

输出:
```
币种          最新价格              更新时间
BTC-USDT     70415.20000000       2026-03-22 00:30:04
ETH-USDT     2148.22000000        2026-03-22 00:30:04
BNB-USDT     641.60000000         2026-03-22 00:30:04
```

时间显示正确,为东八区时间!

## 修改的文件

1. **docker-compose-doris.yml**: 添加 `TZ=Asia/Shanghai` 环境变量
2. **sql/superset_queries.sql**: 修复所有 SQL 查询
   - 修正字段名: `ts` → `timestamp`, `vol_24h` → `volume_24h`
   - 修复语法: 使用窗口函数替代多列 IN 语法

## 注意事项

1. **时区设置生效**: 
   - 已执行的 SQL 设置会立即生效
   - Docker Compose 配置需要重启容器才能生效
   - 建议在 Superset 中刷新页面或重新执行查询

2. **字段名映射**:
   - ODS 层表使用 `ts` 字段
   - DWD 层表使用 `timestamp` 字段
   - 查询时需要根据表层级使用正确的字段名

3. **SQL 兼容性**:
   - Doris 不支持多列 IN 语法
   - 建议使用窗口函数 `ROW_NUMBER()` 获取最新记录
   - 窗口函数性能更好,推荐使用
