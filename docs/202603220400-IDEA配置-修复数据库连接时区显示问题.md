# IDEA 数据库连接 - 修复时区显示问题

## 问题描述

在 IntelliJ IDEA 的 Database 工具中查询 Doris 数据时,时间显示不正确:
- 最早时间显示: `2026-03-18 13:27:02` (错误,应该是 `2026-03-18 21:27:02`)
- 最新时间显示: `2026-03-21 16:30:04` (错误,应该是 `2026-03-22 00:30:04`)

时间相差 8 小时,说明 IDEA 使用了 UTC 时区而不是东八区。

## 问题原因

**JDBC 连接 URL 缺少时区参数**:
- IDEA 的数据库连接默认使用客户端时区(可能是 UTC)
- MySQL/Doris JDBC 驱动需要显式指定时区参数
- 没有配置 `serverTimezone` 参数,导致时间转换错误

## 解决方案

### 方法 1: 修改 IDEA 数据库连接 URL (推荐) ✅

在 IDEA 的 Database 工具中修改连接 URL,添加时区参数:

**原始 URL**:
```
jdbc:mysql://127.0.0.1:9030/crypto_dw
```

**修改后的 URL**:
```
jdbc:mysql://127.0.0.1:9030/crypto_dw?serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=utf8
```

**参数说明**:
- `serverTimezone=Asia/Shanghai`: 指定服务器时区为东八区
- `useUnicode=true`: 启用 Unicode 支持
- `characterEncoding=utf8`: 使用 UTF-8 字符编码

### 方法 2: 在 IDEA 中配置高级选项

如果不想修改 URL,也可以在 IDEA 的数据源配置中添加高级选项:

1. 打开 Database 工具窗口
2. 右键点击数据源 → "Properties"
3. 切换到 "Advanced" 标签页
4. 添加以下属性:
   - `serverTimezone` = `Asia/Shanghai`
   - `useUnicode` = `true`
   - `characterEncoding` = `utf8`

## 详细配置步骤

### 步骤 1: 打开数据源配置

1. 在 IDEA 中打开 Database 工具窗口 (View → Tool Windows → Database)
2. 找到 Doris 数据源连接
3. 右键点击 → "Properties" 或 "Data Source Properties"

### 步骤 2: 修改连接 URL

在 "General" 标签页中:

**原始配置**:
```
Host: 127.0.0.1
Port: 9030
Database: crypto_dw
URL: jdbc:mysql://127.0.0.1:9030/crypto_dw
```

**修改为**:
```
Host: 127.0.0.1
Port: 9030
Database: crypto_dw
URL: jdbc:mysql://127.0.0.1:9030/crypto_dw?serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=utf8
```

### 步骤 3: 测试连接

1. 点击 "Test Connection" 按钮
2. 确认连接成功
3. 点击 "OK" 保存配置

### 步骤 4: 验证时间显示

重新执行查询:
```sql
SELECT 
    '数据验证' AS 类型,
    COUNT(*) AS 总数,
    COUNT(DISTINCT inst_id) AS 币种数,
    MIN(FROM_UNIXTIME(timestamp / 1000)) AS 最早时间,
    MAX(FROM_UNIXTIME(timestamp / 1000)) AS 最新时间
FROM crypto_dw.dwd_crypto_ticker_detail;
```

**预期结果**:
```
类型      总数     币种数   最早时间              最新时间
数据验证  136934   9       2026-03-18 21:27:02   2026-03-22 00:30:04
```

时间应该显示为东八区时间(北京时间)!

## 常见时区参数

| 时区 | 参数值 | 说明 |
|-----|--------|------|
| 中国(东八区) | `Asia/Shanghai` | 北京时间 |
| 日本(东九区) | `Asia/Tokyo` | 东京时间 |
| 美国东部 | `America/New_York` | 纽约时间 |
| 美国西部 | `America/Los_Angeles` | 洛杉矶时间 |
| UTC | `UTC` | 协调世界时 |

## 技术要点

1. **JDBC 时区处理**:
   - MySQL JDBC 驱动会将数据库时间转换为客户端时区
   - 如果不指定 `serverTimezone`,驱动会使用客户端默认时区
   - 显式指定 `serverTimezone` 可以确保时间显示一致

2. **Doris 时区配置**:
   - Doris 服务器时区: `Asia/Shanghai` (已配置)
   - JDBC 连接时区: `Asia/Shanghai` (需要配置)
   - 两者必须一致,才能正确显示时间

3. **时间函数**:
   - `FROM_UNIXTIME()` 会根据 Doris 服务器时区转换时间戳
   - JDBC 驱动会根据 `serverTimezone` 参数显示时间
   - 如果两者不一致,会导致时间显示错误

## 验证配置

### 1. 验证 Doris 服务器时区

```sql
SHOW VARIABLES LIKE 'time_zone';
-- 应该显示: Asia/Shanghai
```

### 2. 验证 JDBC 连接时区

在 IDEA 中执行:
```sql
SELECT NOW() AS '当前时间', 
       @@session.time_zone AS '会话时区',
       @@global.time_zone AS '全局时区';
```

**预期结果**:
```
当前时间: 2026-03-22 03:45:00
会话时区: Asia/Shanghai
全局时区: Asia/Shanghai
```

### 3. 验证时间显示

```sql
SELECT 
    inst_id AS 币种,
    last_price AS 最新价格,
    FROM_UNIXTIME(timestamp / 1000) AS 更新时间
FROM (
    SELECT 
        inst_id,
        last_price,
        timestamp,
        ROW_NUMBER() OVER (PARTITION BY inst_id ORDER BY timestamp DESC) AS rn
    FROM crypto_dw.dwd_crypto_ticker_detail
) t
WHERE rn = 1
ORDER BY last_price DESC
LIMIT 3;
```

**预期结果**:
```
币种          最新价格              更新时间
BTC-USDT     70415.20000000       2026-03-22 00:30:04
ETH-USDT     2148.22000000        2026-03-22 00:30:04
BNB-USDT     641.60000000         2026-03-22 00:30:04
```

时间应该显示为东八区时间!

## 注意事项

1. **修改 URL 后需要重新连接**: 
   - 修改连接 URL 后,需要断开并重新连接数据库
   - 或者重启 IDEA

2. **不同客户端配置方式不同**:
   - IDEA: 修改 URL 或添加高级属性
   - Workbench: 修改连接字符串
   - DBeaver: 添加驱动属性
   - Navicat: 在高级选项中添加

3. **时区参数必须与服务器一致**:
   - Doris 服务器时区: `Asia/Shanghai`
   - JDBC 连接时区: `Asia/Shanghai`
   - 两者必须一致

4. **字符编码也很重要**:
   - 建议同时配置 `useUnicode=true` 和 `characterEncoding=utf8`
   - 避免中文乱码问题

通过在 JDBC 连接 URL 中添加 `serverTimezone=Asia/Shanghai` 参数,可以确保 IDEA 和其他 MySQL 客户端正确显示东八区时间。这个配置与 Doris 服务器的时区设置配合使用,确保整个系统的时间显示一致。
