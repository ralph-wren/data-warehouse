# Grafana 配置 - 添加 Doris 数据源和加密货币监控面板

## 需求描述

在 Grafana 中配置 Doris 数据源,创建一个现代化的加密货币实时数据监控面板,展示各种数据指标和可视化图表。

## 实施方案

### 1. 添加 Doris MySQL 数据源

创建 `monitoring/grafana/provisioning/datasources/doris-mysql.yml`:

```yaml
# Grafana MySQL 数据源配置 - 连接 Doris
apiVersion: 1

datasources:
  # Doris 数据源 (通过 MySQL 协议连接)
  - name: Doris
    type: mysql
    access: proxy
    url: doris-fe:9030
    database: crypto_dw
    user: root
    secureJsonData:
      password: ''
    jsonData:
      maxOpenConns: 10
      maxIdleConns: 2
      connMaxLifetime: 14400
      timezone: 'Asia/Shanghai'  # 时区设置为中国时区
    editable: true
```

### 2. 修复网络配置

**问题**: Grafana 容器无法连接到 Doris,因为它们不在同一个 Docker 网络中。

**解决方案**: 在 `docker-compose-monitoring.yml` 中添加 `doris-net` 网络:

```yaml
services:
  grafana:
    networks:
      - monitoring-net
      - doris-net  # 添加 Doris 网络

networks:
  monitoring-net:
    name: data-warehouse_monitoring-net
    external: true
  doris-net:
    name: data-warehouse_doris-net
    external: true
```

### 3. 创建加密货币监控面板

创建 `monitoring/grafana/dashboards/crypto-data-dashboard.json`,包含以下面板:

#### 面板 1: 总数据量 (Stat)
- 显示 DWD 表中的总记录数
- 使用面积图模式
- 绿色主题

```sql
SELECT COUNT(*) as '总数据量' 
FROM crypto_dw.dwd_crypto_ticker_detail
```

#### 面板 2: 监控币种数量 (Stat)
- 显示监控的币种数量
- 蓝色主题

```sql
SELECT COUNT(DISTINCT inst_id) as '币种数量' 
FROM crypto_dw.dwd_crypto_ticker_detail
```

#### 面板 3-4: BTC/ETH 最新价格 (Stat)
- 显示 BTC 和 ETH 的最新价格
- 使用货币格式 (USD)
- 使用窗口函数获取最新价格

```sql
SELECT last_price as 'BTC 最新价格'
FROM (
  SELECT 
    inst_id,
    last_price,
    ROW_NUMBER() OVER (PARTITION BY inst_id ORDER BY timestamp DESC) AS rn
  FROM crypto_dw.dwd_crypto_ticker_detail
  WHERE inst_id = 'BTC-USDT'
) t
WHERE rn = 1
```

#### 面板 5: 各币种最新价格排行 (Table)
- 显示所有币种的最新价格、成交量和更新时间
- 按价格降序排列
- 支持排序和筛选

```sql
SELECT 
  inst_id AS '币种',
  last_price AS '最新价格',
  volume_24h AS '24小时成交量',
  UNIX_TIMESTAMP(FROM_UNIXTIME(timestamp / 1000)) * 1000 AS '更新时间'
FROM (
  SELECT 
    inst_id,
    last_price,
    volume_24h,
    timestamp,
    ROW_NUMBER() OVER (PARTITION BY inst_id ORDER BY timestamp DESC) AS rn
  FROM crypto_dw.dwd_crypto_ticker_detail
) t
WHERE rn = 1
ORDER BY last_price DESC
```

#### 面板 6: 主流币种价格趋势 (Time Series)
- 显示 BTC、ETH、BNB 的价格趋势
- 最近 1 小时数据
- 多条折线图
- 显示最大值、最小值、最新值

```sql
SELECT 
  UNIX_TIMESTAMP(FROM_UNIXTIME(timestamp / 1000)) * 1000 AS time,
  inst_id AS metric,
  last_price AS value
FROM crypto_dw.dwd_crypto_ticker_detail
WHERE 
  timestamp >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 1 HOUR)) * 1000
  AND inst_id IN ('BTC-USDT', 'ETH-USDT', 'BNB-USDT')
ORDER BY timestamp
```

#### 面板 7: 24小时成交量分布 (Pie Chart)
- 显示各币种的成交量占比
- 饼图展示
- 显示百分比和数值

```sql
SELECT 
  inst_id AS '币种',
  volume_24h AS '24小时成交量'
FROM (
  SELECT 
    inst_id,
    volume_24h,
    ROW_NUMBER() OVER (PARTITION BY inst_id ORDER BY timestamp DESC) AS rn
  FROM crypto_dw.dwd_crypto_ticker_detail
) t
WHERE rn = 1
ORDER BY volume_24h DESC
```

#### 面板 8: 数据更新频率 (Time Series - Bars)
- 显示每分钟的数据点数
- 柱状图展示
- 最近 1 小时数据

```sql
SELECT 
  UNIX_TIMESTAMP(DATE_FORMAT(FROM_UNIXTIME(timestamp / 1000), '%Y-%m-%d %H:%i:00')) * 1000 AS time,
  inst_id AS metric,
  COUNT(*) AS value
FROM crypto_dw.dwd_crypto_ticker_detail
WHERE timestamp >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 1 HOUR)) * 1000
GROUP BY 
  DATE_FORMAT(FROM_UNIXTIME(timestamp / 1000), '%Y-%m-%d %H:%i:00'),
  inst_id
ORDER BY time
```

### 4. 面板配置

**全局配置**:

- 时区: Asia/Shanghai (东八区)
- 时间范围: 最近 1 小时
- 主题: Dark (深色主题)

## 技术要点

### 1. Doris 连接配置

- **协议**: MySQL 协议 (端口 9030)
- **驱动**: Grafana 内置 MySQL 驱动
- **时区**: Asia/Shanghai (东八区)
- **连接池**: 最大 10 个连接,空闲 2 个

### 2. SQL 查询优化

- **窗口函数**: 使用 `ROW_NUMBER()` 获取最新记录,性能优于子查询
- **时间转换**: 使用 `FROM_UNIXTIME(timestamp / 1000)` 转换毫秒时间戳
- **时间格式**: 使用 `UNIX_TIMESTAMP() * 1000` 转换为 Grafana 需要的毫秒时间戳

### 3. 面板类型选择

- **Stat**: 适合显示单个数值指标
- **Table**: 适合显示多行数据,支持排序和筛选
- **Time Series**: 适合显示时序数据,支持多条曲线
- **Pie Chart**: 适合显示占比分布

### 4. 数据刷新策略

- **自动刷新**: 10 秒间隔,适合实时监控
- **查询缓存**: Grafana 会缓存查询结果,减少数据库压力
- **时间范围**: 默认最近 1 小时,可调整

## 常见问题

### 1. 面板显示 "No Data"

**原因**: 
- Doris 数据源未配置
- 网络连接失败
- 数据库中没有数据

**解决方法**:
```bash
# 检查数据源配置
docker exec grafana cat /etc/grafana/provisioning/datasources/doris-mysql.yml

# 测试网络连接
docker exec grafana ping -c 3 doris-fe

# 检查数据
docker exec doris-fe mysql -h127.0.0.1 -P9030 -uroot -e "
  SELECT COUNT(*) FROM crypto_dw.dwd_crypto_ticker_detail"
```

### 2. 时间显示不正确

**原因**: 时区配置错误

**解决方法**:
- 确认 Doris 时区: `SHOW VARIABLES LIKE 'time_zone'`
- 确认 Grafana 数据源时区: `timezone: 'Asia/Shanghai'`
- 确认面板时区: `timezone: "Asia/Shanghai"`

### 3. 查询速度慢

**原因**: 
- 数据量大
- 没有索引
- 查询时间范围过大

**优化方法**:
- 添加索引: `ALTER TABLE ... ADD INDEX ...`
- 减少时间范围: 从 24 小时改为 1 小时
- 使用物化视图: 预计算聚合结果
