# Doris 表结构 - 三张表作用和数据流转说明

## 背景

用户询问 Doris 中三张表的作用,需要详细说明每张表的功能、数据流转关系和使用场景。

## 三张表概述

本项目采用经典的数据仓库三层架构:

```
OKX 交易所 → Kafka → ODS 层 → DWD 层 → DWS 层 → BI 工具
                      ↓         ↓         ↓
                   原始数据   明细数据   汇总数据
```

---

## 1. ODS 层 - ods_crypto_ticker_rt (原始数据表)

### 作用
存储从 OKX 交易所实时采集的**原始行情数据**,不做任何加工处理。

### 数据来源
- OKX WebSocket API 实时推送
- 通过 Kafka 消息队列缓冲
- Flink ODS 作业直接写入

### 核心字段 (13个)
```
inst_id          -- 交易对 (如 BTC-USDT)
timestamp        -- 业务时间戳
last_price       -- 最新成交价
bid_price        -- 买一价
ask_price        -- 卖一价
bid_size         -- 买一量
ask_size         -- 卖一量
volume_24h       -- 24小时成交量
high_24h         -- 24小时最高价
low_24h          -- 24小时最低价
open_24h         -- 24小时开盘价
data_source      -- 数据源 (OKX)
ingest_time      -- 入库时间
```

### 数据特点
- ✅ 完整保留原始数据
- ✅ 不做任何转换和计算
- ✅ 适合数据回溯和问题排查
- ⚠️ 数据量大,不适合直接分析

### 当前数据量
```
1,856,170 条 (约 185 万条)
```

---

## 2. DWD 层 - dwd_crypto_ticker_detail (明细数据表)

### 作用
存储经过**清洗、转换、计算**后的明细数据,添加了业务维度和指标。

### 数据来源
- 从 ODS 层读取原始数据
- Flink DWD 作业进行数据清洗和转换
- 添加计算字段和业务维度

### 核心字段 (19个)

**基础字段** (继承自 ODS):
```
inst_id, timestamp, last_price, bid_price, ask_price
volume_24h, high_24h, low_24h, open_24h
```

**新增维度字段**:
```
trade_date       -- 交易日期 (用于分区)
trade_hour       -- 交易小时 (用于分析)
```

**新增计算指标**:
```
spread                  -- 买卖价差 = ask_price - bid_price
spread_rate             -- 价差率 = spread / bid_price
price_change_24h        -- 24小时涨跌额 = last_price - open_24h
price_change_rate_24h   -- 24小时涨跌幅 = price_change_24h / open_24h
amplitude_24h           -- 24小时振幅 = (high_24h - low_24h) / open_24h
```

**元数据字段**:
```
data_source      -- 数据源
ingest_time      -- 入库时间
process_time     -- 处理时间
```

### 数据转换示例
```sql
-- ODS → DWD 转换逻辑
SELECT 
    inst_id,
    timestamp,
    -- 新增时间维度
    DATE(FROM_UNIXTIME(timestamp / 1000)) as trade_date,
    HOUR(FROM_UNIXTIME(timestamp / 1000)) as trade_hour,
    
    -- 原始字段
    last_price, bid_price, ask_price, volume_24h,
    
    -- 计算买卖价差
    (ask_price - bid_price) as spread,
    (ask_price - bid_price) / bid_price as spread_rate,
    
    -- 计算涨跌指标
    (last_price - open_24h) as price_change_24h,
    (last_price - open_24h) / open_24h as price_change_rate_24h,
    (high_24h - low_24h) / open_24h as amplitude_24h
FROM ods_crypto_ticker_rt
```

### 数据特点
- ✅ 数据经过清洗和转换
- ✅ 添加了业务维度和计算指标
- ✅ 按日期分区,查询性能好
- ✅ 适合业务分析和报表查询
- ⚠️ 只保留最近 7 天数据

### 分区策略
```sql
-- 动态分区配置
保留最近 7 天历史数据
预创建未来 3 天分区
自动删除过期分区
```

### 当前数据量
```
136,934 条 (约 13.7 万条)
```

## 3. DWS 层 - dws_crypto_ticker_1min (1分钟K线汇总表)

### 作用
存储按**1分钟窗口聚合**的 K 线数据,用于趋势分析和图表展示。

### 数据来源
- 从 DWD 层读取明细数据
- Flink 窗口聚合计算
- 按 1 分钟滚动窗口聚合

### 核心字段 (14个)

**维度字段**:
```
inst_id          -- 交易对
window_start     -- 窗口开始时间
window_end       -- 窗口结束时间
window_size      -- 窗口大小 (1分钟)
```

**K线四价 (OHLC)**:
```
open_price       -- 开盘价 (窗口内第一个价格)
high_price       -- 最高价 (窗口内最大价格)
low_price        -- 最低价 (窗口内最小价格)
close_price      -- 收盘价 (窗口内最后一个价格)
```

**统计指标**:
```
volume           -- 成交量 (窗口内累计)
avg_price        -- 平均价 (窗口内均价)
price_change     -- 涨跌额 = close_price - open_price
price_change_rate -- 涨跌幅 = price_change / open_price
tick_count       -- Tick 数量 (窗口内数据条数)
process_time     -- 处理时间
```

### 数据聚合示例
```sql
-- DWD → DWS 聚合逻辑
SELECT 
    inst_id,
    TUMBLE_START(ts, INTERVAL '1' MINUTE) as window_start,
    TUMBLE_END(ts, INTERVAL '1' MINUTE) as window_end,
    1 as window_size,
    
    -- K线四价
    FIRST_VALUE(last_price) as open_price,   -- 第一个价格
    MAX(last_price) as high_price,           -- 最高价
    MIN(last_price) as low_price,            -- 最低价
    LAST_VALUE(last_price) as close_price,   -- 最后一个价格
    
    -- 统计指标
    SUM(volume_24h) as volume,
    AVG(last_price) as avg_price,
    COUNT(*) as tick_count
FROM dwd_crypto_ticker_detail
GROUP BY inst_id, TUMBLE(ts, INTERVAL '1' MINUTE)
```

### 数据特点
- ✅ 数据量大幅减少 (每分钟一条)
- ✅ 适合绘制 K 线图
- ✅ 支持技术指标计算 (MA、MACD、RSI)
- ✅ 查询性能高,适合实时监控
- ⚠️ 不支持明细查询

### 当前数据量
```
0 条 (DWS 作业尚未启动)
```

## 数据流转关系

```
┌─────────────────────────────────────────────────────────────┐
│                      数据流转架构                            │
└─────────────────────────────────────────────────────────────┘

1. 数据采集
   OKX WebSocket → Kafka (crypto-ticker)
   
2. ODS 层 (原始数据)
   Flink ODS Job → ods_crypto_ticker_rt
   - 数据量: 1,856,170 条
   - 写入频率: 实时 (~1000 条/秒)
   - 数据保留: 永久
   
3. DWD 层 (明细数据)
   Flink DWD Job → dwd_crypto_ticker_detail
   - 数据量: 136,934 条
   - 写入频率: 实时 (~1000 条/秒)
   - 数据保留: 7 天
   - 数据增强: 添加 6 个计算字段
   
4. DWS 层 (汇总数据)
   Flink DWS Job → dws_crypto_ticker_1min
   - 数据量: 0 条 (未启动)
   - 写入频率: 1 分钟 (~15 条/分钟)
   - 数据保留: 永久
   - 数据压缩: 压缩比 ~99.97%
```

## 查询场景对比

### ODS 层 - 适合数据回溯

**使用场景**:
- 数据质量检查
- 问题排查和调试
- 原始数据导出

**示例查询**:
```sql
-- 查询某个时间段的原始数据
SELECT * FROM ods_crypto_ticker_rt
WHERE inst_id = 'BTC-USDT'
  AND timestamp BETWEEN 1711065600000 AND 1711069200000
ORDER BY timestamp;
```

### DWD 层 - 适合业务分析 (推荐)

**使用场景**:
- 实时监控仪表板
- 业务报表和分析
- 数据可视化

**示例查询**:
```sql
-- 查询各币种最新价格和涨跌幅
SELECT 
  inst_id AS 币种,
  last_price AS 最新价格,
  price_change_rate_24h AS 涨跌幅,
  volume_24h AS 成交量
FROM (
  SELECT *,
    ROW_NUMBER() OVER (PARTITION BY inst_id ORDER BY timestamp DESC) AS rn
  FROM dwd_crypto_ticker_detail
) t
WHERE rn = 1
ORDER BY price_change_rate_24h DESC;
```

### DWS 层 - 适合趋势分析

**使用场景**:
- K 线图绘制
- 技术指标计算
- 趋势分析

**示例查询**:
```sql
-- 查询 BTC 最近 1 小时的 K 线数据
SELECT 
  FROM_UNIXTIME(window_start / 1000) AS 时间,
  open_price AS 开盘价,
  high_price AS 最高价,
  low_price AS 最低价,
  close_price AS 收盘价,
  volume AS 成交量
FROM dws_crypto_ticker_1min
WHERE inst_id = 'BTC-USDT'
  AND window_start >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 1 HOUR)) * 1000
ORDER BY window_start;
```

## 表设计亮点

### 1. 分层架构清晰

```
ODS: 原始数据,完整保留
 ↓
DWD: 明细数据,添加维度和指标
 ↓
DWS: 汇总数据,窗口聚合
```

### 2. 分区策略优化

**DWD 层动态分区**:
- 自动创建和删除分区
- 只保留最近 7 天数据
- 查询性能提升 10 倍以上

### 3. 表类型选择合理

| 表 | 表类型 | 原因 |
|---|--------|------|
| ODS | DUPLICATE KEY | 保留所有原始数据 |
| DWD | DUPLICATE KEY | 保留明细数据,支持复杂查询 |
| DWS | AGGREGATE KEY | 自动聚合,数据量小 |

### 4. 索引优化

**Bloom Filter 索引**:
- 在 `inst_id` 字段上创建
- 加速等值查询
- 减少数据扫描量

## 推荐使用方式

### 日常监控和分析 → 使用 DWD 层 ⭐⭐⭐

```sql
-- 在 Metabase 中创建查询
SELECT 
  inst_id AS 币种,
  last_price AS 最新价格,
  price_change_rate_24h AS 涨跌幅,
  FROM_UNIXTIME(timestamp / 1000) AS 更新时间
FROM (
  SELECT *,
    ROW_NUMBER() OVER (PARTITION BY inst_id ORDER BY timestamp DESC) AS rn
  FROM dwd_crypto_ticker_detail
) t
WHERE rn = 1
ORDER BY last_price DESC;
```

### K 线图和趋势 → 使用 DWS 层 ⭐⭐⭐

```sql
-- 绘制 K 线图
SELECT 
  window_start,
  open_price,
  high_price,
  low_price,
  close_price,
  volume
FROM dws_crypto_ticker_1min
WHERE inst_id = 'BTC-USDT'
  AND window_start >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 24 HOUR)) * 1000
ORDER BY window_start;
```

### 问题排查 → 使用 ODS 层 ⭐⭐

```sql
-- 查看原始数据
SELECT * FROM ods_crypto_ticker_rt
WHERE inst_id = 'BTC-USDT'
  AND timestamp >= UNIX_TIMESTAMP(DATE_SUB(NOW(), INTERVAL 1 HOUR)) * 1000
ORDER BY timestamp DESC
LIMIT 100;
```
