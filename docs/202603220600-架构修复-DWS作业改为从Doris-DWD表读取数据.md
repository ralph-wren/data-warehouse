# DWS 作业架构修复 - 从 Doris DWD 表读取数据

**时间**: 2026-03-22 06:00  
**问题**: FlinkDWSJob1MinSQL 直接从 Kafka 读取数据,违反数据仓库分层架构  
**状态**: ✅ 已解决

---

## 问题分析

### 发现的问题

**错误的架构设计**:
```
Kafka → DWS 作业 (直接消费 Kafka)
```

**问题**:
1. DWS 作业直接从 Kafka 读取原始数据
2. 跳过了 ODS 和 DWD 层的数据清洗和转换
3. 违反了数据仓库分层架构原则
4. 数据质量无法保证

---

## 正确的架构

### 数据流转

```
Kafka → ODS 作业 → ODS 表 → DWD 作业 → DWD 表 → DWS 作业 → DWS 表
```

### 各层职责

1. **ODS 层** (Operational Data Store - 操作数据存储)
   - 从 Kafka 消费原始数据
   - 进行基本的数据格式转换
   - 写入 Doris ODS 表

2. **DWD 层** (Data Warehouse Detail - 数据仓库明细层)
   - 从 Doris ODS 表读取数据
   - 进行数据清洗和字段补充
   - 计算衍生指标(价差、涨跌幅等)
   - 写入 Doris DWD 表

3. **DWS 层** (Data Warehouse Summary - 数据仓库汇总层)
   - 从 Doris DWD 表读取数据
   - 进行窗口聚合(1分钟 K 线)
   - 计算统计指标(开高低收、成交量等)
   - 写入 Doris DWS 表

## 解决方案

### 修改数据源

**修改前**: 从 Kafka Source 读取
```java
// 错误: DWS 作业直接消费 Kafka
private static String createKafkaSourceDDL(ConfigLoader config) {
    return "CREATE TABLE kafka_ticker_source (\n" +
           "    inst_id STRING,\n" +
           "    `timestamp` BIGINT,\n" +
           "    last_price DECIMAL(20, 8),\n" +
           "    ...\n" +
           ") WITH (\n" +
           "    'connector' = 'kafka',\n" +
           "    'topic' = '" + topic + "',\n" +
           "    ...\n" +
           ")";
}
```

**修改后**: 从 Doris DWD 表读取
```java
// 正确: DWS 作业从 Doris DWD 表读取
private static String createDWDSourceDDL(ConfigLoader config) {
    return "CREATE TABLE doris_dwd_source (\n" +
           "    inst_id STRING,\n" +
           "    `timestamp` BIGINT,\n" +
           "    trade_date DATE,\n" +
           "    trade_hour INT,\n" +
           "    last_price DECIMAL(20, 8),\n" +
           "    bid_price DECIMAL(20, 8),\n" +
           "    ask_price DECIMAL(20, 8),\n" +
           "    spread DECIMAL(20, 8),\n" +
           "    spread_rate DECIMAL(10, 6),\n" +
           "    volume_24h DECIMAL(30, 8),\n" +
           "    high_24h DECIMAL(20, 8),\n" +
           "    low_24h DECIMAL(20, 8),\n" +
           "    open_24h DECIMAL(20, 8),\n" +
           "    price_change_24h DECIMAL(20, 8),\n" +
           "    price_change_rate_24h DECIMAL(10, 6),\n" +
           "    amplitude_24h DECIMAL(10, 6),\n" +
           "    data_source STRING,\n" +
           "    ingest_time BIGINT,\n" +
           "    process_time BIGINT,\n" +
           "    -- 定义事件时间和 Watermark\n" +
           "    event_time AS TO_TIMESTAMP(FROM_UNIXTIME(`timestamp` / 1000)),\n" +
           "    WATERMARK FOR event_time AS event_time - INTERVAL '5' SECOND\n" +
           ") WITH (\n" +
           "    'connector' = 'doris',\n" +
           "    'fenodes' = '" + feNodes + "',\n" +
           "    'table.identifier' = '" + database + "." + dwdTable + "',\n" +
           "    'username' = '" + username + "',\n" +
           "    'password' = '" + password + "'\n" +
           ")";
}
```

### 修改 SQL 查询

**修改前**: 从 kafka_ticker_source 读取
```sql
FROM TABLE(
    TUMBLE(TABLE kafka_ticker_source, DESCRIPTOR(event_time), INTERVAL '1' MINUTE)
)
```

**修改后**: 从 doris_dwd_source 读取
```sql
FROM TABLE(
    TUMBLE(TABLE doris_dwd_source, DESCRIPTOR(event_time), INTERVAL '1' MINUTE)
)
```

### 2. 字段定义

```java

```

## 架构优势

### 1. 分层清晰

```
┌─────────┐     ┌──────────────┐     ┌─────────────┐
│  Kafka  │────▶│  ODS 作业    │────▶│  ODS 表     │
│  Topic  │     │ (DataStream) │     │ (Doris)     │
└─────────┘     └──────────────┘     └─────────────┘
                                            │
                                            ▼
                                     ┌──────────────┐
                                     │  DWD 作业    │
                                     │  (Flink SQL) │
                                     └──────────────┘
                                            │
                                            ▼
                                     ┌─────────────┐
                                     │  DWD 表     │
                                     │  (Doris)    │
                                     └─────────────┘
                                            │
                                            ▼
                                     ┌──────────────┐
                                     │  DWS 作业    │
                                     │  (Flink SQL) │
                                     └──────────────┘
                                            │
                                            ▼
                                     ┌─────────────┐
                                     │  DWS 表     │
                                     │  (Doris)    │
                                     └─────────────┘
```

### 2. 单一数据源

- Kafka 只被 ODS 作业消费一次
- DWD 和 DWS 作业使用 ODS/DWD 层的数据
- 避免重复消费和数据不一致

### 3. 数据质量保证

- ODS 层: 原始数据存储
- DWD 层: 数据清洗和质量检查
- DWS 层: 使用已清洗的高质量数据

### 4. 易于维护

- 每一层的转换逻辑独立
- 可以单独调试和优化
- 支持数据回溯和重新计算

### 5. 支持回溯

- 可以从 ODS 层重新计算 DWD 层
- 可以从 DWD 层重新计算 DWS 层
- 历史数据可以重新处理

## 完整的数据流转

### 1. ODS 作业 (FlinkODSJobDataStream)

```java
// 从 Kafka 读取原始数据
KafkaSource<String> kafkaSource = ...

// 写入 Doris ODS 表
DorisSink<String> dorisSink = ...
```

### 2. DWD 作业 (FlinkDWDJobSQL)

```sql
-- 从 Doris ODS 表读取
CREATE TABLE doris_ods_source (...) WITH ('connector' = 'doris', ...)

-- 数据清洗和字段补充
INSERT INTO doris_dwd_sink
SELECT 
    inst_id,
    `timestamp`,
    CAST(TO_TIMESTAMP(...) AS DATE) as trade_date,
    CAST((ask_price - bid_price) AS DECIMAL(20, 8)) as spread,
    ...
FROM doris_ods_source
```

### 3. DWS 作业 (FlinkDWSJob1MinSQL)

```sql
-- 从 Doris DWD 表读取
CREATE TABLE doris_dwd_source (...) WITH ('connector' = 'doris', ...)

-- 窗口聚合生成 K 线
INSERT INTO doris_dws_1min_sink
SELECT 
    inst_id,
    FIRST_VALUE(last_price) as open_price,
    MAX(last_price) as high_price,
    MIN(last_price) as low_price,
    LAST_VALUE(last_price) as close_price,
    ...
FROM TABLE(
    TUMBLE(TABLE doris_dwd_source, DESCRIPTOR(event_time), INTERVAL '1' MINUTE)
)
GROUP BY inst_id, window_start, window_end
```

## 注意事项

### 1. 数据延迟

- ODS 作业写入 Doris: ~1-5 秒
- DWD 作业从 Doris 读取: ~1-5 秒
- DWS 作业从 Doris 读取: ~1-5 秒
- 总延迟: ~3-15 秒

### 2. 读取模式

- 默认批量读取(一次性读取全部数据)
- 流式读取需要配置 `'scan.mode' = 'stream'`

### 3. 启动顺序

1. 先启动 ODS 作业,写入数据到 ODS 表
2. 再启动 DWD 作业,从 ODS 表读取数据
3. 最后启动 DWS 作业,从 DWD 表读取数据

### 4. 事件时间

- 所有作业都使用相同的事件时间定义
- Watermark 延迟设置为 5 秒
- 保证窗口聚合的正确性

## 技术要点

1. **Doris Flink Connector 支持读取 Doris 表**
   - 使用 `'connector' = 'doris'` 配置
   - 字段名和类型必须与 Doris 表定义一致

2. **事件时间和 Watermark**
   - 可以在 Doris Source 表中定义
   - 支持窗口聚合操作

3. **窗口聚合**
   - 使用 `TUMBLE` 函数定义滚动窗口
   - 窗口大小为 1 分钟
   - 按 inst_id 分组聚合

4. **符合数据仓库分层架构设计原则**
   - ODS: 原始数据存储
   - DWD: 数据清洗和明细
   - DWS: 数据汇总和聚合
