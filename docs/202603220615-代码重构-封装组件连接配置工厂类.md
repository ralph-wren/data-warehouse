## 问题背景

### 重复代码问题

在之前的实现中,每个 Flink 作业都需要手动编写 DDL 语句来创建表:

```java
// FlinkDWDJobSQL.java
private static String createDorisSourceDDL(ConfigLoader config) {
    String feNodes = config.getString("doris.fe.http-url").replace("http://", "");
    String database = config.getString("doris.database");
    String odsTable = config.getString("doris.tables.ods");
    String username = config.getString("doris.fe.username");
    String password = config.getString("doris.fe.password", "");
    
    return "CREATE TABLE doris_ods_source (\n" +
           "    inst_id STRING,\n" +
           "    `timestamp` BIGINT,\n" +
           "    last_price DECIMAL(20, 8),\n" +
           "    ...\n" +  // 大量重复的字段定义
           ") WITH (\n" +
           "    'connector' = 'doris',\n" +
           "    'fenodes' = '" + feNodes + "',\n" +
           "    ...\n" +
           ")";
}

// FlinkDWSJob1MinSQL.java
private static String createDWDSourceDDL(ConfigLoader config) {
    // 几乎相同的代码,只是表名和字段不同
    String feNodes = config.getString("doris.fe.http-url").replace("http://", "");
    String database = config.getString("doris.database");
    String dwdTable = config.getString("doris.tables.dwd");
    ...
}
```

**问题**:
1. 代码重复,每个作业都要写相似的 DDL 创建方法
2. 配置分散,修改配置需要改多个地方
3. 字段定义重复,容易出错
4. 维护困难,添加新字段需要改多个文件

## 解决方案

### 1. 创建工厂类 `FlinkTableFactory`

封装所有表的创建逻辑,统一管理连接配置。

**文件**: `src/main/java/com/crypto/dw/flink/factory/FlinkTableFactory.java`

**核心方法**:

```java
public class FlinkTableFactory {
    
    private final ConfigLoader config;
    
    public FlinkTableFactory(ConfigLoader config) {
        this.config = config;
    }
    
    /**
     * 创建 Kafka Source 表
     * 
     * @param tableName 表名
     * @param schema 字段定义
     * @param withWatermark 是否添加 Watermark
     * @return Kafka Source 表 DDL
     */
    public String createKafkaSourceTable(String tableName, String schema, boolean withWatermark) {
        // 从配置读取 Kafka 连接信息
        String bootstrapServers = config.getString("kafka.bootstrap-servers");
        String topic = config.getString("kafka.topic.crypto-ticker");
        String groupId = config.getString("kafka.consumer.group-id");
        String startupMode = config.getString("kafka.consumer.startup-mode");
        
        // 构建 DDL
        StringBuilder ddl = new StringBuilder();
        ddl.append("CREATE TABLE ").append(tableName).append(" (\n");
        ddl.append(schema);
        
        if (withWatermark) {
            ddl.append(",\n");
            ddl.append("    event_time AS TO_TIMESTAMP(FROM_UNIXTIME(`timestamp` / 1000)),\n");
            ddl.append("    WATERMARK FOR event_time AS event_time - INTERVAL '5' SECOND\n");
        }
        
        ddl.append(") WITH (\n");
        ddl.append("    'connector' = 'kafka',\n");
        ddl.append("    'topic' = '").append(topic).append("',\n");
        ddl.append("    'properties.bootstrap.servers' = '").append(bootstrapServers).append("',\n");
        ddl.append("    'properties.group.id' = '").append(groupId).append("',\n");
        ddl.append("    'scan.startup.mode' = '").append(startupMode).append("',\n");
        ddl.append("    'format' = 'json',\n");
        ddl.append("    'json.fail-on-missing-field' = 'false',\n");
        ddl.append("    'json.ignore-parse-errors' = 'true'\n");
        ddl.append(")");
        
        return ddl.toString();
    }
    
    /**
     * 创建 Doris Source 表
     * 
     * @param tableName Flink 表名
     * @param tableType Doris 表类型(ods/dwd/dws-1min)
     * @param schema 字段定义
     * @return Doris Source 表 DDL
     */
    public String createDorisSourceTable(String tableName, String tableType, String schema) {
        // 从配置读取 Doris 连接信息
        String feNodes = config.getString("doris.fe.http-url").replace("http://", "");
        String database = config.getString("doris.database");
        String dorisTable = getDorisTableName(tableType);
        String username = config.getString("doris.fe.username");
        String password = config.getString("doris.fe.password", "");
        
        // 构建 DDL
        StringBuilder ddl = new StringBuilder();
        ddl.append("CREATE TABLE ").append(tableName).append(" (\n");
        ddl.append(schema);
        ddl.append("\n) WITH (\n");
        ddl.append("    'connector' = 'doris',\n");
        ddl.append("    'fenodes' = '").append(feNodes).append("',\n");
        ddl.append("    'table.identifier' = '").append(database).append(".").append(dorisTable).append("',\n");
        ddl.append("    'username' = '").append(username).append("',\n");
        ddl.append("    'password' = '").append(password).append("'\n");
        ddl.append(")");
        
        return ddl.toString();
    }
    
    /**
     * 创建 Doris Sink 表
     * 
     * @param tableName Flink 表名
     * @param tableType Doris 表类型(ods/dwd/dws-1min)
     * @param schema 字段定义
     * @return Doris Sink 表 DDL
     */
    public String createDorisSinkTable(String tableName, String tableType, String schema) {
        // 从配置读取 Doris 连接信息
        String feNodes = config.getString("doris.fe.http-url").replace("http://", "");
        String beNodes = config.getString("doris.be.nodes", "127.0.0.1:8040");
        String database = config.getString("doris.database");
        String dorisTable = getDorisTableName(tableType);
        String username = config.getString("doris.fe.username");
        String password = config.getString("doris.fe.password", "");
        
        // 构建 DDL
        StringBuilder ddl = new StringBuilder();
        ddl.append("CREATE TABLE ").append(tableName).append(" (\n");
        ddl.append(schema);
        ddl.append("\n) WITH (\n");
        ddl.append("    'connector' = 'doris',\n");
        ddl.append("    'fenodes' = '").append(feNodes).append("',\n");
        ddl.append("    'benodes' = '").append(beNodes).append("',\n");
        ddl.append("    'table.identifier' = '").append(database).append(".").append(dorisTable).append("',\n");
        ddl.append("    'username' = '").append(username).append("',\n");
        ddl.append("    'password' = '").append(password).append("',\n");
        ddl.append("    'sink.label-prefix' = '").append(tableType).append("_").append(System.currentTimeMillis()).append("',\n");
        ddl.append("    'sink.properties.format' = 'json',\n");
        ddl.append("    'sink.properties.read_json_by_line' = 'true'\n");
        ddl.append(")");
        
        return ddl.toString();
    }
    
    /**
     * 根据表类型获取 Doris 表名
     */
    private String getDorisTableName(String tableType) {
        switch (tableType.toLowerCase()) {
            case "ods":
                return config.getString("doris.tables.ods");
            case "dwd":
                return config.getString("doris.tables.dwd");
            case "dws-1min":
                return config.getString("doris.tables.dws-1min");
            default:
                throw new IllegalArgumentException("Unknown table type: " + tableType);
        }
    }
}
```

## 扩展性

### 添加新的表类型

只需要在 `TableSchemas` 中添加新的 Schema 定义:

```java
public class TableSchemas {
    
    /**
     * Doris DWS 5分钟 Sink 表 Schema
     */
    public static final String DORIS_DWS_5MIN_SINK_SCHEMA =
            "    inst_id STRING,\n" +
            "    window_start BIGINT,\n" +
            "    window_end BIGINT,\n" +
            "    window_size INT,\n" +
            "    open_price DECIMAL(20, 8),\n" +
            "    high_price DECIMAL(20, 8),\n" +
            "    low_price DECIMAL(20, 8),\n" +
            "    close_price DECIMAL(20, 8),\n" +
            "    volume DECIMAL(30, 8),\n" +
            "    avg_price DECIMAL(20, 8),\n" +
            "    price_change DECIMAL(20, 8),\n" +
            "    price_change_rate DECIMAL(10, 6),\n" +
            "    tick_count INT,\n" +
            "    process_time BIGINT";
}
```

然后在配置文件中添加表名:

```yaml
doris:
  tables:
    ods: ods_crypto_ticker_rt
    dwd: dwd_crypto_ticker_detail
    dws-1min: dws_crypto_ticker_1min
    dws-5min: dws_crypto_ticker_5min  # 新增
```

使用时:

```java
String dwsSinkDDL = tableFactory.createDorisSinkTable(
    "doris_dws_5min_sink",
    "dws-5min",  // 新的表类型
    TableSchemas.DORIS_DWS_5MIN_SINK_SCHEMA
);
```

## 技术要点

1. **工厂模式**: 使用工厂模式封装对象创建逻辑
2. **配置驱动**: 所有连接信息从配置文件读取
3. **常量定义**: 使用 public static final 定义 Schema 常量
4. **类型安全**: 使用枚举或常量避免字符串拼写错误
5. **可扩展性**: 易于添加新的表类型和字段
