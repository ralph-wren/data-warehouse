# 代码重构 - 使用工厂类简化 Flink 作业代码

**时间**: 2026-03-22 06:30  
**类型**: 代码重构  
**状态**: ✅ 完成

---

## 问题背景

在之前的开发中,我们创建了 `FlinkTableFactory` 工厂类和 `TableSchemas` Schema 定义类来封装组件连接配置。但是 `FlinkDWDJobSQL` 和 `FlinkDWSJob1MinSQL` 两个作业还在使用旧的方式创建表,存在大量重复代码。

### 旧代码问题

每个作业都需要手动编写 100+ 行的 DDL 创建代码:
- 重复的配置读取逻辑
- 重复的字段定义
- 重复的连接参数配置
- 难以维护和修改

---

## 解决方案

### 1. 更新 FlinkDWDJobSQL.java

**修改前**:
```java
// 手动创建 Doris Source 表 (约 50 行代码)
private static String createDorisSourceDDL(ConfigLoader config) {
    String feNodes = config.getString("doris.fe.http-url").replace("http://", "");
    String database = config.getString("doris.database");
    // ... 大量重复代码
    return "CREATE TABLE doris_ods_source (\n" +
           "    inst_id STRING,\n" +
           "    `timestamp` BIGINT,\n" +
           // ... 13 个字段定义
           ") WITH (\n" +
           "    'connector' = 'doris',\n" +
           // ... 5 个连接参数
           ")";
}

// 手动创建 Doris Sink 表 (约 60 行代码)
private static String createDorisSinkDDL(ConfigLoader config) {
    // ... 类似的重复代码
}
```

**修改后**:
```java
// 导入工厂类和 Schema 定义
import com.crypto.dw.flink.factory.FlinkTableFactory;
import com.crypto.dw.flink.schema.TableSchemas;

// 使用工厂类创建表 (只需 3 行代码)
FlinkTableFactory factory = new FlinkTableFactory(config);

String dorisSourceDDL = factory.createDorisSourceTable(
    "doris_ods_source",
    "ods",
    TableSchemas.DORIS_ODS_SOURCE_SCHEMA
);

String dorisSinkDDL = factory.createDorisSinkTable(
    "doris_dwd_sink",
    "dwd",
    TableSchemas.DORIS_DWD_SINK_SCHEMA
);
```

**代码减少**: 从 110 行减少到 10 行,减少了 90% 的代码量

### 2. 更新 FlinkDWSJob1MinSQL.java

**修改前**:
```java
// 手动创建 Doris DWD Source 表 (约 60 行代码,带 Watermark)
private static String createDWDSourceDDL(ConfigLoader config) {
    // ... 大量重复代码
    return "CREATE TABLE doris_dwd_source (\n" +
           "    inst_id STRING,\n" +
           // ... 19 个字段定义
           "    event_time AS TO_TIMESTAMP(FROM_UNIXTIME(`timestamp` / 1000)),\n" +
           "    WATERMARK FOR event_time AS event_time - INTERVAL '5' SECOND\n" +
           ") WITH (\n" +
           // ... 5 个连接参数
           ")";
}

// 手动创建 Doris DWS Sink 表 (约 50 行代码)
private static String createDorisSinkDDL(ConfigLoader config) {
    // ... 类似的重复代码
}
```

**修改后**:
```java
// 导入工厂类和 Schema 定义
import com.crypto.dw.flink.factory.FlinkTableFactory;
import com.crypto.dw.flink.schema.TableSchemas;

// 使用工厂类创建表 (只需 3 行代码)
FlinkTableFactory factory = new FlinkTableFactory(config);

String dwdSourceDDL = factory.createDorisSourceTable(
    "doris_dwd_source",
    "dwd",
    TableSchemas.DORIS_DWD_SOURCE_SCHEMA_WITH_WATERMARK  // 带 Watermark 的 Schema
);

String dorisSinkDDL = factory.createDorisSinkTable(
    "doris_dws_1min_sink",
    "dws-1min",
    TableSchemas.DORIS_DWS_1MIN_SINK_SCHEMA
);
```

**代码减少**: 从 110 行减少到 10 行,减少了 90% 的代码量

## 重构优势

### 1. 代码量大幅减少
- FlinkDWDJobSQL: 从 200 行减少到 110 行 (减少 45%)
- FlinkDWSJob1MinSQL: 从 220 行减少到 120 行 (减少 45%)
- 总计减少约 200 行重复代码

### 2. 维护性提升
- 配置集中管理: 所有连接配置在工厂类中统一管理
- Schema 集中定义: 所有字段定义在 TableSchemas 类中统一维护
- 修改一处生效: 修改工厂类或 Schema 定义,所有作业自动生效

### 3. 可读性提升
- 作业代码更简洁: 只关注业务逻辑,不关注底层配置
- 意图更清晰: 一眼就能看出创建了什么表
- 减少错误: 避免手动编写 DDL 时的拼写错误

### 4. 扩展性提升
- 新增作业更简单: 只需调用工厂方法,无需重复编写 DDL
- 新增表类型更容易: 在工厂类中添加新方法即可
- 支持更多组件: 可以轻松扩展支持其他数据源

## 工厂类使用示例

### 创建 Kafka Source 表
```java
FlinkTableFactory factory = new FlinkTableFactory(config);

String kafkaSourceDDL = factory.createKafkaSourceTable(
    "kafka_source",
    TableSchemas.KAFKA_TICKER_SCHEMA,
    true  // 带 Watermark
);
```

### 创建 Doris Source 表
```java
String dorisSourceDDL = factory.createDorisSourceTable(
    "doris_ods_source",
    "ods",  // 表类型: ods/dwd/dws-1min
    TableSchemas.DORIS_ODS_SOURCE_SCHEMA
);
```

### 创建 Doris Sink 表
```java

    "doris_dwd_sink",
    "dwd",  // 表类型: ods/dwd/dws-1min
    TableSchemas.DORIS_DWD_SINK_SCHEMA
);
```
