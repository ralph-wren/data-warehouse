# ADS 层开发 - 添加实时指标作业和多时间粒度表

**时间**: 2026-03-22 07:00  
**类型**: 功能开发  
**状态**: ✅ 完成

---

## 问题背景

数据仓库已经完成了 ODS、DWD、DWS 三层的开发,但缺少 ADS 应用数据服务层,无法直接为前端/BI 提供业务指标。需要:
1. 添加 ADS 层表结构,存储计算好的业务指标
2. 创建 Flink 作业,从 DWS 层计算实时指标
3. 创建多时间粒度的 K 线表,支持不同时间维度的查询

---

## 解决方案

### 方案设计

采用 **Flink 作业 + 多时间粒度表** 的混合方案:

1. **Flink ADS 作业**: 从 DWS 1分钟 K 线计算实时指标
2. **多时间粒度表**: 创建 5分钟、15分钟、1小时 K 线表
3. **ADS 层表**: 存储实时指标、排行榜、市场概览

**为什么不用物化视图?**
- Doris 物化视图语法限制较多
- 使用 Flink 作业更灵活,可以实现复杂的窗口计算
- 多时间粒度表可以由 Flink 作业直接写入,性能更好

## 实现步骤

### 1. 创建 ADS 层表结构

**文件**: `sql/create_ads_tables.sql`

创建了 3 张 ADS 层表:

#### 1.1 交易对实时指标表 (ads_crypto_ticker_realtime_metrics)

存储每个交易对的实时关键指标:

**价格指标**:
- 当前价格、1分钟前/5分钟前/15分钟前/1小时前价格

**涨跌幅指标**:
- 1分钟/5分钟/15分钟/1小时/24小时涨跌幅

**成交量指标**:
- 1分钟/5分钟/15分钟/1小时/24小时成交量

**波动率指标**:
- 1分钟/5分钟/15分钟/1小时波动率

**价格区间指标**:
- 1分钟/5分钟/1小时的最高价和最低价

**趋势指标**:
- 1分钟/5分钟/1小时趋势 (UP/DOWN/FLAT)

#### 1.2 交易对排行榜表 (ads_crypto_ticker_ranking)

存储各类排行榜数据:
- 涨幅榜 (GAIN)
- 跌幅榜 (LOSS)
- 成交量榜 (VOLUME)
- 波动率榜 (VOLATILITY)

支持多时间窗口: 1MIN/5MIN/15MIN/1HOUR/24HOUR

#### 1.3 市场概览指标表 (ads_crypto_market_overview)

存储整个市场的汇总指标:
- 市场规模: 交易对总数、活跃交易对数、总成交量
- 市场情绪: 上涨/下跌/平盘交易对数量和比例
- 市场波动: 平均/最大/最小波动率
- 价格变动: 平均涨跌幅、最大涨幅、最大跌幅

**表特性**:
- 使用 DUPLICATE KEY 模型 (适合明细数据)
- 动态分区管理 (保留 7 天历史数据)
- LZ4 压缩 (节省存储空间)
- Bloom Filter 索引 (加速查询)

### 3. 创建 Flink ADS 实时指标作业

**文件**: `src/main/java/com/crypto/dw/flink/FlinkADSRealtimeMetricsJob.java`

#### 3.1 作业功能

从 DWS 层读取 1分钟 K 线数据,计算实时指标并写入 ADS 层。

**计算逻辑**:

1. **价格指标**: 使用 LAG 函数获取历史价格
   ```sql
   LAG(close_price, 1) OVER (PARTITION BY inst_id ORDER BY window_end) AS price_1min_ago
   LAG(close_price, 5) OVER (PARTITION BY inst_id ORDER BY window_end) AS price_5min_ago
   ```

2. **涨跌幅指标**: 计算百分比变化
   ```sql
   (close_price - LAG(close_price, 1)) / LAG(close_price, 1) * 100 AS change_rate_1min
   ```

3. **成交量指标**: 使用窗口函数聚合
   ```sql
   SUM(volume) OVER (PARTITION BY inst_id ORDER BY window_end ROWS BETWEEN 4 PRECEDING AND CURRENT ROW) AS volume_5min
   ```

4. **波动率指标**: 使用标准差计算
   ```sql
   (high_price - low_price) / avg_price * 100 AS volatility_1min
   ```

5. **趋势指标**: 使用 CASE WHEN 判断
   ```sql
   CASE 
       WHEN close_price > LAG(close_price, 1) THEN 'UP'
       WHEN close_price < LAG(close_price, 1) THEN 'DOWN'
       ELSE 'FLAT'
   END AS trend_1min
   ```

#### 3.2 代码特点

**使用工厂模式** (减少重复代码):
```java
// 使用 FlinkEnvironmentFactory 创建环境

    "dws_source",
    "dws_crypto_ticker_1min",
    TableSchemas.DORIS_DWS_1MIN_SOURCE_SCHEMA_WITH_WATERMARK
);
```

**使用 TableSchemas 统一管理字段定义**:

## 技术亮点

### 1. 工厂模式封装 ⭐⭐⭐

使用工厂类减少重复代码:
- `FlinkEnvironmentFactory`: 封装 Environment 创建逻辑
- `FlinkTableFactory`: 封装表创建逻辑
- `TableSchemas`: 集中管理字段定义

**代码减少**: 约 150 行 (相比不使用工厂模式)

### 2. 窗口函数应用 ⭐⭐

使用 Flink SQL 窗口函数实现复杂计算:
- LAG 函数: 获取历史价格
- SUM OVER: 计算滚动窗口成交量
- MAX/MIN OVER: 计算滚动窗口价格区间
- CASE WHEN: 判断价格趋势

### 3. 多时间粒度设计 ⭐⭐

支持多种时间维度的查询:
- 1分钟: 实时监控
- 5分钟: 短期趋势
- 15分钟: 中期趋势
- 1小时: 长期趋势

### 4. 动态分区管理 ⭐

使用 Doris 动态分区功能:
- 自动创建新分区
- 自动删除过期分区
- 保留 7 天历史数据
