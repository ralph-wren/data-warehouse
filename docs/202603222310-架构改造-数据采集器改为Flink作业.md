# 架构改造 - 数据采集器改为 Flink 作业

**时间**: 2026-03-22 23:10  
**类型**: 架构改造  
**状态**: ✅ 完成

## 问题描述

原有的 `DataCollectorMain` 是一个独立的 Java 应用程序,通过 WebSocket 从 OKX 获取实时行情数据并发送到 Kafka。存在以下问题:

1. **独立运行**: 需要单独启动和管理,不在 Flink 统一管理范围内
2. **缺少容错**: 没有 Flink 的 Checkpoint 和容错机制
3. **监控困难**: 无法通过 Flink Web UI 监控
4. **资源隔离**: 与 Flink 作业分离,资源管理不统一

## 解决方案

将数据采集器改造为 Flink Source Function,作为 Flink 作业的一部分运行。

### 设计原则 ⭐

遵循单一职责原则:
- **Source Function**: 只负责从 WebSocket 接收数据
- **Kafka Sink**: 只负责将数据写入 Kafka
- **数据流转**: WebSocket → Source → Stream → Sink → Kafka

这样的设计更清晰、更易维护、更符合 Flink 的架构理念。

### 实现步骤

#### 1. 创建 OKXWebSocketSourceFunction

**文件**: `src/main/java/com/crypto/dw/flink/source/OKXWebSocketSourceFunction.java`

**功能**:
- 继承 `RichSourceFunction<String>`
- 封装 WebSocket 客户端逻辑 (内部类 `OKXWebSocketClientInternal`)
- 实现 Flink Source 接口 (`open`, `run`, `cancel`)
- 支持 Flink 的 Checkpoint 机制 (使用 checkpoint lock)
- 支持自动重连和错误处理
- 定期打印统计信息

**关键设计**:
```java
public class OKXWebSocketSourceFunction extends RichSourceFunction<String> {
    
    private final ConfigLoader config;
    private final List<String> symbols;
    
    // WebSocket 客户端 (transient 不参与序列化)
    private transient OKXWebSocketClientInternal wsClient;
    
    // 运行标志
    private final AtomicBoolean running = new AtomicBoolean(true);
    
    @Override
    public void run(SourceContext<String> ctx) throws Exception {
        // 创建 WebSocket 客户端
        wsClient = new OKXWebSocketClientInternal(wsUrl, ctx, symbols);
        wsClient.connectBlocking();
        
        // 保持运行,监控连接状态
        while (running.get() && wsClient.isOpen()) {
            Thread.sleep(1000);
            
            // 自动重连
            if (!wsClient.isOpen() && running.get()) {
                wsClient.reconnectBlocking();
            }
        }
    }
    
    // 内部 WebSocket 客户端
    private class OKXWebSocketClientInternal extends WebSocketClient {
        
        private final SourceContext<String> sourceContext;
        
        @Override
        public void onMessage(String message) {
            // 发送数据到 Flink 数据流
            // 注意: 使用 checkpoint lock 保证 Exactly-Once 语义
            synchronized (sourceContext.getCheckpointLock()) {
                sourceContext.collect(message);
            }
        }
    }
}
```

**重要改进**: Source Function 只负责接收数据,不再直接写入 Kafka

#### 2. 创建 FlinkDataCollectorJob

**文件**: `src/main/java/com/crypto/dw/flink/FlinkDataCollectorJob.java`

**功能**:
- 使用 `FlinkEnvironmentFactory` 创建 Flink 环境
- 添加 `OKXWebSocketSourceFunction` 作为数据源
- 添加 `KafkaSink` 写入数据到 Kafka ⭐
- 支持命令行参数指定交易对
- 自动配置 Web UI (端口 8085)

**关键代码**:
```java
public class FlinkDataCollectorJob {
    
    public static void main(String[] args) throws Exception {
        // 加载配置
        ConfigLoader config = ConfigLoader.getInstance();
        
        // 创建 Flink 环境 (使用工厂类)
        FlinkEnvironmentFactory envFactory = new FlinkEnvironmentFactory(config);
        StreamExecutionEnvironment env = envFactory.createStreamEnvironment(
            "flink-data-collector-job", 8085
        );
        
        // 获取交易对列表
        List<String> symbols = getSymbols(args, config);
        
        // 创建 Kafka Sink (使用 Flink Kafka Connector)
        String kafkaTopic = config.getString("kafka.topic.crypto-ticker");
        String kafkaBootstrapServers = config.getString("kafka.bootstrap-servers");
        
        KafkaSink<String> kafkaSink = KafkaSink.<String>builder()
            .setBootstrapServers(kafkaBootstrapServers)
            .setRecordSerializer(
                KafkaRecordSerializationSchema.builder()
                    .setTopic(kafkaTopic)
                    .setValueSerializationSchema(new SimpleStringSchema())
                    .build()
            )
            .setDeliveryGuarantee(DeliveryGuarantee.AT_LEAST_ONCE)  // 至少一次保证
            .build();
        
        // 创建数据流: WebSocket Source → Kafka Sink
        env.addSource(new OKXWebSocketSourceFunction(config, symbols))
           .name("OKX WebSocket Source")
           .sinkTo(kafkaSink)  // 使用 Kafka Sink 写入数据
           .name("Kafka Sink");
        
        // 执行作业
        env.execute("Flink Data Collector Job");
    }
}
```

**架构优势**: Source 和 Sink 职责分离,符合 Flink 最佳实践

## 技术要点

### 1. RichSourceFunction

`RichSourceFunction` 是 Flink 提供的 Source 接口,支持:
- `open()`: 初始化资源
- `run()`: 持续运行,接收数据
- `cancel()`: 清理资源,优雅关闭

### 2. Checkpoint Lock

```java
// 在 checkpoint lock 中发送数据,保证 Exactly-Once 语义
synchronized (sourceContext.getCheckpointLock()) {
    sourceContext.collect(message);
}
```

- 保证数据发送和 Checkpoint 的原子性
- 避免数据重复或丢失

### 3. transient 关键字

```java
private transient OKXWebSocketClientInternal wsClient;
```

- `transient` 表示该字段不参与序列化
- WebSocket 连接无法序列化
- 在 `run()` 方法中重新创建

### 4. AtomicBoolean 运行标志

```java
private final AtomicBoolean running = new AtomicBoolean(true);
```

- 线程安全的布尔值
- 用于控制 Source Function 的运行状态
- `cancel()` 方法设置为 false,`run()` 方法检查并退出

### 5. Kafka Sink

```java
KafkaSink<String> kafkaSink = KafkaSink.<String>builder()
    .setBootstrapServers(kafkaBootstrapServers)
    .setRecordSerializer(...)
    .setDeliveryGuarantee(DeliveryGuarantee.AT_LEAST_ONCE)
    .build();
```

- 使用 Flink Kafka Connector
- 支持 AT_LEAST_ONCE 保证
- 自动处理序列化和分区

### 6. 自动重连机制

```java
// 如果连接断开,尝试重连
if (!wsClient.isOpen() && running.get()) {
    logger.warn("WebSocket disconnected, attempting to reconnect...");
    wsClient.reconnectBlocking();
}
```

- 每秒检查一次连接状态
- 连接断开时自动重连
- 重连失败抛出异常,触发 Flink 重启

### 7. 统计信息线程

```java
private void startStatisticsThread() {
    Thread statsThread = new Thread(() -> {
        while (running.get()) {
            Thread.sleep(60000); // 每分钟打印一次
            logger.info("Statistics: ...");
        }
    });
    statsThread.setDaemon(true);
    statsThread.start();
}
```

- Daemon 线程,不阻止 JVM 退出
- 每分钟打印一次统计信息
- 包含 WebSocket 和 Kafka 的指标
