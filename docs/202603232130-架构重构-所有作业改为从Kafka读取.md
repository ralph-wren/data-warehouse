# 架构重构 - 所有作业改为从 Kafka 读取

**日期**: 2026-03-23 21:30  
**问题**: DWD/DWS 作业从 Doris 读取导致全表扫描，性能差  
**解决方案**: 所有作业（ODS/DWD/DWS）都改为从 Kafka 读取，实时流式处理  
**状态**: ✅ 已完成

---

## 一、问题背景

### 原架构（错误）❌

```
Kafka → ODS作业 → Doris ODS → DWD作业 → Doris DWD → DWS作业 → Doris DWS
              ↓                    ↓                    ↓
           实时流              全表扫描❌            全表扫描❌
```

**问题**:
1. DWD/DWS 作业每次启动都要扫描全表
2. 性能差，启动时间长（可能几小时）
3. 不是实时处理，需要重启作业才能读取新数据
4. Checkpoint 恢复慢，资源浪费

### 正确架构（实施后）✅

```
                    ┌─→ ODS作业 → Doris ODS (归档)
                    │
Kafka (crypto_ticker)─→ DWD作业 → Doris DWD (清洗)
                    │
                    └─→ DWS作业 → Doris DWS (聚合)
```

**优点**:
- ✅ 所有作业都是实时流式处理
- ✅ 只处理新数据，无全表扫描
- ✅ Checkpoint 恢复快（< 10 秒）
- ✅ 精准一次性保障（Kafka Offset + Doris Label）

## 七、注意事项

### 1. Kafka 数据保留

**配置**:
```yaml
kafka:
  topic:
    crypto-ticker:
      retention.ms: 2592000000  # 30 天
      retention.bytes: 107374182400  # 100 GB
```

### 2. Consumer Group Offset 管理

**查看 Offset**:
```bash
kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --group flink-dwd-consumer \
  --describe
```

**重置 Offset**（重新处理历史数据）:
```bash
kafka-consumer-groups.sh --bootstrap-server localhost:9092 \
  --group flink-dwd-consumer \
  --reset-offsets --to-earliest \
  --topic crypto_ticker \
  --execute
```

### 3. 数据一致性

**问题**: ODS/DWD/DWS 三个作业并行运行，数据可能不一致

**解决方案**:
- 使用相同的 Kafka Offset（从同一个 Topic 读取）
- 使用 Watermark 保证事件时间一致
- 使用 Checkpoint 保证故障恢复一致
