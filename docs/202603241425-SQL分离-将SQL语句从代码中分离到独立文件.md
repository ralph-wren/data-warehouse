# SQL 分离 - 将 SQL 语句从代码中分离到独立文件

**日期**: 2026-03-24 14:25  
**类型**: 代码重构  
**优先级**: 高

## 问题概述

之前 SQL 语句直接编码在 Java 代码中，存在以下问题：
1. SQL 需要转换为字符串格式，加双引号和换行符，不便于编写和修改
2. SQL 格式不清晰，难以阅读和维护
3. SQL 无法使用标准的 SQL 编辑器进行格式化和语法检查
4. SQL 版本管理困难
5. 代码文件过长，可读性差

## 解决方案

将 SQL 语句从 Java 代码中分离出来，放到独立的 `.sql` 文件中，通过工具类加载。

### 3. 创建 SQL 加载工具类

**文件**: `src/main/java/com/crypto/dw/utils/SqlFileLoader.java`

**功能**: 从 classpath 加载 SQL 文件

**核心方法**:
```java
/**
 * 从 classpath 加载 SQL 文件
 * 
 * @param sqlFilePath SQL 文件路径（相对于 classpath）
 * @return SQL 内容
 * @throws RuntimeException 如果文件不存在或读取失败
 */
public static String loadSql(String sqlFilePath) {
    logger.info("加载 SQL 文件: {}", sqlFilePath);
    
    try (InputStream inputStream = SqlFileLoader.class.getClassLoader()
            .getResourceAsStream(sqlFilePath)) {
        if (inputStream == null) {
            throw new RuntimeException("SQL 文件不存在: " + sqlFilePath);
        }
        
        // 读取文件内容
        String sql = new BufferedReader(new InputStreamReader(inputStream, StandardCharsets.UTF_8))
                .lines()
                .collect(Collectors.joining("\n"));
        
        logger.info("✅ SQL 文件加载成功，长度: {} 字符", sql.length());
        return sql;
        
    } catch (Exception e) {
        logger.error("❌ 加载 SQL 文件失败: {}", sqlFilePath, e);
        throw new RuntimeException("加载 SQL 文件失败: " + sqlFilePath, e);
    }
}
```

**特点**:
- 使用 ClassLoader 加载，支持 JAR 包内的文件
- UTF-8 编码，支持中文注释
- 详细的日志输出
- 完善的错误处理

### 5. 更新 Maven 配置

**文件**: `pom.xml`

**修改内容**: 添加 SQL 文件资源配置

```xml
<resources>
    <resource>
        <directory>src/main/resources</directory>
        <includes>
            <include>**/*</include>
        </includes>
    </resource>
    <!-- 将 SQL 文件打包到 classpath -->
    <resource>
        <directory>sql</directory>
        <targetPath>sql</targetPath>
        <includes>
            <include>**/*.sql</include>
        </includes>
    </resource>
</resources>
```

**作用**:
- 将 `sql/` 目录下的所有 `.sql` 文件打包到 JAR 包中
- 打包后路径为 `sql/flink/*.sql`
- 可以通过 ClassLoader 加载

## 技术要点

### 1. ClassLoader 加载资源

```java
InputStream inputStream = SqlFileLoader.class.getClassLoader()
        .getResourceAsStream(sqlFilePath);
```

**说明**:
- 使用 ClassLoader 加载资源
- 支持从 JAR 包中加载
- 路径相对于 classpath

### 2. UTF-8 编码

```java
new InputStreamReader(inputStream, StandardCharsets.UTF_8)
```

**说明**:
- 使用 UTF-8 编码
- 支持中文注释
- 避免乱码问题

### 3. Stream API 读取文件

```java
String sql = new BufferedReader(new InputStreamReader(inputStream, StandardCharsets.UTF_8))
        .lines()
        .collect(Collectors.joining("\n"));
```

**说明**:
- 使用 Stream API 读取所有行
- 使用换行符连接
- 简洁高效

### 4. Maven 资源配置

```xml
<resource>
    <directory>sql</directory>
    <targetPath>sql</targetPath>
    <includes>
        <include>**/*.sql</include>
    </includes>
</resource>
```

**说明**:
- `directory`: 源目录
- `targetPath`: 目标路径（相对于 classpath）
- `includes`: 包含的文件模式
