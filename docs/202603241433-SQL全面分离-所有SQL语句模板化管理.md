# SQL 全面分离 - 所有 SQL 语句模板化管理

**日期**: 2026-03-24 14:33  
**类型**: 代码重构  
**优先级**: 高

## 问题概述

在之前的优化中，我们已经将 INSERT SQL 从代码中分离到独立文件。但是 DDL（CREATE TABLE）语句仍然在 Java 代码中通过字符串拼接生成，存在以下问题：

1. DDL 语句分散在 FlinkTableFactory 中，难以统一管理
2. 字符串拼接方式不便于修改和维护
3. 无法使用 SQL 编辑器进行格式化和语法检查
4. 参数替换逻辑混杂在代码中，不够清晰

## 解决方案

将所有 DDL 语句也分离到 SQL 模板文件中，并增强 SqlFileLoader 支持参数替换功能。

### 1. 创建 DDL 模板文件

#### 1.1 Kafka Source DDL 模板

**文件**: `sql/flink/ddl/kafka_source.sql`

**功能**: 创建 Kafka Source 表的 DDL 模板

**参数**:
- `${tableName}` - 表名
- `${schema}` - 字段定义
- `${watermark}` - Watermark 定义（可选）
- `${bootstrapServers}` - Kafka 服务器地址
- `${topic}` - Kafka Topic
- `${groupId}` - Consumer Group ID
- `${startupMode}` - 启动模式

**内容**:
```sql
CREATE TABLE ${tableName} (
${schema}${watermark}
) WITH (
    'connector' = 'kafka',
    'topic' = '${topic}',
    'properties.bootstrap.servers' = '${bootstrapServers}',
    'properties.group.id' = '${groupId}',
    'scan.startup.mode' = '${startupMode}',
    'format' = 'json',
    'json.fail-on-missing-field' = 'false',
    'json.ignore-parse-errors' = 'true'
)
```

### 2. 增强 SqlFileLoader 工具类

#### 2.1 参数替换方法

```java
/**
 * 从 classpath 加载 SQL 文件并替换参数

 * 参数格式：${paramName}

 * @param sqlFilePath SQL 文件路径
```

**使用示例**:


```java
/**
 * 验证 SQL 中是否还有未替换的参数
 */
public static boolean hasUnresolvedParams(String sql)

/**
 * 获取 SQL 中所有未替换的参数名称
 */
public static List<String> getUnresolvedParams(String sql)
```

## 技术要点

### 1. 参数替换实现

```java
// 替换 ${paramName} 为实际值
String placeholder = "${" + paramName + "}";
result = result.replace(placeholder, paramValue);
```

**特点**:
- 简单高效
- 支持任意参数名
- 不限制参数数量

### 2. 参数验证

```java
// 检查是否还有未替换的参数
public static boolean hasUnresolvedParams(String sql) {
    return sql.contains("${");
}
```

**用途**:
- 防止参数遗漏
- 提供清晰的错误提示
- 提高代码健壮性

### 3. Watermark 条件处理

```java
// 根据条件构建 Watermark 定义
String watermark = "";
if (withWatermark) {
    watermark = ",\n" +
            "    event_time AS TO_TIMESTAMP(...),\n" +
            "    WATERMARK FOR event_time AS ...\n";
} else {
    watermark = "\n";
}
```

**说明**:
- 条件参数在 Java 代码中处理
- SQL 模板保持简洁
- 灵活性和可读性兼顾
