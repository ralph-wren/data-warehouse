# 工厂类增强 - 支持显式传入库名和表名

**日期**: 2026-03-24 14:50  
**类型**: 功能增强  
**优先级**: 中

## 需求概述

用户希望 `FlinkTableFactory` 的 `createDorisSinkTable` 和 `createDorisSourceTable` 方法能够支持显式传入库名和表名，而不是只能通过表类型（ods/dwd/dws-1min）从配置文件读取，以提高灵活性。

---

## 问题分析

### 当前实现的局限性

**原有方法签名**:
```java
// Doris Source
public String createDorisSourceTable(String tableName, String tableType, String schema)

// Doris Sink
public String createDorisSinkTable(String tableName, String tableType, String schema)
```

**局限性**:
1. 只能通过 `tableType`（ods/dwd/dws-1min）从配置文件读取库名和表名
2. 无法动态指定目标数据库和表
3. 无法写入到测试数据库或临时表
4. 无法读取其他项目的表

**使用场景受限**:
- ❌ 无法写入到测试数据库
- ❌ 无法创建临时表
- ❌ 无法跨项目读写数据
- ❌ 无法动态指定表名

---

## 解决方案

### 设计思路

采用**方法重载**的方式，保留原有方法的同时，新增支持显式传入库名和表名的重载方法：

1. **保留原有方法**: 使用表类型（从配置文件读取），保持向后兼容
2. **新增重载方法**: 支持显式传入库名和表名，提高灵活性
3. **代码复用**: 原有方法内部调用新的重载方法，避免代码重复
