## Flink 特性使用

### 1. KeyedState（键控状态）

**用途**: 追踪每个交易对的历史价格

**实现**: `PriceAnomalyDetector`
```java
// 为每个交易对维护独立的状态
private transient ValueState<BigDecimal> lastPriceState;

// 状态会自动持久化到 StateBackend
ValueStateDescriptor<BigDecimal> descriptor = new ValueStateDescriptor<>(
    "last-price",  // 状态名称
    BigDecimal.class  // 状态类型
);
lastPriceState = getRuntimeContext().getState(descriptor);
```

**特点**:
- 每个 key（交易对）有独立的状态
- 自动持久化，支持故障恢复
- 高效的状态访问

### 2. TumblingWindow（滚动窗口）

**用途**: 5 分钟窗口聚合统计

**实现**:
```java
.window(TumblingEventTimeWindows.of(Time.minutes(5)))  // 5 分钟滚动窗口
```

**特点**:
- 基于事件时间（Event Time）
- 窗口不重叠
- 每 5 分钟输出一次结果

**窗口示例**:
```
时间轴: |--W1--|--W2--|--W3--|--W4--|
窗口1:  00:00 - 00:05
窗口2:  00:05 - 00:10
窗口3:  00:10 - 00:15
...
```

---

### 3. Watermark（水位线）

**用途**: 处理乱序数据

**实现**:
```java
WatermarkStrategy
    .<String>forBoundedOutOfOrderness(Duration.ofSeconds(5))
    .withTimestampAssigner(...)
```

**特点**:
- 允许 5 秒的乱序数据
- 超过 5 秒的延迟数据会被丢弃
- 保证窗口计算的正确性

**Watermark 机制**:
```
数据流: [t1, t3, t2, t5, t4, t7, t6]
        ↓
Watermark: 允许 5 秒乱序
        ↓
窗口触发: 当 Watermark >= 窗口结束时间
```

```java
        // 增量累加：每来一条数据就更新累加器

```

**优势**:

**对比**:
| 方式 | 内存占用 | 性能 |
|-----|---------|------|
| 存储所有数据 | O(n) | 低 |
| 增量聚合 | O(1) | 高 ✅ |

## 异常检测算法

### 检测逻辑

**阈值**: 价格变化超过 5%

**公式**:
```
变化率 = |当前价格 - 上一次价格| / 上一次价格

if 变化率 > 0.05:
    标记为异常
```

**示例**:
```
上一次价格: 50000
当前价格: 53000
变化率: |53000 - 50000| / 50000 = 0.06 = 6%
结果: 异常 ⚠️
```

## 指标计算

### 1. 价格波动率

**公式**: 方差（Variance）
```
方差 = E[X²] - E[X]²
     = (Σ price²) / n - ((Σ price) / n)²
```

**说明**:
- 衡量价格的波动程度
- 值越大，波动越剧烈
- 简化计算，不开方（避免精度损失）

---

### 2. 价格变化率

**公式**:
```
变化率 = (最后价格 - 第一个价格) / 第一个价格 × 100%
```

**示例**:
```
窗口内第一个价格: 50000
窗口内最后一个价格: 51000
变化率: (51000 - 50000) / 50000 × 100% = 2%
```

---

### 3. 价格趋势

**判断逻辑**:
```
if 变化率 > 1%:
    趋势 = "上涨"
else if 变化率 < -1%:
    趋势 = "下跌"
else:
    趋势 = "震荡"
```

## 数据库表结构

**表名**: `crypto_dw.ads_market_monitor_5min`

**字段**:
```sql
CREATE TABLE crypto_dw.ads_market_monitor_5min (
    -- 维度字段
    symbol VARCHAR(20) COMMENT '交易对符号',
    window_start BIGINT COMMENT '窗口开始时间',
    window_end BIGINT COMMENT '窗口结束时间',
    
    -- 数据统计
    data_count BIGINT COMMENT '窗口内数据量',
    
    -- 价格指标
    avg_price DECIMAL(20, 8) COMMENT '平均价格',
    max_price DECIMAL(20, 8) COMMENT '最高价格',
    min_price DECIMAL(20, 8) COMMENT '最低价格',
    price_volatility DECIMAL(20, 8) COMMENT '价格波动率',
    price_change_rate DECIMAL(10, 6) COMMENT '价格变化率（%）',
    
    -- 交易量指标
    avg_volume DECIMAL(20, 8) COMMENT '平均交易量',
    max_volume DECIMAL(20, 8) COMMENT '最大交易量',
    
    -- 趋势和异常
    trend VARCHAR(10) COMMENT '价格趋势',
    has_anomaly BOOLEAN COMMENT '是否有异常',
    anomaly_count INT COMMENT '异常数量'
)
DUPLICATE KEY(symbol, window_start)
PARTITION BY RANGE(window_start) ()
DISTRIBUTED BY HASH(symbol) BUCKETS 10;
```

**分区策略**:
- 动态分区，按天分区
- 保留最近 7 天数据
- 自动创建未来 3 天分区

**索引**:
- `idx_symbol`: 交易对索引
- `idx_trend`: 趋势索引

## 学习要点

### Flink 核心概念

1. **Event Time vs Processing Time**:
   - Event Time: 数据产生的时间（使用 ✅）
   - Processing Time: 数据处理的时间

2. **Watermark 机制**:
   - 处理乱序数据
   - 触发窗口计算
   - 平衡延迟和完整性

3. **状态管理**:
   - ValueState: 单个值
   - ListState: 列表
   - MapState: 键值对

4. **窗口类型**:
   - Tumbling: 滚动窗口（不重叠）✅
   - Sliding: 滑动窗口（重叠）
   - Session: 会话窗口（动态）

5. **聚合方式**:
   - AggregateFunction: 增量聚合 ✅
   - ReduceFunction: 归约聚合
   - ProcessWindowFunction: 全量聚合
