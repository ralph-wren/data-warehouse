**类型**: 功能开发

## 问题描述

需要创建一个复杂的 Flink 流处理作业，展示以下技术特性：
- 双流 Join（现货 + 期货）
- Interval Join（时间窗口关联）
- 广播流（Redis 白名单）
- 维度关联（过滤数据）
- 性能优化思考

## 业务场景

计算加密货币现货和期货之间的价差套利机会：
- 实时监控现货价格和期货价格
- 计算价差和价差率
- 判断套利方向
- 估算利润（扣除手续费）
- 使用白名单过滤交易对

## 技术架构

### 数据流图

```
Kafka(现货) ──┐
              ├─→ Interval Join ──→ 计算套利空间 ──┐
Kafka(期货) ──┘                                   │
                                                  ├─→ 广播 Join ──→ 过滤 ──→ Doris
Redis(白名单) ──→ 广播流 ─────────────────────────┘
```

### 技术特性

1. **双流 Join**
   - 现货价格流 + 期货价格流
   - 使用 Interval Join 在时间窗口内关联
   - 时间窗口：±10 秒

2. **Interval Join**
   - 优势：相比 Window Join，状态更小，性能更好
   - 只保留时间窗口内的数据
   - 自动清理过期数据

3. **广播流**
   - 从 Redis 读取交易对白名单
   - 定期刷新（每 60 秒）
   - 分发到所有并行实例

4. **维度关联**
   - 使用广播状态过滤数据
   - 避免每条数据查询 Redis
   - 大幅提升性能

5. **性能优化**
   - 使用 Interval Join 减少状态存储
   - 广播流避免频繁查询 Redis
   - 连接池管理 Redis 连接
   - 合理设置并行度

### 2. 套利机会计算器

**功能**:
- 计算价差：`spread = futures_price - spot_price`
- 计算价差率：`spread_rate = spread / spot_price * 100`
- 判断套利方向：
  - 期货价格 > 现货价格：做空期货/做多现货
  - 现货价格 > 期货价格：做多期货/做空现货
- 估算利润：`profit = |spread| - fee * 2`（扣除双边手续费）

**代码**:
```java
public static class ArbitrageCalculator 
        extends ProcessJoinFunction<SpotPrice, FuturesPrice, ArbitrageOpportunity> {
    
    @Override
    public void processElement(
            SpotPrice spot,
            FuturesPrice futures,
            Context ctx,
            Collector<ArbitrageOpportunity> out) {
        
        // 计算价差
        BigDecimal spread = futures.price.subtract(spot.price);
        
        // 计算价差率
        BigDecimal spreadRate = spread.divide(spot.price, 6, RoundingMode.HALF_UP);
        
        // 判断是否有套利机会（价差率绝对值超过 0.5%）
        if (spreadRate.abs().compareTo(ARBITRAGE_THRESHOLD) > 0) {
            ArbitrageOpportunity opportunity = new ArbitrageOpportunity();
            opportunity.symbol = spot.symbol;
            opportunity.spotPrice = spot.price;
            opportunity.futuresPrice = futures.price;
            opportunity.spread = spread;
            opportunity.spreadRate = spreadRate.multiply(new BigDecimal("100"));
            
            // 判断套利方向
            if (spread.compareTo(BigDecimal.ZERO) > 0) {
                opportunity.arbitrageDirection = "做空期货/做多现货";
            } else {
                opportunity.arbitrageDirection = "做多期货/做空现货";
            }
            
            // 估算利润（扣除 0.1% 手续费）
            BigDecimal fee = spot.price.multiply(new BigDecimal("0.001"));
            opportunity.profitEstimate = spread.abs().subtract(fee.multiply(new BigDecimal("2")));
            
            opportunity.timestamp = System.currentTimeMillis();
            
            out.collect(opportunity);
        }
    }
}
```

### 3. 白名单过滤器

**功能**:
- 接收广播的白名单数据
- 更新广播状态
- 使用广播状态过滤数据流

**代码**:
```java
public static class WhitelistFilter 
        extends BroadcastProcessFunction<ArbitrageOpportunity, Tuple2<String, Boolean>, ArbitrageOpportunity> {
    
    @Override
    public void processElement(
            ArbitrageOpportunity opportunity,
            ReadOnlyContext ctx,
            Collector<ArbitrageOpportunity> out) throws Exception {
        
        // 读取广播状态
        ReadOnlyBroadcastState<String, Boolean> whitelistState = 
            ctx.getBroadcastState(whitelistStateDescriptor);
        
        // 检查是否在白名单中
        Boolean isWhitelisted = whitelistState.get(opportunity.symbol);
        
        if (isWhitelisted != null && isWhitelisted) {
            out.collect(opportunity);
        }
    }
    
    @Override
    public void processBroadcastElement(
            Tuple2<String, Boolean> whitelistEntry,
            Context ctx,
            Collector<ArbitrageOpportunity> out) throws Exception {
        
        // 更新广播状态
        BroadcastState<String, Boolean> whitelistState = 
            ctx.getBroadcastState(whitelistStateDescriptor);
        
        whitelistState.put(whitelistEntry.f0, whitelistEntry.f1);
    }
}
```

### 4. Redis 白名单数据源

**功能**:
- 定期从 Redis 读取交易对白名单
- 发送到广播流
- 使用连接池管理 Redis 连接

**Redis 数据结构**:
- Key: `crypto:whitelist`
- Type: Set
- Value: `["BTC-USDT", "ETH-USDT", "SOL-USDT", ...]`

**代码**:
```java
public static class RedisWhitelistSource 
        extends RichMapFunction<Long, Tuple2<String, Boolean>> 
        implements SourceFunction<Tuple2<String, Boolean>> {
    
    private transient JedisPool jedisPool;
    private volatile boolean isRunning = true;
    private static final long REFRESH_INTERVAL_MS = 60000;  // 60 秒
    
    @Override
    public void open(Configuration parameters) {
        // 初始化 Redis 连接池
        JedisPoolConfig poolConfig = new JedisPoolConfig();
        poolConfig.setMaxTotal(10);
        poolConfig.setMaxIdle(5);
        poolConfig.setMinIdle(1);
        
        jedisPool = new JedisPool(poolConfig, redisHost, redisPort);
    }
    
    @Override
    public void run(SourceContext<Tuple2<String, Boolean>> ctx) throws Exception {
        while (isRunning) {
            // 从 Redis 读取白名单
            Set<String> whitelist = fetchWhitelistFromRedis();
            
            // 发送到广播流
            for (String symbol : whitelist) {
                ctx.collect(new Tuple2<>(symbol, true));
            }
            
            // 等待下一次刷新
            Thread.sleep(REFRESH_INTERVAL_MS);
        }
    }
    
    private Set<String> fetchWhitelistFromRedis() {
        try (Jedis jedis = jedisPool.getResource()) {
            return jedis.smembers("crypto:whitelist");
        }
    }
}
```

## 性能优化

### 1. Interval Join vs Window Join

**Interval Join 优势**:
- 状态更小：只保留时间窗口内的数据
- 性能更好：自动清理过期数据
- 延迟更低：不需要等待窗口关闭

**对比**:
```
Window Join:
- 状态大小：窗口大小 × 数据量
- 延迟：窗口大小 + 处理时间

Interval Join:
- 状态大小：时间窗口 × 数据量（通常更小）
- 延迟：处理时间（无需等待窗口关闭）
```

### 2. 广播流 vs 查询 Redis

**广播流优势**:
- 避免每条数据查询 Redis
- 减少网络开销
- 提高吞吐量

**性能对比**:
```
查询 Redis:
- 每条数据：1 次网络请求
- 延迟：~1ms
- 吞吐量：~1000 条/秒

广播流:
- 每条数据：0 次网络请求
- 延迟：~0.1ms
- 吞吐量：~10000 条/秒
```

### 3. 连接池管理

**优势**:
- 避免频繁创建连接
- 复用连接，减少开销
- 提高稳定性

**配置**:
```java
JedisPoolConfig poolConfig = new JedisPoolConfig();
poolConfig.setMaxTotal(10);  // 最大连接数
poolConfig.setMaxIdle(5);    // 最大空闲连接数
poolConfig.setMinIdle(1);    // 最小空闲连接数
poolConfig.setTestOnBorrow(true);  // 获取连接时测试
```

### 4. 并行度设置

**建议**:
- 并行度 = CPU 核心数 × 1-2
- 根据 Kafka 分区数调整
- 监控背压，动态调整

**配置**:
```yaml
flink:
  execution:
    parallelism: 4  # 根据 CPU 核心数调整
```
