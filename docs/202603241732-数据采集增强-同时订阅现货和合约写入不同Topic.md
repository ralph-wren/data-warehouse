# 数据采集增强 - 同时订阅现货和合约写入不同 Topic

**时间**: 2026-03-24 17:32  
**类型**: 功能增强  
**状态**: ✅ 已完成

## 问题描述

原有的数据采集器只订阅了现货（SPOT）数据，需要增强为同时订阅现货和合约（SWAP/永续合约），并写入不同的 Kafka Topic，以便后续进行套利计算。

**用户需求**:
- 欧意交易所有现货和合约（永续合约），不是期货
- 需要同时订阅现货和合约
- 写入不同的 Kafka Topic
- 在 Flink 作业中进行 Join 计算套利机会

## 技术方案

### 1. OKX 交易对命名规则

**现货（SPOT）**:
- 格式：`BTC-USDT`
- 说明：直接交易，即时交割

**合约（SWAP/永续合约）**:
- 格式：`BTC-USDT-SWAP`
- 说明：永续合约，无交割日期

### 2. 数据流架构

```
OKX WebSocket
    │
    ├─→ 订阅现货: BTC-USDT
    │       ↓
    │   Kafka Topic: crypto-ticker-spot
    │       ↓
    │   Flink: 现货价格流
    │
    └─→ 订阅合约: BTC-USDT-SWAP
            ↓
        Kafka Topic: crypto-ticker-swap
            ↓
        Flink: 合约价格流
            ↓
        Interval Join ──→ 计算套利机会 ──→ Doris
```

**1.2 修改订阅逻辑**:
```java
/**
 * 订阅 Ticker 频道
 * 
 * 同时订阅现货和合约：
 * - 现货：BTC-USDT（SPOT）
 * - 合约：BTC-USDT-SWAP（永续合约）
 */
private void subscribe() {
    StringBuilder sb = new StringBuilder();
    sb.append("{\"op\":\"subscribe\",\"args\":[");
    
    int argCount = 0;
    
    // 订阅现货和合约
    for (String symbol : symbols) {
        // 现货：BTC-USDT
        if (argCount > 0) {
            sb.append(",");
        }
        sb.append("{\"channel\":\"tickers\",\"instId\":\"").append(symbol).append("\"}");
        argCount++;
        
        // 合约：BTC-USDT-SWAP
        sb.append(",");
        sb.append("{\"channel\":\"tickers\",\"instId\":\"").append(symbol).append("-SWAP\"}");
        argCount++;
    }
    
    sb.append("]}");
    
    logger.info("Subscribing to tickers (SPOT + SWAP): {}", sb.toString());
    logger.info("Total subscriptions: {} (SPOT: {}, SWAP: {})", 
        argCount, symbols.size(), symbols.size());
    
    send(sb.toString());
}
```

**1.3 修改数据处理逻辑**:
```java
/**
 * 处理 Ticker 数据
 * 
 * 根据 instId 判断是现货还是合约：
 * - 现货：BTC-USDT → 发送到 crypto-ticker-spot
 * - 合约：BTC-USDT-SWAP → 发送到 crypto-ticker-swap
 */
private void processTickerData(JsonNode dataNode) {
    try {
        TickerData tickerData = objectMapper.treeToValue(dataNode, TickerData.class);
        
        String instId = tickerData.getInstId();
        String jsonData = objectMapper.writeValueAsString(tickerData);
        
        // 判断是现货还是合约
        boolean isSwap = instId.endsWith("-SWAP");
        String topic = isSwap ? swapTopic : spotTopic;
        String type = isSwap ? "SWAP" : "SPOT";
        
        // 更新统计信息
        if (isSwap) {
            swapMessageCount.incrementAndGet();
        } else {
            spotMessageCount.incrementAndGet();
        }
        
        log.info("Processing ticker [{}]: {}, Price: {}", type, instId, tickerData.getLast());
        
        // 发送到 Kafka（指定 Topic）
        kafkaProducer.send(topic, instId, jsonData)
            .thenAccept(metadata -> {
                log.info("✓ Sent to Kafka [{}]: {} → Topic: {}, Partition: {}, Offset: {}", 
                    type, instId, topic, metadata.partition(), metadata.offset());
            })
            .exceptionally(ex -> {
                log.error("❌ Failed to send to Kafka [{}]: {}, Error: {}", 
                    type, instId, ex.getMessage());
                return null;
            });
        
        // 定期打印统计信息
        long count = messageCount.get();
        if (count % 100 == 0) {
            logger.info("Statistics - Total: {}, SPOT: {}, SWAP: {}, Errors: {}", 
                count, spotMessageCount.get(), swapMessageCount.get(), errorCount.get());
        }
        
    } catch (Exception e) {
        logger.error("Error processing ticker data: {}", e.getMessage(), e);
    }
}
```

### 2. 增强 KafkaProducerManager

**文件**: `src/main/java/com/crypto/dw/kafka/KafkaProducerManager.java`

**修改内容**:

添加重载方法支持指定 Topic：

```java
/**
 * 发送消息到 Kafka（使用默认 Topic）
 */
public CompletableFuture<RecordMetadata> send(String key, String value) {
    return send(topic, key, value);
}

/**
 * 发送消息到 Kafka（指定 Topic）
 * 
 * @param topic 目标 Topic
 * @param key 消息键（用于分区）
 * @param value 消息内容（JSON 格式）
 * @return CompletableFuture<RecordMetadata>
 */
public CompletableFuture<RecordMetadata> send(String topic, String key, String value) {
    CompletableFuture<RecordMetadata> future = new CompletableFuture<>();
    
    ProducerRecord<String, String> record = new ProducerRecord<>(topic, key, value);
    
    producer.send(record, (metadata, exception) -> {
        if (exception == null) {
            successCount.incrementAndGet();
            future.complete(metadata);
            
            if (logger.isDebugEnabled()) {
                logger.debug("Message sent successfully. Topic: {}, Key: {}, Partition: {}, Offset: {}", 
                    topic, key, metadata.partition(), metadata.offset());
            }
        } else {
            failureCount.incrementAndGet();
            future.completeExceptionally(exception);
            logger.error("Failed to send message. Topic: {}, Key: {}, Error: {}", 
                topic, key, exception.getMessage());
        }
    });
    
    return future;
}
```

DataStream<FuturesPrice> futuresStream = env.fromSource(
    swapKafkaSource,
    WatermarkStrategy.<String>forBoundedOutOfOrderness(Duration.ofSeconds(5)),
    "Swap Kafka Source"
)
.map(json -> {
    TickerData ticker = OBJECT_MAPPER.readValue(json, TickerData.class);
    FuturesPrice futures = new FuturesPrice();
    
    // 从 instId 中提取交易对（去掉 -SWAP 后缀）
    // 例如：BTC-USDT-SWAP → BTC-USDT

        futures.symbol = instId;

    futures.price = ticker.getLastPrice();  // 合约价格
    futures.timestamp = ticker.getTimestamp();
    return futures;
})
.filter(futures -> futures.price != null && futures.price.compareTo(BigDecimal.ZERO) > 0)
.name("Parse Swap Price");

## 技术要点

### 1. OKX 交易对命名

- **现货**: `BTC-USDT`（直接交易）
- **合约**: `BTC-USDT-SWAP`（永续合约）
- **判断方法**: `instId.endsWith("-SWAP")`

### 2. Kafka Topic 设计

- **现货 Topic**: `crypto-ticker-spot`
- **合约 Topic**: `crypto-ticker-swap`
- **优势**: 数据隔离，便于独立消费和处理

### 3. 数据流关联

- **Interval Join**: 时间窗口内关联（±10 秒）
- **Key**: 交易对符号（去掉 -SWAP 后缀）
- **优势**: 状态小，性能好

### 4. 套利计算

- **价差**: `spread = swap_price - spot_price`
- **价差率**: `spread_rate = spread / spot_price * 100`
- **套利方向**:
  - 合约价格 > 现货价格：做空合约/做多现货
  - 现货价格 > 合约价格：做多合约/做空现货
- **利润估算**: `profit = |spread| - fee * 2`（扣除双边手续费）

### 5. 性能优化

- **并行订阅**: 同时订阅现货和合约，提高数据采集效率
- **Topic 隔离**: 不同 Topic 可以独立扩展分区
- **Interval Join**: 减少状态存储，提高性能
- **广播流**: 避免频繁查询 Redis
