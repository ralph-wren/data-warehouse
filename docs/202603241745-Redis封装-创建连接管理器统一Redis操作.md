# Redis 封装 - 创建连接管理器统一 Redis 操作

**时间**: 2026-03-24 17:45  
**类型**: 代码重构  
**状态**: ✅ 已完成

## 问题描述

在 `FlinkADSArbitrageJob` 中直接使用 `JedisPool` 和 `Jedis` 进行 Redis 操作，代码重复且不便于维护。需要创建一个统一的 Redis 连接管理器，封装常用的 Redis 操作。

**存在的问题**:
1. Redis 连接池配置代码重复
2. 每次操作都需要手动获取和释放连接
3. 异常处理分散在各处
4. 缺少统一的日志记录
5. 不便于复用和测试

## 解决方案

创建 `RedisConnectionManager` 类，封装 Redis 连接池管理和常用操作。

### 设计原则

1. **单一职责**: 只负责 Redis 连接和操作
2. **自动管理**: 自动获取和释放连接
3. **异常处理**: 统一的异常处理和日志记录
4. **易于使用**: 提供简洁的 API
5. **性能优化**: 使用连接池，避免频繁创建连接

```java
public class RedisConnectionManager implements Serializable {
    
    private transient JedisPool jedisPool;
    private final String host;
    private final int port;
    private final int timeout;
    private final String password;
    
    /**
     * 构造函数（从配置加载）
     */
    public RedisConnectionManager(ConfigLoader config) {
        this.host = config.getString("redis.host", "localhost");
        this.port = config.getInt("redis.port", 6379);
        this.timeout = config.getInt("redis.timeout", 2000);
        this.password = config.getString("redis.password", null);
        
        initPool();
    }
    
    /**
     * 初始化连接池
     */
    private void initPool() {
        JedisPoolConfig poolConfig = new JedisPoolConfig();
        poolConfig.setMaxTotal(10);        // 最大连接数
        poolConfig.setMaxIdle(5);          // 最大空闲连接数
        poolConfig.setMinIdle(1);          // 最小空闲连接数
        poolConfig.setTestOnBorrow(true);  // 获取连接时测试
        poolConfig.setTestOnReturn(true);  // 归还连接时测试
        poolConfig.setTestWhileIdle(true); // 空闲时测试
        poolConfig.setMaxWaitMillis(3000); // 最大等待时间
        
        if (password != null && !password.isEmpty()) {
            jedisPool = new JedisPool(poolConfig, host, port, timeout, password);
        } else {
            jedisPool = new JedisPool(poolConfig, host, port, timeout);
        }
        
        // 测试连接
        try (Jedis jedis = jedisPool.getResource()) {
            String pong = jedis.ping();
            logger.info("✓ Redis 连接测试成功: {}", pong);
        }
    }
}
```

```java
public static class RedisWhitelistSource ... {
    private transient RedisConnectionManager redisManager;
    
    @Override
    public void open(Configuration parameters) {
        // 使用 RedisConnectionManager
        redisManager = new RedisConnectionManager(config);
        
        // 测试连接
        if (redisManager.testConnection()) {
            logger.info("✓ Redis 连接测试成功");
        } else {
            logger.warn("⚠ Redis 连接测试失败，将使用默认白名单");
        }
    }
    
    private Set<String> fetchWhitelistFromRedis() {
        // 使用封装的方法
        Set<String> whitelist = redisManager.getSet("crypto:whitelist");
        
        if (whitelist == null || whitelist.isEmpty()) {
            logger.warn("Redis 中没有白名单数据，使用默认白名单");
            whitelist = getDefaultWhitelist();
        }
        
        return whitelist;
    }
    
    @Override
    public void close() {
        if (redisManager != null) {
            logger.info("关闭 Redis 连接管理器...");
            logger.info(redisManager.getPoolStatus());
            redisManager.close();
        }
    }
}
```

## 技术要点

### 1. 连接池配置

**参数说明**:
- `maxTotal`: 最大连接数（10）
- `maxIdle`: 最大空闲连接数（5）
- `minIdle`: 最小空闲连接数（1）
- `testOnBorrow`: 获取连接时测试（true）
- `testOnReturn`: 归还连接时测试（true）
- `testWhileIdle`: 空闲时测试（true）
- `maxWaitMillis`: 最大等待时间（3000ms）

**优化建议**:
- 开发环境：maxTotal=10, maxIdle=5
- 生产环境：maxTotal=50, maxIdle=20
- 根据并发量调整参数

### 2. 自动资源管理

**try-with-resources**:
```java
// 自动释放连接
try (Jedis jedis = getConnection()) {
    return jedis.get(key);
}
// 连接自动归还到连接池
```

**优势**:
- 自动释放连接，防止连接泄漏
- 代码简洁，易于维护
- 异常安全

### 3. 异常处理

**统一处理**:
```java
public String get(String key) {
```

**优势**:

## 常见问题

### 1. 连接超时

**问题**: `JedisConnectionException: Could not get a resource from the pool`

**原因**: 
- Redis 服务未启动
- 网络不通
- 连接池耗尽

**解决方法**:
```bash
# 检查 Redis 是否运行
redis-cli ping

# 检查网络连接
telnet localhost 6379

# 增加连接池大小
poolConfig.setMaxTotal(20);
```

### 2. 连接泄漏

**问题**: 连接池耗尽，无法获取新连接

**原因**: 没有正确释放连接

**解决方法**:
```java
// ❌ 错误：没有释放连接
Jedis jedis = redisManager.getConnection();
jedis.get("key");
// 忘记调用 jedis.close()

// ✅ 正确：使用 try-with-resources
try (Jedis jedis = redisManager.getConnection()) {
    jedis.get("key");
}  // 自动释放连接
```

### 3. 密码认证失败

**问题**: `JedisDataException: NOAUTH Authentication required`

**原因**: Redis 配置了密码，但没有提供

**解决方法**:
```yaml
# 配置文件中添加密码
redis:
  password: your_redis_password
```
