# RocksDB 依赖 - 解决状态后端工厂类找不到错误

## 问题描述

运行 Flink 作业时遇到以下错误:

```
Exception in thread "main" org.apache.flink.util.WrappingRuntimeException: 
org.apache.flink.util.DynamicCodeLoadingException: 
Cannot find configured state backend factory class: rocksdb
```

## 问题原因

配置文件 `application-dev.yml` 中指定了 `state.backend: rocksdb`,但 `pom.xml` 中缺少 RocksDB 状态后端依赖。

Flink 在启动时无法找到 RocksDB 状态后端的工厂类,导致作业启动失败。

## 解决方案

### 1. 添加 RocksDB 依赖

在 `pom.xml` 中添加 RocksDB 状态后端依赖:

```xml
<!-- Flink RocksDB State Backend - 生产环境推荐的状态后端 -->
<!-- 支持增量 checkpoint、大状态存储、本地恢复等特性 -->
<dependency>
    <groupId>org.apache.flink</groupId>
    <artifactId>flink-statebackend-rocksdb</artifactId>
    <version>${flink.version}</version>
</dependency>
```

### 2. 配置状态后端

在 `application-dev.yml` 中配置 RocksDB 状态后端:

```yaml
flink:
  # 状态后端配置
  state:
    backend: rocksdb  # 状态后端类型：rocksdb (生产推荐) / hashmap (开发测试)
    checkpoint-dir: file:///tmp/flink-checkpoints  # Checkpoint 存储目录
    savepoint-dir: file:///tmp/flink-savepoints  # Savepoint 存储目录
    incremental: true  # RocksDB 增量 Checkpoint，减少状态快照体积
    local-recovery: true  # 启用本地恢复，加快失败恢复速度
```

### 3. 更新 FlinkEnvironmentFactory

在 `FlinkEnvironmentFactory.java` 中添加状态后端配置逻辑:

```java
/**
 * 配置状态后端
 *
 * 说明：
 * 1. 从项目配置文件读取 flink.state.* 配置
 * 2. 转换为 Flink 1.17 官方配置项
 * 3. 避免配置文件中声明了状态后端，但运行时没有真正生效
 *
 * @param flinkConfig Flink 配置
 */
private void configureStateBackend(Configuration flinkConfig) {
    // 从 YAML 读取状态后端类型，例如 rocksdb / hashmap
    String configuredBackend = config.getString("flink.state.backend", "");
    String normalizedBackend = normalizeStateBackend(configuredBackend);
    if (!normalizedBackend.isEmpty()) {
        flinkConfig.setString("state.backend.type", normalizedBackend);
    }

    // 从 YAML 读取 checkpoint 存储目录
    String checkpointDir = config.getString("flink.state.checkpoint-dir", "").trim();
    if (!checkpointDir.isEmpty()) {
        flinkConfig.setString("state.checkpoints.dir", checkpointDir);
    }

    // 从 YAML 读取 savepoint 存储目录
    String savepointDir = config.getString("flink.state.savepoint-dir", "").trim();
    if (!savepointDir.isEmpty()) {
        flinkConfig.setString("state.savepoints.dir", savepointDir);
    }

    // 可选：RocksDB 增量 checkpoint
    if (config.get("flink.state.incremental") != null) {
        flinkConfig.setBoolean(
            "state.backend.incremental",
            config.getBoolean("flink.state.incremental", false)
        );
    }

    // 可选：本地恢复
    if (config.get("flink.state.local-recovery") != null) {
        flinkConfig.setBoolean(
            "state.backend.local-recovery",
            config.getBoolean("flink.state.local-recovery", false)
        );
    }
}
```

### 4. 重新编译和打包

```bash
# 重新编译项目
mvn clean compile -DskipTests

# 重新打包项目
mvn clean package -DskipTests
```

## RocksDB 状态后端优势

### 1. 支持大状态存储
- RocksDB 将状态存储在本地磁盘,不受 JVM 堆内存限制
- 适合状态数据量超过几 GB 的场景
- HashMap 状态后端将状态存储在 JVM 堆内存,受内存限制

### 2. 支持增量 Checkpoint
- 只保存自上次 Checkpoint 以来的增量变化
- 大幅减少 Checkpoint 时间和存储空间
- HashMap 状态后端只支持全量 Checkpoint

### 3. 支持本地恢复
- 失败恢复时优先从本地磁盘恢复状态
- 加快恢复速度,减少对远程存储的依赖
- 提高作业可用性

### 4. 生产环境推荐
- Flink 官方推荐在生产环境使用 RocksDB 状态后端
- 更稳定、更可靠、更高效
- 支持更多高级特性

## 状态后端对比

| 特性 | RocksDB | HashMap |
|------|---------|---------|
| 存储位置 | 本地磁盘 | JVM 堆内存 |
| 状态大小限制 | 磁盘容量 | 堆内存大小 |
| Checkpoint 类型 | 增量/全量 | 全量 |
| 本地恢复 | 支持 | 不支持 |
| 性能 | 中等 | 高 |
| 适用场景 | 生产环境、大状态 | 开发测试、小状态 |

## 注意事项

### 1. 依赖 scope

RocksDB 依赖的 scope 不要设置为 `provided`,应该打包到 JAR 中:

```xml
<!-- 正确 -->
<dependency>
    <groupId>org.apache.flink</groupId>
    <artifactId>flink-statebackend-rocksdb</artifactId>
    <version>${flink.version}</version>
</dependency>

<!-- 错误 -->
<dependency>
    <groupId>org.apache.flink</groupId>
    <artifactId>flink-statebackend-rocksdb</artifactId>
    <version>${flink.version}</version>
    <scope>provided</scope>  <!-- 不要设置为 provided -->
</dependency>
```

### 2. 版本匹配

RocksDB 依赖版本必须与 Flink 版本匹配:

- Flink 1.17.x → flink-statebackend-rocksdb:1.17.x
- Flink 1.18.x → flink-statebackend-rocksdb:1.18.x

### 3. 存储目录

- 开发环境可以使用本地文件系统: `file:///tmp/flink-checkpoints`
- 生产环境建议使用分布式文件系统: `hdfs://namenode:9000/flink/checkpoints`

### 4. 增量 Checkpoint

- 增量 Checkpoint 只对 RocksDB 状态后端有效
- HashMap 状态后端不支持增量 Checkpoint
- 建议在生产环境开启增量 Checkpoint

## 相关配置

### Checkpoint 配置

```yaml
flink:
  checkpoint:
    interval: 60000  # 60 秒
    mode: EXACTLY_ONCE
    timeout: 300000  # 5 分钟
    min-pause: 10000  # 10 秒
    max-concurrent: 1
    tolerable-failed-checkpoints: 3
    externalized-retention: DELETE_ON_CANCELLATION
    snapshot-compression: true  # 建议开启，减少快照体积
    checkpoints-after-tasks-finish: true
```

### 重启策略配置

```yaml
flink:
  restart:
    strategy: fixed-delay  # 开发环境更适合固定延迟重启
    fixed-delay:
      attempts: 3  # 最多重试 3 次
      delay-ms: 10000  # 每次重试前等待 10 秒
```

## 总结

通过添加 RocksDB 状态后端依赖,成功解决了 "Cannot find configured state backend factory class: rocksdb" 错误。

RocksDB 状态后端是 Flink 生产环境推荐的状态后端,支持大状态存储、增量 Checkpoint、本地恢复等高级特性,能够显著提升作业的稳定性和可靠性。
