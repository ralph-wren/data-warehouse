# Flink SQL 解析 - 移除 DDL 模板文件注释解决解析错误

## 问题描述

Flink 作业启动时报 SQL 解析错误:
```
org.apache.flink.table.api.SqlParserException: SQL parse failed. 
Non-query expression encountered in illegal context
at org.apache.flink.table.planner.parse.CalciteParser.parseSqlList(CalciteParser.java:82)
```

## 问题现象

1. **SQL 解析失败**: Flink Table API 无法解析 DDL 语句
2. **错误位置**: `FlinkODSJobSQL.java` 第 50 行 `tableEnv.executeSql(dorisSinkDDL)`
3. **错误信息**: "Non-query expression encountered in illegal context"

## 根本原因

**SQL 模板文件包含注释导致解析错误** ⭐⭐⭐

1. **SQL 模板文件包含注释**:
   ```sql
   -- Doris Sink 表 DDL 模板
   -- 功能：创建 Doris Sink 表，用于向 Doris 写入数据
   -- 参数说明：...
   
   CREATE TABLE ${tableName} (
   ...
   ```

2. **Flink SQL Parser 的限制**:
   - `executeSql()` 方法只能执行单条 SQL 语句
   - SQL 注释（`--` 开头）会被当作独立的语句
   - 导致解析器认为遇到了多条语句

3. **错误触发**:
   - 解析器先遇到注释行
   - 然后遇到 CREATE TABLE 语句
   - 认为这是在非法上下文中的非查询表达式

## 解决方案

**移除 SQL 模板文件中的注释** ✅

## 技术细节

### Flink SQL Parser 的限制

**1. executeSql() 只能执行单条语句**:
```java
// ✅ 正确 - 单条 CREATE TABLE 语句
tableEnv.executeSql("CREATE TABLE my_table (id INT) WITH (...)");

// ❌ 错误 - 包含注释
tableEnv.executeSql("-- 注释\nCREATE TABLE my_table (id INT) WITH (...)");

// ❌ 错误 - 多条语句
tableEnv.executeSql("CREATE TABLE t1 (...); CREATE TABLE t2 (...)");
```

**2. 注释的处理**:
- SQL 注释（`--` 开头）会被解析器识别
- 但 `executeSql()` 不支持包含注释的 SQL
- 需要在传入前移除注释

**3. 多语句执行**:
```java
// 如果需要执行多条语句，使用 executeSql() 多次调用
tableEnv.executeSql("CREATE TABLE t1 (...)");
tableEnv.executeSql("CREATE TABLE t2 (...)");

// 或者使用 sqlQuery() 和 executeInsert()
tableEnv.sqlQuery("SELECT * FROM t1").executeInsert("t2");
```

### SQL 模板最佳实践

**1. 不要在模板文件中添加注释**:
```sql
-- ❌ 错误 - 注释会导致解析错误
-- 这是一个表
CREATE TABLE my_table (...)

-- ✅ 正确 - 直接写 SQL
CREATE TABLE my_table (...)
```

**2. 在 Java 代码中添加注释**:
```java
// ✅ 在代码中添加注释说明
// 创建 Kafka Source 表，用于从 Kafka 读取数据
String kafkaSourceDDL = tableFactory.createKafkaSourceTable(...);
tableEnv.executeSql(kafkaSourceDDL);
```

**3. 使用 README 文件说明模板**:
```
sql/flink/ddl/
├── README.md          # 说明各个模板的用途和参数
├── kafka_source.sql   # Kafka Source 表模板（无注释）
├── doris_source.sql   # Doris Source 表模板（无注释）
└── doris_sink.sql     # Doris Sink 表模板（无注释）
```
