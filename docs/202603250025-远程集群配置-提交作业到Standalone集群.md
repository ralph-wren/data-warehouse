# 远程集群配置 - 提交作业到 Standalone 集群

## 问题描述

需要将 Flink 作业提交到本地的 Standalone 集群 (localhost:8081),而不是使用内嵌的本地模式运行。

## 问题背景

当前代码使用 `StreamExecutionEnvironment.getExecutionEnvironment()` 创建本地模式环境,作业在 JVM 进程内运行。要提交到 Standalone 集群,需要使用 `StreamExecutionEnvironment.createRemoteEnvironment()` 创建远程环境。

## 解决方案

### 1. 添加集群模式配置

在 `application-dev.yml` 中添加集群配置:

```yaml
# Flink 配置
flink:
  # 集群配置
  cluster:
    mode: local  # 运行模式: local（本地模式）, remote（远程集群）
    remote:
      host: localhost  # 远程集群 JobManager 地址
      port: 8081  # 远程集群 JobManager 端口
      jar-path: target/realtime-crypto-datawarehouse-1.0.0.jar  # JAR 包路径
```

**配置说明**:
- `mode`: 运行模式
  - `local`: 本地模式,在当前 JVM 进程中运行
  - `remote`: 远程模式,提交到已存在的 Flink 集群
- `remote.host`: JobManager 的主机地址
- `remote.port`: JobManager 的 REST API 端口(默认 8081)
- `remote.jar-path`: 作业 JAR 包的路径(相对于项目根目录)

### 2. 修改 FlinkEnvironmentFactory

在 `FlinkEnvironmentFactory.java` 中添加远程集群支持:

**修改 createStreamEnvironment 方法**:

```java
public StreamExecutionEnvironment createStreamEnvironment(String jobName, int webUIPort) {
    logger.info("==========================================");
    logger.info("创建 Flink Stream Environment");
    logger.info("作业名称: {}", jobName);
    logger.info("==========================================");
    
    // 读取集群模式配置
    String clusterMode = config.getString("flink.cluster.mode", "local");
    logger.info("集群模式: {}", clusterMode);
    
    StreamExecutionEnvironment env;
    
    if ("remote".equalsIgnoreCase(clusterMode)) {
        // 远程集群模式
        String remoteHost = config.getString("flink.cluster.remote.host", "localhost");
        int remotePort = config.getInt("flink.cluster.remote.port", 8081);
        String jarPath = config.getString("flink.cluster.remote.jar-path", 
            "target/realtime-crypto-datawarehouse-1.0.0.jar");
        
        logger.info("连接到远程 Flink 集群:");
        logger.info("  JobManager: {}:{}", remoteHost, remotePort);
        logger.info("  JAR 路径: {}", jarPath);
        
        // 创建 Flink 配置
        Configuration flinkConfig = createFlinkConfiguration(jobName, webUIPort);
        
        // 创建远程执行环境
        env = StreamExecutionEnvironment.createRemoteEnvironment(
            remoteHost,
            remotePort,
            flinkConfig,
            jarPath
        );
    } else {
        // 本地模式
        logger.info("使用本地模式");
        
        // 创建 Flink 配置
        Configuration flinkConfig = createFlinkConfiguration(jobName, webUIPort);
        
        // 创建本地执行环境
        env = StreamExecutionEnvironment.getExecutionEnvironment(flinkConfig);
    }
    
    // 统一应用执行配置
    int parallelism = configureExecutionEnvironment(env);
    long checkpointInterval = configureCheckpointEnvironment(env);
    
    // 打印环境信息
    logEnvironmentInfo(parallelism, checkpointInterval, webUIPort);
    
    return env;
}
```

**同样修改 createTableEnvironment 方法**,添加相同的远程集群支持逻辑。

### 3. 使用方法

#### 方法一: 修改配置文件(推荐)

1. **修改 application-dev.yml**:
```yaml
flink:
  cluster:
    mode: remote  # 改为 remote
    remote:
      host: localhost
      port: 8081
      jar-path: target/realtime-crypto-datawarehouse-1.0.0.jar
```

2. **打包项目**:
```bash
mvn clean package -DskipTests
```

3. **运行作业**:
```bash
java -cp target/realtime-crypto-datawarehouse-1.0.0.jar \
    com.crypto.dw.jobs.FlinkODSJobDataStream \
    --APP_ENV dev
```

#### 方法二: 使用提交脚本

创建 `run-flink-ods-remote.sh` 脚本:

```bash
#!/bin/bash

echo "=========================================="
echo "Flink ODS Job - 提交到远程集群"
echo "=========================================="

# 1. 打包项目
echo "步骤 1: 打包项目..."
mvn clean package -DskipTests

if [ $? -ne 0 ]; then
    echo "❌ 打包失败"
    exit 1
fi

echo "✅ 打包成功"

# 2. 检查 JAR 文件
JAR_FILE="target/realtime-crypto-datawarehouse-1.0.0.jar"
if [ ! -f "$JAR_FILE" ]; then
    echo "❌ JAR 文件不存在: $JAR_FILE"
    exit 1
fi

echo "✅ JAR 文件存在: $JAR_FILE"

# 3. 运行作业
echo "步骤 2: 提交作业到远程集群..."
java -cp "$JAR_FILE" \
    com.crypto.dw.jobs.FlinkODSJobDataStream \
    --APP_ENV dev

echo "=========================================="
echo "作业已提交到远程集群"
echo "查看作业状态: http://localhost:8081"
echo "=========================================="
```

运行脚本:
```bash
bash run-flink-ods-remote.sh
```

#### 方法三: 使用 Flink CLI(传统方式)

```bash
# 1. 打包项目
mvn clean package -DskipTests

# 2. 使用 flink run 命令提交
flink run \
    -c com.crypto.dw.jobs.FlinkODSJobDataStream \
    target/realtime-crypto-datawarehouse-1.0.0.jar \
    --APP_ENV dev
```

## 配置对比

### 本地模式 vs 远程模式

| 特性 | 本地模式 (local) | 远程模式 (remote) |
|-----|----------------|------------------|
| 运行位置 | 当前 JVM 进程 | Flink 集群 |
| Web UI | 内嵌 Web UI | 集群 Web UI |
| 资源管理 | 本地资源 | 集群资源 |
| 适用场景 | 开发调试 | 生产部署 |
| 并行度 | 受限于本地资源 | 可扩展 |
| 高可用 | 不支持 | 支持 |

### 配置示例

**本地模式配置**:
```yaml
flink:
  cluster:
    mode: local
  execution:
    parallelism: 4
```

**远程模式配置**:
```yaml
flink:
  cluster:
    mode: remote
    remote:
      host: localhost
      port: 8081
      jar-path: target/realtime-crypto-datawarehouse-1.0.0.jar
  execution:
    parallelism: 4  # 集群会根据可用资源调整
```

## 常见问题

### 问题 1: 连接被拒绝

**错误信息**:
```
java.net.ConnectException: Connection refused: localhost/127.0.0.1:8081
```

**原因**: Flink 集群未启动或端口配置错误

**解决方法**:
```bash
# 检查集群是否运行
docker ps | grep flink

# 检查端口是否监听
netstat -an | grep 8081

# 重启集群
docker-compose -f docker-compose-flink.yml restart
```

### 问题 2: JAR 文件找不到

**错误信息**:
```
java.io.FileNotFoundException: target/realtime-crypto-datawarehouse-1.0.0.jar
```

**原因**: JAR 文件路径配置错误或未打包

**解决方法**:
```bash
# 检查 JAR 文件是否存在
ls -lh target/realtime-crypto-datawarehouse-1.0.0.jar

# 重新打包
mvn clean package -DskipTests

# 使用绝对路径
jar-path: /absolute/path/to/target/realtime-crypto-datawarehouse-1.0.0.jar
```

### 问题 3: 类找不到

**错误信息**:
```
java.lang.ClassNotFoundException: com.crypto.dw.jobs.FlinkODSJobDataStream
```

**原因**: 
- JAR 包不完整
- 依赖缺失

**解决方法**:
```bash
# 检查 JAR 包内容
jar tf target/realtime-crypto-datawarehouse-1.0.0.jar | grep FlinkODSJobDataStream

# 使用 maven-shade-plugin 打包所有依赖
mvn clean package -DskipTests

# 检查 pom.xml 中是否配置了 maven-shade-plugin
```

### 问题 4: 配置文件读取失败

**错误信息**:
```
Failed to load configuration file: config/application-dev.yml
```

**原因**: 配置文件未打包到 JAR 中

**解决方法**:
```bash
# 检查配置文件是否在 JAR 中
jar tf target/realtime-crypto-datawarehouse-1.0.0.jar | grep application-dev.yml

# 确保 pom.xml 中配置了资源打包
<resources>
    <resource>
        <directory>src/main/resources</directory>
        <includes>
            <include>**/*.yml</include>
            <include>**/*.properties</include>
        </includes>
    </resource>
</resources>
```

## 技术要点

### 1. createRemoteEnvironment 方法

```java
StreamExecutionEnvironment.createRemoteEnvironment(
    String host,           // JobManager 主机地址
    int port,              // JobManager 端口
    Configuration config,  // Flink 配置
    String... jarFiles     // 作业 JAR 文件路径
)
```

**关键参数**:
- `host`: JobManager 的 REST API 地址
- `port`: JobManager 的 REST API 端口(默认 8081)
- `config`: Flink 配置对象,包含并行度、Checkpoint 等配置
- `jarFiles`: 作业 JAR 文件路径,可以是多个

### 2. JAR 文件路径

JAR 文件路径可以是:
- **相对路径**: `target/realtime-crypto-datawarehouse-1.0.0.jar`
  - 相对于当前工作目录
  - 适合开发环境
  
- **绝对路径**: `/home/user/project/target/realtime-crypto-datawarehouse-1.0.0.jar`
  - 完整的文件系统路径
  - 适合生产环境

### 3. 配置优先级

配置加载顺序:
1. 代码中的配置 (Configuration 对象)
2. 配置文件 (application-dev.yml)
3. 环境变量
4. 系统属性

### 4. 远程模式限制

远程模式的一些限制:
- 不支持内嵌 Web UI (使用集群的 Web UI)
- 需要确保 JAR 文件包含所有依赖
- 需要确保集群可以访问外部资源(Kafka、Doris 等)

## 最佳实践

### 1. 开发环境

**推荐配置**:
```yaml
flink:
  cluster:
    mode: local  # 使用本地模式,方便调试
  execution:
    parallelism: 2  # 较低并行度,节省资源
```

**优点**:
- 快速启动
- 方便调试
- 不需要额外的集群

### 2. 测试环境

**推荐配置**:
```yaml
flink:
  cluster:
    mode: remote  # 使用远程集群
    remote:
      host: localhost
      port: 8081
  execution:
    parallelism: 4  # 中等并行度
```

**优点**:
- 接近生产环境
- 可以测试集群功能
- 资源隔离

### 3. 生产环境

**推荐配置**:
```yaml
flink:
  cluster:
    mode: remote
    remote:
      host: flink-jobmanager  # 使用服务名称
      port: 8081
  execution:
    parallelism: 8  # 高并行度
```

**优点**:
- 高可用
- 可扩展
- 资源管理
