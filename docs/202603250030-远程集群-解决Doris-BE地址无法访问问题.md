# 远程集群 - 解决 Doris BE 地址无法访问问题

## 问题描述

将 Flink 作业提交到远程 Standalone 集群后,出现以下错误:

```
org.apache.doris.flink.exception.DorisRuntimeException: no available backend.
at org.apache.doris.flink.sink.writer.DorisWriter.initializeLoad
```

## 问题现象

- 作业提交成功,但在初始化 Doris Sink 时失败
- 错误信息: "no available backend"
- TaskManager 无法连接到 Doris BE 节点

## 根本原因

### 原因 1: 无效的 BE 节点注册 ⭐⭐⭐

Doris FE 中注册了多个 BE 节点,但只有一个是正常的:

```
BackendId: 1773924732414, Host: 172.25.0.3, Alive: true   ✅ 正常
BackendId: 1774106122264, Host: host.docker.internal, Alive: false  ❌ 无效
BackendId: 1774106122265, Host: 127.0.0.1, Alive: false  ❌ 无效
```

当 Flink Connector 尝试连接 BE 时,可能会选到这些无效的节点,导致连接失败。

### 原因 2: BE 地址配置错误 ⭐⭐⭐

配置文件中的 BE 地址是 `127.0.0.1:8040`,这对于运行在 Docker 容器内的 TaskManager 来说是无效的:

```yaml
# ❌ 错误配置 (本地模式)
doris:
  be:
    nodes: "127.0.0.1:8040"  # TaskManager 在容器内无法访问
```

**问题说明**:
- **本地模式**: Flink 作业在宿主机 JVM 进程中运行,可以访问 `127.0.0.1`
- **远程集群模式**: TaskManager 在 Docker 容器内运行,`127.0.0.1` 指向容器自己,无法访问宿主机的 Doris

### 原因 3: Docker 网络配置

虽然 TaskManager 已经连接到 `doris-net` 网络,但配置文件中没有使用容器名称,导致无法利用 Docker 的 DNS 解析。

## 解决方案

### 方案 1: 清理无效的 BE 节点 ✅

使用 `DROPP` 命令(双P)强制删除无效的 BE 节点:

```bash
# 删除 127.0.0.1 节点
docker exec doris-fe mysql -h 127.0.0.1 -P 9030 -u root \
  -e "ALTER SYSTEM DROPP BACKEND '127.0.0.1:9050';"

# 删除 host.docker.internal 节点
docker exec doris-fe mysql -h 127.0.0.1 -P 9030 -u root \
  -e "ALTER SYSTEM DROPP BACKEND 'host.docker.internal:9050';"
```

**验证结果**:
```bash
docker exec doris-fe mysql -h 127.0.0.1 -P 9030 -u root \
  -e "SHOW BACKENDS\G" | grep -E "(BackendId|Host|Alive)"

# 应该只剩一个节点:
# BackendId: 1773924732414
# Host: 172.25.0.3
# Alive: true
```

### 方案 2: 修改 BE 地址配置 ✅ (推荐)

修改 `application-dev.yml`,使用 Docker 容器名称:

```yaml
# ✅ 正确配置 (远程集群模式)
doris:
  # FE 节点配置 (宿主机访问)
  fe:
    http-url: http://127.0.0.1:8030
    jdbc-url: jdbc:mysql://127.0.0.1:9030
    username: root
    password: ""
  
  # BE 节点配置 (容器访问)
  # 重要说明:
  # - 本地模式: 使用 127.0.0.1:8040 (宿主机访问)
  # - 远程集群模式: 使用 doris-be:8040 (Docker 容器访问)
  be:
    nodes: "doris-be:8040"  # Docker 容器名称
```

**为什么使用容器名称?**
- Docker 提供内置 DNS 解析
- 容器名称会自动解析为容器的 IP 地址
- 不受容器重启影响(IP 可能变化,但名称不变)

### 方案 3: 验证网络连通性 ✅

确保 TaskManager 可以访问 Doris BE:

```bash
# 1. 检查 TaskManager 是否在 doris-net 网络中
docker network inspect data-warehouse_doris-net \
  --format '{{range .Containers}}{{.Name}} {{end}}' | grep flink-taskmanager

# 2. 测试 TaskManager 访问 Doris BE
docker exec flink-taskmanager-1 curl -s http://doris-be:8040/api/health

# 3. 查看网络中的所有容器
docker network inspect data-warehouse_doris-net \
  --format '{{range .Containers}}{{.Name}}: {{.IPv4Address}}{{"\n"}}{{end}}'
```

## 技术要点

### 1. Docker 网络模式

**桥接网络 (Bridge Network)**:
- 容器之间可以通过容器名称互相访问
- Docker 提供内置 DNS 解析
- 适合多容器应用

**网络配置**:
```yaml
# docker-compose-flink.yml
services:
  flink-jobmanager:
    networks:
      - flink-net
      - doris-net  # 连接到 doris-net
  
  flink-taskmanager:
    networks:
      - flink-net
      - doris-net  # 连接到 doris-net

networks:
  doris-net:
    external: true
    name: data-warehouse_doris-net
```

### 2. Doris BE 地址解析

**Doris Flink Connector 的 BE 地址解析流程**:

1. **从配置读取 BE 地址**:
   - 如果配置了 `benodes`,直接使用
   - 否则,从 FE 查询 BE 列表

2. **选择可用的 BE**:
   - 遍历 BE 列表
   - 测试连接
   - 选择第一个可用的 BE

3. **连接 BE**:
   - 使用 HTTP 协议连接 BE 的 8040 端口
   - 执行 Stream Load 操作

**配置优先级**:
```java
// 1. 配置文件中的 benodes (最高优先级)
'benodes' = 'doris-be:8040'

// 2. 从 FE 查询 BE 列表 (如果没有配置 benodes)
SHOW BACKENDS;
```

### 3. 本地模式 vs 远程集群模式

| 特性 | 本地模式 | 远程集群模式 |
|-----|---------|-------------|
| Flink 运行位置 | 宿主机 JVM | Docker 容器 |
| BE 地址配置 | 127.0.0.1:8040 | doris-be:8040 |
| 网络访问 | 宿主机网络 | Docker 网络 |
| DNS 解析 | 系统 DNS | Docker DNS |

### 4. DROPP vs DROP 命令

**DROP 命令** (单P):
- 安全删除,会检查数据
- 如果 BE 上有数据,会拒绝删除
- 推荐用于生产环境

**DROPP 命令** (双P):
- 强制删除,不检查数据
- 会丢失 BE 上的所有数据
- 适合清理无效节点

```sql
-- 安全删除 (推荐)
ALTER SYSTEM DROP BACKEND '127.0.0.1:9050';

-- 强制删除 (清理无效节点)
ALTER SYSTEM DROPP BACKEND '127.0.0.1:9050';
```

## 常见问题

### 问题 1: 为什么会有多个 BE 节点?

**原因**:
- 之前尝试过不同的 BE 地址配置
- 每次尝试都会在 FE 中注册一个新的 BE 节点
- 旧的节点没有被清理

**预防方法**:
- 使用正确的 BE 地址配置
- 定期检查和清理无效节点
- 使用 Docker 容器名称而不是 IP 地址

### 问题 2: 如何判断 BE 节点是否有效?

**检查方法**:
```bash
# 查看 BE 节点状态
docker exec doris-fe mysql -h 127.0.0.1 -P 9030 -u root \
  -e "SHOW BACKENDS\G"

# 关键字段:
# - Alive: true/false (是否存活)
# - LastHeartbeat: 最后心跳时间
# - TabletNum: Tablet 数量 (有数据的节点 > 0)
```

**有效节点特征**:
- `Alive: true`
- `LastHeartbeat` 是最近的时间
- `TabletNum > 0` (如果有数据)
- `ErrMsg` 为空

**无效节点特征**:
- `Alive: false`
- `LastHeartbeat: NULL`
- `TabletNum: 0`
- `ErrMsg` 有错误信息

### 问题 3: 本地模式和远程模式如何切换?

**方法 1: 使用不同的配置文件**

创建两个配置文件:
- `application-local.yml`: 本地模式配置
- `application-remote.yml`: 远程集群模式配置

```bash
# 本地模式
java -cp target/xxx.jar com.crypto.dw.flink.FlinkODSJobDataStream --APP_ENV local

# 远程模式
java -cp target/xxx.jar com.crypto.dw.flink.FlinkODSJobDataStream --APP_ENV remote
```

**方法 2: 使用环境变量覆盖**

```bash
# 本地模式
export DORIS_BE_NODES="127.0.0.1:8040"
java -cp target/xxx.jar com.crypto.dw.flink.FlinkODSJobDataStream

# 远程模式
export DORIS_BE_NODES="doris-be:8040"
java -cp target/xxx.jar com.crypto.dw.flink.FlinkODSJobDataStream
```

**方法 3: 动态检测 (推荐)**

在代码中检测运行环境:

```java
String clusterMode = config.getString("flink.cluster.mode", "local");
String beNodes;

if ("remote".equalsIgnoreCase(clusterMode)) {
    // 远程集群模式,使用容器名称
    beNodes = "doris-be:8040";
} else {
    // 本地模式,使用宿主机地址
    beNodes = "127.0.0.1:8040";
}
```
