## 配置验证

### 1. 算子链配置

**配置文件**: `src/main/resources/config/application-dev.yml`

```yaml
flink:
  execution:
    operator-chaining: false  # 禁用算子链,Web UI 显示详细算子
```

**验证方法**:
- 启动作业后,访问 Flink Web UI
- 查看作业的算子拓扑图
- 应该能看到多个独立的算子(Source、Map、Filter、Sink 等)

### 2. 远程集群配置

**配置文件**: `src/main/resources/config/application-dev.yml`

```yaml
flink:
  cluster:
    mode: remote  # 远程集群模式
    remote:
      host: localhost  # JobManager 地址
      port: 8081  # JobManager 端口
      jar-path: target/realtime-crypto-datawarehouse-1.0.0.jar
```

**验证方法**:
```bash
# 检查 Flink 集群状态
curl -s http://localhost:8081/overview
```

预期输出:
```json
{
  "taskmanagers": 2,
  "slots-total": 8,
  "slots-available": 8,
  "jobs-running": 0
}
```

### 3. Doris BE 地址配置

**配置文件**: `src/main/resources/config/application-dev.yml`

```yaml
doris:
  fe:
    http-url: http://127.0.0.1:8030
    jdbc-url: jdbc:mysql://127.0.0.1:9030
    username: root
    password: ""
  
  be:
    nodes: "doris-be:8040"  # ✅ 使用 Docker 容器名称
```

**关键说明**:
- FE 地址使用 `127.0.0.1` (宿主机访问)
- BE 地址使用 `doris-be` (容器名称,TaskManager 在容器内访问)

## 常见问题排查

### 问题 1: 作业提交失败

**错误信息**:
```
java.net.ConnectException: Connection refused: localhost/127.0.0.1:8081
```

**排查步骤**:
```bash
# 1. 检查 Flink 集群是否运行
docker ps --filter "name=flink"

# 2. 检查端口是否监听
netstat -an | grep 8081

# 3. 测试 REST API
curl http://localhost:8081/overview

# 4. 重启 Flink 集群
docker-compose -f docker-compose-flink.yml restart
```

### 问题 2: TaskManager 无法访问 Doris BE

**错误信息**:
```
org.apache.doris.flink.exception.DorisRuntimeException: no available backend
```

**排查步骤**:
```bash
# 1. 检查 BE 节点状态
docker exec doris-fe mysql -h 127.0.0.1 -P 9030 -u root \
  -e "SHOW BACKENDS\G"

# 2. 测试 TaskManager 访问 BE
docker exec flink-taskmanager-1 curl http://doris-be:8040/api/health

# 3. 检查网络配置
docker network inspect data-warehouse_doris-net

# 4. 运行修复脚本
bash test/fix-doris-be-remote.sh
```

### 问题 3: 算子链未禁用

**现象**: Web UI 只显示一个或少数几个算子

**排查步骤**:
```bash
# 1. 检查配置文件
grep "operator-chaining" src/main/resources/config/application-dev.yml

# 2. 检查日志
grep "算子链" logs/app/flink-app.log

# 3. 重新打包
mvn clean package -DskipTests

# 4. 重新提交作业
bash run-flink-ods-remote.sh
```

### 问题 4: JAR 文件找不到

**错误信息**:
```
java.io.FileNotFoundException: target/realtime-crypto-datawarehouse-1.0.0.jar
```

**排查步骤**:
```bash
# 1. 检查 JAR 文件
ls -lh target/*.jar

# 2. 重新打包
mvn clean package -DskipTests

# 3. 使用绝对路径
pwd
# 修改配置文件中的 jar-path 为绝对路径
```

## 性能监控

### 1. Flink 指标

**访问 Metrics API**:
```bash
# 获取作业 ID
JOB_ID=$(curl -s http://localhost:8081/jobs | jq -r '.jobs[0].id')

# 查看作业指标
curl -s "http://localhost:8081/jobs/$JOB_ID/metrics" | jq
```

**关键指标**:
- `numRecordsIn`: 输入记录数
- `numRecordsOut`: 输出记录数
- `numBytesIn`: 输入字节数
- `numBytesOut`: 输出字节数

### 2. Doris 数据量

**查询 ODS 表数据量**:
```bash
docker exec doris-fe mysql -h 127.0.0.1 -P 9030 -u root -e "
SELECT COUNT(*) as total 
FROM crypto_dw.ods_crypto_ticker_rt;
"
```

**查询最新数据**:
```bash
docker exec doris-fe mysql -h 127.0.0.1 -P 9030 -u root -e "
SELECT * 
FROM crypto_dw.ods_crypto_ticker_rt 
ORDER BY ts DESC 
LIMIT 10;
"
```

### 3. 资源使用

**查看容器资源使用**:

docker stats --no-stream --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}"

flink-taskmanager-1   15.0%     2.0GiB / 4GiB

4. **算子链**: 开发环境禁用,生产环境启用
