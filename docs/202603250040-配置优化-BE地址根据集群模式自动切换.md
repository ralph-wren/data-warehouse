# 配置优化 - BE 地址根据集群模式自动切换

## 问题描述

之前的配置需要手动修改 BE 地址来适配不同的运行模式:
- 本地模式: 需要使用 `127.0.0.1:8040`
- 远程集群模式: 需要使用 `doris-be:8040`

这种方式容易出错,每次切换模式都需要手动修改配置文件。

## 优化目标

实现 BE 地址根据集群模式自动切换,无需手动修改配置文件。

## 解决方案

### 1. 修改配置文件结构

将 BE 地址配置改为嵌套结构,分别配置本地模式和远程模式的地址:

**application-dev.yml**:
```yaml
doris:
  fe:
    http-url: http://127.0.0.1:8030
    jdbc-url: jdbc:mysql://127.0.0.1:9030
    username: root
    password: ""
  
  # BE 节点配置 (根据集群模式自动选择)
  # 说明：
  # - local 模式：使用 be.nodes.local (宿主机地址)
  # - remote 模式：使用 be.nodes.remote (Docker 容器名称)
  # 代码会根据 flink.cluster.mode 自动选择正确的地址
  be:
    nodes:
      local: "127.0.0.1:8040"   # 本地模式 - 宿主机访问
      remote: "doris-be:8040"   # 远程集群模式 - 容器访问
```

**application-docker.yml**:
```yaml
doris:
  fe:
    http-url: http://doris-fe:8030
    jdbc-url: jdbc:mysql://doris-fe:9030
    username: root
    password: ""
  
  # BE 节点配置 (根据集群模式自动选择)
  be:
    nodes:
      local: "127.0.0.1:8040"   # 本地模式 - 宿主机访问
      remote: "doris-be:8040"   # 远程集群模式 - 容器访问
```

### 2. 修改代码实现自动选择

在 `FlinkTableFactory.java` 的 `createDorisSinkTable()` 方法中添加自动选择逻辑:

```java
/**
 * 创建 Doris Sink 表 DDL - 显式传入库名和表名（推荐使用）
 * 
 * BE 地址自动选择说明：
 * - local 模式：使用 doris.be.nodes.local (宿主机地址 127.0.0.1:8040)
 * - remote 模式：使用 doris.be.nodes.remote (容器名称 doris-be:8040)
 * - 代码会根据 flink.cluster.mode 自动选择正确的地址
 */
public String createDorisSinkTable(String tableName, String database, String dorisTable, 
                                   String labelPrefix, String schema) {
    // 读取 Doris 连接配置
    String feNodes = config.getString("doris.fe.http-url").replace("http://", "");
    String username = config.getString("doris.fe.username");
    String password = config.getString("doris.fe.password", "");
    
    // 根据集群模式自动选择 BE 地址
    String clusterMode = config.getString("flink.cluster.mode", "local");
    String beNodes;
    
    if ("remote".equalsIgnoreCase(clusterMode)) {
        // 远程集群模式：使用容器名称
        beNodes = config.getString("doris.be.nodes.remote", "doris-be:8040");
        logger.info("远程集群模式：使用容器名称访问 BE: {}", beNodes);
    } else {
        // 本地模式：使用宿主机地址
        beNodes = config.getString("doris.be.nodes.local", "127.0.0.1:8040");
        logger.info("本地模式：使用宿主机地址访问 BE: {}", beNodes);
    }
    
    logger.info("创建 Doris Sink 表: {}", tableName);
    logger.info("  集群模式: {}", clusterMode);
    logger.info("  FE Nodes: {}", feNodes);
    logger.info("  BE Nodes: {}", beNodes);
    logger.info("  Database: {}", database);
    logger.info("  Table: {}", dorisTable);
    logger.info("  Label Prefix: {}", labelPrefix);
    
    // ... 后续代码
}
```

## 工作原理

### 1. 配置读取流程

```
1. 读取 flink.cluster.mode 配置
   ├─ local: 本地模式
   └─ remote: 远程集群模式

2. 根据模式选择 BE 地址
   ├─ local 模式: 读取 doris.be.nodes.local
   └─ remote 模式: 读取 doris.be.nodes.remote

3. 使用选择的 BE 地址创建 Doris Sink
```

### 2. 模式切换

**切换到本地模式**:
```yaml
flink:
  cluster:
    mode: local  # 修改为 local
```

代码会自动使用 `doris.be.nodes.local` (127.0.0.1:8040)

**切换到远程集群模式**:
```yaml
flink:
  cluster:
    mode: remote  # 修改为 remote
```

代码会自动使用 `doris.be.nodes.remote` (doris-be:8040)

## 使用方法

### 1. 本地模式运行

**配置** (`application-dev.yml`):
```yaml
flink:
  cluster:
    mode: local  # 本地模式
```

**运行**:
```bash
# 打包
mvn clean package -DskipTests

# 运行
java -cp target/realtime-crypto-datawarehouse-1.0.0.jar \
    com.crypto.dw.flink.FlinkODSJobDataStream \
    --APP_ENV dev
```

**结果**: 自动使用 `127.0.0.1:8040` 访问 BE

### 2. 远程集群模式运行

**配置** (`application-dev.yml`):
```yaml
flink:
  cluster:
    mode: remote  # 远程集群模式
    remote:
      host: localhost
      port: 8081
      jar-path: target/realtime-crypto-datawarehouse-1.0.0.jar
```

**运行**:
```bash
# 打包
mvn clean package -DskipTests

# 提交到远程集群
bash run-flink-ods-remote.sh
```

**结果**: 自动使用 `doris-be:8040` 访问 BE

## 技术要点

### 1. 配置嵌套结构

使用嵌套结构存储不同模式的配置:

```yaml
doris:
  be:
    nodes:
      local: "127.0.0.1:8040"
      remote: "doris-be:8040"
```

读取方式:
```java
// 读取 local 模式地址
String localNodes = config.getString("doris.be.nodes.local");

// 读取 remote 模式地址
String remoteNodes = config.getString("doris.be.nodes.remote");
```

### 2. 模式判断

根据 `flink.cluster.mode` 判断当前模式:

```java
String clusterMode = config.getString("flink.cluster.mode", "local");

if ("remote".equalsIgnoreCase(clusterMode)) {
    // 远程集群模式
    beNodes = config.getString("doris.be.nodes.remote");
} else {
    // 本地模式
    beNodes = config.getString("doris.be.nodes.local");
}
```

### 3. 默认值

提供合理的默认值,避免配置缺失:

```java
// 本地模式默认值
beNodes = config.getString("doris.be.nodes.local", "127.0.0.1:8040");

// 远程模式默认值
beNodes = config.getString("doris.be.nodes.remote", "doris-be:8040");
```
