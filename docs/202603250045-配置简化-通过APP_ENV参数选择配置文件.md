# 配置简化 - 通过 APP_ENV 参数选择配置文件

## 问题描述

之前的方案是在配置文件中嵌套 `local/remote` 配置,然后在代码中根据 `flink.cluster.mode` 判断选择哪个配置。这种方式虽然可行,但不够简洁:

```yaml
# 之前的方案 - 配置嵌套
doris:
  be:
    nodes:
      local: "127.0.0.1:8040"
      remote: "doris-be:8040"

kafka:
  bootstrap-servers:
    local: "localhost:9093"
    remote: "kafka:9093"
```

## 优化方案

采用更简洁的方式:**通过 `--APP_ENV` 参数直接选择不同的配置文件**:

- **本地模式**: `--APP_ENV dev` → 使用 `application-dev.yml` (宿主机地址)
- **远程集群模式**: `--APP_ENV docker` → 使用 `application-docker.yml` (容器名称)

这样配置文件更简洁,不需要嵌套,也不需要在代码中判断。

## 实现方案

### 1. 配置文件结构

**application-dev.yml** (本地模式):
```yaml
# 本地开发环境配置
# 使用方式: --APP_ENV dev

kafka:
  bootstrap-servers: localhost:9093  # 宿主机地址

doris:
  fe:
    http-url: http://127.0.0.1:8030
    jdbc-url: jdbc:mysql://127.0.0.1:9030
  be:
    nodes: "127.0.0.1:8040"  # 宿主机地址
```

**application-docker.yml** (远程集群模式):
```yaml
# Docker/远程集群环境配置
# 使用方式: --APP_ENV docker

kafka:
  bootstrap-servers: kafka:9093  # 容器名称

doris:
  fe:
    http-url: http://doris-fe:8030
    jdbc-url: jdbc:mysql://doris-fe:9030
  be:
    nodes: "doris-be:8040"  # 容器名称
```

### 2. 代码实现

代码保持简单,直接读取配置即可,不需要判断:

```java
// FlinkTableFactory.java
public String createDorisSinkTable(...) {
    // 直接读取配置,不需要判断模式
    String beNodes = config.getString("doris.be.nodes", "127.0.0.1:8040");
    
    logger.info("创建 Doris Sink 表: {}", tableName);
    logger.info("  BE Nodes: {}", beNodes);
    
    // ... 后续代码
}
```

### 3. 使用方法

**本地模式运行**:
```bash

    com.crypto.dw.jobs.FlinkODSJobDataStream \
    --APP_ENV dev
```

**远程集群模式运行**:

```yaml
  be:
    nodes: "doris-be:8040"

  bootstrap-servers: kafka:9093
```

**代码**:

## 工作原理

### 1. ConfigLoader 加载流程

```
1. 读取 APP_ENV 参数
   ├─ 命令行参数: --APP_ENV dev
   └─ 环境变量: export APP_ENV=dev

2. 加载对应的配置文件
   ├─ APP_ENV=dev → application-dev.yml
   └─ APP_ENV=docker → application-docker.yml

3. 代码直接读取配置
   ├─ config.getString("doris.be.nodes")
   └─ config.getString("kafka.bootstrap-servers")
```

### 2. 配置文件选择

| APP_ENV | 配置文件 | 适用场景 | 地址类型 |
|---------|---------|---------|---------|
| dev | application-dev.yml | 本地开发 | 宿主机地址 (localhost/127.0.0.1) |
| docker | application-docker.yml | 远程集群/Docker | 容器名称 (kafka/doris-be) |

### 3. 运行模式对应关系

| Flink 运行模式 | APP_ENV | 配置文件 | 说明 |
|--------------|---------|---------|------|
| 本地模式 (local) | dev | application-dev.yml | 在宿主机 JVM 中运行 |
| 远程集群模式 (remote) | docker | application-docker.yml | 提交到 Docker 集群运行 |

## 使用示例

### 示例 1: 本地开发

```bash
# 1. 设置环境
export APP_ENV=dev

# 2. 打包项目
mvn clean package -DskipTests

# 3. 运行作业
java -cp target/realtime-crypto-datawarehouse-1.0.0.jar \
    com.crypto.dw.jobs.FlinkODSJobDataStream \
    --APP_ENV dev

# 4. 查看日志
tail -f logs/app/flink-app.log | grep "BE Nodes"
# 输出: BE Nodes: 127.0.0.1:8040
```

### 示例 2: 远程集群

```bash
# 1. 设置环境
export APP_ENV=docker

# 2. 打包项目
mvn clean package -DskipTests

# 3. 提交到远程集群
bash run-flink-ods-remote.sh

# 4. 查看日志
docker logs flink-taskmanager-1 | grep "BE Nodes"
# 输出: BE Nodes: doris-be:8040
```

```bash
    com.crypto.dw.jobs.FlinkODSJobDataStream \
```

## 扩展性

### 添加新环境

如果需要添加测试环境:

**1. 创建配置文件**:
```bash
cp src/main/resources/config/application-dev.yml \
   src/main/resources/config/application-test.yml
```

**2. 修改配置**:
```yaml
# application-test.yml
kafka:
  bootstrap-servers: test-kafka:9093

doris:
  be:
    nodes: "test-doris-be:8040"
```

**3. 使用新环境**:
```bash
java -cp target/xxx.jar \
    com.crypto.dw.jobs.FlinkODSJobDataStream \
    --APP_ENV test
```

还可以使用环境变量覆盖配置:
