# 技术指标系统-集成策略工厂支持动态策略配置

## 时间
2026-03-25 14:10

## 问题描述
需要修改 `FlinkADSTechnicalIndicatorsJob` 类，让它支持从 strategy 目录配置的指标输出交易信号。原有实现中需要手动为每个指标类型编写计算逻辑和策略创建代码，不够灵活。

## 解决思路

### 1. 核心改进
- 移除手动编写的指标计算方法（calculateMA、calculateEMA等）
- 直接使用 `StrategyRegisterCenter` 来获取策略
- 通过策略判断K线是否满足触发信号
- 简化代码，提高可维护性

### 2. 架构设计
```
K线数据 -> BarSeries -> StrategyRegisterCenter.createStrategy() -> Strategy.shouldEnter/shouldExit() -> 交易信号
```

## 实施步骤

### 1. 添加策略注册中心依赖
在 `FlinkADSTechnicalIndicatorsJob` 中导入 `StrategyRegisterCenter`：
```java
import com.crypto.dw.strategy.StrategyRegisterCenter;
```

### 2. 扩展指标配置实体
在 `IndicatorConfig` 类中添加 `strategyType` 字段：
```java
public String strategyType;  // 策略类型（用于STRATEGY类型，对应StrategyRegisterCenter中的策略）
```

### 3. 重构指标计算方法
简化 `calculateIndicator` 方法，直接使用策略工厂：
```java
// 从配置中获取策略类型
String strategyType = config.strategyType;
if (strategyType == null || strategyType.isEmpty()) {
    strategyType = config.indicatorType;
}

// 使用StrategyRegisterCenter创建策略
Strategy strategy = StrategyRegisterCenter.createStrategy(series, strategyType);

// 使用策略判断是否满足触发信号
int endIndex = series.getEndIndex();
boolean shouldEnter = strategy.shouldEnter(endIndex);
boolean shouldExit = strategy.shouldExit(endIndex);

// 生成交易信号
if (shouldEnter) {
    result.signalType = "BUY";
    result.signalReason = strategy.getName() + " entry signal";
    result.signalStrength = new BigDecimal(80);
} else if (shouldExit) {
    result.signalType = "SELL";
    result.signalReason = strategy.getName() + " exit signal";
    result.signalStrength = new BigDecimal(80);
} else {
    result.signalType = "HOLD";
    result.signalReason = "No signal from " + strategy.getName();
    result.signalStrength = BigDecimal.ZERO;
}
```

### 4. 修复编译错误

#### 4.1 修复 import 错误
StrategyFactory2/3/4 中有错误的 import 语句：
```java
// 错误
import static com.okx.trading.strategy.StrategyRegisterCenter.addExtraStopRule;

// 正确
import static com.crypto.dw.strategy.StrategyRegisterCenter.addExtraStopRule;
```

#### 4.2 修复 ta4j 0.18 API 变化
ta4j 0.18 版本的 API 有以下变化：

1. 移除了 `series.numOf()` 方法，使用 `DecimalNum.valueOf()` 替代
2. 移除了 `BaseBarSeriesBuilder.withNumTypeOf()` 方法
3. `BaseBar` 构造函数参数变化：
   - 需要 `Instant` 而不是 `ZonedDateTime`
   - 需要额外的 `amount` 和 `trades` 参数

修复示例：
```java
// 创建BarSeries
series = new BaseBarSeriesBuilder()
    .withName(symbol)
    .build();  // 移除 withNumTypeOf

// 创建Bar
return new BaseBar(
    duration,
    minuteTime.toInstant(),  // ZonedDateTime转Instant
    DecimalNum.valueOf(open.doubleValue()),
    DecimalNum.valueOf(high.doubleValue()),
    DecimalNum.valueOf(low.doubleValue()),
    DecimalNum.valueOf(close.doubleValue()),
    DecimalNum.valueOf(volume.doubleValue()),
    DecimalNum.valueOf(0),  // amount
    0L  // trades
);

// 替换 series.numOf()
// 错误
Num middle = highest.plus(lowest).dividedBy(series.numOf(2));
// 正确
Num middle = highest.plus(lowest).dividedBy(DecimalNum.valueOf(2));
```

#### 4.3 添加缺失的 import
在 CustomPriceChannelIndicator 和 CustomVolumeWeightedMA 中添加：
```java
import org.ta4j.core.num.DecimalNum;
```

### 5. 配置示例

在 Redis 中配置策略：
```json
{
  "config_id": "strategy_rsi_btc",
  "symbol": "BTC-USDT",
  "indicator_type": "STRATEGY",
  "indicator_name": "RSI策略",
  "strategy_type": "STRATEGY_RSI",
  "params": {
    "period": "14"
  },
  "enabled": true
}
```

支持的策略类型包括：
- 移动平均线策略：STRATEGY_SMA, STRATEGY_EMA, STRATEGY_WMA 等
- 震荡指标策略：STRATEGY_RSI, STRATEGY_MACD, STRATEGY_STOCHASTIC 等
- 趋势指标策略：STRATEGY_ADX, STRATEGY_AROON, STRATEGY_ICHIMOKU 等
- 波动指标策略：STRATEGY_BOLLINGER_BANDS, STRATEGY_ATR, STRATEGY_KELTNER_CHANNEL 等
- 组合策略：STRATEGY_TURTLE_TRADING, STRATEGY_MEAN_REVERSION 等
- 共130+种策略（详见 StrategyRegisterCenter）

## 技术要点

### 1. 策略工厂模式
使用 `StrategyRegisterCenter` 统一管理所有策略，支持：
- 130+ 种预定义策略
- 动态策略注册
- 策略参数配置
- 止损止盈规则

### 2. 广播状态模式
使用 Flink 广播状态实现动态配置更新：
- 配置流：从 Redis 定期读取策略配置
- 数据流：处理 K线数据
- 广播连接：动态应用最新配置

### 3. K线聚合优化
- 同一分钟的多条 ticker 聚合为一根 K线
- 避免重复添加 K线
- 只在 K线完成时计算指标

### 4. 信号生成逻辑
- BUY：策略的 shouldEnter() 返回 true
- SELL：策略的 shouldExit() 返回 true
- HOLD：两者都为 false

## 优势

### 1. 代码简化
- 移除了大量重复的指标计算代码
- 统一使用策略工厂创建策略
- 减少维护成本

### 2. 灵活性提升
- 支持 130+ 种策略
- 通过 Redis 动态配置
- 无需重启作业即可添加/删除策略

### 3. 可扩展性
- 新增策略只需在 StrategyFactory 中实现
- 自动注册到 StrategyRegisterCenter
- 立即可用于信号生成

### 4. 统一管理
- 所有策略逻辑集中在 strategy 包
- 统一的止损止盈规则
- 便于回测和优化

## 注意事项

1. **Java 版本**：ta4j 0.18 需要 Java 21
2. **策略类型**：strategyType 必须是 StrategyRegisterCenter 中注册的策略名称
3. **K线数据量**：至少需要 2 根 K线才能计算指标
4. **信号强度**：当前使用固定值 80，后续可以根据策略类型动态调整
5. **Redis 配置**：确保 Redis 中的配置格式正确
