# ta4j 从 0.18 升级到 0.21

## 升级时间
2026-03-25 16:00

## 升级原因
1. ta4j 0.21 是最新的稳定版本（2025年11月29日发布）
2. 提供统一的返回表示系统，性能指标更准确
3. 新增多个技术指标，功能更强大
4. 修复了已知的 bug 和性能问题

## 版本对比

### ta4j 0.18（旧版本）
- 发布时间：较早
- 功能：基础技术指标库
- 问题：部分 API 设计不合理，性能指标计算不准确

### ta4j 0.21（新版本）
- 发布时间：2025年11月29日
- 主要优势：
  - 统一返回表示系统（Unified Return Representation）
  - 更准确的性能指标计算
  - 新增技术指标（如 Chande Momentum Oscillator 等）
  - 改进的 API 设计

### 破坏性变更
1. **EMA 指标在不稳定期返回 NaN**
   - 影响：EMA 指标在前几根 K 线可能返回 NaN
   - 解决：需要检查 NaN 值并处理

2. **TransformIndicator 被移除**
   - 影响：所有使用 TransformIndicator 的代码需要修改
   - 解决：创建 IndicatorTransform 工具类替代

3. **BaseBar 构造函数变更**
   - 影响：需要同时提供 beginTime 和 endTime
   - 解决：修改 KLineAggregator.toBar() 方法

### 2. 创建 IndicatorTransform 工具类
由于 ta4j 0.21 移除了 `TransformIndicator`，我们创建了兼容性工具类：

**文件位置**：`src/main/java/com/crypto/dw/indicators/IndicatorTransform.java`

**主要功能**：
- `multiply(indicator, multiplier)` - 乘法转换
- `divide(indicator, divisor)` - 除法转换
- `add(indicator, addend)` - 加法转换
- `subtract(indicator, subtrahend)` - 减法转换
- `transform(indicator, function)` - 自定义转换

**关键实现**：
```java
public static Indicator<Num> multiply(Indicator<Num> indicator, double multiplier) {
    return new CachedIndicator<Num>(indicator.getBarSeries()) {
        @Override
        protected Num calculate(int index) {
            Num value = indicator.getValue(index);
            // ta4j 0.21: 使用 DecimalNum.valueOf() 创建 Num 对象
            return value.multipliedBy(DecimalNum.valueOf(multiplier));
        }
        
        // ta4j 0.21: 必须实现 getCountOfUnstableBars() 方法
        @Override
        public int getCountOfUnstableBars() {
            return indicator.getCountOfUnstableBars();
        }
    };
}
```

**关键变更点**：
1. `CachedIndicator` 需要实现 `getCountOfUnstableBars()` 方法
2. 使用 `DecimalNum.valueOf()` 而不是 `Num.numOf()` 或 `BarSeries.numOf()`
3. 返回类型从 `TransformIndicator` 改为 `Indicator<Num>`

### 3. 批量替换 TransformIndicator 引用
使用 sed 命令批量替换所有策略文件中的 TransformIndicator：

```bash
# 替换 TransformIndicator.multiply 为 IndicatorTransform.multiply
find src/main/java/com/crypto/dw/strategy -name "*.java" -exec sed -i '' 's/TransformIndicator\.multiply/IndicatorTransform.multiply/g' {} \;

# 替换 new TransformIndicator 为 IndicatorTransform.transform
find src/main/java/com/crypto/dw/strategy -name "*.java" -exec sed -i '' 's/new TransformIndicator/IndicatorTransform.transform/g' {} \;
```

### 5. 修复 BaseBar 构造函数
ta4j 0.21 中 `BaseBar` 构造函数需要同时提供 `beginTime` 和 `endTime`：

**修改文件**：`src/main/java/com/crypto/dw/flink/FlinkADSTechnicalIndicatorsJob.java`

**修改位置**：`KLineAggregator.toBar()` 方法

```java
// 修改前（ta4j 0.18）
public Bar toBar(Duration duration) {
    return new BaseBar(
        duration,
        minuteTime,  // 只需要一个时间参数
        DecimalNum.valueOf(open.doubleValue()),
        // ...
    );
}

// 修改后（ta4j 0.21）
public Bar toBar(Duration duration) {
    // ta4j 0.21 变更：BaseBar 构造函数需要 beginTime 和 endTime 两个 Instant 参数
    Instant endTime = minuteTime.toInstant();
    Instant beginTime = endTime.minus(duration);  // 开始时间 = 结束时间 - 周期
    
    return new BaseBar(
        duration,
        beginTime,  // 开始时间（ta4j 0.21 新增）
        endTime,    // 结束时间
        DecimalNum.valueOf(open.doubleValue()),
        // ...
    );
}
```

## 升级后的注意事项

### 1. EMA 指标 NaN 处理
ta4j 0.21 中，EMA 指标在不稳定期（前几根 K 线）会返回 NaN。需要在使用 EMA 时检查：

```java
// 建议在 calculateIndicator 方法中添加 NaN 检查
Num emaValue = ema.getValue(index);
if (emaValue.isNaN()) {
    // 跳过或使用默认值
    return null;
}
```

### 2. 性能指标更准确
ta4j 0.21 使用统一的返回表示系统，回测结果可能与 0.18 版本略有不同，这是正常的，新版本的结果更准确。

### 3. 新增指标
ta4j 0.21 新增了一些技术指标，可以在后续开发中使用：
- Chande Momentum Oscillator
- 改进的 Bollinger Bands
- 更多的统计指标

## 遇到的问题和解决方案

### 问题 1: CachedIndicator 编译错误
**错误信息**：
```
The type new CachedIndicator<Num>(){} must implement the inherited abstract method 
Indicator<Num>.getCountOfUnstableBars()
```

**原因**：ta4j 0.21 中 `CachedIndicator` 必须实现 `getCountOfUnstableBars()` 方法

**解决方案**：
```java
@Override
public int getCountOfUnstableBars() {
    return indicator.getCountOfUnstableBars();
}
```

### 问题 2: numOf() 方法不存在
**错误信息**：
```
The method numOf(double) is undefined for the type Num
The method numOf(double) is undefined for the type BarSeries
```

**原因**：ta4j 0.21 中 API 变更，不再有 `Num.numOf()` 或 `BarSeries.numOf()` 方法

**解决方案**：使用 `DecimalNum.valueOf()` 替代
```java
// 错误写法
value.multipliedBy(value.numOf(multiplier))
value.multipliedBy(getBarSeries().numOf(multiplier))

// 正确写法
value.multipliedBy(DecimalNum.valueOf(multiplier))
```

### 问题 3: TransformIndicator 找不到符号
**错误信息**：
```
找不到符号
符号:   类 TransformIndicator
位置: 类 com.crypto.dw.strategy.StrategyFactory3
```

**原因**：变量类型声明仍然使用 `TransformIndicator`

**解决方案**：改为 `Indicator<Num>`
```java
// 错误写法
TransformIndicator volumeThreshold = IndicatorTransform.multiply(avgVolume, 1.2);

// 正确写法
Indicator<Num> volumeThreshold = IndicatorTransform.multiply(avgVolume, 1.2);
```
