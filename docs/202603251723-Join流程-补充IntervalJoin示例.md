## 关键说明（示例）

- **KeyBy 分区**：现货流与合约流先按交易对（如 `BTC-USDT`）分区，确保同 key 数据进入同一算子。
- **时间区间**：设置 `between(-10s, +10s)`，表示允许两条流事件时间相差 ±10 秒。
- **匹配方式**：某一侧事件到达时，在另一侧缓存状态里查找时间范围内事件，匹配成功才触发计算。
- **示例**：
  - 现货事件时间 `10:00:05`，合约事件时间 `10:00:12`，差 7 秒 → 匹配成功
  - 现货事件时间 `10:00:05`，合约事件时间 `10:00:30`，差 25 秒 → 不匹配
