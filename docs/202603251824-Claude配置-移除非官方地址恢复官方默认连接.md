# Claude 配置修复：移除非官方地址，恢复官方默认连接

## 1. 问题现象

用户反馈：

- 本地运行 Claude 时，使用的好像不是官方地址

## 2. 排查结果

本次排查发现两个关键信息：

1. 当前 shell 中没有 `claude` 命令
   - `which claude` 返回 `claude not found`
   - `type claude` 返回 `claude not found`
2. Claude 本地配置文件中存在自定义地址覆盖
   - 文件：`~/.claude/settings.json`
   - 原配置中存在：

```json
"ANTHROPIC_BASE_URL": "https://claudecode.top/ca-agent-claude"
```

这个地址不是 Anthropic 官方 API 域名，因此会导致 Claude 请求被转发到非官方服务。

## 3. 修复动作

本次已经完成以下修复：

1. 备份原始配置文件
   - `~/.claude/settings.json.bak-20260325`
2. 修改 `~/.claude/settings.json`
3. 删除自定义环境变量：
   - `ANTHROPIC_BASE_URL`

修复后的配置文件中，`ANTHROPIC_BASE_URL` 已不存在。

## 4. 修复原理

Claude 本地配置里如果显式设置了：

- `ANTHROPIC_BASE_URL`

那么客户端会优先使用这个地址，而不是官方默认地址。

删除该配置后，Claude 会回退到官方默认连接地址，而不是继续使用第三方代理地址。

## 5. 当前状态

已确认：

1. 非官方地址覆盖已移除
2. 原配置已备份
3. 当前 `~/.claude/settings.json` 中不再包含 `ANTHROPIC_BASE_URL`

## 6. 安全建议

这次还有一个重要的安全点：

你原来的 `~/.claude/settings.json` 中除了自定义 Base URL，还包含 `ANTHROPIC_AUTH_TOKEN`。

因为这个 token 之前是配合非官方地址使用的，所以存在以下风险：

1. token 可能已经发给过第三方服务
2. 该 token 后续继续使用存在泄露风险

### 建议操作

建议你尽快做两件事：

1. 在 Anthropic / Claude 侧重新生成新的 token
2. 废弃旧 token

## 7. 后续建议

如果你后面还想继续排查 Claude 本地运行问题，建议按这个顺序检查：

1. `~/.claude/settings.json`
2. shell 环境变量
   - `ANTHROPIC_BASE_URL`
   - `http_proxy`
   - `https_proxy`
3. shell 启动脚本
   - `~/.zshrc`
   - `~/.zprofile`
4. 本地是否真的安装了 `claude` 命令

## 8. 一句话总结

本次问题的根因是：

**`~/.claude/settings.json` 中把 `ANTHROPIC_BASE_URL` 指到了非官方地址。现在已经移除这个覆盖，Claude 会恢复使用官方默认连接方式。**
