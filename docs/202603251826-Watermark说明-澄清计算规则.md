# Watermark 计算规则补充说明

## 目标

澄清 “每个 subtask 的 watermark 不是取最大事件时间” 的计算规则。

## 处理步骤

1. 在 `FlinkADSArbitrageJob` 注释中补充 watermark 计算来源说明。
2. 明确“最大事件时间 - 乱序容忍”的基本逻辑，以及多输入取最小值的规则。

## 关键说明

- **subtask 级别**：每个并行子任务维护自己的 watermark。
- **计算逻辑**：watermark 通常为“该子任务已看到的最大事件时间 - 乱序容忍”。
- **多输入规则**：如果有多个上游输入分区，实际生效 watermark 为这些输入 watermark 的最小值。

## 结果

已补充 watermark 计算规则说明，避免误解为“取所有 key 最大事件时间”。
