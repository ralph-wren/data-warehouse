## 问题背景

在之前的 SQL 分离工作中（202603241433），我们已经将 DWD 和 DWS 层的 SQL 语句分离到独立文件，但是 ODS 层和 ADS 层的 SQL 仍然硬编码在 Java 代码中：

1. **FlinkODSJobSQL.java** - `createInsertSQL()` 方法中有硬编码的 INSERT SQL
2. **FlinkADSRealtimeMetricsJob.java** - `createInsertSQL()` 方法中有复杂的 INSERT SQL

这导致：
- SQL 维护困难，需要修改 Java 代码
- SQL 逻辑和业务逻辑耦合
- 无法独立测试和验证 SQL
- 不符合"SQL 全面分离"的架构规范

## 解决方案

### 1. 创建 SQL 文件

#### 1.1 创建 ODS INSERT SQL 文件

**文件**: `sql/flink/ods_insert.sql`

**功能**:
- 从 Kafka 读取原始行情数据
- 添加数据源标识（data_source = 'OKX'）
- 添加数据摄入时间（ingest_time）
- 写入 Doris ODS 层

**SQL 内容**:
```sql
INSERT INTO doris_ods_sink
SELECT 
    inst_id,
    `timestamp`,
    last_price,
    bid_price,
    ask_price,
    bid_size,
    ask_size,
    volume_24h,
    high_24h,
    low_24h,
    open_24h,
    'OKX' as data_source,
    UNIX_TIMESTAMP() * 1000 as ingest_time
FROM kafka_source
```

#### 1.2 创建 ADS Realtime Metrics INSERT SQL 文件

**文件**: `sql/flink/ads_realtime_metrics_insert.sql`

**功能**:
- 从 DWS 层读取 1分钟 K 线数据
- 使用 LAG 函数计算历史价格
- 计算多时间窗口涨跌幅（1分钟/5分钟/15分钟/1小时）
- 计算成交量指标（使用窗口函数聚合）
- 计算波动率指标（使用标准差）
- 判断价格趋势（上涨/下跌/震荡）
- 写入 Doris ADS 层

**SQL 特点**:
- 复杂的窗口函数计算
- 多层嵌套的 CASE WHEN 逻辑
- LAG 函数用于时间序列分析
- 约 150 行 SQL 代码

### 2. 修改 Java 代码

#### 2.1 修改 FlinkODSJobSQL.java

**修改前**:
```java
// 执行 INSERT INTO 语句
String insertSQL = createInsertSQL();
logger.info("Executing INSERT SQL...");

// ... 

private static String createInsertSQL() {
    return "INSERT INTO doris_ods_sink\n" +
           "SELECT \n" +
           "    inst_id,\n" +
           // ... 硬编码的 SQL
}
```

**修改后**:
```java
// 执行 INSERT INTO 语句（从外部 SQL 文件加载）
// 重构说明: SQL 语句已分离到独立文件，便于维护和版本控制
logger.info("Loading INSERT SQL from file...");
String insertSQL = SqlFileLoader.loadSql("sql/flink/ods_insert.sql");
logger.info("Executing INSERT SQL...");

// 删除 createInsertSQL() 方法
```

**改进点**:
- 添加 `SqlFileLoader` 导入
- 使用 `SqlFileLoader.loadSql()` 加载外部 SQL 文件
- 删除硬编码的 `createInsertSQL()` 方法
- 添加注释说明 SQL 文件位置

## 架构改进

### 1. 完整的 SQL 分离

现在所有 Flink 作业的 SQL 都已分离到独立文件：

| 作业 | SQL 文件 | 状态 |
|------|----------|------|
| ODS 作业 | `sql/flink/ods_insert.sql` | ✅ 已分离 |
| DWD 作业 | `sql/flink/dwd_insert.sql` | ✅ 已分离 |
| DWS 作业 | `sql/flink/dws_1min_insert.sql` | ✅ 已分离 |
| ADS 作业 | `sql/flink/ads_realtime_metrics_insert.sql` | ✅ 已分离 |

### 2. 统一的 SQL 加载机制

所有作业都使用 `SqlFileLoader` 加载 SQL：

```java
// 统一的加载方式
String sql = SqlFileLoader.loadSql("sql/flink/xxx_insert.sql");
tableEnv.executeSql(sql);
```

### 3. 清晰的目录结构

```
sql/
├── flink/              # Flink SQL（按层级组织）
│   ├── ddl/            # DDL 定义
│   ├── ods_insert.sql  # ODS 层
│   ├── dwd_insert.sql  # DWD 层
│   ├── dws_1min_insert.sql  # DWS 层
│   └── ads_realtime_metrics_insert.sql  # ADS 层
└── doris/              # Doris 建表 SQL
```

## 注意事项

### 1. SQL 文件编码

- 所有 SQL 文件使用 UTF-8 编码
- 使用 Unix 风格换行符（LF）

### 2. SQL 文件路径

- 路径相对于 classpath
- 编译后位于 `target/classes/sql/`
- JAR 包中位于 `sql/` 目录

### 3. 修改 SQL 后的操作

```bash
# 重新编译（SQL 文件会自动打包）
mvn clean compile -DskipTests

# 验证 SQL 文件是否打包
ls -lh target/classes/sql/flink/
```

### 4. SQL 注释规范

每个 SQL 文件开头必须包含：
- 功能说明
- 数据流转说明
- 计算逻辑说明
- 注意事项
