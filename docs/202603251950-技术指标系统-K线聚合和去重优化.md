# 技术指标系统 - K线聚合和去重优化

## 概述

**优化时间**: 2026-03-25 19:50  
**问题**: 同一分钟的多条ticker数据会创建多根K线，导致数据重复  
**解决方案**: 实现K线聚合器，将同一分钟的ticker数据聚合为一根K线

## 问题分析

### 原始问题

**问题1**: Redis存储的是什么？
- ✅ **答案**: Redis存储的是**指标配置**，不是指标名称
- 配置包括：指标类型、参数、启用状态等

**问题2**: 输出的是什么？
- ✅ **答案**: 输出的是**技术指标值**，不是交易信号
- 技术指标：MA、RSI、MACD等数值
- 交易信号：BUY/SELL/HOLD（在另一个系统中）

**问题3**: K线去重问题
- ⚠️ **问题**: 同一分钟的多条ticker数据会创建多根K线
- ❌ **原始代码**: 每条ticker直接创建一根K线，没有聚合
- ✅ **优化方案**: 实现K线聚合器，按分钟聚合

## 系统架构对比

### 原始架构（有问题）

```
Ticker数据流
  ↓
每条ticker创建一根K线 ❌
  ↓
K线序列（重复数据）
  ↓
计算指标（不准确）
  ↓
输出到Doris
```

**问题**:
- 同一分钟可能有10条ticker → 创建10根K线
- K线数据重复，指标计算不准确
- 浪费存储空间和计算资源

### 优化后架构（正确）

```
Ticker数据流
  ↓
K线聚合器（按分钟聚合）✅
  ↓
  同一分钟的ticker聚合为一根K线
  - 开盘价：第一条ticker的价格
  - 最高价：所有ticker中的最高价
  - 最低价：所有ticker中的最低价
  - 收盘价：最后一条ticker的价格
  - 成交量：所有ticker的成交量累加
  ↓
K线序列（无重复）
  ↓
计算指标（准确）
  ↓
输出到Doris
```

**优势**:
- 每分钟只有一根K线
- 数据准确，符合K线定义
- 节省存储和计算资源

## 核心实现

### 1. K线聚合器类

```java
/**
 * K线聚合器
 * 用于聚合同一分钟内的多条ticker数据为一根K线
 */
public static class KLineAggregator implements java.io.Serializable {
    public long minuteTimestamp;        // 分钟时间戳（对齐到分钟）
    public ZonedDateTime minuteTime;    // 分钟时间
    public BigDecimal open;             // 开盘价
    public BigDecimal high;             // 最高价
    public BigDecimal low;              // 最低价
    public BigDecimal close;            // 收盘价
    public BigDecimal volume;           // 成交量
    public int tickCount;               // ticker数量
    
    /**
     * 更新K线数据
     * 聚合新的ticker数据
     */
    public void update(TickerData ticker) {
        if (tickCount == 0) {
            // 第一条数据：初始化所有值
            this.open = ticker.price;
            this.high = ticker.high;
            this.low = ticker.low;
            this.close = ticker.price;
            this.volume = ticker.volume;
        } else {
            // 后续数据：更新最高价、最低价、收盘价、成交量
            if (ticker.high.compareTo(this.high) > 0) {
                this.high = ticker.high;
            }
            if (ticker.low.compareTo(this.low) < 0) {
                this.low = ticker.low;
            }
            this.close = ticker.price;
            this.volume = this.volume.add(ticker.volume);
        }
        this.tickCount++;
    }
    
    /**
     * 转换为TA4J的Bar对象
     */
    public Bar toBar(Duration duration) {
        return new BaseBar(
            duration,
            minuteTime,
            DecimalNum.valueOf(open),
            DecimalNum.valueOf(high),
            DecimalNum.valueOf(low),
            DecimalNum.valueOf(close),
            DecimalNum.valueOf(volume),
            DecimalNum.valueOf(0)
        );
    }
}
```

### 2. 状态管理

```java
// 使用MapState存储每个交易对的K线序列
private transient MapState<String, BarSeries> barSeriesState;

// 使用ValueState存储当前正在聚合的K线数据
private transient ValueState<KLineAggregator> currentKLineState;
```

**两个状态的作用**:
- `barSeriesState`: 存储完整的K线序列（最多500根）
- `currentKLineState`: 存储当前正在聚合的K线（未完成）

### 3. 处理逻辑

```java
@Override
public void processElement(
        TickerData tickerData,
        ReadOnlyContext ctx,
        Collector<IndicatorResult> out) throws Exception {
    
    // 1. 计算当前数据所属的分钟时间戳（对齐到分钟）
    long minuteTimestamp = (tickerData.timestamp / 60000) * 60000;
    
    // 2. 获取当前正在聚合的K线
    KLineAggregator aggregator = currentKLineState.value();
    
    // 3. 判断是否需要创建新的K线
    boolean needNewBar = false;
    if (aggregator == null || aggregator.minuteTimestamp != minuteTimestamp) {
        // 如果有旧的K线，先添加到序列中
        if (aggregator != null) {
            Bar completedBar = aggregator.toBar(Duration.ofMinutes(1));
            series.addBar(completedBar);
        }
        
        // 创建新的K线聚合器
        aggregator = new KLineAggregator(minuteTimestamp, minuteTime);
        needNewBar = true;
    }
    
    // 4. 更新K线数据（聚合同一分钟的多条ticker）
    aggregator.update(tickerData);
    currentKLineState.update(aggregator);
    
    // 5. 如果是新K线且序列中已有数据，则计算指标
    if (needNewBar && series.getBarCount() > 0) {
        // 计算所有配置的指标
        // ...
    }
}
```

## 聚合规则详解

### K线OHLCV计算

| 字段 | 计算规则 | 说明 |
|-----|---------|------|
| Open（开盘价） | 第一条ticker的价格 | 分钟开始时的价格 |
| High（最高价） | 所有ticker中的最高价 | 分钟内的最高价格 |
| Low（最低价） | 所有ticker中的最低价 | 分钟内的最低价格 |
| Close（收盘价） | 最后一条ticker的价格 | 分钟结束时的价格 |
| Volume（成交量） | 所有ticker的成交量累加 | 分钟内的总成交量 |

### 时间对齐

```java
// 原始时间戳：1711334567890（某个具体时刻）
// 对齐到分钟：1711334520000（该分钟的开始时刻）

long minuteTimestamp = (tickerData.timestamp / 60000) * 60000;
```

**示例**:
- 输入：2026-03-25 10:15:37.890
- 对齐：2026-03-25 10:15:00.000

### 聚合示例

**输入**（同一分钟的3条ticker）:
```
10:15:10 - price: 50000, high: 50100, low: 49900, volume: 10
10:15:30 - price: 50200, high: 50300, low: 50000, volume: 15
10:15:50 - price: 50100, high: 50200, low: 50000, volume: 12
```

**输出**（聚合后的K线）:
```
时间: 10:15:00
开盘价: 50000  （第一条的price）
最高价: 50300  （max(50100, 50300, 50200)）
最低价: 49900  （min(49900, 50000, 50000)）
收盘价: 50100  （最后一条的price）
成交量: 37     （10 + 15 + 12）
```

## 指标计算时机

### 优化前（错误）

```
每条ticker到达 → 立即计算指标
```

**问题**:
- 同一分钟计算多次
- 指标值不稳定
- 浪费计算资源

### 优化后（正确）

```
新的分钟开始 → 上一分钟K线完成 → 计算指标
```

**优势**:
- 每分钟只计算一次
- 指标值稳定准确
- 节省计算资源

## 性能对比

- **K线数量**: 每分钟10-100根（取决于ticker频率）
- **指标计算次数**: 每分钟10-100次
- **存储空间**: 10-100倍
- **计算资源**: 10-100倍

### 优化后

- **K线数量**: 每分钟1根 ✅
- **指标计算次数**: 每分钟1次 ✅
- **存储空间**: 正常 ✅
- **计算资源**: 正常 ✅

**性能提升**: 10-100倍！

## 注意事项

### 1. 状态大小

```java
series.setMaximumBarCount(500);  // 保留最近500根K线
```

- 500根K线 = 500分钟 ≈ 8.3小时
- 每个交易对约占用 50KB 状态空间
- 100个交易对 ≈ 5MB 状态空间

### 2. 延迟问题

- K线在下一分钟开始时才完成
- 指标计算有1分钟延迟
- 这是正常的，符合K线定义

### 3. 乱序数据

- 使用Watermark处理乱序（允许5秒）
- 超过5秒的乱序数据会被丢弃
- 可以根据需要调整Watermark延迟
