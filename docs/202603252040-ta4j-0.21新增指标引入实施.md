## 背景
根据之前的分析文档 `202603252030-ta4j-0.21新增指标分析和引入建议.md`,开始实施 ta4j 0.21 新增指标的引入工作。

## 实施内容

### 1. 创建策略工厂5 (StrategyFactory5.java)

创建了专门使用 ta4j 0.21 新增指标的策略工厂,包含以下策略:

#### 已成功实现的策略 (4个)

1. **SchaffTrendCycleStrategy - 沙夫趋势周期策略** ⭐⭐⭐⭐⭐
   - 使用 `SchaffTrendCycleIndicator`
   - 比 MACD 更敏感,减少假信号
   - 入场: STC 突破 25 且价格在 20 日均线上方
   - 出场: STC 跌破 75 或价格跌破均线
   - 构造函数: `new SchaffTrendCycleIndicator(closePrice, 23, 50, 10, 3)`

2. **PercentRankStrategy - 百分位排名策略** ⭐⭐⭐⭐
   - 使用 `PercentRankIndicator`
   - 标准化指标值,识别极端值
   - 入场: RSI 百分位 < 20% 且 成交量百分位 > 80%
   - 出场: RSI 百分位 > 80%
   - 构造函数: `new PercentRankIndicator(indicator, barCount)`

3. **NetMomentumStrategy - 净动量策略** ⭐⭐⭐
   - 使用 `NetMomentumIndicator`
   - 简单直观的动量度量
   - 入场: 净动量转正 且 价格在 50 日均线上方
   - 出场: 净动量转负
   - 构造函数: `new NetMomentumIndicator(closePrice, 20, 0.0, 0.0)`
   - 注意: 阈值参数需要使用 `Double` 类型,不能使用 `Num` 类型

4. **StochasticIndicatorStrategy - 通用随机指标策略** ⭐⭐⭐
   - 使用 `StochasticIndicator`
   - 对任意指标进行随机化处理
   - 本策略对 RSI 进行随机化
   - 入场: 随机 RSI 突破 20
   - 出场: 随机 RSI 跌破 80
   - 构造函数: `new StochasticIndicator(indicator, barCount)`

#### 暂时注释的策略 (4个)

以下策略因为相关指标在 ta4j 0.21 中可能还未完全实现或 API 有变化,暂时注释:

1. **AlligatorStrategy - Bill Williams 鳄鱼策略**
   - 需要 `AlligatorIndicator`
   - 编译时找不到该类

2. **FractalStrategy - 分形支撑阻力策略**
   - 需要 `FractalHighIndicator`, `FractalLowIndicator`
   - 需要 `RecentFractalSwingHighIndicator`, `RecentFractalSwingLowIndicator`
   - 编译时找不到这些类

3. **GatorOscillatorStrategy - 鳄鱼振荡器策略**
   - 需要 `GatorOscillatorIndicator`
   - 编译时找不到该类

4. **BillWilliamsCompleteStrategy - Bill Williams 完整策略**
   - 综合使用上述所有 Bill Williams 指标
   - 依赖的指标类都不可用

## 遇到的问题和解决方案

### 问题1: 缺少 import 语句

**错误信息**:
```
找不到符号: 类 SMAIndicator
```

**解决方案**:
添加缺失的 import:
```java
import org.ta4j.core.indicators.averages.SMAIndicator;
```

### 问题2: SchaffTrendCycleIndicator 构造函数参数错误

**错误信息**:
```
构造器 SchaffTrendCycleIndicator(ClosePriceIndicator, int, int, int) 不适用
(实际参数列表和形式参数列表长度不同)
```

**解决方案**:
查看 ta4j 0.21 文档,发现需要 5 个参数:
```java
// 错误: 4 个参数
new SchaffTrendCycleIndicator(closePrice, 23, 50, 10);

// 正确: 5 个参数
new SchaffTrendCycleIndicator(closePrice, 23, 50, 10, 3);
```

### 问题3: NetMomentumIndicator 构造函数参数类型错误

**错误信息**:
```
不兼容的类型: org.ta4j.core.num.Num无法转换为java.lang.Number
```

**解决方案**:
阈值参数需要使用 `Number` 类型(如 `Double`),不能使用 `Num` 类型:
```java
// 错误: 使用 Num 类型
new NetMomentumIndicator(closePrice, 20, Ta4jNumUtil.valueOf(0), Ta4jNumUtil.valueOf(0));

// 正确: 使用 Double 类型
new NetMomentumIndicator(closePrice, 20, Double.valueOf(0.0), Double.valueOf(0.0));
```

### 问题4: Bill Williams 指标类不存在

**错误信息**:
```
找不到符号: 类 AlligatorIndicator
找不到符号: 类 FractalHighIndicator
找不到符号: 类 GatorOscillatorIndicator
```

**原因分析**:
这些指标在 ta4j 0.21 中可能:
1. 还未完全实现
2. API 有变化,类名或包名不同
3. 需要额外的依赖

**解决方案**:
暂时注释相关策略,等待 ta4j 后续版本完全支持。

## 技术要点

### 1. ta4j 0.21 指标构造函数变化

ta4j 0.21 版本的指标构造函数可能与之前版本不同,需要注意:

- **SchaffTrendCycleIndicator**: 需要 5 个参数(closePrice, shortPeriod, longPeriod, cyclePeriod, smoothingPeriod)
- **NetMomentumIndicator**: 阈值参数使用 `Number` 类型,不是 `Num` 类型
- **PercentRankIndicator**: 标准的 2 个参数(indicator, barCount)
- **StochasticIndicator**: 标准的 2 个参数(indicator, barCount)

### 2. 策略命名规范

新策略使用大写下划线命名:
- `SCHAFF_TREND_CYCLE`
- `PERCENT_RANK`
- `NET_MOMENTUM`
- `STOCHASTIC_INDICATOR`

### 3. 策略注册方式

使用方法引用注册策略:
```java
strategyCreators.put("STRATEGY_NAME", StrategyFactory5::createStrategyMethod);
```
