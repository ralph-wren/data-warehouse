# Windows 路径 - 解决 260 字符路径长度限制

## 问题描述

Flink 作业在 Windows 上运行时,RocksDB Checkpoint 失败:

```
org.rocksdb.RocksDBException: Failed to create a directory: 
C:\Users\ralph\AppData\Local\Temp\minicluster_...\chk_2\d5edf7f507614b41ae7c1031b84cd176.tmp: 
文件名或扩展名太长
```

### 问题现象

- RocksDB 创建 Checkpoint 目录失败
- 错误信息显示"文件名或扩展名太长"
- Checkpoint 无法完成,作业失败

### 根本原因

**Windows 路径长度限制** ⭐⭐⭐

1. **历史限制**: Windows 传统的 MAX_PATH 限制为 260 字符
2. **路径过长**: RocksDB Checkpoint 路径包含多层嵌套目录:
   ```
   C:\Users\ralph\AppData\Local\Temp\
   minicluster_a57161c16eb756e5e20b4404ac71c523\
   tm_0\localState\
   aid_cd512b7a47ca1242cffb2996d1c61616\
   jid_89814f5631a45ac505075c102d4a07b9\
   vtx_ea632d67b7d595e5b851708ae9ad79d6_sti_2\
   chk_2\
   d5edf7f507614b41ae7c1031b84cd176.tmp
   ```
3. **超过限制**: 完整路径超过 260 字符,导致创建失败

## 解决方案

### 方案 1: 使用短路径 (快速方案) ✅ ⭐⭐⭐

**优点**: 立即生效,无需系统配置,兼容性最好

修改配置文件使用短路径:

```yaml
flink:
  state:
    # 使用短路径避免 Windows 260 字符路径长度限制
    checkpoint-dir: file:///c/flink-state/checkpoints
    savepoint-dir: file:///c/flink-state/savepoints
    local-recovery: false  # 禁用本地恢复,避免路径过长
```

**创建目录**:
```bash
mkdir -p /c/flink-state/checkpoints
mkdir -p /c/flink-state/savepoints
```

**路径对比**:
- ❌ 长路径: `C:\Users\ralph\IdeaProject\data-warehouse\rocksdb-dir\checkpoints` (60+ 字符基础路径)
- ✅ 短路径: `C:\flink-state\checkpoints` (25 字符基础路径)

### 方案 2: 启用 Windows 长路径支持 (系统级方案) ⭐⭐

**优点**: 一劳永逸,支持所有应用程序

**要求**: Windows 10 (1607+) 或 Windows 11

#### 2.1 使用 PowerShell 脚本 (推荐)

```powershell
# 以管理员身份运行 PowerShell
# 执行脚本
.\test\enable-windows-long-paths.ps1
```

脚本会自动:
1. 检查管理员权限
2. 修改注册表启用长路径
3. 配置 Git 长路径支持
4. 验证配置结果

#### 2.2 手动修改注册表

1. 按 `Win + R`,输入 `regedit`,打开注册表编辑器
2. 导航到: `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\FileSystem`
3. 找到或创建 `LongPathsEnabled` (DWORD 32位)
4. 设置值为 `1`
5. 重启应用程序 (不需要重启系统)

#### 2.3 使用组策略 (Windows 专业版/企业版)

1. 按 `Win + R`,输入 `gpedit.msc`,打开组策略编辑器
2. 导航到: `计算机配置` → `管理模板` → `系统` → `文件系统`
3. 找到 `启用 Win32 长路径`
4. 设置为 `已启用`
5. 点击 `确定`
6. 重启应用程序

#### 2.4 为 Git 启用长路径

```bash
# 全局启用
git config --global core.longpaths true

# 验证配置
git config --global core.longpaths
# 输出: true
```

### 方案 3: 使用 UNC 路径 (高级方案)

Windows 支持使用 UNC 路径绕过 MAX_PATH 限制:

```yaml
flink:
  state:
    # 使用 UNC 路径 (\\?\C:\...)
    checkpoint-dir: file:///\\?\C:\flink-state\checkpoints
    savepoint-dir: file:///\\?\C:\flink-state\savepoints
```

**注意**: 某些应用程序可能不支持 UNC 路径

## 推荐方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| 短路径 | 立即生效,兼容性好,无需权限 | 需要修改配置 | ⭐⭐⭐ |
| 启用长路径 | 一劳永逸,支持所有应用 | 需要管理员权限,需要重启应用 | ⭐⭐ |
| UNC 路径 | 绕过限制 | 兼容性差,某些应用不支持 | ⭐ |

## 技术要点

### Windows 路径限制历史

1. **MAX_PATH 常量**: 定义为 260 字符 (包括驱动器号、路径分隔符和 NULL 终止符)
2. **历史原因**: 源自 MS-DOS 时代的限制
3. **现代支持**: Windows 10 (1607+) 开始支持长路径,但需要显式启用

### 路径长度计算

```
完整路径 = 驱动器 (3) + 目录路径 + 文件名 + NULL (1)
例如: C:\ (3) + Users\ralph\... (路径) + file.tmp (文件名) + \0 (1)
```

### RocksDB Checkpoint 路径结构

```
<checkpoint-dir>/
  ├── <job-id>/
  │   ├── <operator-id>/
  │   │   ├── <checkpoint-id>/
  │   │   │   ├── <state-file-1>
  │   │   │   ├── <state-file-2>
  │   │   │   └── ...
```

每层目录都使用 UUID 或长哈希值,导致路径快速增长。

### local-recovery 的影响

启用 `local-recovery` 会在本地创建额外的状态副本:

```
<temp-dir>/
  ├── localState/
  │   ├── <aid>/
  │   │   ├── <jid>/
  │   │   │   ├── <vtx>/
  │   │   │   │   ├── <chk>/
```

这会进一步增加路径长度,因此在 Windows 上建议禁用。

## 注意事项

1. **重启应用**: 修改注册表后需要重启应用程序才能生效
2. **兼容性**: 某些旧版本的应用程序可能仍然不支持长路径
3. **最佳实践**: 即使启用了长路径支持,仍建议使用短路径以获得最佳兼容性
4. **Git 配置**: 如果项目使用 Git,记得同时配置 Git 长路径支持
5. **Docker 环境**: Docker 容器内部使用 Linux 文件系统,没有 260 字符限制

## 总结

**推荐方案**: 使用短路径 (`C:\flink-state`) ⭐⭐⭐

- 立即生效,无需系统配置
- 兼容性最好,适用于所有 Windows 版本
- 维护简单,易于理解

**可选方案**: 启用 Windows 长路径支持

- 一劳永逸,支持所有应用程序
- 需要管理员权限和重启应用
- 适合长期开发使用

两种方案可以同时使用,以获得最佳效果!
