# 数据采集作业增强 - 启动时自动计算价差订阅 Top10 币对

## 问题背景

原来的 `FlinkTickerCollectorJob` 需要手动配置要订阅的币对,不方便。而 `FlinkADSArbitrageJob` 需要手动去找现货、合约价差较大的币对做套利。

## 需求

修改 `FlinkTickerCollectorJob`,在启动时:
1. 自动调用 OKX REST API 获取所有现货和合约价格
2. 计算价差,找出价差最大的前10个币种
3. 直接订阅这10个币对的现货和合约数据
4. 无需 Redis,无需重启

## 解决方案

### 1. 创建 REST API Source

创建 `OKXRestApiSource.java`,用于获取所有现货和合约价格:

**文件位置**: `src/main/java/com/crypto/dw/flink/source/OKXRestApiSource.java`

**核心功能**:
- 调用 OKX REST API 获取所有 USDT 交易对价格
- 支持现货 (SPOT) 和合约 (SWAP)
- 计算价差并按价差率绝对值排序
- 返回价差最大的前 N 个币种

**关键方法**:
```java
// 获取所有现货价格
public Map<String, BigDecimal> fetchSpotPrices() throws Exception

// 获取所有合约价格
public Map<String, BigDecimal> fetchSwapPrices() throws Exception

// 计算价差并排序
public List<PriceSpreadInfo> calculateSpreads(
    Map<String, BigDecimal> spotPrices, 
    Map<String, BigDecimal> swapPrices)
```

### 2. 修改 FlinkTickerCollectorJob

修改 `FlinkTickerCollectorJob.java`,在启动时计算价差:

**核心逻辑**:
```java
// 步骤1: 启动时计算一次价差,获取价差最大的前10个币对
List<String> symbols = getSymbolsWithSpreadCalculation(args, config);

// 步骤2: 订阅这些币对的现货和合约数据
DataStream<String> sourceStream = env.addSource(
    new OKXWebSocketSourceFunction(config, symbols)
).name("OKX WebSocket Source");
```

**获取币对的优先级**:
1. 如果命令行提供了交易对,直接使用 (手动指定)
2. 否则,调用 REST API 计算价差,取前10个 (自动计算)
3. 如果 API 调用失败,使用配置文件中的交易对 (配置文件)
4. 如果配置文件也没有,使用默认交易对 (默认值)

### 3. 价差计算方法

```java
private static List<String> calculateTopSpreadSymbols(ConfigLoader config, int topN) {
    // 1. 创建 REST API Source 实例
    OKXRestApiSource apiSource = new OKXRestApiSource(config, 0, topN);
    
    // 2. 初始化 (配置代理等)
    apiSource.open(flinkConfig);
    
    // 3. 获取现货价格
    Map<String, BigDecimal> spotPrices = apiSource.fetchSpotPrices();
    
    // 4. 获取合约价格
    Map<String, BigDecimal> swapPrices = apiSource.fetchSwapPrices();
    
    // 5. 计算价差并排序
    List<PriceSpreadInfo> spreadList = apiSource.calculateSpreads(spotPrices, swapPrices);
    
    // 6. 取前 N 个
    List<String> topSymbols = new ArrayList<>();
    for (int i = 0; i < Math.min(topN, spreadList.size()); i++) {
        PriceSpreadInfo info = spreadList.get(i);
        topSymbols.add(info.symbol + "-USDT");
    }
    
    return topSymbols;
}
```

## 优势

1. **自动化**: 无需手动配置币对,启动时自动计算
2. **智能选择**: 自动追踪价差最大的币对,提高套利机会
3. **简单易用**: 一键启动,无需额外配置
4. **灵活性**: 支持自动计算、手动指定、配置文件三种模式
5. **无依赖**: 不需要 Redis,不需要重启

## 技术细节

### API 端点

- 现货 Ticker: `https://www.okx.com/api/v5/market/tickers?instType=SPOT`
- 合约 Ticker: `https://www.okx.com/api/v5/market/tickers?instType=SWAP`

### 价差计算公式

```
价差 = 合约价格 - 现货价格
价差率 = 价差 / 现货价格
```

### 排序规则

按价差率的绝对值降序排列,取前 N 个。

### 代理支持

如果需要代理访问 OKX API,在配置文件中启用:
```yaml
okx:
  proxy:
    enabled: true
    host: localhost
    port: 10809
```

## 注意事项

1. **API 限流**: OKX REST API 有频率限制,启动时只调用一次,不会触发限流
2. **网络问题**: 如果 API 调用失败,会自动降级到配置文件或默认币对
3. **币对格式**: 返回的币对格式为 `BTC-USDT`,会同时订阅现货和合约 (`BTC-USDT-SWAP`)
4. **Top N 数量**: 默认取前10个,可以在代码中修改 `topN` 参数
