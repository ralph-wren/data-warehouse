# 交易服务修复 - 解决 401 认证失败和 SSL 握手错误

## 问题描述

`FlinkADSArbitrageJob` 套利作业在尝试下单时出现以下错误:

```
ERROR com.crypto.dw.trading.OKXTradingService - ✗ 下单请求失败: 401
ERROR com.crypto.dw.trading.OKXTradingService - 做空现货失败: Remote host terminated the handshake
```

## 问题分析

### 1. 错误码 401 - 认证失败

**原因**:
- OKX API 认证失败
- 可能是 API Key、Secret 或 Passphrase 配置错误
- 或者签名算法有问题

**排查步骤**:
1. 检查环境变量是否设置
2. 检查 API Key 是否有效
3. 检查签名算法是否正确

### 2. SSL 握手失败 - Remote host terminated the handshake

**原因**:
- 国内访问 OKX API 需要代理
- `OKXTradingService` 的 `OkHttpClient` 没有配置代理
- 导致 SSL 握手失败

**对比**:
- `OKXRestApiSource` 已经配置了代理,可以正常访问
- `OKXTradingService` 缺少代理配置

## 解决方案

### 1. 添加 API 配置检查

在 `OKXTradingService` 构造函数中添加配置检查:

```java
// 检查 API 配置
if (!isConfigured()) {
    logger.error("✗ OKX API 未配置! 请设置环境变量: OKX_API_KEY, OKX_SECRET_KEY, OKX_PASSPHRASE");
    logger.error("✗ 当前配置: apiKey={}, secretKey={}, passphrase={}", 
        apiKey.isEmpty() ? "未设置" : "已设置", 
        secretKey.isEmpty() ? "未设置" : "已设置",
        passphrase.isEmpty() ? "未设置" : "已设置");
} else {
    logger.info("✓ OKX API 配置已加载");
}
```

### 2. 配置 OkHttpClient 代理

修改 `initHttpClient()` 方法,添加代理支持:

```java
private void initHttpClient() {
    if (httpClient == null) {
        OkHttpClient.Builder builder = new OkHttpClient.Builder()
            .connectTimeout(10, TimeUnit.SECONDS)
            .readTimeout(10, TimeUnit.SECONDS)
            .writeTimeout(10, TimeUnit.SECONDS);
        
        // 配置代理（如果启用）
        ConfigLoader config = ConfigLoader.getInstance();
        boolean proxyEnabled = config.getBoolean("okx.proxy.enabled", false);
        if (proxyEnabled) {
            String proxyHost = config.getString("okx.proxy.host", "localhost");
            int proxyPort = config.getInt("okx.proxy.port", 10809);
            
            java.net.Proxy proxy = new java.net.Proxy(
                java.net.Proxy.Type.HTTP,
                new java.net.InetSocketAddress(proxyHost, proxyPort)
            );
            builder.proxy(proxy);
            
            logger.info("✓ HTTP 客户端代理已启用: {}:{}", proxyHost, proxyPort);
        }
        
        httpClient = builder.build();
    }
}
```

## 配置说明

确保设置了以下环境变量:

```bash

export OKX_PASSPHRASE="your-passphrase"
```

在 `application-dev.yml` 中启用代理:

```yaml

  proxy:
    enabled: true  # 启用代理
    host: localhost  # 代理服务器地址
    port: 10809  # 代理服务器端口
```

## 常见问题

### Q1: 仍然出现 401 错误

**可能原因**:
1. API Key 无效或已过期
2. API Key 权限不足（需要交易权限）
3. IP 白名单限制

**解决方法**:
1. 登录 OKX 官网,检查 API Key 状态
2. 确认 API Key 有交易权限
3. 添加当前 IP 到白名单

### Q2: 仍然出现 SSL 握手错误

**可能原因**:
1. 代理服务未启动
2. 代理配置错误
3. 代理不支持 HTTPS

**解决方法**:
1. 确认代理服务正在运行
2. 检查代理地址和端口
3. 使用支持 HTTPS 的代理（如 V2Ray、Clash）

### Q3: 如何测试 API 是否可用

**测试脚本**:
```bash
# 测试公共 API（不需要认证）
curl https://www.okx.com/api/v5/public/time

# 测试私有 API（需要认证）
# 需要生成签名,建议使用 Postman 或 OKX 官方 SDK
```

## 注意事项

1. **API 密钥安全**: 不要将 API Key 提交到代码仓库
2. **模拟交易**: 建议先使用模拟交易测试
3. **风险控制**: 设置合理的交易金额和止损
4. **频率限制**: OKX API 有频率限制,避免频繁请求
