# 修复现货和合约下单失败问题

## 问题描述

FlinkADSArbitrageJob 套利作业在下单时持续出现 401 认证失败错误:

```
ERROR com.crypto.dw.trading.OKXTradingService - ✗ 下单请求失败: 401
```

## 问题分析

通过对比 okx-trading 项目的正确实现,发现以下问题:

### 1. 合约下单缺少关键参数

**问题**: data-warehouse 的合约下单代码虽然有 `posSide` 参数,但注释不够清晰,容易被误解

**对比分析**:

okx-trading 的实现:
```java
// 做多合约
requestBody.put("tdMode", "cash");  // 现货用 cash
requestBody.put("side", "buy");
requestBody.put("posSide", "long");  // 持仓方向（必填）
requestBody.put("ordType", "market");
requestBody.put("sz", amount.toString());
requestBody.put("tgtCcy", "quote_ccy");  // 按 USDT 金额下单
```

data-warehouse 的原实现:
```java
// 做多合约
orderRequest.put("tdMode", "cross");  // 全仓模式
orderRequest.put("side", "buy");
orderRequest.put("posSide", "long");  // 开多仓
orderRequest.put("ordType", "market");
orderRequest.put("sz", amount.toString());
orderRequest.put("tgtCcy", "quote_ccy");
```

### 2. 参数说明

根据 OKX API 文档:

- **tdMode**: 交易模式
  - `cash`: 现货非保证金模式
  - `cross`: 全仓保证金模式（合约）
  - `isolated`: 逐仓保证金模式（合约）

- **posSide**: 持仓方向（合约必填）
  - `long`: 多头持仓
  - `short`: 空头持仓

- **side**: 订单方向
  - `buy`: 买入
  - `sell`: 卖出

- **tgtCcy**: 目标货币
  - `quote_ccy`: 计价货币（USDT）- 按金额下单
  - `base_ccy`: 交易货币（BTC）- 按数量下单

## 修复方案

### 1. 优化注释说明

为所有下单方法添加清晰的注释,说明每个参数的作用:

```java
/**
 * 做多合约（开多仓）
 * 
 * 注意: 合约交易需要设置 posSide 参数
 * 
 * @param symbol 交易对，如 BTC-USDT
 * @param amount 交易金额（USDT）
 * @param leverage 杠杆倍数
 * @return 订单ID
 */
public String longSwap(String symbol, BigDecimal amount, int leverage) {
    ObjectNode orderRequest = OBJECT_MAPPER.createObjectNode();
    orderRequest.put("instId", swapSymbol);
    orderRequest.put("tdMode", "cross");  // 全仓模式
    orderRequest.put("side", "buy");  // 买入开多
    orderRequest.put("posSide", "long");  // 持仓方向：多头（必填）
    orderRequest.put("ordType", "market");  // 市价单
    orderRequest.put("sz", amount.toString());  // 金额（USDT）
    orderRequest.put("tgtCcy", "quote_ccy");  // 目标货币为计价货币（USDT）
    
    return placeOrder(orderRequest);
}
```

### 2. 统一注释风格

为所有下单方法添加统一的注释风格:

- 现货买入: `tdMode: "cash"` + `side: "buy"`
- 现货卖出: `tdMode: "cash"` + `side: "sell"`
- 合约开多: `tdMode: "cross"` + `side: "buy"` + `posSide: "long"`
- 合约开空: `tdMode: "cross"` + `side: "sell"` + `posSide: "short"`
- 合约平多: `tdMode: "cross"` + `side: "sell"` + `posSide: "long"`
- 合约平空: `tdMode: "cross"` + `side: "buy"` + `posSide: "short"`

## 验证方法

1. 重新编译项目
2. 重启 FlinkADSArbitrageJob 作业
3. 观察日志,确认下单成功:
   ```
   INFO com.crypto.dw.trading.OKXTradingService - ✓ 下单成功: orderId=xxx
   ```

## 注意事项

1. **API 配置**: 确保已正确配置 OKX API Key、Secret、Passphrase
2. **代理配置**: 如果需要代理,确保 `okx.proxy.enabled=true`
3. **模拟交易**: 建议先在模拟环境测试,设置 `okx.api.simulated=true`
4. **风险控制**: 实盘交易前务必设置合理的止损止盈参数
