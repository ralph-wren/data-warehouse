# 交易日志 - 添加失败黑名单和 CSV 日志记录

## 时间
2026-04-11 02:00

## 问题描述
需要实现两个功能:
1. 如果币种首次下单失败,停止后续再次下单
2. 把下单返回值存储到 logs 目录的 CSV 文件,每天一个文件

## 解决方案

### 1. 失败黑名单机制

在 `TradingDecisionProcessor` 类中添加失败币种黑名单:

```java
// 失败币种黑名单,如果首次下单失败,不再尝试
private transient Set<String> failedSymbols;
```

在 `open()` 方法中初始化:

```java
// 初始化失败币种黑名单
failedSymbols = new HashSet<>();
```

在 `openPosition()` 方法开头检查黑名单:

```java
// 检查是否在失败黑名单中
if (failedSymbols.contains(opp.symbol)) {
    logger.warn("⚠ 跳过交易: {} 在失败黑名单中", opp.symbol);
    return;
}
```

下单失败时加入黑名单:

```java
// 加入失败黑名单
failedSymbols.add(opp.symbol);
logger.warn("🚫 {} 已加入失败黑名单,不再尝试交易", opp.symbol);
```

### 2. CSV 日志记录

#### 文件格式

- 文件路径: `logs/trading_YYYYMMDD.csv`
- 每天一个文件,自动按日期切换
- CSV 格式,包含以下字段:
  * 时间
  * 币种
  * 操作 (OPEN/CLOSE)
  * 方向 (LONG_SPOT_SHORT_SWAP/SHORT_SPOT_LONG_SWAP)
  * 现货订单ID
  * 合约订单ID
  * 数量
  * 现货价格
  * 合约价格
  * 价差率
  * 状态 (SUCCESS/FAILED)
  * 错误信息

#### 实现代码

添加字段:

```java
// CSV 日志记录器
private transient BufferedWriter csvWriter;
private transient String currentLogDate;  // 当前日志文件的日期
```

初始化方法:

```java
/**
 * 初始化 CSV 日志记录器
 * 每天一个文件,格式: logs/trading_YYYYMMDD.csv
 */
private void initCsvWriter() throws IOException {
    String today = LocalDate.now().format(DateTimeFormatter.ofPattern("yyyyMMdd"));
    
    // 如果日期变了,关闭旧文件,创建新文件
    if (csvWriter != null && !today.equals(currentLogDate)) {
        csvWriter.close();
        csvWriter = null;
    }
    
    if (csvWriter == null) {
        currentLogDate = today;
        
        // 确保 logs 目录存在
        File logsDir = new File("logs");
        if (!logsDir.exists()) {
            logsDir.mkdirs();
        }
        
        File csvFile = new File(logsDir, "trading_" + today + ".csv");
        boolean isNewFile = !csvFile.exists();
        
        csvWriter = new BufferedWriter(new FileWriter(csvFile, true));
        
        // 如果是新文件,写入表头
        if (isNewFile) {
            csvWriter.write("时间,币种,操作,方向,现货订单ID,合约订单ID,数量,现货价格,合约价格,价差率,状态,错误信息\n");
            csvWriter.flush();
        }
        
        logger.info("✓ CSV 日志文件已创建: {}", csvFile.getAbsolutePath());
    }
}
```

记录日志方法:

```java
/**
 * 记录交易日志到 CSV 文件
 */
private void logTradeToCsv(
        String symbol,
        String action,
        String direction,
        String spotOrderId,
        String swapOrderId,
        BigDecimal amount,
        BigDecimal spotPrice,
        BigDecimal swapPrice,
        BigDecimal spreadRate,
        String status,
        String errorMsg
) {
    try {
        // 检查是否需要切换日志文件
        initCsvWriter();
        
        String timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
        
        csvWriter.write(String.format("%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s\n",
            timestamp,
            symbol,
            action,
            direction != null ? direction : "",
            spotOrderId != null ? spotOrderId : "",
            swapOrderId != null ? swapOrderId : "",
            amount != null ? amount.toPlainString() : "",
            spotPrice != null ? spotPrice.toPlainString() : "",
            swapPrice != null ? swapPrice.toPlainString() : "",
            spreadRate != null ? spreadRate.multiply(new BigDecimal("100")).toPlainString() + "%" : "",
            status,
            errorMsg != null ? errorMsg.replace(",", ";") : ""  // 替换逗号避免 CSV 格式错误
        ));
        csvWriter.flush();
        
    } catch (IOException e) {
        logger.error("写入 CSV 日志失败: {}", e.getMessage(), e);
    }
}
```

关闭方法:

```java
@Override
public void close() throws Exception {
    // 关闭 CSV 日志记录器
    if (csvWriter != null) {
        try {
            csvWriter.close();
            logger.info("✓ CSV 日志记录器已关闭");
        } catch (IOException e) {
            logger.error("关闭 CSV 日志记录器失败: {}", e.getMessage(), e);
        }
    }
    super.close();
}
```

## 失败场景

以下情况会触发失败黑名单:

1. **无法计算合约张数**: `calculateContractSize()` 返回 `null`
   - 原因: 无法获取交易对信息 (API 调用失败)
   
2. **计算的张数为 0**: `calculateContractSize()` 返回 `0`
   - 原因: 交易金额太小,计算出的张数小于最小下单数量
   
3. **订单提交失败**: `spotOrderId` 或 `swapOrderId` 为 `null`
   - 原因: OKX API 返回错误 (余额不足、交易对暂停等)
   
4. **下单异常**: 捕获到异常
   - 原因: 网络错误、API 错误等

## 注意事项

1. **日志文件自动切换**: 每天 00:00 后第一次写入时会自动创建新文件
2. **CSV 格式**: 错误信息中的逗号会被替换为分号,避免破坏 CSV 格式
3. **黑名单持久化**: 当前黑名单只在内存中,重启作业后会清空
4. **日志目录**: logs 目录已在 .gitignore 中,不会提交到 Git
5. **文件编码**: 使用 UTF-8 编码,支持中文

## 改进建议

1. **黑名单持久化**: 可以将黑名单存储到 Redis 或 Flink State,重启后仍然有效
2. **黑名单过期**: 可以设置黑名单过期时间,例如 24 小时后自动移除
3. **日志压缩**: 可以定期压缩旧的日志文件,节省磁盘空间
4. **日志分析**: 可以编写脚本分析 CSV 日志,统计成功率、失败原因等
