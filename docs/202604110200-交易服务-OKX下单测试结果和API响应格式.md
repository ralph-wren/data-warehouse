# OKX 交易服务测试结果和 API 响应格式

## 测试时间
2026-04-11 02:00 - 02:10

## 问题描述
OKXTradingService 下单失败,错误码 50112: `Invalid OK-ACCESS-TIMESTAMP`

## 问题原因
时间戳格式错误。原代码使用 `Instant.now().toString()` 生成 ISO 8601 格式,但 OKX API 要求的格式为 `yyyy-MM-dd'T'HH:mm:ss.SSS'Z'`

## 解决方案

### 1. 修复时间戳格式
参考 okx-trading 项目的 `SignatureUtil.getIsoTimestamp()` 方法,使用 `DateTimeFormatter` 生成正确格式的时间戳:

```java
/**
 * 获取 OKX API 要求的 ISO 时间戳
 * 
 * @return ISO 8601 格式的时间戳,例如: 2023-01-09T08:15:39.924Z
 */
private String getIsoTimestamp() {
    // OKX API 要求的时间戳格式为:yyyy-MM-dd'T'HH:mm:ss.SSSZ
    // 使用 DateTimeFormatter 确保生成精确的毫秒格式
    return DateTimeFormatter
            .ofPattern("yyyy-MM-dd'T'HH:mm:ss.SSS'Z'")
            .withZone(ZoneOffset.UTC)
            .format(Instant.now());
}
```

### 2. 修复合约交易参数
合约交易不支持 `tgtCcy` 参数,需要移除该参数:

```java
// ❌ 错误:合约不支持 tgtCcy
orderRequest.put("sz", amount.toString());
orderRequest.put("tgtCcy", "quote_ccy");

// ✅ 正确:合约只需要 sz(张数)
orderRequest.put("sz", amount.toString());  // 张数(1张 = 1 USD 价值)
```
