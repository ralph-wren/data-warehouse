# 缓存优化 - 交易对信息改为 Redis 存储

## 时间
2026-04-11 02:10

## 问题描述
`OKXTradingService` 中的 `instrumentCache` 使用内存 HashMap 存储交易对信息,存在以下问题:
1. 每个 Flink 任务实例都有独立的缓存,无法共享
2. 重启后缓存丢失,需要重新查询 OKX API
3. 多个任务实例会重复查询相同的交易对信息

## 解决方案

将 `instrumentCache` 改为使用 Redis Hash 存储,实现:
1. 多个任务实例共享缓存
2. 缓存持久化,重启后仍然有效
3. 减少对 OKX API 的调用次数

### 1. 修改字段定义

**之前 (内存 HashMap):**
```java
// 缓存交易对信息,避免重复查询
// Key: instId (如 "COMP-USDT-SWAP"), Value: InstrumentInfo
private final Map<String, InstrumentInfo> instrumentCache = new ConcurrentHashMap<>();
```

**之后 (Redis):**
```java
// Redis 连接管理器,用于缓存交易对信息
private transient RedisConnectionManager redisManager;

// Redis Hash Key,用于存储交易对信息
private static final String INSTRUMENT_CACHE_KEY = "okx:instrument:cache";
```

### 2. 初始化 Redis 连接

在构造函数中初始化 Redis 连接管理器:

```java
// 初始化 Redis 连接管理器
try {
    this.redisManager = new RedisConnectionManager(config);
    logger.info("✓ Redis 连接管理器初始化成功,用于缓存交易对信息");
} catch (Exception e) {
    logger.error("✗ Redis 连接管理器初始化失败: {}", e.getMessage(), e);
    this.redisManager = null;
}
```

### 3. 修改缓存逻辑

**查询流程:**
1. 先从 Redis Hash 中查找
2. 如果找到,反序列化为 `InstrumentInfo` 对象
3. 如果没找到,调用 OKX API 查询
4. 将查询结果序列化为 JSON,存储到 Redis Hash

**代码实现:**

```java
public InstrumentInfo getInstrumentInfo(String symbol) {
    try {
        String instId = symbol + "-USDT-SWAP";
        
        // 1. 先从 Redis 缓存中查找
        if (redisManager != null) {
            String cachedJson = redisManager.hget(INSTRUMENT_CACHE_KEY, instId);
            if (cachedJson != null && !cachedJson.isEmpty()) {
                try {
                    InstrumentInfo info = OBJECT_MAPPER.readValue(cachedJson, InstrumentInfo.class);
                    logger.debug("✓ 从 Redis 缓存获取交易对信息: {}", info);
                    return info;
                } catch (Exception e) {
                    logger.warn("解析 Redis 缓存失败,将重新查询: {}", e.getMessage());
                }
            }
        }
        
        // 2. 缓存中没有,调用 API 查询
        // ... (查询 OKX API 的代码)
        
        // 3. 缓存结果到 Redis
        if (redisManager != null) {
            try {
                String infoJson = OBJECT_MAPPER.writeValueAsString(info);
                redisManager.hset(INSTRUMENT_CACHE_KEY, instId, infoJson);
                logger.info("✓ 获取交易对信息并缓存到 Redis: {}", info);
            } catch (Exception e) {
                logger.warn("缓存交易对信息到 Redis 失败: {}", e.getMessage());
                logger.info("✓ 获取交易对信息: {}", info);
            }
        } else {
            logger.info("✓ 获取交易对信息 (Redis 未连接): {}", info);
        }
        
        return info;
    } catch (Exception e) {
        logger.error("获取交易对信息异常: {}", e.getMessage(), e);
        return null;
    }
}
```

## Redis 数据结构

使用 Redis Hash 存储交易对信息:

```
Key: okx:instrument:cache
Type: Hash

Field                   Value (JSON)
----------------------  --------------------------------------------------
BTC-USDT-SWAP          {"instId":"BTC-USDT-SWAP","ctVal":0.01,"lotSz":0.01,"minSz":0.01}
ETH-USDT-SWAP          {"instId":"ETH-USDT-SWAP","ctVal":0.1,"lotSz":0.01,"minSz":0.01}
COMP-USDT-SWAP         {"instId":"COMP-USDT-SWAP","ctVal":0.1,"lotSz":1,"minSz":1}
```

## 优势

1. **共享缓存**: 多个 Flink 任务实例共享同一个 Redis 缓存
2. **持久化**: 缓存数据持久化到 Redis,重启后仍然有效
3. **减少 API 调用**: 避免重复查询 OKX API,降低 API 调用频率
4. **容错性**: 如果 Redis 连接失败,仍然可以正常工作(直接查询 API)

## 查看缓存

使用 Redis CLI 查看缓存内容:

```bash
# 连接 Redis
redis-cli

# 查看所有交易对
HKEYS okx:instrument:cache

# 查看特定交易对信息
HGET okx:instrument:cache BTC-USDT-SWAP

# 查看所有交易对信息
HGETALL okx:instrument:cache

# 删除特定交易对缓存
HDEL okx:instrument:cache COMP-USDT-SWAP

# 清空所有缓存
DEL okx:instrument:cache
```

## 注意事项

1. **Redis 连接失败**: 如果 Redis 连接失败,会降级为直接查询 API,不影响正常功能
2. **缓存过期**: 当前实现没有设置过期时间,交易对信息会永久缓存
3. **缓存更新**: 如果 OKX 修改了交易对参数,需要手动清除缓存
4. **序列化**: 使用 Jackson 将 `InstrumentInfo` 对象序列化为 JSON 字符串

## 改进建议

1. **设置过期时间**: 可以为缓存设置过期时间,例如 24 小时
2. **缓存预热**: 在启动时预先查询常用交易对的信息
3. **缓存刷新**: 定期刷新缓存,确保数据是最新的
4. **监控**: 添加缓存命中率监控,优化缓存策略
