# OKX 交易服务下单功能修复和测试结果

## 问题描述

data-warehouse 项目中的 `OKXTradingService` 现货和合约下单功能失败,需要参考 okx-trading 项目的正确实现进行修复。

## 问题分析

### 1. 时间戳格式错误

**错误信息**: `Invalid OK-ACCESS-TIMESTAMP` (错误码 50112)

**原因**: 
- 原代码使用 `Instant.now().toString()` 生成时间戳
- 生成的格式类似 `2026-04-11T02:00:10.358Z` (ISO 8601 标准格式)
- OKX API 要求的格式为 `yyyy-MM-dd'T'HH:mm:ss.SSS'Z'` (精确到毫秒)

**解决方案**:
```java
private String getIsoTimestamp() {
    return DateTimeFormatter
            .ofPattern("yyyy-MM-dd'T'HH:mm:ss.SSS'Z'")
            .withZone(ZoneOffset.UTC)
            .format(Instant.now());
}
```

### 2. 合约交易参数错误

**错误信息**: `The instrument corresponding to this BTC-USDT-SWAP does not support the tgtCcy parameter` (错误码 59110)

**原因**:
- 现货交易支持 `tgtCcy: "quote_ccy"` 参数,可以按 USDT 金额下单
- 合约交易不支持 `tgtCcy` 参数,`sz` 参数表示张数而不是金额
- 1 张 = 1 USD 价值的合约

**解决方案**:
- 移除合约交易方法中的 `tgtCcy` 参数
- 在注释中说明 `sz` 参数的含义

### 3. 错误日志不清晰

**原问题**: 错误日志只打印原始错误代码和消息,不易理解

**解决方案**:
- 优化错误日志格式,使用树形结构显示
- 添加 `translateErrorMessage()` 方法,将常见错误代码翻译成中文说明
- 支持的错误代码:
  - 51001: 交易对不存在
  - 51008: 账户余额不足
  - 51169: 没有可平仓的持仓
  - 59110: 合约不支持 tgtCcy 参数
  - 50112: 时间戳格式错误
  - 50113: 请求过期
  - 50111: 签名验证失败
  - 51020/51021: 订单金额限制
  - 51119/51120: 仓位模式/杠杆错误

## 修复内容

### 1. 修复时间戳生成

**文件**: `src/main/java/com/crypto/dw/trading/OKXTradingService.java`

- 添加 `getIsoTimestamp()` 方法
- 在 `placeOrder()` 和 `setLeverage()` 方法中使用正确的时间戳格式
- 添加必要的 import: `ZoneOffset`, `DateTimeFormatter`

### 2. 修复合约交易参数

**修改的方法**:
- `longSwap()` - 做多合约
- `shortSwap()` - 做空合约
- `closeLongSwap()` - 平多仓
- `closeShortSwap()` - 平空仓

**修改内容**:
- 移除 `orderRequest.put("tgtCcy", "quote_ccy")`
- 更新注释说明 `sz` 参数表示张数

### 3. 优化错误日志

**新增方法**: `translateErrorMessage(String errorCode, String originalMsg)`

**日志格式优化**:
```
✗ 下单失败!
  └─ 错误代码: 1 (详细代码: 51001)
  └─ 错误原因: 交易对不存在 - 该币种在 OKX 上不存在或交易对名称错误
  └─ 原始消息: All operations failed | Instrument ID doesn't exist
```

## 下一步优化建议

### 1. 交易对验证
- 在套利作业启动时,调用 OKX REST API 获取可用交易对列表
- 过滤掉不存在的交易对,避免下单失败
- 定期更新交易对列表

### 2. 合约交易金额转换
- 当前合约交易的 `sz` 参数直接使用 USDT 金额
- 应该根据当前价格计算需要的张数
- 公式: `张数 = 金额 / 当前价格`

### 3. 账户余额检查
- 在下单前检查账户余额是否充足
- 避免因余额不足导致下单失败

### 4. 重试机制
- 对于网络错误、临时性错误,添加重试机制
- 对于业务错误(如交易对不存在),不应重试
