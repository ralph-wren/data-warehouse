# 持仓限制 - 实现最大持仓数量限制功能

## 问题描述

用户配置了 `max-positions: 1`，但仍有多个币种同时下单。

## 根本原因

1. Flink 使用 KeyedStream 按 symbol 分组，每个币种独立处理
2. 开仓逻辑中没有检查全局持仓数量
3. 需要使用 Redis 实现跨 Flink 任务实例的全局持仓计数

## 解决方案

### 1. 在 RedisConnectionManager 中添加原子性计数器操作

添加了三个方法支持原子性的计数器操作：

```java
// 原子性递增计数器
public long incr(String key)

// 原子性递减计数器  
public long decr(String key)

// 获取计数器的值
public long getCounter(String key)
```

这些方法使用 Redis 的 INCR/DECR 命令，保证了并发安全。

### 2. 在 TradingDecisionProcessor 中实现持仓计数

#### 2.1 添加字段和初始化

```java
private static final String POSITION_COUNT_KEY = "okx:arbitrage:position:count";
private final int maxPositions;
private transient RedisConnectionManager redisManager;
```

在 `open()` 方法中初始化 Redis 连接：

```java
@Override
public void open(Configuration parameters) throws Exception {
    super.open(parameters);
    
    // 初始化 Redis 连接管理器
    redisManager = new RedisConnectionManager(config);
    logger.info("✓ Redis 连接管理器已初始化");
    
    // ... 其他初始化代码
}
```

#### 2.2 开仓前检查持仓数量

在 `openPosition()` 方法开头添加检查逻辑：

```java
// 检查是否达到最大持仓数量
long currentPositionCount = redisManager.getCounter(POSITION_COUNT_KEY);
if (currentPositionCount >= maxPositions) {
    logger.info("⏸ 已达到最大持仓数量 {}/{}，跳过开仓: {}", 
        currentPositionCount, maxPositions, opp.symbol);
    return;
}
```

#### 2.3 开仓成功后增加计数

在订单提交成功后，使用原子性操作增加持仓计数：

```java
// 增加全局持仓计数 (原子性操作)
long currentCount = redisManager.incr(POSITION_COUNT_KEY);
logger.info("📈 持仓数量增加: {} -> {}/{}", opp.symbol, currentCount, maxPositions);
```

#### 2.4 平仓成功后减少计数

在平仓订单提交成功后，使用原子性操作减少持仓计数：

```java
// 减少全局持仓计数 (原子性操作)
long currentCount = redisManager.decr(POSITION_COUNT_KEY);
logger.info("📉 持仓数量减少: {} -> {}/{}", pos.getSymbol(), currentCount, maxPositions);
```

## 技术要点

### 1. 原子性保证

使用 Redis 的 INCR/DECR 命令，这些命令是原子性的，即使多个 Flink 任务实例并发执行也能保证计数准确。

### 2. 全局共享

Redis Key `okx:arbitrage:position:count` 在所有 Flink 任务实例之间共享，实现了真正的全局持仓限制。

### 3. 时序保证

- 开仓前检查：防止超过限制
- 开仓后增加：确保计数准确
- 平仓后减少：释放持仓名额

### 4. 日志输出

每次持仓数量变化都会输出清晰的日志：
- `📈 持仓数量增加: COMP -> 1/1`
- `📉 持仓数量减少: COMP -> 0/1`
- `⏸ 已达到最大持仓数量 1/1，跳过开仓: PENDLE`

## 配置说明

在 `application-dev.yml` 中配置最大持仓数量：

```yaml
arbitrage:
  max-positions: 1  # 最大持仓数量
```

## 测试验证

1. 配置 `max-positions: 1`
2. 启动 Flink 作业
3. 观察日志输出：
   - 第一个币种开仓成功，持仓计数变为 1
   - 第二个币种尝试开仓时被拦截，日志显示"已达到最大持仓数量"
   - 第一个币种平仓后，持仓计数变为 0
   - 其他币种可以继续开仓

## 注意事项

1. Redis 必须正常运行，否则持仓限制功能无法工作
2. 如果 Redis 重启，持仓计数会重置为 0，需要手动同步实际持仓状态
3. 建议在作业启动时检查 Redis 中的持仓计数是否与实际持仓一致
