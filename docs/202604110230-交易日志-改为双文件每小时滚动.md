# 交易日志 - 改为双文件每小时滚动

## 需求描述

将交易日志改为两个独立的 CSV 文件，每小时滚动一次，存放在 `trading` 目录下：

1. **订单明细日志**：每条订单一行，包含所有详细信息（数量、成交价格、手续费等）
2. **持仓汇总日志**：开仓一条，平仓一条，包含做多/做空的汇总信息

## 实现方案

### 1. 文件命名规则

**目录结构**：
```
trading/
├── order_detail_2026041113.csv      # 订单明细 (2026-04-11 13:00-13:59)
├── order_detail_2026041114.csv      # 订单明细 (2026-04-11 14:00-14:59)
├── position_summary_2026041113.csv  # 持仓汇总 (2026-04-11 13:00-13:59)
└── position_summary_2026041114.csv  # 持仓汇总 (2026-04-11 14:00-14:59)
```

**命名格式**：
- 订单明细：`order_detail_YYYYMMDDHH.csv`
- 持仓汇总：`position_summary_YYYYMMDDHH.csv`

### 2. 订单明细日志格式

**文件名**：`order_detail_YYYYMMDDHH.csv`

**表头**：
```csv
时间,币种,订单ID,订单类型,方向,数量,价格,成交价格,成交数量,手续费,手续费币种,状态,错误信息
```

**字段说明**：
- **时间**：订单时间（yyyy-MM-dd HH:mm:ss）
- **币种**：交易币种（如 BTC、ETH）
- **订单ID**：OKX 订单 ID
- **订单类型**：SPOT_BUY, SPOT_SELL, SWAP_LONG, SWAP_SHORT, SWAP_CLOSE_LONG, SWAP_CLOSE_SHORT
- **方向**：BUY, SELL
- **数量**：下单数量
- **价格**：下单价格
- **成交价格**：实际成交价格
- **成交数量**：实际成交数量
- **手续费**：交易手续费
- **手续费币种**：手续费使用的币种
- **状态**：SUCCESS, FAILED, PENDING
- **错误信息**：失败原因

**示例数据**：
```csv
2026-04-11 13:29:34,OL,3468957034484097024,SPOT_BUY,BUY,14,0.00691,0.00691,14,0.0001,USDT,SUCCESS,
2026-04-11 13:29:34,OL,3468957043141140480,SWAP_SHORT,SELL,14,0.00695,0.00695,14,0.0002,USDT,SUCCESS,
```

### 3. 持仓汇总日志格式

**文件名**：`position_summary_YYYYMMDDHH.csv`

**表头**：
```csv
时间,币种,操作,方向,现货订单ID,合约订单ID,数量,开仓现货价,开仓合约价,平仓现货价,平仓合约价,价差率,盈亏,持仓时长(秒),状态
```

**字段说明**：
- **时间**：操作时间（yyyy-MM-dd HH:mm:ss）
- **币种**：交易币种
- **操作**：OPEN（开仓）, CLOSE（平仓）
- **方向**：LONG_SPOT_SHORT_SWAP（做多现货做空合约）, SHORT_SPOT_LONG_SWAP（做空现货做多合约）
- **现货订单ID**：现货订单 ID
- **合约订单ID**：合约订单 ID
- **数量**：交易数量
- **开仓现货价**：开仓时的现货价格
- **开仓合约价**：开仓时的合约价格
- **平仓现货价**：平仓时的现货价格
- **平仓合约价**：平仓时的合约价格
- **价差率**：价差率（百分比）
- **盈亏**：盈亏金额（USDT）
- **持仓时长(秒)**：持仓时长（秒）
- **状态**：SUCCESS, FAILED

**示例数据**：
```csv
2026-04-11 13:29:34,OL,OPEN,LONG_SPOT_SHORT_SWAP,3468957034484097024,3468957043141140480,14,0.00691,0.00695,,,57.89%,,,SUCCESS
2026-04-11 13:35:20,OL,CLOSE,LONG_SPOT_SHORT_SWAP,3468960123456789012,3468960234567890123,14,0.00691,0.00695,0.00688,0.00692,-52.30%,0.42,346,SUCCESS
```

## 代码实现

### 1. 字段定义

```java
// 新增: CSV 日志记录器 - 改为两个文件
private transient BufferedWriter orderDetailWriter;  // 订单明细日志
private transient BufferedWriter positionSummaryWriter;  // 持仓汇总日志
private transient String currentLogDate;  // 当前日志文件的日期
private transient int currentLogHour;  // 当前日志文件的小时
```

### 2. 初始化方法

```java
/**
 * 初始化 CSV 日志记录器
 * 每小时2个文件:
 * 1. trading/order_detail_YYYYMMDDHH.csv - 订单明细
 * 2. trading/position_summary_YYYYMMDDHH.csv - 持仓汇总
 */
private void initCsvWriters() throws IOException {
    LocalDateTime now = LocalDateTime.now();
    String today = now.format(DateTimeFormatter.ofPattern("yyyyMMdd"));
    int hour = now.getHour();
    
    // 如果日期或小时变了,关闭旧文件,创建新文件
    if (orderDetailWriter != null && (!today.equals(currentLogDate) || hour != currentLogHour)) {
        orderDetailWriter.close();
        orderDetailWriter = null;
    }
    if (positionSummaryWriter != null && (!today.equals(currentLogDate) || hour != currentLogHour)) {
        positionSummaryWriter.close();
        positionSummaryWriter = null;
    }
    
    if (orderDetailWriter == null || positionSummaryWriter == null) {
        currentLogDate = today;
        currentLogHour = hour;
        
        // 确保 trading 目录存在
        File tradingDir = new File("trading");
        if (!tradingDir.exists()) {
            tradingDir.mkdirs();
        }
        
        String hourStr = String.format("%02d", hour);
        
        // 1. 订单明细日志
        File orderDetailFile = new File(tradingDir, "order_detail_" + today + hourStr + ".csv");
        boolean isNewOrderFile = !orderDetailFile.exists();
        orderDetailWriter = new BufferedWriter(new FileWriter(orderDetailFile, true));
        
        if (isNewOrderFile) {
            orderDetailWriter.write("时间,币种,订单ID,订单类型,方向,数量,价格,成交价格,成交数量,手续费,手续费币种,状态,错误信息\n");
            orderDetailWriter.flush();
        }
        
        // 2. 持仓汇总日志
        File positionSummaryFile = new File(tradingDir, "position_summary_" + today + hourStr + ".csv");
        boolean isNewSummaryFile = !positionSummaryFile.exists();
        positionSummaryWriter = new BufferedWriter(new FileWriter(positionSummaryFile, true));
        
        if (isNewSummaryFile) {
            positionSummaryWriter.write("时间,币种,操作,方向,现货订单ID,合约订单ID,数量,开仓现货价,开仓合约价,平仓现货价,平仓合约价,价差率,盈亏,持仓时长(秒),状态\n");
            positionSummaryWriter.flush();
        }
        
        logger.info("✓ CSV 日志文件已创建:");
        logger.info("  订单明细: {}", orderDetailFile.getAbsolutePath());
        logger.info("  持仓汇总: {}", positionSummaryFile.getAbsolutePath());
    }
}
```

## 技术要点

### 1. 每小时自动滚动

- 每次写入前检查当前日期和小时
- 如果日期或小时变化，自动关闭旧文件，创建新文件
- 文件名包含日期和小时，便于按时间查找

### 2. 自动创建目录

```java
File tradingDir = new File("trading");
if (!tradingDir.exists()) {
    tradingDir.mkdirs();
}
```

### 3. 追加模式写入

```java
new BufferedWriter(new FileWriter(file, true))  // true 表示追加模式
```

### 4. 自动写入表头

```java
if (isNewFile) {
    writer.write("表头内容\n");
    writer.flush();
}
```

### 5. 兼容旧代码

保留了旧的 `logTradeToCsv` 方法，内部调用新方法，确保兼容性。

## 优势

1. **结构清晰**：订单明细和持仓汇总分开，便于分析
2. **自动滚动**：每小时自动创建新文件，避免单个文件过大
3. **易于查找**：文件名包含日期和小时，便于按时间查找
4. **详细信息**：订单明细包含所有交易信息，便于对账
5. **汇总信息**：持仓汇总一目了然，便于统计盈亏
