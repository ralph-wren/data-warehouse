## 时间
2026-04-11 02:35 (初始实现)

## 问题描述
用户要求将交易日志改为双文件每小时滚动:
1. 日志放到 `logs/trading` 目录 (更新)
2. 每小时 2 个 CSV 文件
3. 订单明细文件: 每条订单一行,包含数量、成交价格、手续费、手续费币种等
4. 持仓汇总文件: 开仓一条,平仓一条
5. 现货下单失败时,记录失败原因到 CSV 文件 (新增)

## 日志记录流程

### 开仓流程
1. `openPosition()`: 下单时记录订单明细(状态: SUBMITTED)
2. `handleOrderFilled()`: 接收订单成交推送,保存手续费信息
3. `confirmOpen()`: 两个订单都成交后,记录完整的订单明细(状态: FILLED)和持仓汇总

### 平仓流程
1. `closePosition()`: 下单时记录订单明细(状态: SUBMITTED)
2. `handleOrderFilled()`: 接收订单成交推送,保存手续费信息
3. `confirmClose()`: 两个订单都成交后,记录完整的订单明细(状态: FILLED)和持仓汇总(包含盈亏和持仓时长)

## 关键改进
1. 订单明细记录两次: 下单时(SUBMITTED)和成交时(FILLED)
2. 持仓汇总只记录一次: 成交确认时
3. 手续费信息从 OKX WebSocket 订单推送中获取
4. 盈亏计算: (平仓价差 - 开仓价差) × 数量
5. 持仓时长: 平仓时间 - 开仓时间(秒)

### 2. 记录现货下单失败原因
在现货下单失败时,立即记录订单明细到 CSV 文件:
```java

        null,
        null,

```

**改进点**:
- 现货下单失败时,立即记录到 CSV 文件
- 订单ID 为空,状态为 FAILED
- 错误信息字段记录失败原因: "现货下单失败,跳过合约下单"
- 方便排查问题,了解哪些币种下单失败
