# 现货杠杆下单失败问题修复

**日期**: 2026-04-11 15:30  
**问题**: OKX 现货杠杆下单一直失败,错误代码 51020/59110/50014  
**状态**: ✅ 已解决

---

## 问题描述

在 `FlinkADSArbitrageJob` 中使用现货杠杆模式下单时,一直收到以下错误:

```
错误 51020: Your order should meet or exceed the minimum order amount.
错误 59110: The instrument corresponding to this BTC-USDT does not support the tgtCcy parameter.
错误 50014: Parameter ccy can not be empty.
```

---

## 问题分析

通过创建测试脚本逐步诊断,发现了三个关键问题:

### 1. 不支持 `tgtCcy` 参数 (错误 59110)

**原因**: 现货杠杆模式 (`tdMode: cross`) 不支持 `tgtCcy` 参数

**原代码**:
```java
orderRequest.put("tgtCcy", "base_ccy");  // ❌ 杠杆模式不支持
```

**修复**:
```java
// 移除 tgtCcy 参数,直接使用 sz 表示币数量
```

### 2. 科学计数法格式问题 (错误 51000)

**原因**: `BigDecimal.toString()` 可能返回科学计数法(如 `1E-5`),OKX API 不接受

**原代码**:
```java
orderRequest.put("sz", size.toString());  // ❌ 可能产生 1E-5
```

**修复**:
```java
orderRequest.put("sz", size.toPlainString());  // ✓ 使用普通格式 0.00001
```

### 3. 缺少保证金币种参数 (错误 50014)

**原因**: 杠杆模式必须指定保证金币种 `ccy` 参数

**原代码**:
```java
// 缺少 ccy 参数
```

**修复**:
```java
orderRequest.put("ccy", "USDT");  // ✓ 指定保证金币种
```

## 解决方案

### 修改文件: `OKXTradingService.java`

#### 1. 修复 `buySpot()` 方法

```java
public String buySpot(String symbol, BigDecimal size) {
    try {
        // 拼接完整交易对: BTC → BTC-USDT
        String instId = symbol + "-USDT";
        logger.info("📈 做多现货（买入）: {} | 数量: {} {}", instId, size, symbol);
        
        ObjectNode orderRequest = OBJECT_MAPPER.createObjectNode();
        orderRequest.put("instId", instId);
        orderRequest.put("tdMode", "cross");  // 全仓杠杆模式
        orderRequest.put("ccy", "USDT");  // ✅ 保证金币种(杠杆模式必需)
        orderRequest.put("side", "buy");
        orderRequest.put("ordType", "market");
        // ✅ 移除 tgtCcy 参数
        // ✅ 使用 toPlainString() 避免科学计数法
        orderRequest.put("sz", size.toPlainString());
        
        return placeOrder(orderRequest);
        
    } catch (Exception e) {
        logger.error("做多现货失败: {}", e.getMessage(), e);
        return null;
    }
}
```

## OKX 现货杠杆下单参数说明

### 市价单参数 (Java 代码使用)

```json
{
  "instId": "BTC-USDT",      // 交易对
  "tdMode": "cross",         // 交易模式: cross(全仓杠杆) / isolated(逐仓杠杆) / cash(普通现货)
  "ccy": "USDT",             // 保证金币种(杠杆模式必需)
  "side": "buy",             // 方向: buy / sell
  "ordType": "market",       // 订单类型: market(市价单)
  "sz": "0.0001"             // 数量(必须使用普通格式,不能用科学计数法)
}
```

### 限价单参数 (测试使用)

```json
{
  "instId": "BTC-USDT",      // 交易对
  "tdMode": "cross",         // 交易模式
  "ccy": "USDT",             // 保证金币种
  "side": "buy",             // 方向
  "ordType": "limit",        // 订单类型: limit(限价单)
  "px": "72800.0",           // 限价(限价单必需)
  "sz": "0.0001"             // 数量
}
```

### 市价单 vs 限价单

| 特性 | 市价单 (market) | 限价单 (limit) |
|------|----------------|---------------|
| 成交速度 | 立即成交 | 等待价格达到限价 |
| 价格参数 | 不需要 `px` | 必需 `px` 参数 |
| 成交保证 | 保证成交 | 不保证成交 |
| 价格控制 | 按市场价 | 按指定价格 |
| 适用场景 | 快速成交 | 价格控制 |

### 不支持的参数

- `tgtCcy` - 杠杆模式不支持此参数(仅普通现货支持)

## 注意事项

1. **数量格式**: 必须使用 `toPlainString()` 避免科学计数法
2. **保证金币种**: 杠杆模式必须指定 `ccy` 参数
3. **最小下单金额**: 根据交易对不同,通常为 5-10 USDT
4. **账户等级**: 需要账户等级 >= 2 (单币种保证金模式)
5. **API 权限**: 确保 API Key 有交易权限
