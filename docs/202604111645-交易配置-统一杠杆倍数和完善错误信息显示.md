# 交易配置优化 - 统一杠杆倍数和完善错误信息显示

## 问题描述

用户反馈了三个问题：

1. **杠杆倍数不一致**：现货使用5倍杠杆，合约使用1倍杠杆，配置不统一
2. **错误信息不显示**：订单失败时，order_detail.csv 中没有显示 OKX 返回的详细错误原因（sMsg 字段）
3. **持仓限制失效**：ZK 币种现货卖出、合约买入后还没平仓，就开始开仓其他币种，违反了"同时只允许开仓1个币种"的配置

## 解决方案

### 1. 统一杠杆倍数配置

#### 配置文件修改

在 `application-dev.yml` 中添加统一的杠杆倍数配置：

```yaml
arbitrage:
  trading:
    leverage: 1  # 杠杆倍数(默认1倍,现货和合约统一使用)
```

#### 代码修改

**FlinkADSArbitrageJob.java**：
- 在 `TradingDecisionProcessor` 类中添加 `leverage` 字段
- 在构造函数中读取配置：`this.leverage = config.getInt("arbitrage.trading.leverage", 1);`
- 修改 `openPosition()` 方法中的合约下单调用，使用配置的杠杆倍数：
  ```java
  swapOrderId = tradingService.shortSwap(opp.symbol, contractSize, leverage);
  swapOrderId = tradingService.longSwap(opp.symbol, contractSize, leverage);
  ```

**TradingDecisionProcessor.java**（旧版本）：
- 添加 `leverage` 字段
- 在构造函数中读取配置
- 修改硬编码的杠杆倍数 `1` 为配置的 `leverage`

### 2. 完善错误信息显示

#### OKXTradingService.java

在 `placeOrder()` 方法中，已经实现了错误信息的提取和保存：

```java
// 检查 data 数组中的详细错误
if (jsonResponse.has("data") && jsonResponse.get("data").isArray() 
    && jsonResponse.get("data").size() > 0) {
    JsonNode dataNode = jsonResponse.get("data").get(0);
    String sCode = dataNode.get("sCode").asText();
    String sMsg = dataNode.get("sMsg").asText();
    
    // 保存错误信息供外部获取
    lastErrorMessage = sMsg;
    
    logger.error("✗ 下单失败!");
    logger.error("  └─ 错误代码: {} (详细代码: {})", errorCode, sCode);
    logger.error("  └─ 错误原因: {}", translateErrorMessage(sCode, sMsg));
    logger.error("  └─ 原始消息: {} | {}", errorMsg, sMsg);
}
```

#### FlinkADSArbitrageJob.java

在 `openPosition()` 方法中，下单失败时获取详细错误信息：

```java
if (spotOrderId == null) {
    // 获取详细错误信息

    }
    logger.error("❌ {}: {}", opp.symbol, errorMsg);

        "BUY",
        contractSize,
        null,
        null,
        null,
        null,
        null,
        "FAILED",
        errorMsg  // 详细错误信息
    );
}
```

### 3. 修复持仓限制问题

#### 问题根源

在 `closePosition()` 方法中，平仓订单提交后立即减少持仓计数，但此时订单还未真正成交。这导致：
1. 平仓订单提交 → 持仓计数减少（从1变为0）
2. 系统认为没有持仓，允许开新仓
3. 但实际上旧的持仓还没平掉

#### 解决方案

**修改 closePosition() 方法**：
- 移除订单提交时立即减少持仓计数的逻辑
- 添加注释说明应该在 `confirmClose()` 中确认成交后再减少

```java
// 注意: 不在这里减少持仓计数,而是在 confirmClose 中确认成交后再减少
// 这样可以避免订单还没成交就开新仓的问题
```

**修改 confirmClose() 方法**：
- 在平仓确认成交后，减少全局持仓计数

```java
// 平仓成功后，减少全局持仓计数
if (redisManager != null) {
    try {
        long currentCount = redisManager.decr(POSITION_COUNT_KEY);
        logger.info("📉 持仓数量减少: {} -> {}/{}", pending.symbol, currentCount, maxPositions);
    } catch (Exception e) {
        logger.error("减少持仓计数失败: {}", e.getMessage(), e);
    }
}
```

## 注意事项

1. **杠杆倍数配置**：
   - 默认值为 1 倍（无杠杆）
   - 可以根据风险承受能力调整，但要注意爆仓风险
   - 现货和合约使用相同的杠杆倍数，确保策略一致性

2. **错误信息记录**：
   - 错误信息来自 OKX API 返回的 `data[0].sMsg` 字段
   - 如果没有 `sMsg`，则使用顶层的 `msg` 字段
   - 错误信息中的逗号会被替换为分号，避免破坏 CSV 格式

3. **持仓限制**：
   - 使用 Redis 原子操作（incr/decr）确保并发安全
   - 开仓时先预留名额（incr），失败时回退（decr）
   - 平仓时在订单成交确认后才释放名额（decr）
   - 这样可以严格控制同时持仓数量，避免超限
