# 杠杆检查 - 修复策略跳过不加入黑名单问题

## 问题描述

在套利交易中，当币对不支持杠杆交易时，策略B（做空现货+做多合约）会被跳过，但币种被错误地加入了失败黑名单，导致策略A（做多现货+做空合约）也无法执行。

### 具体表现

```
查询杠杆支持失败: Instrument ID doesn't exist
⚠ SAHARA: 币对不支持杠杆交易,无法做空现货
⚠ 订单提交失败: spotOrderId=null, swapOrderId=null
↩️ 开仓失败，回退持仓名额: SAHARA | 当前持仓: 0/1
🚫 SAHARA 已加入失败黑名单,不再尝试交易
```

### 问题分析

1. 不支持杠杆的币种（如 SAHARA）在策略B失败后，被加入了黑名单
2. 但实际上这些币种仍然可以执行策略A（做多现货+做空合约）
3. 策略选择失败（不适用）和真正的下单失败应该区分对待

## 解决方案

### 1. 修复 checkMarginSupport 方法的 instId 拼接问题

**文件**: `src/main/java/com/crypto/dw/trading/OKXTradingService.java`

**问题**: `symbol` 可能已经包含 `-USDT` 后缀，导致拼接后变成 `BTC-USDT-USDT`

**修复**:
```java
// 修复前
String instId = symbol + "-USDT";

// 修复后
String instId = symbol.contains("-") ? symbol : (symbol + "-USDT");
```

同时优化错误处理：
- 当 API 返回 "Instrument ID doesn't exist" 时，直接返回 `false`（不支持杠杆）
- 避免将 API 错误误认为是支持杠杆

### 2. 区分"策略跳过"和"下单失败"

**文件**: `src/main/java/com/crypto/dw/flink/FlinkADSArbitrageJob.java`

**核心改动**:

1. 添加 `strategySkipped` 标志变量：
```java
boolean strategySkipped = false;  // 标记策略是否因不适用而跳过
```

2. 当策略B因不支持杠杆而跳过时，设置标志：
```java
if (!marginSupported) {
    errorMsg = "币对不支持杠杆交易,无法做空现货";
    logger.warn("⚠ {}: {}", opp.symbol, errorMsg);
    
    // 标记为策略跳过（不是真正的下单失败）
    strategySkipped = true;
    
    // 记录状态改为 SKIPPED
    logOrderDetail(..., "SKIPPED", errorMsg);
    
    // 不执行下单，直接跳过
    spotOrderId = null;
    swapOrderId = null;
}
```

3. 只有真正的下单失败才加入黑名单：
```java
if (!orderSuccess) {
    logTradeToCsv(
        ...,
        strategySkipped ? "SKIPPED" : "FAILED",  // 区分跳过和失败
        errorMsg
    );
    
    // 只有真正的下单失败才加入黑名单
    if (!strategySkipped) {
        failedSymbols.add(opp.symbol);
        logger.warn("🚫 {} 已加入失败黑名单,不再尝试交易", opp.symbol);
    } else {
        logger.info("ℹ️ {} 策略B跳过（不支持杠杆），但仍可执行策略A", opp.symbol);
    }
}
```

## 效果

### 修复前
- 不支持杠杆的币种被加入黑名单
- 策略A（做多现货+做空合约）也无法执行
- 减少了可交易的币种数量

### 修复后
- 不支持杠杆的币种只跳过策略B
- 策略A（做多现货+做空合约）仍然可以执行
- 订单状态记录为 `SKIPPED` 而不是 `FAILED`
- 日志清晰显示：`ℹ️ SAHARA 策略B跳过（不支持杠杆），但仍可执行策略A`

## 策略说明

### 策略A：做多现货 + 做空合约
- 现货使用 `cash` 模式（现货模式）
- 不需要借币，不需要杠杆
- 所有币种都可以执行

### 策略B：做空现货 + 做多合约
- 现货使用 `cross` 模式（杠杆模式）
- 需要借币，需要杠杆支持
- 只有支持杠杆的币种才能执行

## 验证方法

1. 观察日志，确认不支持杠杆的币种：
   - 策略B被跳过，状态为 `SKIPPED`
   - 不会被加入黑名单
   - 仍然可以执行策略A

2. 检查 `order_detail.csv` 文件：
   - 策略跳过的记录状态为 `SKIPPED`
   - 真正下单失败的记录状态为 `FAILED`

3. 检查 Redis 黑名单：
   - 只有真正下单失败的币种才在黑名单中
   - 策略跳过的币种不在黑名单中

## 总结

通过区分"策略不适用"和"真正的下单失败"，避免了将不支持杠杆的币种错误地加入黑名单，提高了系统的交易机会和灵活性。
