# 杠杆检查 - 开仓前检查币对是否支持杠杆交易

## 问题描述

在实际交易中发现，很多币种（如 NMR、SAHARA）不支持杠杆交易，导致下单时报错：

```
"sCode":"54000","sMsg":"Margin trading is not supported."
```

这些币种只能使用现货模式（`tdMode: cash`），不能使用杠杆模式（`tdMode: cross`）。

## 需求分析

用户需求：
1. **开仓前检查币对是否支持杠杆交易**
2. **如果不支持杠杆**：只允许"现货买入+合约卖出"策略（使用现货模式 `cash`）
3. **如果支持杠杆**：允许两种策略
   - 现货买入+合约卖出（现货模式 `cash`）
   - 现货卖出+合约买入（杠杆模式 `cross`，需要借币）

## 解决方案

### 1. 添加杠杆支持检查方法

在 `OKXTradingService.java` 中添加 `checkMarginSupport()` 方法：

```java
/**
 * 查询币种是否支持杠杆交易
 * 
 * @param symbol 币种符号（如 BTC）
 * @return true=支持杠杆，false=不支持杠杆
 */
public boolean checkMarginSupport(String symbol) {
    try {
        initHttpClient();
        
        // 查询现货杠杆交易对信息
        // API: GET /api/v5/public/instruments?instType=MARGIN&instId=BTC-USDT
        String instId = symbol + "-USDT";
        String url = baseUrl + "/api/v5/public/instruments?instType=MARGIN&instId=" + instId;
        
        Request request = new Request.Builder()
            .url(url)
            .get()
            .build();
        
        try (Response response = httpClient.newCall(request).execute()) {
            String responseBody = response.body() != null ? response.body().string() : "";
            
            if (response.isSuccessful() && !responseBody.isEmpty()) {
                JsonNode jsonResponse = OBJECT_MAPPER.readTree(responseBody);
                
                if ("0".equals(jsonResponse.get("code").asText())) {
                    JsonNode dataArray = jsonResponse.get("data");
                    
                    // 如果 data 数组不为空，说明支持杠杆交易
                    if (dataArray != null && dataArray.isArray() && dataArray.size() > 0) {
                        logger.info("✓ {} 支持杠杆交易", symbol);
                        return true;
                    } else {
                        logger.info("✗ {} 不支持杠杆交易", symbol);
                        return false;
                    }
                } else {
                    logger.warn("查询杠杆支持失败: {}", jsonResponse.get("msg").asText());
                    return false;
                }
            } else {
                logger.error("查询杠杆支持请求失败: HTTP {}", response.code());
                return false;
            }
        }
        
    } catch (Exception e) {
        logger.error("查询杠杆支持异常: {}", e.getMessage(), e);
        return false;
    }
}
```

### 2. 修改开仓逻辑

在 `FlinkADSArbitrageJob.java` 的 `openPosition()` 方法中，添加杠杆检查：

```java
try {
    // 检查币对是否支持杠杆交易
    boolean marginSupported = tradingService.checkMarginSupport(opp.symbol);
    
    if (opp.arbitrageDirection.contains("做多现货")) {
        // 策略 A: 做多现货 + 做空合约（现货模式，不需要杠杆）
        direction = "LONG_SPOT_SHORT_SWAP";
        
        // 先下现货单（传入价格用于检查订单金额）
        spotOrderId = tradingService.buySpot(opp.symbol, contractSize, opp.spotPrice);
        
        // ... 后续逻辑
    } else {
        // 策略 B: 做空现货 + 做多合约（需要杠杆模式）
        
        // 检查是否支持杠杆交易
        if (!marginSupported) {
            errorMsg = "币对不支持杠杆交易,无法做空现货";
            logger.warn("⚠ {}: {}", opp.symbol, errorMsg);
            
            // 记录失败日志
            logOrderDetail(
                opp.symbol,
                null,
                "SPOT",
                "SELL",
                contractSize,
                null,
                null,
                null,
                null,
                null,
                "FAILED",
                errorMsg
            );
            
            // 不执行下单，直接跳过
            spotOrderId = null;
            swapOrderId = null;
        } else {
            // 支持杠杆，可以做空现货
            direction = "SHORT_SPOT_LONG_SWAP";
            
            // 先下现货单（会自动查询借币利息，并检查订单金额）
            spotOrderId = tradingService.sellSpot(opp.symbol, contractSize, opp.spotPrice);
            
            // ... 后续逻辑
        }
    }
}
```

### 3. 交易模式说明

#### 现货模式（cash）
- **使用场景**：现货买入（不需要杠杆）
- **特点**：
  - 不需要借币
  - 所有币种都支持
  - 适用于"做多现货+做空合约"策略

#### 杠杆模式（cross）
- **使用场景**：现货卖出（需要借币做空）
- **特点**：
  - 需要借币
  - 只有部分币种支持
  - 适用于"做空现货+做多合约"策略
  - 需要支付借币利息

## 策略选择逻辑

```
开仓前检查
    ↓
是否支持杠杆？
    ├─ 是 → 允许两种策略
    │       ├─ 策略A：现货买入（cash模式）+ 合约卖出
    │       └─ 策略B：现货卖出（cross模式）+ 合约买入
    │
    └─ 否 → 只允许策略A
            └─ 策略A：现货买入（cash模式）+ 合约卖出
            └─ 策略B：跳过（记录失败日志）
```

## OKX API 说明

### 查询杠杆支持

**API**: `GET /api/v5/public/instruments`

**参数**:
- `instType`: MARGIN（杠杆交易）
- `instId`: BTC-USDT（交易对）

**响应示例**（支持杠杆）:
```json
{
  "code": "0",
  "msg": "",
  "data": [
    {
      "instId": "BTC-USDT",
      "instType": "MARGIN",
      "lever": "10",
      ...
    }
  ]
}
```

**响应示例**（不支持杠杆）:
```json
{
  "code": "0",
  "msg": "",
  "data": []
}
```

## 注意事项

1. **API 调用频率**：
   - 杠杆检查是公共 API，不需要签名
   - 每次开仓前都会调用一次
   - 建议后续优化：缓存结果，避免重复查询

2. **错误处理**：
   - 如果查询失败，默认返回 `false`（不支持杠杆）
   - 这样可以避免误操作，保证安全

3. **策略选择**：
   - 不支持杠杆的币种只能做多现货
   - 支持杠杆的币种可以做多或做空现货
   - 系统会自动选择合适的交易模式

4. **借币利息**：
   - 只有使用杠杆模式（cross）时才需要支付借币利息
   - 现货模式（cash）不需要借币，也不需要支付利息

## 后续优化建议

1. **缓存杠杆支持信息**：
   - 将查询结果缓存到 Redis
   - 避免每次开仓都查询 API
   - 设置合理的过期时间（如 1 小时）

2. **启动时预加载**：
   - 在程序启动时批量查询所有交易对的杠杆支持情况
   - 存储到内存或 Redis
   - 提高开仓速度

3. **定期更新**：
   - 定时任务定期更新杠杆支持信息
   - 应对 OKX 交易所的配置变更
