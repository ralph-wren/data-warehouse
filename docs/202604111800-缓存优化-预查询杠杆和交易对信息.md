# 缓存优化 - 预查询杠杆和交易对信息

## 问题背景

在套利交易过程中,每次开仓都需要查询:
1. 币对是否支持杠杆交易 (`checkMarginSupport`)
2. 交易对信息(最小张数、合约面值等) (`getInstrumentInfo`)

这导致频繁调用 OKX API,可能触发限流,同时增加交易延迟。

从日志 `order_detail_2026041117.csv` 和 `position_summary_2026041118.csv` 可以看到,系统在频繁尝试下单,特别是对于不支持杠杆的币对(如 NMR),仍然在重复尝试做空现货操作。

## 解决方案

### 核心改进

1. **提前检查杠杆支持**: 在开仓之前就检查 Redis 缓存中的杠杆支持信息,如果不支持则直接跳过,避免后续的计算和下单操作
2. **数据预查询和缓存**: 将数据查询工作前置到 `FlinkTickerCollectorJob`,定期查询价差最高的币对并缓存相关信息到 Redis
3. **兜底机制**: 如果缓存中没有数据,仍然会调用 API 查询,保证系统可用性

### 1. 配置文件修改

在 `application-dev.yml` 中添加配置:

```yaml
arbitrage:
  # 监控配置
```

### 2. FlinkADSArbitrageJob 改造 - 提前检查杠杆支持

在 `openPosition` 方法开头添加杠杆支持检查:

```java
private void openPosition(ArbitrageOpportunity opp, Collector<TradeRecord> out) 
        throws Exception {
    
    // 确定策略方向
    String strategyDirection = opp.arbitrageDirection.contains("做多现货") ? 
        "LONG_SPOT_SHORT_SWAP" : "SHORT_SPOT_LONG_SWAP";
    
    // 检查该币种的该策略是否在失败黑名单中
    Set<String> failedStrategiesForSymbol = failedStrategies.get(opp.symbol);
    if (failedStrategiesForSymbol != null && failedStrategiesForSymbol.contains(strategyDirection)) {
        return;
    }
    
    // ========================================
    // 提前检查: 如果是做空现货策略,检查是否支持杠杆
    // ========================================
    if (strategyDirection.equals("SHORT_SPOT_LONG_SWAP")) {
        // 从 Redis 缓存读取杠杆支持信息
        boolean marginSupported = false;
        try {
            String flags = redisManager.hget("okx:margin:support", opp.symbol);
            if (flags != null && !flags.isEmpty()) {
                marginSupported = flags.contains("1");
            } else {
                // 缓存中没有数据,调用 API 查询（兜底方案）
                marginSupported = tradingService.checkMarginSupport(opp.symbol);
            }
        } catch (Exception e) {
            // 读取失败,调用 API 查询（兜底方案）
            marginSupported = tradingService.checkMarginSupport(opp.symbol);
        }
        
        // 如果不支持杠杆,直接跳过,不进行后续操作
        if (!marginSupported) {
            logger.warn("⚠ {} 不支持杠杆交易,无法执行策略A(做空现货+做多合约),跳过", opp.symbol);
            
            // 记录跳过日志
            logTradeToCsv(
                opp.symbol,
                "OPEN",
                strategyDirection,
                null,
                null,
                null,
                opp.spotPrice,
                opp.futuresPrice,
                opp.spreadRate,
                "SKIPPED",
                "不支持杠杆交易"
            );
            
            // 不加入黑名单,因为该币种仍可执行策略B
            return;
        }
    }
    
    // 后续的持仓检查、计算张数、下单等操作...
}
```

### 3. FlinkTickerCollectorJob 改造

#### 3.1 添加定时缓存任务

```java
private static void startCacheRefreshTask(ConfigLoader config) {
    int topSymbolsCount = config.getInt("arbitrage.monitor.top-symbols-count", 10);
    int refreshIntervalSeconds = config.getInt("arbitrage.cache.refresh-interval-seconds", 300);
    
    ScheduledExecutorService scheduler = Executors.newSingleThreadScheduledExecutor();
    scheduler.scheduleAtFixedRate(() -> {
        refreshCache(config, topSymbolsCount);
    }, 0, refreshIntervalSeconds, TimeUnit.SECONDS);
}
```

#### 3.2 缓存刷新逻辑

```java
private static void refreshCache(ConfigLoader config, int topN) {
    // 1. 计算价差最大的币对
    List<String> topSymbols = calculateTopSpreadSymbols(config, topN);
    
    // 2. 创建 OKXTradingService 实例
    OKXTradingService tradingService = new OKXTradingService(config);
    
    // 3. 创建 Redis 连接管理器
    RedisConnectionManager redisManager = new RedisConnectionManager(config);
    
    try (Jedis jedis = redisManager.getConnection()) {
        // 4. 遍历每个币对,查询并缓存信息
        for (String symbolWithSuffix : topSymbols) {
            String symbol = symbolWithSuffix.replace("-USDT", "");
            
            // 4.1 查询杠杆支持信息
            boolean supportsMargin = tradingService.checkMarginSupport(symbol);
            String flags = supportsMargin ? "12" : "2";  // "12"=都支持, "2"=只支持策略B
            jedis.hset("okx:margin:support", symbol, flags);
            
            // 4.2 查询交易对信息
            InstrumentInfo instrumentInfo = tradingService.getInstrumentInfo(symbol);
            if (instrumentInfo != null) {
                String json = mapper.writeValueAsString(instrumentInfo);
                jedis.hset("okx:instrument:cache", symbol, json);
            }
        }
        
        // 5. 设置缓存过期时间
        int expireSeconds = refreshIntervalSeconds * 2;
        jedis.expire("okx:margin:support", expireSeconds);
        jedis.expire("okx:instrument:cache", expireSeconds);
    }
}
```

### 4. Redis 缓存格式

#### 4.1 杠杆支持信息

- Redis Key: `okx:margin:support` (Hash)
- Hash Field: 币种名称 (如 `BTC`)
- Hash Value: 策略标识 (flags)
  - `"1"`: 支持策略A (现货卖出+合约买入,需要杠杆)
  - `"2"`: 支持策略B (现货买入+合约卖出,不需要杠杆)
  - `"12"`: 都支持

#### 4.2 交易对信息

- Redis Key: `okx:instrument:cache` (Hash)
- Hash Field: 币种名称 (如 `BTC`)
- Hash Value: JSON 字符串
  ```json
  {
    "minSz": "1",
    "ctVal": "0.01",
    "lotSz": "1"
  }
  ```

### 5. OKXTradingService 改造

添加重载方法支持从 Redis 缓存读取交易对信息:

```java
public BigDecimal calculateContractSize(String symbol, BigDecimal usdtAmount, BigDecimal price,
        RedisConnectionManager redisManager) {
    // 优先从 Redis 缓存读取
    InstrumentInfo info = null;
    if (redisManager != null) {
        try {
            String json = redisManager.hget("okx:instrument:cache", symbol);
            if (json != null && !json.isEmpty()) {
                info = mapper.readValue(json, InstrumentInfo.class);
            }
        } catch (Exception e) {
            // 缓存读取失败,调用 API 查询
        }
    }
    
    // 如果缓存中没有,调用 API 查询（兜底方案）
    if (info == null) {
        info = getInstrumentInfo(symbol);
    }
    
    // ... 后续计算逻辑
}
```

## 优势

1. **提前拦截**: 在开仓之前就检查杠杆支持,避免不必要的计算和下单操作
2. **减少 API 调用**: 只在定时任务中查询,避免每次交易都调用 API
3. **降低延迟**: 从 Redis 读取比调用 API 快得多
4. **避免限流**: 减少 API 调用频率,避免触发 OKX 限流
5. **兜底机制**: 如果缓存中没有数据,仍然会调用 API 查询,保证系统可用性
6. **自动过期**: 缓存设置过期时间,避免数据过期
7. **避免重复尝试**: 对于不支持杠杆的币对,直接跳过,不会重复尝试

## 工作流程

```
FlinkTickerCollectorJob (定时任务,每5分钟)
    ↓
1. 计算价差最大的10个币对
    ↓
2. 查询每个币对的杠杆支持信息
    ↓
3. 查询每个币对的交易对信息
    ↓
4. 存入 Redis (okx:margin:support, okx:instrument:cache)
    ↓
5. 设置过期时间 (10分钟)

FlinkADSArbitrageJob (交易时)
    ↓
1. 检查策略方向
    ↓
2. 如果是做空现货策略,从 Redis 读取杠杆支持信息
    ↓
3. 如果不支持杠杆,直接跳过,记录 SKIPPED 日志
    ↓
4. 如果支持杠杆,继续后续操作
    ↓
5. 从 Redis 读取交易对信息
    ↓
6. 如果缓存中没有,调用 API 查询（兜底）
    ↓
7. 执行交易
```

## 注意事项

1. **缓存刷新间隔**: 默认5分钟,可根据实际情况调整
2. **缓存过期时间**: 设置为刷新间隔的2倍,避免缓存失效
3. **兜底机制**: 如果缓存中没有数据,仍然会调用 API 查询
4. **监控币对数量**: 默认10个,可根据实际情况调整
5. **定时任务线程**: 设置为守护线程,主线程退出时自动退出
6. **提前检查**: 在预留持仓名额之前就检查杠杆支持,避免不必要的资源占用
