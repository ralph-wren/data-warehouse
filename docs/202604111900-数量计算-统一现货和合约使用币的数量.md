# 数量计算优化 - 统一现货和合约使用币的数量

## 问题背景

之前的实现中，现货和合约使用不同的计算方式：
- 现货：使用 `calculateSpotSize()` 计算币的数量
- 合约：使用 `calculateContractSize()` 计算张数（考虑 ctVal 和 lotSz）

这导致了以下问题：
1. 现货下单金额 = 币数量 × 价格
2. 合约下单金额 = 张数 × ctVal × 价格
3. 两者金额不一致，可能导致下单失败（余额不足）

## 解决方案

### 统一计算方式

现货和合约都使用币的数量，计算公式：
```
币数量 = USDT金额 / 价格（向上取整）
```

向上取整的原因：
- 确保订单金额不会因为精度问题低于最小金额
- 例如：100 USDT / 50000 USDT = 0.002 BTC
- 向上取整后仍然是 0.002 BTC

### 代码修改

#### 1. OKXTradingService.java

修改 `calculateSpotSize()` 方法：
```java
public BigDecimal calculateSpotSize(BigDecimal usdtAmount, BigDecimal price) {
    // 计算币数量（向上取整）
    BigDecimal coinAmount = usdtAmount.divide(price, 8, java.math.RoundingMode.UP);
    
    logger.info("  └─ 计算交易数量:");
    logger.info("     - 交易金额: {} USDT", usdtAmount);
    logger.info("     - 当前价格: {} USDT", price);
    logger.info("     - 币数量: {} (现货和合约统一使用)", coinAmount);
    
    return coinAmount;
}
```

#### 2. FlinkADSArbitrageJob.java

##### openPosition() 方法

修改前：
```java
// 分别计算现货数量和合约张数
BigDecimal spotSize = tradingService.calculateSpotSize(tradeAmountUsdt, opp.spotPrice);
BigDecimal contractSize = tradingService.calculateContractSize(opp.symbol, tradeAmountUsdt, opp.spotPrice, redisManager);

// 现货下单使用 spotSize
spotOrderId = tradingService.buySpot(opp.symbol, spotSize, opp.spotPrice);

// 合约下单使用 contractSize
swapOrderId = tradingService.shortSwap(opp.symbol, contractSize, leverage);
```

修改后：
```java
// 统一计算交易数量（现货和合约都使用币的数量）
BigDecimal tradeSize = tradingService.calculateSpotSize(tradeAmountUsdt, opp.spotPrice);

// 现货和合约都使用相同的币数量
spotOrderId = tradingService.buySpot(opp.symbol, tradeSize, opp.spotPrice);
swapOrderId = tradingService.shortSwap(opp.symbol, tradeSize, leverage);
```

##### closePosition() 方法

修改前：
```java
// 只支持策略 A 平仓
spotOrderId = tradingService.sellSpot(pos.getSymbol(), pos.getAmount(), opp.spotPrice, leverage);
swapOrderId = tradingService.closeShortSwap(pos.getSymbol(), pos.getAmount());
```

修改后：
```java
// 根据持仓方向平仓
if ("LONG_SPOT_SHORT_SWAP".equals(pos.getDirection())) {
    // 策略 B 平仓: 卖出现货 + 平空合约
    spotOrderId = tradingService.sellSpot(pos.getSymbol(), pos.getAmount(), opp.spotPrice, leverage);
    swapOrderId = tradingService.closeShortSwap(pos.getSymbol(), pos.getAmount());
} else {
    // 策略 A 平仓: 买入现货 + 平多合约
    spotOrderId = tradingService.buySpot(pos.getSymbol(), pos.getAmount(), opp.spotPrice);
    swapOrderId = tradingService.closeLongSwap(pos.getSymbol(), pos.getAmount());
}
```

#### 3. PositionState.java

添加字段说明：
```java
/**
 * 持仓金额（USDT）
 * @deprecated 使用 spotSize 和 contractSize 代替
 */
@Deprecated
private BigDecimal amount;

/**
 * 现货数量（币的数量）
 * 例如: 5 BTC, 100 ETH

 * 合约张数
```

注意：虽然添加了新字段，但实际上现在统一使用 `amount` 字段保存币的数量，`spotSize` 和 `contractSize` 字段暂时未使用。

## 优势

1. **简化逻辑**：现货和合约使用相同的数量，代码更简洁
2. **避免金额不一致**：不会因为 ctVal 导致现货和合约金额差异过大
3. **更容易理解**：币的数量比合约张数更直观
4. **减少计算错误**：不需要查询 ctVal 和 lotSz，减少 API 调用

## 注意事项

1. **向上取整**：使用 `RoundingMode.UP` 确保订单金额不会低于最小金额
2. **精度控制**：保留 8 位小数，满足大部分币种的精度要求
3. **兼容性**：`amount` 字段现在保存的是币的数量，不是 USDT 金额

## 测试建议

1. 测试不同价格的币种（高价币如 BTC，低价币如 DOGE）
2. 测试不同交易金额（小额如 10 USDT，大额如 1000 USDT）
3. 验证现货和合约订单金额是否一致
4. 检查是否还有余额不足的错误
