# 交易日志改为线程池同步写入并修复现货失败逻辑

## 问题描述

1. **日志写入问题**: `AsyncCsvWriter` 使用异步队列写入,日志一直没有写入文件
2. **交易逻辑问题**: 现货下单失败后,合约仍然会下单,导致单边持仓风险

## 解决方案

### 1. 创建 SyncCsvWriter 类

创建了新的 `SyncCsvWriter` 类,使用线程池同步写入 CSV 文件:

**文件**: `data-warehouse/src/main/java/com/crypto/dw/flink/async/SyncCsvWriter.java`

**核心特性**:
- 使用 `ExecutorService` 线程池管理写入任务
- 每个写入任务在线程池中同步执行,确保写入成功
- 自动按小时滚动日志文件
- 线程安全,支持并发写入
- 优雅关闭,等待所有任务完成

**与 AsyncCsvWriter 的区别**:
```java
// AsyncCsvWriter: 使用队列异步写入,可能丢失数据
queue.offer(logLine);  // 非阻塞,可能失败

// SyncCsvWriter: 使用线程池同步写入,确保写入成功
executorService.submit(() -> {
    writer.write(logLine);
    writer.flush();  // 立即刷新
});
```

### 3. 修复现货下单失败逻辑

**修改 openPosition() 方法**:

```java
// 策略 A: 做多现货 + 做空合约
if (opp.arbitrageDirection.contains("做多现货")) {
    direction = "LONG_SPOT_SHORT_SWAP";
    
    // 先下现货单
    spotOrderId = tradingService.buySpot(opp.symbol, tradeAmount, opp.spotPrice);
    
    // 检查现货单是否成功
    if (spotOrderId == null) {
        logger.error("❌ 现货下单失败,取消合约下单");
        recordFailure(opp.symbol, "现货下单失败");
        
        // 写入失败日志
        String timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
        String errorLog = String.format("%s,%s,OPEN,%s,SPOT,N/A,N/A,N/A,FAILED,现货下单失败",
            timestamp, opp.symbol, direction);
        orderDetailWriter.write(errorLog);
        return;  // 直接返回,不下合约单
    }
    
    // 现货单成功,再下合约单
    swapOrderId = tradingService.shortSwap(opp.symbol, tradeAmount, leverage);
}

// 策略 B: 做空现货 + 做多合约
else {
    direction = "SHORT_SPOT_LONG_SWAP";
    
    // 先下现货单
    spotOrderId = tradingService.sellSpot(opp.symbol, tradeAmount, opp.spotPrice, leverage);
    
    // 检查现货单是否成功
    if (spotOrderId == null) {
        logger.error("❌ 现货下单失败,取消合约下单");
        
        // 获取详细错误信息
        String errorMsg = tradingService.getLastErrorMessage();
        if (errorMsg != null) {
            if (errorMsg.contains("Insufficient") || errorMsg.contains("51008")) {
                recordFailure(opp.symbol, "余额不足或借币额度不足");
            } else if (errorMsg.contains("51001")) {
                recordFailure(opp.symbol, "订单数量不符合要求");
            } else if (errorMsg.contains("51004")) {
                recordFailure(opp.symbol, "交易对不存在或已下线");
            } else {
                recordFailure(opp.symbol, errorMsg);
            }
        } else {
            recordFailure(opp.symbol, "现货下单失败");
        }
        
        // 写入失败日志
        String timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
        String errorLog = String.format("%s,%s,OPEN,%s,SPOT,N/A,N/A,N/A,FAILED,%s",
            timestamp, opp.symbol, direction, 
            errorMsg != null ? errorMsg.replace(",", ";") : "现货下单失败");
        orderDetailWriter.write(errorLog);
        return;  // 直接返回,不下合约单
    }
    
    // 现货单成功,再下合约单
    swapOrderId = tradingService.longSwap(opp.symbol, tradeAmount, leverage);
}
```

**关键改进**:
1. 先下现货单,检查是否成功
2. 现货单失败则直接返回,不下合约单
3. 获取详细错误信息并记录到黑名单
4. 写入详细的失败日志

## 技术要点

### 1. 线程池配置

```java
// 创建固定大小的线程池
this.executorService = Executors.newFixedThreadPool(threadPoolSize, r -> {
    Thread thread = new Thread(r, "SyncCsvWriter-" + filePrefix);
    thread.setDaemon(false);  // 非守护线程,确保日志写完
    return thread;
});
```

**为什么使用非守护线程?**
- 守护线程会在主线程结束时立即终止,可能导致日志丢失
- 非守护线程会等待任务完成后才终止,确保日志写入成功

### 2. 优雅关闭

```java
@Override
public void close() {
    // 等待所有任务完成
    executorService.shutdown();
    if (!executorService.awaitTermination(30, TimeUnit.SECONDS)) {
        logger.warn("日志写入超时,强制关闭");
        executorService.shutdownNow();
    }
    
    // 关闭文件
    if (writer != null) {
        writer.flush();
        writer.close();
    }
}
```

**关键步骤**:
1. 调用 `shutdown()` 停止接收新任务
2. 等待最多 30 秒让现有任务完成
3. 超时则强制关闭
4. 最后刷新并关闭文件

### 3. 线程安全

```java
public void write(String logLine) {
    executorService.submit(() -> {
        try {
            synchronized (lock) {  // 加锁保证线程安全
                checkAndRotateFile();
                writer.write(logLine);
                writer.flush();
            }
        } catch (IOException e) {
            logger.error("写入日志失败: {}", e.getMessage(), e);
        }
    });
}
```

**为什么需要加锁?**
- 多个线程可能同时写入同一个文件
- 文件滚动时需要关闭旧文件、创建新文件
- 加锁确保这些操作的原子性

## 注意事项

1. **线程池大小**: 建议设置为 2-4,太大会增加内存开销,太小会影响写入性能
2. **缓冲区大小**: 使用 64KB 缓冲区,平衡内存和性能
3. **立即刷新**: 每次写入后立即 `flush()`,确保数据持久化
4. **错误处理**: 现货下单失败后,获取详细错误信息并记录到黑名单
