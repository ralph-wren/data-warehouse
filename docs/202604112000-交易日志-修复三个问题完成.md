### 3. 修复杠杆倍数硬编码

**问题**: 代码中多处使用了 `leverage` 变量，但没有明确使用 `this.leverage`，导致可能使用了错误的值

**修改位置**: 所有调用 `tradingService` 方法的地方

**修改内容**:
1. `openPosition()` 方法中的现货做空：`this.leverage`
2. `openPosition()` 方法中的合约做多：`this.leverage`
3. `openPosition()` 方法中的合约做空：`this.leverage`
4. `closePosition()` 方法中的现货卖出：`this.leverage`

**修改示例**:
```java
// 修改前
spotOrderId = tradingService.sellSpot(opp.symbol, tradeAmount, opp.spotPrice, leverage);

// 修改后
spotOrderId = tradingService.sellSpot(opp.symbol, tradeAmount, opp.spotPrice, this.leverage);
```

**配置文件**:
```yaml
arbitrage:
  trading:
    leverage: 1  # 杠杆倍数(默认1倍,现货和合约统一使用)
```

### 4. 修复黑名单跟踪器清理

**问题**: 币对加入黑名单后，定时器仍然输出"套利机会跟踪: 观察中"的状态日志

**原因**: 
- 黑名单检查时只是 `return`，没有清除 `opportunityTracker` 状态
- 定时器会读取 `opportunityTracker` 状态并输出日志
- 导致黑名单币对仍然显示"观察中"

**修改位置**: `TradingDecisionProcessor.processElement1()` 方法

**修改内容**:
```java
// 修改前
if (isInBlacklist(opportunity.symbol)) {
    // 在黑名单中,跳过
    return;
}

// 修改后
if (isInBlacklist(opportunity.symbol)) {
    // 在黑名单中,清除跟踪器并跳过
    if (tracker != null && tracker.isActive()) {
        opportunityTracker.clear();
    }
    return;
}
```

**效果**:
- ✅ 币对加入黑名单后，立即清除套利机会跟踪器
- ✅ 定时器输出的状态日志显示"套利机会跟踪: 无活跃跟踪"
- ✅ 避免误导性的"观察中"状态

### 5. 优化状态日志输出

**问题**: 加入黑名单的币对仍然每30秒打印"📊 币对状态报告"，造成日志噪音

**原因**: 
- 定时器会定期触发 `onTimer()` 方法
- `onTimer()` 方法没有检查黑名单状态
- 导致黑名单币对仍然输出状态日志

**修改位置**: `TradingDecisionProcessor.onTimer()` 方法

**修改内容**:
```java
// 修改前
// 输出状态日志
printStatusLog(symbol, position, tracker, now);

// 修改后
// 检查是否在黑名单中
if (isInBlacklist(symbol)) {
    // 在黑名单中,不输出状态日志
    // 注册下一个定时器
    long nextTimerTime = now + STATUS_LOG_INTERVAL_MS;
    ctx.timerService().registerProcessingTimeTimer(nextTimerTime);
    return;
}

// 只有持仓状态的币对才输出状态日志
if (position != null && position.isOpen()) {
    printStatusLog(symbol, position, tracker, now);
}
```

**效果**:
- ✅ 黑名单币对不再输出状态日志
- ✅ 只有持仓状态的币对才输出状态日志
- ✅ 减少日志噪音，提高日志可读性

## 日志文件结构

现在系统会生成三个日志文件：

### 1. order_detail_YYYYMMDD_HH.csv
记录所有订单的详细信息（开仓、平仓、失败等）

### 2. position_YYYYMMDD_HH.csv
记录所有持仓的详细信息（开仓、平仓、未实现利润等）

### 3. summary_YYYYMMDD_HH.csv（新增）
记录所有交易的汇总信息（利润、利润率、持仓时间等）
