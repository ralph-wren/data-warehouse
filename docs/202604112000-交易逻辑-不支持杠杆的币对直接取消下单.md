# 交易逻辑优化 - 不支持杠杆的币对直接取消下单

## 问题描述

在做空现货交易时,如果币对不支持杠杆交易,系统仍然会尝试下单,导致以下问题:

```
2026-04-11 20:00:50.017 [Trading Decision] INFO  com.crypto.dw.trading.OKXTradingService - 
📉 做空现货（卖出）: SAHARA-USDT | 数量: 10 SAHARA | 杠杆: 1x

2026-04-11 20:00:50.017 [Trading Decision] WARN  com.crypto.dw.trading.OKXTradingService - 
⚠ SAHARA-USDT 不支持杠杆交易，将使用现货模式（可能失败）

2026-04-11 20:00:50.017 [Trading Decision] WARN  com.crypto.dw.trading.OKXTradingService - 
⚠ 订单金额 0.22 USDT < 最小要求 10 USDT,自动调整数量
```

**问题现象**:
1. 不支持杠杆的币对仍然尝试下单
2. 使用"现货模式"下单,但实际上会失败
3. 浪费 API 调用次数
4. 可能触发黑名单机制

## 根本原因

在 `OKXTradingService.sellSpot()` 方法中,当检测到币对不支持杠杆时,只是打印警告,然后继续执行下单逻辑:

```java
// 旧逻辑
if (supportsMargin) {
    setMarginLeverage(instId, leverage);
    logger.info("✓ {} 支持杠杆交易，已设置杠杆倍数: {}x", instId, leverage);
} else {
    logger.warn("⚠ {} 不支持杠杆交易，将使用现货模式（可能失败）", instId);
    // 继续执行下单逻辑 ❌
}
```

**为什么会失败?**
- 做空现货必须使用杠杆模式（借币卖出）
- 不支持杠杆的币对无法借币
- 即使调整订单金额,也会因为无法借币而失败

## 解决方案

修改 `sellSpot()` 方法,在不支持杠杆时直接返回 null,不再尝试下单:

```java
// 新逻辑
// 如果不支持杠杆，直接返回 null（不下单）
if (!supportsMargin) {
    logger.warn("⚠ {} 不支持杠杆交易，取消下单", instId);
    return null;
}

// 支持杠杆，设置杠杆倍数
setMarginLeverage(instId, leverage);
logger.info("✓ {} 支持杠杆交易，已设置杠杆倍数: {}x", instId, leverage);
```

## 技术要点

### 1. 杠杆交易的必要性

**做空现货的原理**:
1. 借入币种（如 BTC）
2. 卖出借入的币种,获得 USDT
3. 等待价格下跌
4. 买回币种,归还借币
5. 赚取差价

**为什么必须支持杠杆?**
- 做空需要借币
- 只有支持杠杆的币对才能借币
- 不支持杠杆的币对无法借币,做空必然失败

### 2. 杠杆支持检查

**检查流程**:
```java
// 1. 优先从内存缓存查询（性能最好）
if (marginCache != null) {
    supportsMargin = marginCache.supportsMargin(instId);
    logger.debug("从内存缓存查询杠杆支持: {} -> {}", instId, supportsMargin);
}

// 2. 兜底方案：调用 API 查询（性能较差）
else {
    logger.warn("⚠ 未提供杠杆缓存，将调用 API 查询（性能较差）");
    supportsMargin = checkMarginSupport(symbol);
}
```

**缓存来源**:
- `FlinkTickerCollectorJob` 启动时查询所有币对的杠杆支持信息
- 写入 Redis: `okx:margin:support`
- `MarginSupportCache` 从 Redis 加载到内存
- 每 5 分钟刷新一次

### 3. 错误处理

**上层调用处理**:
```java
// TradingDecisionProcessor.openPosition()
spotOrderId = tradingService.sellSpot(opp.symbol, tradeAmount, opp.spotPrice, leverage);

// 检查现货单是否成功
if (spotOrderId == null) {
    logger.error("❌ 现货下单失败,取消合约下单");
    
    // 获取详细错误信息
    String errorMsg = tradingService.getLastErrorMessage();
    if (errorMsg != null) {
        // 根据错误码分类处理
        if (errorMsg.contains("Insufficient") || errorMsg.contains("51008")) {
            recordFailure(opp.symbol, "余额不足或借币额度不足");
        } else if (errorMsg.contains("51001")) {
            recordFailure(opp.symbol, "订单数量不符合要求");
        } else if (errorMsg.contains("51004")) {
            recordFailure(opp.symbol, "交易对不存在或已下线");
        } else {
            recordFailure(opp.symbol, errorMsg);
        }
    } else {
        recordFailure(opp.symbol, "现货下单失败");
    }
    
    // 写入失败日志
    String timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
    String errorLog = String.format("%s,%s,OPEN,%s,SPOT,N/A,N/A,N/A,FAILED,%s",
        timestamp, opp.symbol, direction, 
        errorMsg != null ? errorMsg.replace(",", ";") : "现货下单失败");
    orderDetailWriter.write(errorLog);
    return;  // 直接返回,不下合约单
}
```

## 注意事项

1. **杠杆支持信息的时效性**:
   - 杠杆支持信息每 5 分钟刷新一次
   - 如果币对刚上线或下线,可能有延迟
   - 建议在交易前再次确认

2. **黑名单机制**:
   - 不支持杠杆的币对会被记录失败
   - 连续失败 3 次会加入黑名单
   - 黑名单持续 60 分钟

3. **日志记录**:
   - 所有失败都会记录到 CSV 日志
   - 包括失败原因和时间戳
   - 方便后续分析和排查
