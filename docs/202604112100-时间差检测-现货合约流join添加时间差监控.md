# 时间差检测 - 现货合约流 join 添加时间差监控

## 问题描述

需要在现货和合约两条流 join 时添加时间差检测，监控数据延迟情况。

## 需求

在 `FlinkADSArbitrageJob` 的现货和合约流 join 处添加时间差检测：

1. 检测现货和合约两条数据的时间差是否超过 2 秒
2. 检测系统当前时间与现货数据时间差是否超过 2 秒
3. 检测系统当前时间与合约数据时间差是否超过 2 秒

## 解决方案

### 1. 定位 join 位置

在 `FlinkADSArbitrageJob.java` 中找到现货和合约流的 join 位置：

```java
// 第 182-186 行
DataStream<ArbitrageOpportunity> arbitrageStream = spotStream
    .keyBy(spot -> spot.symbol)
    .intervalJoin(futuresStream.keyBy(futures -> futures.symbol))
    .between(Time.seconds(-2), Time.seconds(2))
    .process(new ArbitrageCalculator())
    .name("Calculate Arbitrage");
```

join 使用了 `ArbitrageCalculator` 处理函数，需要在这个类中添加时间差检测逻辑。

### 2. 修改 ArbitrageCalculator 类

在 `ArbitrageCalculator.processElement()` 方法中添加时间差检测：

```java
public class ArbitrageCalculator 
        extends ProcessJoinFunction<SpotPrice, FuturesPrice, ArbitrageOpportunity> {
    
    private static final Logger logger = LoggerFactory.getLogger(ArbitrageCalculator.class);
    
    // 套利阈值：价差超过 0.5% 才认为有套利机会
    private static final BigDecimal ARBITRAGE_THRESHOLD = new BigDecimal("0.005");
    
    // 时间差阈值：2 秒（毫秒）
    private static final long TIME_DIFF_THRESHOLD_MS = 2000;
    
    @Override
    public void processElement(
            SpotPrice spot,
            FuturesPrice futures,
            Context ctx,
            Collector<ArbitrageOpportunity> out) {
        
        // ========== 时间差检测 ==========
        long currentTime = System.currentTimeMillis();
        long spotTime = spot.timestamp;
        long futuresTime = futures.timestamp;
        
        // 1. 检测现货和合约数据的时间差
        long spotFuturesTimeDiff = Math.abs(spotTime - futuresTime);
        if (spotFuturesTimeDiff > TIME_DIFF_THRESHOLD_MS) {
            logger.warn("⚠️ 现货和合约时间差超过 2 秒: symbol={}, 现货时间={}, 合约时间={}, 时间差={}ms",
                    spot.symbol, spotTime, futuresTime, spotFuturesTimeDiff);
        }
        
        // 2. 检测系统时间与现货数据的时间差
        long currentSpotTimeDiff = Math.abs(currentTime - spotTime);
        if (currentSpotTimeDiff > TIME_DIFF_THRESHOLD_MS) {
            logger.warn("⚠️ 系统时间与现货数据时间差超过 2 秒: symbol={}, 系统时间={}, 现货时间={}, 时间差={}ms",
                    spot.symbol, currentTime, spotTime, currentSpotTimeDiff);
        }
        
        // 3. 检测系统时间与合约数据的时间差
        long currentFuturesTimeDiff = Math.abs(currentTime - futuresTime);
        if (currentFuturesTimeDiff > TIME_DIFF_THRESHOLD_MS) {
            logger.warn("⚠️ 系统时间与合约数据时间差超过 2 秒: symbol={}, 系统时间={}, 合约时间={}, 时间差={}ms",
                    spot.symbol, currentTime, futuresTime, currentFuturesTimeDiff);
        }
        
        // 后续套利计算逻辑...
    }
}
```

### 3. 关键改动

1. 添加 Logger 实例
2. 添加时间差阈值常量 `TIME_DIFF_THRESHOLD_MS = 2000`（2 秒）
3. 在 `processElement()` 方法开头添加三个时间差检测：
   - 现货和合约数据时间差
   - 系统时间与现货数据时间差
   - 系统时间与合约数据时间差
4. 使用 `logger.warn()` 打印警告日志，包含详细信息：
   - 币对符号
   - 相关时间戳
   - 时间差（毫秒）

## 监控价值

通过这个时间差检测，可以：

1. 监控数据采集延迟：如果系统时间与数据时间差过大，说明数据采集有延迟
2. 监控流同步问题：如果现货和合约时间差过大，说明两条流的数据不同步
3. 及时发现问题：延迟过大可能导致套利机会计算不准确，需要及时排查
