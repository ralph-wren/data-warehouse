# 交易优化 - 添加失败黑名单和修复异步日志配置

## 时间
2026-04-11 21:30

## 问题背景

用户反馈两个问题:
1. **异步日志没有写入 CSV 文件** - 日志目录配置缺失
2. **下单失败(余额不足)没有加入黑名单,一直重复下单** - 之前移除了黑名单逻辑

## 问题分析

### 问题 1: 异步日志未写入
- `TradingDecisionProcessor` 初始化 `AsyncCsvWriter` 时使用配置项 `arbitrage.trading.log-dir`
- 但配置文件中没有这个配置项
- 导致日志目录为 `null`,无法创建日志文件

### 问题 2: 重复下单
- 之前为了简化代码,移除了黑名单过滤逻辑
- 导致下单失败(如余额不足)后,系统会一直重试
- 错误日志显示: `"code":"1","sCode":"51008","sMsg":"Order failed. Insufficient USDT margin in account"`
- 这是致命错误,应该加入黑名单,避免重复下单

## 解决方案

### 1. 添加日志目录配置

在 `application-dev.yml` 中添加:
```yaml
arbitrage:
  trading:
    log-dir: ./logs/trading  # 交易日志目录
```

### 2. 添加失败黑名单机制

#### 配置项
```yaml
arbitrage:
  trading:
    blacklist:
      enabled: true                  # 是否启用黑名单(默认启用)
      duration-minutes: 60           # 黑名单持续时间(分钟,默认60分钟)
      max-failures: 3                # 最大失败次数(超过此次数加入黑名单)
```

#### 实现逻辑

1. **状态管理**
```java
// 失败黑名单状态
private transient MapState<String, Integer> failureCount;  // 失败次数统计
private transient MapState<String, Long> blacklistExpiry;  // 黑名单过期时间
```

2. **黑名单检查**
```java
private boolean isInBlacklist(String symbol) throws Exception {
    if (!blacklistEnabled) {
        return false;
    }
    
    Long expiry = blacklistExpiry.get(symbol);
    if (expiry == null) {
        return false;
    }
    
    long now = System.currentTimeMillis();
    if (now > expiry) {
        // 黑名单已过期,清除
        blacklistExpiry.remove(symbol);
        failureCount.remove(symbol);
        logger.info("⏰ 黑名单过期: {} | 已移出黑名单", symbol);
        return false;
    }
    
    return true;
}
```

3. **失败记录**
```java
private void recordFailure(String symbol, String reason) throws Exception {
    if (!blacklistEnabled) {
        return;
    }
    
    // 增加失败次数
    Integer count = failureCount.get(symbol);
    count = (count == null) ? 1 : count + 1;
    failureCount.put(symbol, count);
    
    logger.warn("⚠ 交易失败: {} | 失败次数: {} / {} | 原因: {}", 
        symbol, count, maxFailures, reason);
    
    // 检查是否需要加入黑名单
    if (count >= maxFailures) {
        long expiry = System.currentTimeMillis() + blacklistDurationMs;
        blacklistExpiry.put(symbol, expiry);
        
        logger.error("🚫 加入黑名单: {} | 失败次数: {} | 持续时间: {} 分钟", 
            symbol, count, blacklistDurationMs / 60000);
    }
}
```

4. **成功后清除失败计数**
```java
private void clearFailureCount(String symbol) throws Exception {
    if (!blacklistEnabled) {
        return;
    }
    
    Integer count = failureCount.get(symbol);
    if (count != null && count > 0) {
        failureCount.remove(symbol);
        logger.info("✓ 清除失败计数: {} | 之前失败次数: {}", symbol, count);
    }
}
```

### 3. 错误分类处理

根据 OKX API 错误码进行分类处理:
- `51008`: 余额不足 - 加入黑名单
- `51001`: 订单数量不符合要求 - 加入黑名单
- `51004`: 交易对不存在或已下线 - 加入黑名单
- 其他错误: 记录失败次数

```java
catch (Exception e) {
    String errorMsg = e.getMessage();
    if (errorMsg != null) {
        if (errorMsg.contains("Insufficient") || errorMsg.contains("51008")) {
            recordFailure(opp.symbol, "余额不足");
        } else if (errorMsg.contains("51001")) {
            recordFailure(opp.symbol, "订单数量不符合要求");
        } else if (errorMsg.contains("51004")) {
            recordFailure(opp.symbol, "交易对不存在或已下线");
        } else {
            recordFailure(opp.symbol, errorMsg);
        }
    }
}
```

## 工作流程

### 正常流程
1. 发现套利机会
2. 检查是否在黑名单中 → 不在
3. 尝试开仓
4. 开仓成功
5. 清除失败计数

### 失败流程
1. 发现套利机会
2. 检查是否在黑名单中 → 不在
3. 尝试开仓
4. 开仓失败(余额不足)
5. 记录失败次数 +1
6. 如果失败次数 >= 3 → 加入黑名单(持续 60 分钟)
7. 下次发现套利机会时,检查黑名单 → 在黑名单中 → 跳过

### 黑名单过期
1. 60 分钟后,黑名单过期
2. 下次检查时自动移出黑名单
3. 清除失败计数
4. 可以重新尝试交易

## 注意事项

1. **黑名单持续时间**: 默认 60 分钟,可根据实际情况调整
2. **最大失败次数**: 默认 3 次,可根据实际情况调整
3. **日志目录**: 确保 `./logs/trading` 目录有写权限
4. **状态持久化**: 黑名单状态保存在 Flink 状态后端,重启后会恢复
5. **手动清除**: 如需手动清除黑名单,需要重启 Flink 作业并清除状态
