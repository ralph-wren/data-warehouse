# 数量计算 - 修复现货和合约下单 USDT 金额转换错误

## 问题描述

### 问题 1: 现货下单数量错误

COMP 现货做空时，下单数量是 5.5 个币，价格约 20 USDT，总金额 = 5.5 × 20 = 110 USDT，导致保证金不足错误：

```
{"instId":"COMP-USDT","tdMode":"cross","ccy":"USDT","side":"sell","ordType":"market","sz":"5.5"}
{"code":"1","data":[{"sCode":"51008","sMsg":"Order failed. Insufficient USDT margin in account"}]}
```

### 问题 2: 合约下单张数为 0

RSR 现货下单成功后，合约下单失败，错误是"计算的合约张数为0"：

```
✓ 下单成功: orderId=3469996735873703936
❌ 计算的合约张数为0: symbol=RSR, spotCoinAmount=5.5
❌ 合约下单失败,现货单已提交: 3469996735873703936
```

## 根本原因

### 原因 1: 现货下单参数理解错误

`OKXTradingService.sellSpot()` 方法中，参数 `size` 应该是 **USDT 金额**（如 5.5 USDT），但代码把它当成了 **币的数量**（5.5 个币）。

### 原因 2: 合约下单参数传递错误

`TradingDecisionProcessor.openPosition()` 方法中，传递给 `longSwap()` 的参数是 **USDT 金额**（5.5），但 `longSwap()` 期望的是 **币的数量**。

### 原因 3: 持仓状态保存错误

开仓成功后，`PositionState.amount` 保存的是 **USDT 金额**，但平仓时需要的是 **币的数量**。

## 解决方案

### 1. 修改 `OKXTradingService.sellSpot()` 方法

**文件**: `src/main/java/com/crypto/dw/trading/OKXTradingService.java`

**修改内容**:

```java
// 修改前：参数名为 size，被当成币的数量
public String sellSpot(String symbol, BigDecimal size, BigDecimal price, int leverage) {
    // ...
    BigDecimal orderValue = size.multiply(price);  // 错误：把 USDT 金额当成币数量
    // ...
    orderRequest.put("sz", size.toPlainString());  // 错误：直接使用 USDT 金额作为币数量
}

// 修改后：参数名为 amountUsdt，明确是 USDT 金额
public String sellSpot(String symbol, BigDecimal amountUsdt, BigDecimal price, int leverage) {
    // ...
    // 根据 USDT 金额和价格计算币的数量: 数量 = USDT金额 / 价格（向上取整）
    BigDecimal size = amountUsdt.divide(price, 8, RoundingMode.UP);
    logger.info("💰 计算数量: {} USDT ÷ {} = {} {}", amountUsdt, price, size, symbol);
    // ...
    orderRequest.put("sz", size.toPlainString());  // 正确：使用计算后的币数量
}
```

### 2. 修改 `TradingDecisionProcessor.openPosition()` 方法

**文件**: `src/main/java/com/crypto/dw/processor/TradingDecisionProcessor.java`

**修改内容**:

```java
// 修改前：直接传递 USDT 金额给合约下单
swapOrderId = tradingService.longSwap(opp.symbol, tradeAmount, opp.futuresPrice, this.leverage);

// 修改后：先计算币数量，再传递给合约下单
// 根据 USDT 金额和现货价格计算币的数量（用于合约下单）
BigDecimal coinAmount = tradeAmount.divide(opp.spotPrice, 8, RoundingMode.UP);
logger.info("💰 交易金额: {} USDT | 现货价格: {} | 计算币数量: {} {}", 
    tradeAmount, opp.spotPrice, coinAmount, opp.symbol);

// 现货下单（传入 USDT 金额）
spotOrderId = tradingService.sellSpot(opp.symbol, tradeAmount, opp.spotPrice, this.leverage);

// 合约下单（传入币数量）
swapOrderId = tradingService.longSwap(opp.symbol, coinAmount, opp.futuresPrice, this.leverage);
```

### 3. 修改持仓状态保存逻辑

**修改内容**:

```java
// 修改前：保存 USDT 金额
tempPosition.setAmount(tradeAmount);

// 修改后：保存币的数量（用于平仓）
tempPosition.setAmount(coinAmount);
```

同时修改日志记录：

```java
// 修改前：日志记录 USDT 金额
String spotLog = String.format("%s,%s,OPEN,%s,SPOT,%s,%s,%s,PENDING,订单已提交",
    timestamp, opp.symbol, direction, spotOrderId, 
    opp.spotPrice, tradeAmount);

// 修改后：日志记录币的数量
String spotLog = String.format("%s,%s,OPEN,%s,SPOT,%s,%s,%s,PENDING,订单已提交",
    timestamp, opp.symbol, direction, spotOrderId, 
    opp.spotPrice, coinAmount);
```

## 测试验证

### 预期结果

以 RSR 为例（价格 0.01 USDT，配置金额 5.5 USDT）：

1. 计算币数量：`5.5 ÷ 0.01 = 550 个币`
2. 现货下单：`{"sz":"550"}`（币数量）
3. 合约下单：`{"sz":"550"}`（张数，USDT 合约 1 张 = 1 币）
4. 持仓状态：`amount = 550`（币数量）

### 日志输出

```
💰 交易金额: 5.5 USDT | 现货价格: 0.01 | 计算币数量: 550 RSR
📉 做空现货（卖出）: RSR-USDT | 金额: 5.5 USDT | 价格: 0.01 | 杠杆: 1x
💰 计算数量: 5.5 USDT ÷ 0.01 = 550 RSR
✓ 下单成功: orderId=xxx
📈 做多合约: RSR-USDT-SWAP | 现货币数量: 550 RSR | 合约张数: 550 | 实际币数量: 550 RSR | 杠杆: 1x
✓ 下单成功: orderId=yyy
```

## 影响范围

- 修改文件：
  - `src/main/java/com/crypto/dw/trading/OKXTradingService.java`
  - `src/main/java/com/crypto/dw/processor/TradingDecisionProcessor.java`
- 影响方法：
  - `OKXTradingService.sellSpot()`
  - `TradingDecisionProcessor.openPosition()`
- 影响功能：
  - 现货做空下单
  - 合约做多下单
  - 持仓状态保存
  - 平仓逻辑

## 注意事项

1. 配置文件中的 `trade-amount-usdt` 是 USDT 金额，不是币的数量
2. 现货下单使用 USDT 金额，合约下单使用币数量
3. 持仓状态保存币数量，用于平仓时计算
4. 数量计算使用 `RoundingMode.UP`（向上取整），确保订单金额不小于配置值
5. 保留 8 位小数，满足大部分币种的精度要求
