# 错误翻译 - 修复借币额度不足的错误消息翻译

## 问题描述

用户报告错误消息翻译不准确:

```
错误原因: 账户余额不足 - 请检查账户中的 USDT 余额是否充足
原始消息: All operations failed | Your order failed because it requires borrowing LPT, and the platform's loan quota is insufficient.
```

原始消息明确说明是"借币额度不足"(loan quota is insufficient),但被翻译成了"账户余额不足",导致用户误解。

## 根本原因

`translateErrorMessage` 方法只根据错误码 `51008` 进行翻译,没有考虑原始消息的具体内容。

错误码 `51008` 是一个通用错误码,可能包含多种情况:
- 账户余额不足
- 借币额度不足
- 其他资金相关问题

## 解决方案

修改 `translateErrorMessage` 方法,优先检查原始消息中的关键词,提供更精确的翻译:

```java
/**
 * 翻译 OKX 错误信息为中文说明
 * 
 * @param errorCode OKX 错误代码
 * @param originalMsg 原始错误消息
 * @return 中文错误说明
 */
private String translateErrorMessage(String errorCode, String originalMsg) {
    // 优先检查原始消息中的关键词,提供更精确的翻译
    if (originalMsg != null) {
        String lowerMsg = originalMsg.toLowerCase();
        
        // 借币相关错误
        if (lowerMsg.contains("loan quota") || lowerMsg.contains("borrowing")) {
            if (lowerMsg.contains("insufficient")) {
                return "借币额度不足 - 平台的借币额度已用完,无法借入该币种进行做空";
            }
        }
        
        // 余额不足相关错误
        if (lowerMsg.contains("insufficient balance") || lowerMsg.contains("not enough")) {
            return "账户余额不足 - 请检查账户中的 USDT 或对应币种余额是否充足";
        }
        
        // 最小数量错误
        if (lowerMsg.contains("minimum") && lowerMsg.contains("size")) {
            return "订单数量低于最小限额 - 请增加交易数量";
        }
        
        // 最大数量错误
        if (lowerMsg.contains("maximum") && lowerMsg.contains("size")) {
            return "订单数量超过最大限额 - 请减少交易数量";
        }
    }
    
    // 根据错误码翻译
    switch (errorCode) {
        case "51121":
            return "订单数量不符合最小交易单位 - 数量必须是 lot size 的整数倍,请调整交易数量";
        case "51001":
            return "交易对不存在 - 该币种在 OKX 上不存在或交易对名称错误,请检查交易对是否正确";
        case "51008":
            // 51008 是一个通用错误码,可能是余额不足或借币额度不足
            // 如果原始消息没有匹配到关键词,返回通用说明
            return "余额或额度不足 - 可能是账户余额不足或借币额度不足";
        // ... 其他错误码
        default:
            return originalMsg;  // 未知错误,返回原始消息
    }
}
```

## 关键改进

### 1. 优先检查原始消息关键词

在根据错误码翻译之前,先检查原始消息中的关键词:

- `loan quota` + `insufficient` → "借币额度不足"
- `insufficient balance` → "账户余额不足"
- `minimum` + `size` → "订单数量低于最小限额"
- `maximum` + `size` → "订单数量超过最大限额"

### 2. 更精确的错误说明

针对不同的错误情况,提供更精确的说明:

- **借币额度不足**: 明确说明是平台的借币额度已用完,无法借入该币种进行做空
- **账户余额不足**: 说明是账户中的 USDT 或对应币种余额不足

### 3. 通用错误码的处理

对于 `51008` 这种通用错误码,如果原始消息没有匹配到关键词,返回通用说明:

```
余额或额度不足 - 可能是账户余额不足或借币额度不足
```

## 注意事项

1. **关键词匹配顺序**
   - 优先检查原始消息中的关键词
   - 如果没有匹配到关键词,再根据错误码翻译

2. **大小写不敏感**
   - 使用 `toLowerCase()` 转换为小写后再匹配
   - 避免因大小写问题导致匹配失败

3. **多个关键词组合**
   - 使用多个关键词组合匹配,提高准确性
   - 例如: `loan quota` + `insufficient` 才判断为借币额度不足

4. **未知错误处理**
   - 如果错误码和关键词都没有匹配到,返回原始消息
   - 方便后续分析和添加新的错误翻译

## 总结

通过优先检查原始消息中的关键词,提供了更精确的错误翻译。特别是对于 `51008` 这种通用错误码,能够根据具体情况区分"借币额度不足"和"账户余额不足",避免用户误解。
