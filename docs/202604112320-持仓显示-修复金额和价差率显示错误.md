# 持仓显示 - 修复金额和价差率显示错误

## 问题描述

持仓状态日志显示有两个问题：

1. **金额显示错误**：显示"金额: 300 USDT"，但配置的交易金额是 5.5 USDT
2. **价差率显示错误**：显示"开仓价差率: null%"，应该显示实际的价差率

示例日志：
```
📊 币对状态报告: SPK
⏰ 时间: 2026-04-11 23:44:51
========================================
📈 持仓状态: 已开仓
  方向: SHORT_SPOT_LONG_SWAP
  金额: 300 USDT
  开仓价格: 现货=0.02094, 合约=0.02083
  开仓价差率: null%
  💰 预估利润: -0.6000 USDT (利润率: -0.20%)
  持仓时间: 17秒
  订单ID: 现货=3470195253489783490, 合约=3470195262885224448
```

## 根本原因

### 问题 1：金额显示错误

`PositionState.amount` 字段存储的是币的数量（如 300 个 SPK），而不是 USDT 金额。但是在 `printStatusLog` 方法中，直接显示为"USDT"，导致误导。

实际情况：
- SPK 价格约 0.02 USDT
- 币数量：300 个 SPK
- 实际 USDT 金额：300 × 0.02 = 6 USDT（接近配置的 5.5 USDT）

### 问题 2：价差率显示错误

在 `openPosition` 方法中，创建 `PositionState` 时没有设置 `entrySpreadRate` 字段，导致显示为 `null`。

## 解决方案

### 修复 1：正确显示金额

修改 `printStatusLog` 方法，计算实际的 USDT 金额并同时显示币数量：

```java
// 修复：amount 存储的是币的数量，不是 USDT 金额
// 计算实际的 USDT 金额 = 币数量 × 现货价格
BigDecimal usdtAmount = position.getAmount().multiply(position.getEntrySpotPrice());
status.append(String.format("  金额: %s %s (约 %s USDT)\n", 
    position.getAmount().setScale(4, RoundingMode.HALF_UP), 
    symbol,
    usdtAmount.setScale(2, RoundingMode.HALF_UP)));
```

### 修复 2：保存并显示价差率

在 `openPosition` 方法中，创建 `PositionState` 时设置 `entrySpreadRate`：

```java
// 立即创建临时持仓状态,防止重复开仓
PositionState tempPosition = new PositionState();
tempPosition.setSymbol(opp.symbol);
tempPosition.setOpen(true);
tempPosition.setDirection(direction);
tempPosition.setAmount(coinAmount);  // 保存币的数量（用于平仓）
tempPosition.setEntrySpotPrice(opp.spotPrice);
tempPosition.setEntrySwapPrice(opp.futuresPrice);
tempPosition.setEntrySpreadRate(opp.spreadRate);  // 保存开仓价差率
tempPosition.setOpenTime(System.currentTimeMillis());
tempPosition.setSpotOrderId(spotOrderId);
tempPosition.setSwapOrderId(swapOrderId);
tempPosition.setLastLogTime(System.currentTimeMillis());
positionState.update(tempPosition);
```

同时修改 `printStatusLog` 方法，正确显示价差率：

```java
// 修复：显示实际的开仓价差率
if (position.getEntrySpreadRate() != null) {
    status.append(String.format("  开仓价差率: %s%%\n", 
        position.getEntrySpreadRate().multiply(new BigDecimal("100")).setScale(3, RoundingMode.HALF_UP)));
} else {
    status.append("  开仓价差率: 未记录\n");
}
```

- `data-warehouse/src/main/java/com/crypto/dw/processor/TradingDecisionProcessor.java`

编译成功，无错误。

## 注意事项

1. `PositionState.amount` 字段存储的是币的数量，不是 USDT 金额
2. 实际的 USDT 金额 = 币数量 × 现货价格
3. 开仓价差率需要在创建持仓状态时保存，否则会显示为 `null`
4. 价差率显示时需要乘以 100 转换为百分比
