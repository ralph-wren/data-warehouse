# 持仓监控 - 币对状态报告显示当前价差和订单详情CSV存储增强

## 问题描述

用户反馈两个问题:
1. 根据订单ID查询的明细数据没有存储到CSV文件
2. 币对状态报告没有打印当前价差信息

## 问题分析

### 问题1: 订单详情CSV存储

代码已经实现了订单详情查询和CSV保存功能,但用户反馈"没有存储到csv"。可能的原因:
- 异步线程执行失败,没有详细日志
- CSV文件写入器未正确初始化
- 文件写入权限问题

### 问题2: 当前价差显示

代码已经在 `processElement1` 方法中更新了 `currentOpportunity` 状态,但 `printStatusLog` 方法没有使用这个状态来显示当前价差。

## 解决方案

### 1. 增强订单详情查询日志

在 `handleOrderFilled` 方法中添加详细日志:
- 查询开始日志
- 查询成功日志
- 查询失败日志
- 保存CSV日志

在 `saveOrderDetailToCsv` 方法中添加:
- CSV行内容日志
- orderDetailFullWriter 空值检查
- 详细的成功/失败日志

### 2. 使用 SyncCsvWriter 的线程池写入

重要说明:
- `orderQueryExecutor` 线程池用于异步查询订单详情(调用OKX API)
- `saveOrderDetailToCsv` 方法内部调用 `orderDetailFullWriter.write()`,它使用 `SyncCsvWriter` 的线程池写入CSV
- 这样设计的好处:
  - 查询和写入分离,互不阻塞
  - `SyncCsvWriter` 内部使用线程池,确保写入成功(同步等待)
  - 避免重复创建线程池

### 3. 修改 printStatusLog 方法签名

添加 `ArbitrageOpportunity` 参数,用于显示当前价差:

```java
private void printStatusLog(String symbol, PositionState position, OpportunityTracker tracker, 
                            ArbitrageOpportunity opportunity, long now) throws Exception
```

### 4. 在状态报告中显示当前价差

在持仓状态部分添加当前价差显示:

```java
// 新增：显示当前价差（如果有最新的套利机会数据）
if (opportunity != null) {
    status.append(String.format("  📊 当前价差: 现货=%s, 合约=%s, 价差率=%s%%\n",
        opportunity.spotPrice.setScale(6, RoundingMode.HALF_UP),
        opportunity.futuresPrice.setScale(6, RoundingMode.HALF_UP),
        opportunity.spreadRate));
}
```

### 5. 修改 onTimer 方法

获取 `currentOpportunity` 状态并传递给 `printStatusLog`:

```java
ArbitrageOpportunity opportunity = currentOpportunity.value();  // 获取当前套利机会
printStatusLog(symbol, position, tracker, opportunity, now);  // 传入当前套利机会
```

## 代码修改

### 文件: TradingDecisionProcessor.java

1. **printStatusLog 方法**: 添加 `ArbitrageOpportunity` 参数,显示当前价差
2. **onTimer 方法**: 获取 `currentOpportunity` 状态并传递
3. **handleOrderFilled 方法**: 添加详细的查询和保存日志,说明使用 SyncCsvWriter 的线程池
4. **saveOrderDetailToCsv 方法**: 添加空值检查和详细日志

## 线程池架构

```
订单成交 → orderQueryExecutor (异步查询OKX API)
           ↓
        查询成功 → saveOrderDetailToCsv()
                   ↓
                orderDetailFullWriter.write() → SyncCsvWriter 的线程池 → 写入CSV文件
```

- `orderQueryExecutor`: 2个线程,用于异步查询订单详情
- `SyncCsvWriter`: 2个线程,用于同步写入CSV文件
- 总共4个线程,查询和写入分离,互不阻塞

## 效果

### 订单详情CSV存储

通过详细日志可以排查问题:
- 如果看到"📡 正在查询订单详情"但没有"✅ 查询成功",说明查询失败
- 如果看到"✅ 查询成功"但没有"✅ 已保存到CSV",说明保存失败
- 如果看到"❌ orderDetailFullWriter 为空",说明CSV写入器未初始化

### 币对状态报告显示当前价差

状态报告现在会显示:

## 注意事项

1. **异步查询**: 订单详情查询使用 `orderQueryExecutor` 异步执行,不会阻塞主流程
2. **同步写入**: CSV写入使用 `SyncCsvWriter` 的线程池,确保数据写入成功
3. **日志级别**: 查询和保存日志使用 `info` 级别,方便排查问题
4. **空值检查**: 添加了 `orderDetailFullWriter` 空值检查,避免空指针异常
5. **当前价差**: 只有在有最新套利机会数据时才显示当前价差

## 测试建议

1. 观察日志中是否有"📡 正在查询订单详情"
2. 检查是否有"✅ 已保存订单详细信息到CSV"
3. 查看 `logs/trading/order_detail_full_YYYYMMDDHH.csv` 文件是否生成
4. 观察状态报告是否显示"📊 当前价差"
