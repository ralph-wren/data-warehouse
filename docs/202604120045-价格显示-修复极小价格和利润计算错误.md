# 价格显示 - 修复极小价格和利润计算错误

## 问题描述

用户反馈状态报告显示异常:
```
📊 当前价差: 现货=0.000000, 合约=0.000000, 价差率=-0.543000%
💰 预估利润: -1000000.0000 USDT (利润率: -0.20%)
```

问题:
1. 当前价差显示为 0.000000,无法看到实际价格
2. 预估利润显示 -1000000 USDT,明显错误

## 问题分析

### 问题1: 价格显示为 0

SATS 的价格非常小(科学计数法 `1.105E-8`),使用 `setScale(6)` 会四舍五入为 0:
- 现货价格: `1.105E-8` = `0.00000001105`
- 使用 `setScale(6)` → `0.000000`

### 问题2: 利润计算错误

原代码计算未实现利润时:
```java
// 错误的计算方式
BigDecimal fee = pos.getAmount().multiply(new BigDecimal("0.002"));
BigDecimal unrealizedProfit = spreadDiff.subtract(fee);
```

对于 SATS:
- `pos.getAmount()` = 500000000 (5亿个币)
- `fee` = 500000000 × 0.002 = 1000000 (错误!)
- 应该基于 USDT 金额计算手续费

正确的计算方式:
```java
// 计算 USDT 金额
BigDecimal usdtAmount = pos.getAmount().multiply(pos.getEntrySpotPrice());
// 基于 USDT 金额计算手续费
BigDecimal fee = usdtAmount.multiply(new BigDecimal("0.002"));
```

## 解决方案

### 1. 添加 formatPrice 方法

自适应精度显示价格:
- 价格 < 0.000001: 使用科学计数法 `%.2E`
- 价格 < 0.01: 使用 8 位小数
- 价格 >= 0.01: 使用 6 位小数

```java
private String formatPrice(BigDecimal price) {
    if (price == null) {
        return "N/A";
    }
    
    // 如果价格小于 0.000001，使用科学计数法
    if (price.compareTo(new BigDecimal("0.000001")) < 0) {
        return String.format("%.2E", price);  // 科学计数法，2位小数
    }
    // 如果价格小于 0.01，使用 8 位小数
    else if (price.compareTo(new BigDecimal("0.01")) < 0) {
        return price.setScale(8, RoundingMode.HALF_UP).stripTrailingZeros().toPlainString();
    }
    // 否则使用 6 位小数
    else {
        return price.setScale(6, RoundingMode.HALF_UP).stripTrailingZeros().toPlainString();
    }
}
```

### 2. 修复 updateUnrealizedProfit 方法

正确计算未实现利润:
```java
// 计算未实现利润 = 价差变化 × 币数量
BigDecimal unrealizedProfitFromSpread = spreadDiff.multiply(pos.getAmount());

// 计算手续费（基于 USDT 金额）
BigDecimal usdtAmount = pos.getAmount().multiply(pos.getEntrySpotPrice());
BigDecimal fee = usdtAmount.multiply(new BigDecimal("0.002"));

// 最终未实现利润 = 价差利润 - 手续费
BigDecimal unrealizedProfit = unrealizedProfitFromSpread.subtract(fee);

// 计算利润率（基于 USDT 金额）
BigDecimal profitRate = unrealizedProfit.divide(usdtAmount, 6, RoundingMode.HALF_UP)
    .multiply(new BigDecimal("100"));
```

### 3. 修复 shouldClose 方法

同样修复止损计算:
```java
// 计算未实现利润 = 价差变化 × 币数量
BigDecimal unrealizedProfitFromSpread = spreadDiff.multiply(pos.getAmount());

// 计算手续费（基于 USDT 金额）
BigDecimal usdtAmount = pos.getAmount().multiply(pos.getEntrySpotPrice());
BigDecimal fee = usdtAmount.multiply(new BigDecimal("0.002"));

// 最终未实现利润
BigDecimal unrealizedProfit = unrealizedProfitFromSpread.subtract(fee);
boolean stopLoss = unrealizedProfit.compareTo(maxLossPerTrade.negate()) < 0;
```

### 4. 修复 printStatusLog 方法

利润率计算基于 USDT 金额:
```java
// 计算 USDT 金额用于利润率计算
BigDecimal positionUsdtAmount = position.getAmount().multiply(position.getEntrySpotPrice());
BigDecimal profitRate = position.getUnrealizedProfit()
    .divide(positionUsdtAmount, 6, RoundingMode.HALF_UP)
    .multiply(new BigDecimal("100"));
```

## 计算示例

以 SATS 为例:
- 币数量: 500000000 (5亿个)
- 现货价格: 1.105E-8 USDT
- USDT 金额: 500000000 × 1.105E-8 = 5.525 USDT
- 手续费: 5.525 × 0.002 = 0.01105 USDT
- 价差变化: (1.099E-8 - 1.105E-8) × 500000000 = -0.03 USDT
- 未实现利润: -0.03 - 0.01105 = -0.04105 USDT
- 利润率: -0.04105 / 5.525 × 100 = -0.74%

## 注意事项

1. **科学计数法**: 对于极小价格(< 0.000001),使用科学计数法显示
2. **USDT 金额**: 所有手续费和利润率计算都基于 USDT 金额,不是币数量
3. **精度**: 不同价格范围使用不同精度,确保显示清晰
4. **stripTrailingZeros**: 去除尾部的 0,使显示更简洁
