# 价格显示修复 - 修复开仓价差率和当前价差率显示错误

## 问题描述

用户反馈持仓状态报告中显示的价差率异常：
- RSR 的开仓价差率显示为 -52.390%（实际应该是 -0.524%）
- 当前价差率显示为 -0.523900%（格式正确但精度过高）

## 问题分析

通过代码审查发现两个关键问题：

1. **开仓价差率显示错误**：
   - `ArbitrageOpportunity.spreadRate` 字段存储的是百分比值（如 -0.524 表示 -0.524%）
   - 在 `openPosition` 方法中，`entrySpreadRate` 直接保存了 `opp.spreadRate`
   - 但在 `printStatusLog` 方法中，代码将 `entrySpreadRate` 乘以 100 再显示
   - 导致显示为 -52.390%（实际应该是 -0.524%）

2. **当前价差率显示错误**：
   - 在 `printStatusLog` 方法中，直接显示 `opportunity.spreadRate`
   - 但这个值已经是百分比了，不需要再乘以 100
   - 导致显示格式不一致

## 解决方案

修改 `TradingDecisionProcessor.printStatusLog()` 方法：

```java
// 修复前：
status.append(String.format("  开仓价差率: %s%%\n", 
    position.getEntrySpreadRate().multiply(new BigDecimal("100")).setScale(3, RoundingMode.HALF_UP)));

status.append(String.format("  📊 当前价差: 现货=%s, 合约=%s, 价差率=%s%%\n",
    spotPriceStr,
    futuresPriceStr,
    opportunity.spreadRate));

// 修复后：
status.append(String.format("  开仓价差率: %s%%\n", 
    position.getEntrySpreadRate().setScale(3, RoundingMode.HALF_UP)));

status.append(String.format("  📊 当前价差: 现货=%s, 合约=%s, 价差率=%s%%\n",
    spotPriceStr,
    futuresPriceStr,
    opportunity.spreadRate.setScale(3, RoundingMode.HALF_UP)));
```

## 修改内容

1. 移除开仓价差率的 `multiply(new BigDecimal("100"))` 操作
2. 为当前价差率添加 `setScale(3, RoundingMode.HALF_UP)` 统一精度

## 验证结果

修复后的显示效果：
- 开仓价差率：-0.524%（正确）
- 当前价差率：-0.524%（正确，精度统一为 3 位小数）

## 相关文件

- `data-warehouse/src/main/java/com/crypto/dw/processor/TradingDecisionProcessor.java`

## 注意事项

1. `ArbitrageOpportunity.spreadRate` 字段存储的是百分比值，不需要再乘以 100
2. 所有价差率显示都应该统一精度为 3 位小数
3. 开仓时保存的 `entrySpreadRate` 直接来自 `opp.spreadRate`，已经是百分比值
