# 订单查询修复 - 添加 instId 参数解决查询失败问题

**日期**: 2026-04-12 02:00  
**问题**: 查询订单详情时报错 "Parameter instId can not be empty"  
**状态**: ✅ 已解决

---

## 问题描述

交易程序在查询订单详情时失败,错误日志:

```
❌ 查询订单详情失败: orderId=3470457545767018496, instType=SPOT, code=50014, msg=Parameter instId can not be empty.
```

## 根本原因

OKX API 查询订单详情接口 `/api/v5/trade/order` 需要 3 个必需参数:
- `ordId`: 订单ID
- `instId`: 交易对名称 (如 `ACH-USDT`, `ACH-USDT-SWAP`)
- `instType`: 订单类型 (SPOT/SWAP)

但代码中只传了 `ordId` 和 `instType`,缺少 `instId` 参数。

## 解决方案

### 1. 修改 `OKXTradingService.queryOrderDetail` 方法

**修改前**:
```java
public OrderDetailResponse queryOrderDetail(String orderId, String instType)
```

**修改后**:
```java
public OrderDetailResponse queryOrderDetail(String orderId, String instId, String instType)
```

添加 `instId` 参数,并在 API 请求路径中包含该参数:

```java
String path = String.format("/api/v5/trade/order?ordId=%s&instId=%s&instType=%s", 
    orderId, instId, instType);
```

### 2. 修改 `TradingDecisionProcessor` 中的调用

根据订单类型构造正确的 `instId`:

```java
// 现货订单: symbol + "-USDT"
String spotInstId = symbol + "-USDT";
OrderDetailResponse spotDetail = tradingService.queryOrderDetail(
    spotOrderId, spotInstId, "SPOT");

// 合约订单: symbol + "-USDT-SWAP"
String swapInstId = symbol + "-USDT-SWAP";
OrderDetailResponse swapDetail = tradingService.queryOrderDetail(
    swapOrderId, swapInstId, "SWAP");
```
