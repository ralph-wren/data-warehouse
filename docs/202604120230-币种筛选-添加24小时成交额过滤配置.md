# 币种筛选 - 添加24小时成交额过滤配置

## 问题描述

在 `FlinkTickerCollectorJob` 筛选币种时,需要过滤掉24小时成交额小于30万 USDT 的币种,并将阈值做成配置项,避免订阅成交量过低的币种。

## 解决方案

### 1. 添加配置项

在 `application-dev.yml` 中添加 `ticker.filter.min-volume-24h` 配置:

```yaml
ticker:
  subscribe:
    number: 20
  filter:
    margin-only: true  # 是否只订阅支持现货杠杆的币对
    min-volume-24h: "300000"  # 最小24小时成交额(USDT),默认30万
```

### 2. 修改现货价格筛选逻辑

在 `OKXRestApiSource.fetchSpotPrices()` 方法中添加成交额过滤:

```java
// ⭐ 过滤24小时成交额小于配置值的币种
if (item.has("volCcy24h")) {
    String volCcy24hStr = item.get("volCcy24h").asText();
    try {
        BigDecimal volCcy24h = new BigDecimal(volCcy24hStr);
        if (volCcy24h.compareTo(minVolume24h) < 0) {
            filteredByVolume++;
            continue;  // 跳过成交额不足的币种
        }
    } catch (NumberFormatException e) {
        logger.warn("解析24小时成交额失败: {} - {}", symbol, volCcy24hStr);
        continue;
    }
}
```

### 3. 修改合约价格筛选逻辑

在 `OKXRestApiSource.fetchSwapPrices()` 方法中添加相同的成交额过滤逻辑:

```java
// ⭐ 过滤24小时成交额小于配置值的币种
if (item.has("volCcy24h")) {
    String volCcy24hStr = item.get("volCcy24h").asText();
    try {
        BigDecimal volCcy24h = new BigDecimal(volCcy24hStr);
        if (volCcy24h.compareTo(minVolume24h) < 0) {
            filteredByVolume++;
            continue;  // 跳过成交额不足的币种
        }
    } catch (NumberFormatException e) {
        logger.warn("解析24小时成交额失败: {} - {}", symbol, volCcy24hStr);
        continue;
    }
}
```

### 4. 添加统计日志

在两个方法中都添加了统计日志,显示筛选结果:

```java
logger.info("现货价格筛选: 总数={}, 成交额过滤={}, 保留={}", 
    totalCount, filteredByVolume, prices.size());

logger.info("合约价格筛选: 总数={}, 成交额过滤={}, 保留={}", 
    totalCount, filteredByVolume, prices.size());
```

## 技术细节

### 成交额字段说明

- `volCcy24h`: 24小时成交额,以计价货币(USDT)计
- 单位: USDT
- 来源: OKX Ticker API 返回的实时数据

### 配置读取

在 `OKXRestApiSource` 构造函数中读取配置:

```java
// 读取24小时成交额过滤阈值(默认30万USDT)
String minVolume24hStr = config.getProperty("ticker.filter.min-volume-24h", "300000");
this.minVolume24h = new BigDecimal(minVolume24hStr);
```

### 过滤逻辑

1. 从 Ticker API 响应中读取 `volCcy24h` 字段
2. 转换为 `BigDecimal` 类型
3. 与配置的 `minVolume24h` 比较
4. 小于阈值的币种跳过,不加入价格列表
5. 统计过滤数量并记录日志

## 注意事项

1. 成交额单位是 USDT,不是币的数量
2. 配置值使用字符串格式,避免精度丢失
3. 解析失败的币种会跳过并记录警告日志
4. 统计日志会显示总数、过滤数、保留数,方便监控
