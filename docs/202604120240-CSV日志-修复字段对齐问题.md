# CSV日志 - 修复字段对齐问题

## 问题描述

`order_detail_full_2026041202.csv` 文件第三行出现字段对不齐的问题,前面有很多逗号,导致数据错位。

## 问题现象

CSV文件内容:
```csv
timestamp,symbol,order_id,inst_type,inst_id,side,pos_side,order_type,price,avg_price,size,filled_size,state,fee,fee_ccy,fill_time,create_time,update_time
2026-04-12 02:12:36,RSR,3470493230871896064,SPOT,RSR-USDT,sell,net,market,,0.001502031662877,3700,3700,filled,-0.005557517152645,USDT,2026-04-12 02:12:38,2026-04-12 02:12:38,2026-04-12 02:12:38
2026-04-12 02:12:36,RSR,3470493238857850884,SWAP,RSR-USDT-SWAP,buy,long,market,,0.0015,37,37,filled,-0.002775,USDT,2026-04-12 02:12:38,2026-04-12 02:12:38,2026-04-12 02:12:38
,,,,,,,,,,2026-04-12 02:35:07,DASH,3470538570257948672,SPOT,DASH-USDT,sell,net,market,,47.03,0.12,0.12,filled,-0.0056436,USDT,2026-04-12 02:35:09,2026-04-12 02:35:09,2026-04-12 02:35:09
```

第4行(数据的第3行)前面有10个空字段(逗号),导致数据从第11个字段开始,字段完全错位。

## 根本原因

在 `TradingDecisionProcessor.saveOrderDetailToCsv()` 方法中,有一行日志输出:

```java
logger.info("📄 CSV行内容: {}", csvLine);
```

这行日志可能在某些情况下被误写入了CSV文件,导致字段对不齐。虽然日志配置正确,但为了避免任何潜在的问题,应该删除这行不必要的日志输出。

## 解决方案

删除可能导致问题的日志输出:

```java
// 修改前
String csvLine = String.format("%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s",
    timestamp, symbol, orderId, instType, instId, side, posSide,
    orderType, price, avgPrice, size, filledSize, state,
    fee, feeCcy, fillTimeStr, createTimeStr, updateTimeStr);

logger.info("📄 CSV行内容: {}", csvLine);  // ❌ 删除这行

if (orderDetailFullWriter == null) {
    logger.error("❌ orderDetailFullWriter 为空,无法写入CSV!");
    return;
}

orderDetailFullWriter.write(csvLine);

// 修改后
String csvLine = String.format("%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s,%s",
    timestamp, symbol, orderId, instType, instId, side, posSide,
    orderType, price, avgPrice, size, filledSize, state,
    fee, feeCcy, fillTimeStr, createTimeStr, updateTimeStr);

// ✅ 删除了日志输出

if (orderDetailFullWriter == null) {
    logger.error("❌ orderDetailFullWriter 为空,无法写入CSV!");
    return;
}

orderDetailFullWriter.write(csvLine);
```

## 技术要点

1. **CSV写入应该避免日志干扰**: 在写入CSV之前,不要输出包含CSV内容的日志,避免日志被误写入
2. **SyncCsvWriter是线程安全的**: 使用线程池同步写入,确保数据写入成功
3. **日志输出应该简洁**: 只输出必要的信息,避免输出大量数据
