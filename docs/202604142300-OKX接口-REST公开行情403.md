# OKX 公开 REST 行情返回 HTTP 403

**时间**: 2026-04-14  
**现象**: `FlinkTickerCollectorJob` 日志出现 `价差计算失败: HTTP 请求失败, 状态码: 403`，订阅列表为空，缓存刷新跳过。

## 原因说明

1. **WAF / CDN 拦截（最常见）**  
   `HttpURLConnection` 若不设置 `User-Agent`，JVM 会带上类似 `Java/17.0.x` 的默认 UA，部分交易所边缘防护会直接返回 **403**，与是否签名无关（`/api/v5/market/tickers` 为公开接口）。

2. **网络与地区**  
   机房/云主机出口 IP、公司代理、跨境线路可能被风控；此时即使用浏览器 UA 仍可能 403，需换网络、代理或按官方文档使用可访问的域名。

3. **排查方式**  
   同一台机器执行：  
   `curl -sS -o /dev/null -w "%{http_code}\n" "https://www.okx.com/api/v5/market/tickers?instType=SPOT"`  
   若 curl 为 200 而 Java 为 403，多半是 UA 或 JVM 代理差异。

## 代码侧改动

- `OKXRestApiSource`：默认设置浏览器风格 `User-Agent`、`Accept: application/json`；支持 `okx.rest.base-url`、`okx.rest.user-agent`；非 200 时打日志并附带响应体片段。
- `FlinkTickerCollectorJob`：REST 失败时回退 `okx.symbols` / `okx.symbols.spot`；支持从命令行传入币对（跳过 `--env`）。

## 配置示例

```yaml
okx:
  rest:
    base-url: https://www.okx.com
    # user-agent: "自定义 UA 字符串"  # 可选
```

## 验证

```bash
mvn -q compile -DskipTests
```

本地再跑一次采集作业，关注日志中的 `OKX REST 请求: baseUrl=...` 与是否仍出现 403。
