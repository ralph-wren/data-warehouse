# 状态日志 - 控制台报告改为真实成交口径

## 问题描述

`logs/trading/status_log_*.csv` 中的投入、开仓价、利润率已经优先使用真实成交明细，但控制台“币对状态报告”仍然显示下单时价格，导致人工核对时出现口径不一致。

## 核查现象

以 `KMNO` 为例：

- `order_detail_full_2026041622.csv` 中真实成交均价为：
  - 现货：`0.02143`
  - 合约：`0.02141`
- `status_log_2026041622.csv` 中成本和利润计算也基于上述真实成交价
- 但控制台日志仍显示：
  - 现货：`0.02157`
  - 合约：`0.02145`

这两个值是下单提交时价格，不是真实成交均价。

## 根本原因

`TradingDecisionProcessor.printStatusLog()` 中：

1. `金额` 计算使用的是 `entrySpotPrice`
2. `开仓价格` 展示使用的是 `entrySpotPrice / entrySwapPrice`
3. 没有复用 `writeStatusLogToCsv()` 中“优先真实成交价，否则回退委托价”的逻辑

因此控制台报告与 CSV 状态日志出现双重口径。

## 解决方案

将控制台状态报告改为与 CSV 状态日志一致：

1. `金额` 优先使用 `actualSpotPrice`
2. `开仓价格` 优先显示 `actualSpotPrice / actualSwapPrice`
3. 若真实成交价尚未回填，再回退到 `entrySpotPrice / entrySwapPrice`

## 修改点

文件：

- `src/main/java/com/crypto/dw/processor/TradingDecisionProcessor.java`

修改逻辑：

```java
BigDecimal amountPrice = position.getActualSpotPrice() != null
    ? position.getActualSpotPrice()
    : position.getEntrySpotPrice();

String entrySpotPrice = position.getActualSpotPrice() != null
    ? formatPrice(position.getActualSpotPrice())
    : formatPrice(position.getEntrySpotPrice());

String entrySwapPrice = position.getActualSwapPrice() != null
    ? formatPrice(position.getActualSwapPrice())
    : formatPrice(position.getEntrySwapPrice());
```

## 修复效果

修复后，以下三处口径一致：

1. 交易所订单详情
2. `order_detail_full_*.csv` / `status_log_*.csv`
3. 控制台“币对状态报告”

这样在核对投入、成本、数量、利润时，不会再出现“CSV 正确但控制台价格偏旧”的误导。

## 验证建议

1. 重新观察新一笔开仓后的控制台“币对状态报告”
2. 对比 `order_detail_full_*.csv` 中的 `avg_price`
3. 确认控制台中的 `金额 / 开仓价格 / 利润率` 与 `status_log_*.csv` 一致
