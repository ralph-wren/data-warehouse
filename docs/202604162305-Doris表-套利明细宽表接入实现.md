# Doris表 - 套利明细宽表接入实现

## 目标

将 `ads_arbitrage_opportunities` 从“只保存发现阶段机会”的简表，改为保存：

1. 开仓提交事件
2. 持仓状态快照
3. 平仓提交事件
4. 下单/平仓失败事件

并尽量覆盖控制台状态日志、订单日志、成本费用与利润字段。

## 实现方式

### 1. 扩展 `TradeRecord`

为 `TradeRecord` 增加以下宽表字段：

- 事件基础信息：`eventType`、`eventStage`、`positionStatus`
- 发现阶段字段：`discoverSpotPrice`、`discoverSwapPrice`、`discoverSpreadRate`
- 配置快照：`tradeAmountUsdt`、`openThreshold`、`closeThreshold`、`maxLossUsdt`
- 订单与成交字段：`spotOrderId`、`swapOrderId`、`actualSpotPrice`、`actualSwapPrice`
- 成本与费用：`spotCost`、`swapCost`、`totalCost`、`totalFee`
- 持仓状态：`currentSpotPrice`、`currentSwapPrice`、`currentSpreadRate`、`unrealizedProfit`
- 跟踪器与日志摘要：`trackerActive`、`statusMessage`、`logSource`

### 2. 在 `TradingDecisionProcessor` 输出 Doris 明细事件

新增以下事件构造逻辑：

- `OPEN_PENDING`
- `POSITION_STATUS`
- `CLOSE_PENDING`
- `REJECTED`
- `ERROR`

这些事件通过 `Collector<TradeRecord>` 输出，形成统一明细流。

### 3. `FlinkADSArbitrageJob` 改为使用详细事件流写入 Doris

原来 `ads_arbitrage_opportunities` 使用 `arbitrageStream` 直接落库，只包含：

- `symbol`
- `spot_price`
- `futures_price`
- `spread`
- `spread_rate`
- `profit_estimate`

现在改为使用 `tradeStream` 映射为完整 JSON 后写入 `ads_arbitrage_opportunities`。

## 当前可落库事件

1. 开仓订单已提交
2. 持仓中状态快照（定时器输出）
3. 平仓订单已提交
4. 开仓/平仓失败事件

## 当前限制

平仓后异步查询到的“最终已实现盈亏汇总”目前仍只写到 `summary.csv`，还没有回灌到 Doris 宽表。

原因是该汇总发生在异步线程中，当前 Doris 明细流仍由 Flink 主线程的 `Collector` 输出。
