# 主线程性能优化：REST 调用改为线程池异步执行

## 背景

套利作业存在 Kafka lag 偏高问题。排查发现主线程在关键路径中执行外部 REST 调用，会导致 `processElement` 处理延迟上升并触发反压。

## 本次改造

### 1) 持仓阶段订单详情回填异步化

- 主线程不再直接执行订单详情查询 REST。
- 改为线程池执行查询，查询结果通过并发容器回传。
- 主线程仅在处理循环中消费结果并更新 Flink state。

### 2) 开仓/平仓下单异步化

- 开仓和平仓触发时，主线程只提交异步命令，不阻塞。
- 线程池执行 `calculateValidCoinAmount` 与下单 REST。
- 主线程统一消费异步下单结果，负责：
  - 更新 `PositionState`
  - 写入 CSV 日志
  - 输出 Doris 事件（`TradeRecord`）

### 3) 防重复提交

- 增加 in-flight 标记，防止同一 symbol 的 OPEN/CLOSE 命令并发重复提交。

## 结果

- REST 外部 I/O 从主线程剥离，降低消费主链路阻塞风险。
- Kafka lag 场景下，主线程吞吐与稳定性预期提升。
