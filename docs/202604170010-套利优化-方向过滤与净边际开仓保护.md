# 套利策略优化：方向过滤与净边际开仓保护

## 背景

近期 `STATUS` 事件中的浮动盈亏多数为负，且 `SHORT_SPOT_LONG_SWAP` 方向表现明显偏弱。  
同时，原开仓判定仅基于裸价差阈值，未显式扣除手续费、滑点与安全缓冲，容易出现“看起来有价差但净边际不足”的开仓。

## 本次改动

## 1) 方向过滤（Direction Filter）

新增配置：

- `arbitrage.trading.direction-filter.enable-long-spot-short-swap`
- `arbitrage.trading.direction-filter.enable-short-spot-long-swap`

默认策略：

- 开启 `LONG_SPOT_SHORT_SWAP`
- 关闭 `SHORT_SPOT_LONG_SWAP`

效果：先屏蔽历史表现较差方向，降低回撤与亏损概率。

## 2) 净边际开仓保护（Open Guard）

新增配置：

- `arbitrage.trading.open-guard.fee-rate-estimate`
- `arbitrage.trading.open-guard.slippage-rate-estimate`
- `arbitrage.trading.open-guard.safety-buffer-rate`

计算逻辑：

- `minNetEdgeRate = feeRateEstimate + slippageRateEstimate + safetyBufferRate`
- 开仓需同时满足：
  1. `abs(spreadRate) > openThreshold`
  2. `abs(spreadRate) - minNetEdgeRate > 0`

效果：把开仓从“裸价差判断”升级为“净边际判断”。

## 配置落地

已在以下环境配置添加：

- `src/main/resources/config/application-dev.yml`
- `src/main/resources/config/application-docker.yml`
- `src/main/resources/config/application-prod.yml`

默认值：

- `fee-rate-estimate: 0.0015`
- `slippage-rate-estimate: 0.0015`
- `safety-buffer-rate: 0.0005`

## 说明

- 新配置项为必填，缺失时会在作业启动阶段显式报错，避免“配置未生效”。
- 异步下单路径中也统一使用 `resolveOpenDirection`，保证判定与执行方向口径一致。
