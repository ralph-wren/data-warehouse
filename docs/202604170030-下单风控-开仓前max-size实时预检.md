# 下单风控增强：开仓前 max-size 实时预检

## 背景

近期出现多次下单失败，典型错误为：

- `Insufficient USDT margin in account`
- `order amount exceeds max order amount`

即使账户总权益较高，也可能因为可用保证金、账户上限或交易模式限制导致下单失败。

## 本次改动

在开仓异步执行链路中新增“实时可下单量预检”：

1. 先查询账户 `max-size`（spot/swap，`tdMode=cross`）
2. 计算本次需要下单的现货数量与合约张数
3. 若 `required > maxBuy/maxSell`，直接拦截并返回明确失败原因
4. 不再进入真实下单 API，避免盲下后报错

## 技术实现

### `OKXTradingService`

新增：

- `MaxSizeInfo` 数据结构
- `queryMaxSize(instId, tdMode)` 方法  
  对接接口：`GET /api/v5/account/max-size`

### `TradingDecisionProcessor`

新增：

- `precheckOpenOrderCapacity(symbol, direction, coinAmount)`  
  在 `executeOpenOrderCommand` 中下单前调用

覆盖校验：

- `LONG_SPOT_SHORT_SWAP`: 现货 `maxBuy` + 合约 `maxSell`
- `SHORT_SPOT_LONG_SWAP`: 现货 `maxSell` + 合约 `maxBuy`

## 预期收益

- 提前识别“可下单量不足”并快速失败
- 减少无效下单请求与错误噪音
- 提高失败日志可解释性（明确 required 与 max 值）
