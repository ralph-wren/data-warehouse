## 快速开始

### 1. 环境准备

```bash
# 检查 Java 版本
java -version  # 应该是 Java 11

# 检查 Docker
docker --version

# 检查 Maven
mvn --version
```

### 2. 启动 Doris 3.1.x

```bash
# 停止旧版本 (如果有)
docker stop doris-fe doris-be 2>/dev/null
docker rm doris-fe doris-be 2>/dev/null

# 启动 Doris 3.1.x
docker-compose up -d doris-fe doris-be

# 或手动启动
docker run -d --name doris-fe \
  -p 8030:8030 -p 9030:9030 \
  apache/doris:fe-3.1.0

docker run -d --name doris-be \
  -p 8040:8040 -p 9050:9050 \
  apache/doris:be-3.1.0

# 等待启动完成 (约 30 秒)
sleep 30
```

### 3. 初始化数据库

```bash
# 创建数据库
mysql -h 127.0.0.1 -P 9030 -u root -e "CREATE DATABASE IF NOT EXISTS crypto_dw;"

# 创建表
mysql -h 127.0.0.1 -P 9030 -u root crypto_dw < sql/create_tables.sql

# 验证
mysql -h 127.0.0.1 -P 9030 -u root -e "SHOW TABLES FROM crypto_dw;"
```

### 4. 启动 Kafka

```bash
# 使用项目提供的脚本
./kafka-setup.sh

# 或使用 Docker Compose
docker-compose up -d kafka zookeeper
```

### 5. 编译项目

```bash
# 清理并编译
mvn clean compile -DskipTests

# 打包
mvn package -DskipTests
```

### 6. 启动数据采集

```bash
# 启动 OKX WebSocket 数据采集器
./run-collector.sh
```

### 7. 启动 Flink ODS 作业

```bash
# 使用官方 Doris Connector (推荐)
./run-flink-ods-datastream2.sh
```

### 8. 验证数据

```bash
# 查看数据量
mysql -h 127.0.0.1 -P 9030 -u root -e "
SELECT COUNT(*) as total 
FROM crypto_dw.ods_crypto_ticker_rt;
"

# 查看最新数据
mysql -h 127.0.0.1 -P 9030 -u root -e "
SELECT * 
FROM crypto_dw.ods_crypto_ticker_rt 
ORDER BY ingest_time DESC 
LIMIT 10;
"
```

## 核心配置

### application.yml

```yaml
# Kafka 配置
kafka:
  bootstrap-servers: localhost:9092
  topic:
    crypto-ticker: crypto_ticker

# Doris 配置
doris:
  fe:
    http-url: http://127.0.0.1:8030
    username: root
    password: ""
  database: crypto_dw
  tables:
    ods: ods_crypto_ticker_rt
  stream-load:
    batch-size: 1000
    batch-interval-ms: 5000
    max-retries: 3

# Flink 配置
flink:
  execution:
    parallelism: 2
  checkpoint:
    interval: 30000
```

## 性能调优

### 1. Flink 并行度

根据 CPU 核心数调整:
```yaml
flink:
  execution:
    parallelism: 4  # 建议 = CPU 核心数
```

### 2. Doris 批量大小

根据数据量调整:
```yaml
doris:
  stream-load:
    batch-size: 2000  # 数据量大时增加
    batch-interval-ms: 3000  # 延迟敏感时减少
```

### 3. Checkpoint 间隔

根据一致性要求调整:
```yaml
flink:
  checkpoint:
    interval: 60000  # 一致性要求高时减少
```

## 生产环境建议

### 1. 高可用部署

- Doris: 至少 3 个 FE 节点,3 个 BE 节点
- Kafka: 至少 3 个 Broker,副本数 = 3
- Flink: 使用 Standalone 或 Kubernetes 集群模式

### 2. 资源配置

**Doris BE**:
- CPU: 8 核+
- 内存: 16GB+
- 磁盘: SSD,500GB+

**Flink TaskManager**:
- CPU: 4 核+
- 内存: 8GB+

**Kafka Broker**:
- CPU: 4 核+
- 内存: 8GB+
- 磁盘: SSD,200GB+

### 3. 监控告警

- Flink: 使用 Prometheus + Grafana
- Doris: 监控 BE 存活状态,查询延迟
- Kafka: 监控消费延迟,磁盘使用率

### 4. 备份策略

- Doris: 定期备份元数据和数据
- Kafka: 配置合理的保留时间
- Flink: 定期保存 Savepoint

创建 `start-all.sh`:

```bash

echo "启动实时加密货币数据仓库"
echo "=========================================="
echo ""

# 1. 启动 Doris

echo "3. 启动 Kafka..."
./kafka-setup.sh

echo "5. 启动 Flink ODS 作业..."

echo "  mysql -h 127.0.0.1 -P 9030 -u root crypto_dw"

```
