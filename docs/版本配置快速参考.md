## 常见问题

### Q1: 为什么选择 Flink 1.17 而不是 1.18?

**答**: 
- Flink 1.17.2 是长期支持版本 (LTS),经过充分验证
- Flink 1.18 是较新版本,可能存在未知问题
- Doris Connector 1.6.2 与 Flink 1.17 配合最稳定
- 生产环境优先选择稳定性,而非新特性

### Q2: 遇到 DECIMAL128 类型错误怎么办?

**答**:
1. 使用自定义 DorisStreamLoadSink (已实现)
2. 或者升级到 Doris Connector 24.0.x (不推荐)
3. 或者修改表结构,降低 DECIMAL 精度

### Q3: 日志不输出怎么办?

**答**:
1. 确保添加了 `log4j2.properties` 配置文件
2. 确保添加了 Log4j2 依赖
3. 检查日志级别配置是否正确
4. 查看 `logs/` 目录下的日志文件

### Q4: 如何升级版本?

**答**:
```bash
# 1. 修改 pom.xml 中的版本号
# 2. 清理旧的依赖
mvn clean

# 3. 重新编译
mvn compile

# 4. 运行测试
mvn test

# 5. 打包
mvn package
```

## 性能调优参数

### Flink 配置

```yaml
# config/application.yml
flink:
  execution:
    parallelism: 4  # 并行度,根据 CPU 核心数调整
  checkpoint:
    interval: 60000  # Checkpoint 间隔 (ms)
```

### Doris Stream Load 配置

```yaml
doris:
  stream-load:
    batch-size: 1000  # 批量大小,建议 500-2000
    batch-interval-ms: 5000  # 批量间隔,建议 3000-10000
    max-retries: 3  # 最大重试次数
```

## 监控指标

### 关键指标

1. **吞吐量**: records/second
2. **延迟**: end-to-end latency
3. **错误率**: failed records / total records
4. **资源使用**: CPU, Memory, Network

### 监控命令

```bash
# 查看 Flink 作业状态
flink list

# 查看 Flink 作业详情
flink info <job-id>

# 取消 Flink 作业
flink cancel <job-id>
```

## 故障排查

### 常见错误

1. **ClassNotFoundException**
   - 检查依赖是否正确打包
   - 使用 `mvn dependency:tree` 查看依赖

2. **Connection refused**
   - 检查 Kafka/Doris 服务是否启动
   - 检查网络连接和防火墙

3. **OutOfMemoryError**
   - 增加 JVM 堆内存: `-Xmx2g`
   - 减少并行度或批量大小

4. **Stream Load failed**
   - 查看详细日志
   - 检查 Doris 表结构
   - 检查数据格式
