# Flink 监控系统使用指南

## 快速开始

### 1. 启动监控服务

```bash
# 启动 Prometheus + Grafana + Pushgateway
bash start-monitoring.sh
```

等待约 30 秒，确保所有服务启动完成。

### 2. 启动 Flink 作业

```bash
# 启动 ODS 作业
bash run-flink-ods-datastream.sh

# 启动 DWD 作业（可选）
bash run-flink-dwd-sql.sh
```

### 3. 验证监控系统

```bash
# 运行测试脚本
bash test/test-monitoring.sh
```

### 4. 访问监控界面

- **Prometheus**: http://localhost:9090
- **Grafana**: http://localhost:3000 (admin/admin)
- **Pushgateway**: http://localhost:9091
- **Flink ODS Job**: http://localhost:8081
- **Flink DWD Job**: http://localhost:8082

## Prometheus 使用

### 查看 Targets

1. 访问 http://localhost:9090/targets
2. 确认 Pushgateway 状态为 UP

### 查询指标

1. 访问 http://localhost:9090/graph
2. 输入查询语句，例如：

```promql
# 查看所有 Flink 指标
{job=~"flink-.*"}

# 查看输入速率
flink_taskmanager_job_task_numRecordsInPerSecond

# 查看输出速率
flink_taskmanager_job_task_numRecordsOutPerSecond

# 查看 JVM 堆内存使用
flink_taskmanager_Status_JVM_Memory_Heap_Used

# 查看 Checkpoint 耗时
flink_jobmanager_job_lastCheckpointDuration

# 按作业名称过滤
flink_taskmanager_job_task_numRecordsInPerSecond{job="flink-ods-job"}
```

### 常用查询示例

```promql
# 1. 总吞吐量（所有任务）
sum(rate(flink_taskmanager_job_task_numRecordsIn[1m]))

# 2. 平均延迟
avg(flink_taskmanager_job_task_currentInputWatermark)

# 3. JVM 内存使用率
flink_taskmanager_Status_JVM_Memory_Heap_Used / flink_taskmanager_Status_JVM_Memory_Heap_Max * 100

# 4. Checkpoint 成功率
rate(flink_jobmanager_job_numberOfCompletedCheckpoints[5m]) / 
(rate(flink_jobmanager_job_numberOfCompletedCheckpoints[5m]) + rate(flink_jobmanager_job_numberOfFailedCheckpoints[5m])) * 100

# 5. GC 时间占比
rate(flink_taskmanager_Status_JVM_GarbageCollector_G1_Young_Generation_Time[1m]) / 1000 * 100
```

## Grafana 使用

### 首次登录

1. 访问 http://localhost:3000
2. 使用默认账号登录：
   - 用户名: `admin`
   - 密码: `admin`
3. 首次登录后会提示修改密码（可跳过）

### 查看 Dashboard

1. 点击左侧菜单 "Dashboards"
2. 选择 "Flink Monitoring"
3. 查看实时监控数据

### Dashboard 面板说明

| 面板名称 | 说明 | 指标 |
|---------|------|------|
| Records In/Out Rate | 输入/输出速率 | numRecordsInPerSecond, numRecordsOutPerSecond |
| JVM Memory | JVM 内存使用 | Heap Used, Heap Max |
| CPU Load | CPU 负载 | JVM.CPU.Load |
| Checkpoint Duration | Checkpoint 耗时 | lastCheckpointDuration |
| Checkpoint Count | Checkpoint 数量 | numberOfCompletedCheckpoints, numberOfFailedCheckpoints |
| GC Time | GC 耗时 | GarbageCollector.*.Time |
| Task Status | 任务状态 | Task.*.Status |

### 自定义 Dashboard

1. 点击右上角 "Add panel"
2. 选择 "Add a new panel"
3. 在 Query 中输入 PromQL 查询语句
4. 配置可视化选项（图表类型、颜色等）
5. 点击 "Apply" 保存

### 设置告警

1. 编辑 Panel
2. 点击 "Alert" 标签
3. 配置告警规则：
   - Condition: 触发条件
   - Evaluate every: 评估间隔
   - For: 持续时间
4. 配置通知渠道（Email、Slack 等）
5. 保存

## Pushgateway 使用

### 查看指标

访问 http://localhost:9091 查看所有推送的指标。

### 手动推送指标（测试）

```bash
# 推送单个指标
echo "test_metric 123" | curl --data-binary @- http://localhost:9091/metrics/job/test

# 推送多个指标
cat <<EOF | curl --data-binary @- http://localhost:9091/metrics/job/test
# TYPE test_counter counter
test_counter 42
# TYPE test_gauge gauge
test_gauge 3.14
EOF
```

### 删除指标

```bash
# 删除指定 job 的所有指标
curl -X DELETE http://localhost:9091/metrics/job/test

# 删除指定 job 和 instance 的指标
curl -X DELETE http://localhost:9091/metrics/job/test/instance/localhost
```

## 故障排查

### 1. 监控服务无法访问

**检查 Docker 容器状态**:
```bash
docker ps | grep -E "prometheus|grafana|pushgateway"
```

**查看容器日志**:
```bash
docker logs prometheus
docker logs grafana
docker logs pushgateway
```

**重启监控服务**:
```bash
bash stop-monitoring.sh
bash start-monitoring.sh
```

### 2. Flink 指标未显示

**检查 Pushgateway**:
```bash
curl http://localhost:9091/metrics | grep flink_
```

**检查 Flink 日志**:
```bash
tail -f logs/flink/flink.log | grep -i "prometheus"
```

**检查 Flink 配置**:
```bash
# 查看 Metrics Reporter 配置
grep -r "metrics.reporter" logs/flink/flink.log
```

**常见原因**:
- Flink 作业未启动
- Metrics Reporter 配置错误
- Pushgateway 地址不正确
- 推送间隔未到（等待 15 秒）

### 3. Grafana Dashboard 无数据

**检查数据源**:
1. 访问 Grafana
2. Configuration > Data Sources
3. 测试 Prometheus 连接

**检查查询语句**:
1. 打开 Dashboard
2. 编辑 Panel
3. 查看 Query 是否正确
4. 在 Prometheus 中测试查询

**检查时间范围**:
1. 确认 Dashboard 时间范围设置正确
2. 尝试选择 "Last 5 minutes"

### 4. Prometheus 无法抓取 Pushgateway

**检查 Prometheus 配置**:
```bash
# 查看配置文件
cat monitoring/prometheus/prometheus.yml

# 查看 Targets 状态
curl http://localhost:9090/api/v1/targets
```

**检查网络连接**:
```bash
# 从 Prometheus 容器内测试
docker exec prometheus wget -O- http://pushgateway:9091/metrics
```

## 性能优化

### 1. 减少 Metrics 开销

**调整推送间隔**:
```java
// 从 15 秒增加到 30 秒
config.setInteger("metrics.reporter.promgateway.interval", 30);
```

**禁用不需要的指标**:
```java
// 禁用系统资源指标
config.setBoolean(MetricOptions.SYSTEM_RESOURCE_METRICS, false);
```

### 2. 优化 Prometheus 存储

**调整数据保留时间**:
```yaml
# monitoring/prometheus/prometheus.yml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

# 添加存储配置
storage:
  tsdb:
    retention.time: 7d  # 保留 7 天数据
    retention.size: 10GB  # 最大 10GB
```

### 3. 优化 Grafana 查询

**使用聚合函数**:
```promql
# 不好：查询所有时间序列
flink_taskmanager_job_task_numRecordsInPerSecond

# 好：聚合后查询
sum(flink_taskmanager_job_task_numRecordsInPerSecond)
```

**使用 rate() 而不是 irate()**:
```promql
# rate() 更平滑，适合长期趋势
rate(flink_taskmanager_job_task_numRecordsIn[5m])

# irate() 更敏感，适合短期波动
irate(flink_taskmanager_job_task_numRecordsIn[5m])
```

## 最佳实践

### 1. 监控指标选择

**必须监控**:
- 吞吐量: `numRecordsInPerSecond`, `numRecordsOutPerSecond`
- 延迟: `currentInputWatermark`
- Checkpoint: `lastCheckpointDuration`, `numberOfFailedCheckpoints`
- 资源: `JVM.Memory.Heap.Used`, `JVM.CPU.Load`

**可选监控**:
- GC: `GarbageCollector.*.Time`
- 网络: `numBytesInPerSecond`, `numBytesOutPerSecond`
- 任务状态: `Task.*.Status`

### 2. 告警规则设置

**高优先级告警**:
- Checkpoint 失败率 > 10%
- JVM 堆内存使用率 > 90%
- 作业重启次数 > 3 次/小时

**中优先级告警**:
- 吞吐量下降 > 50%
- 延迟增加 > 2 倍
- GC 时间占比 > 30%

**低优先级告警**:
- CPU 使用率 > 80%
- 网络流量异常

### 3. Dashboard 设计

**布局建议**:
1. 第一行：关键指标（吞吐量、延迟、错误率）
2. 第二行：资源使用（CPU、内存、网络）
3. 第三行：Checkpoint 信息
4. 第四行：详细指标（GC、任务状态等）

**时间范围**:
- 实时监控：Last 5 minutes
- 问题排查：Last 1 hour
- 趋势分析：Last 24 hours
